## 摘要
在多模态多跳问答中，我们通过两个不同的任务关注初始检索阶段：（1）证据集补全，即给定上下文检索缺失的证据；（2）序列池构建，即从头开始迭代构建top-K池。在这些设置下，我们指出传统的迭代检索框架常常遭受语义锚定（Semantic Anchoring）问题，即先前获取的证据会困住检索器并产生以实体为中心的冗余。为了打破这一困境，我们提出了GRAIL（通过自适应隐式定位的差距感知检索），一种直接在嵌入层进行隐式查询重写的范式。通过上下文减法查询引导，GRAIL在组合性跨模态推理方面表现出色，而加法嵌入更新则在局部信息聚合上展示出优势。通过根据任务类型动态路由查询，我们的混合框架在MultimodalQA上实现了40.3%的宏观平均性能提升。大量评估表明，序列化GRAIL以优越且抗噪声的方式检索，通过迭代的差距感知优化显著扩展了搜索视野。

## Abstract
In multimodal multi-hop question answering, we focus on the initial retrieval stage via two distinct tasks: (1) evidence set completion, retrieving missing evidence given context, and (2) sequential pool construction, iteratively building the top-$K$ pool from the scratch. Under these settings, we point out that conventional iterative retrieval frameworks often suffer from Semantic Anchoring, where previously fetched evidence traps the retriever and yields entity-centric redundancy. To break this trap, we propose GRAIL (Gap-aware Retrieval via Adaptive Implicit Localization), a paradigm that performs implicit query rewriting directly at the embedding level. By context-subtractive query steering, GRAIL excels at compositional cross-modal reasoning, while additive embedding updates show strength on localized information aggregation. By dynamically routing queries based on task type, our Hybrid Framework achieves a 40.3\% macro-averaged performance gain on MultimodalQA. Extensive evaluations demonstrate that sequential GRAIL retrieves in a superior, noise-resilient manner, significantly expanding the search horizon through iterative gap-aware optimization.

- **背景**：在开放域多模态场景下，预定义静态查询分解不可行（因为事先不知道目标事实的模态）；而基于显式文本生成的查询改写（如使用LLM的IRCoT）容易引入语义漂移、语言偏置和高计算成本。因此需要一种在嵌入空间隐式进行查询改写、动态识别信息间隙的方法。

## 2. 方法论：核心思想与关键技术

### 核心思想
- **GRAIL（Gap-aware Retrieval via Adaptive Implicit Localization）**：在统一的多模态隐空间中，通过**上下文减法**（context-subtractive）改写查询，即从原始查询嵌入中减去已覆盖上下文的信息，从而将检索向量推向尚未回答的语义间隙。
- 同时引入**任务自适应混合框架**，根据问题类型动态路由：加法嵌入更新（Additive）更适合局部信息聚合任务（如单模态、交集类问题），减法更新（Gap-aware）更适合组合推理任务（如比较、组合类问题）。

### 关键技术细节
1. **统一语义对齐（Unified Semantic Alignment）**：
   - 将文本、表格、图像嵌入到共享隐空间（BGE-large 和 CLIP 分别编码后通过投影层对齐）。
   - 采用**查询-证据（Query-evidence）对齐策略**：以查询嵌入为锚点，利用InfoNCE损失函数拉近同一证据集内的成员，推远其他样本。
   - 实验中该策略在冻结骨干网络（frozen）时表现最佳，避免微调破坏预训练对齐。

2. **差距感知搜索机制（Gap-aware Search Mechanism）**：
   - **上下文聚合**：对已有证据集 $E_p$ 中每个证据 $e_i$ 加权求和得到上下文向量 $h_{ctx}$（注意力权重由查询与证据的相似度决定）。
   - **上下文减法**：计算可学习的门控系数 $g = \sigma(W_{gap}[q; h_{ctx}])$，生成净查询意图 $q_{gap} = q - g \cdot \left( \frac{q \cdot h_{ctx}}{\|h_{ctx}\|^2} \right) h_{ctx}$。
   - **动态混合**：将 $q_{gap}$ 和 $h_{ctx}$ 通过可学习权重 $[w_1, w_2]$ 混合得到最终检索向量 $h_{req} = LN(w_1 q_{gap} + w_2 h_{ctx})$。
   - 利用动态温度 $τ_{dyn}$ 的InfoNCE损失进行端到端训练。

3. **混合框架**：
   - 轻量级查询选择器 $\Phi(q)$（基于BERT）判定当前问题应使用加法还是减法模型。
   - 训练时独立优化选择器（准确率97.71%），推理时冻结。

### 算法流程（文字说明）
- **证据集补全**：给定查询 $q$ 和部分证据 $E_p$，执行上述减法改写得到 $h_{req}$，检索缺失证据。
- **序列池构建**：从零开始，先以原始查询检索初始基础，然后在每一轮中利用累积上下文进行减法改写，迭代补齐剩余证据（步骤数及每步检索量可配置，如 3+2+3+2）。

## 6. 主要结论与发现
1. **语义锚定普遍存在**：加法型方法在所有任务类型上的 $\Delta_{esc}$ 均为负，说明查询被上下文“困住”；减法型方法在组合推理任务（Compare, Compose）上 $\Delta_{esc}$ 为正。
2. **GRAIL显著优于加法与LLM基线**：在MultimodalQA上，混合框架宏平均Recall@1取得20.53%（对比Additive 11.57%，IRCoT 13.42%）。
3. **序列池构建更抗噪声**：即使初始检索全部错误（NRM为正），GRAIL仍能有效纠正，而加法模型误差传播严重。
4. **混合框架实现最佳综合性能**：选择器准确率97.71%，使各任务类型下都能调用最合适的专家。
5. **隐式改写优于显式LLM生成**：避免了语言损耗、语义漂移和高延迟，且完全在嵌入空间进行，不依赖外部API。

## 7. 优点
- **方法创新**：首次将“减法”思想引入多模态多跳检索，以投影方式滤除已覆盖信息，直接定位信息间隙。
- **高效性**：无需显式LLM调用，推理延迟仅几毫秒，API成本为零。
- **鲁棒性**：对噪声初始证据具有天然抗性，不会累积误差。
- **可解释性**：通过 $\Delta_{esc}$、Jump、NRM等诊断指标揭示了检索行为的几何意义。
- **完备实验**：
  - 覆盖两个大规模多模态数据集，对比多种基线（包括当前最先进的LLM）。
  - 深入分析跨模态对齐、训练策略、门控机制等内部模块。
  - 提供消融实验验证动态混合模块的必要性（虽不重要但整体略有帮助）。

## 8. 不足与局限
1. **固定步长调度**：序列池构建中步长和每步检索数量是手动预设的（如3+2+3+2），未实现自适应动态控制。
2. **无下游生成测试**：只聚焦检索阶段，未与生成式读者（如LMM）联合评估最终答案质量。
3. **可扩展性未验证**：实验限于中等规模语料（MultimodalQA约42万、WebQA约105万），未在web级开放域（如Wiki+多模态Web数据）中测试，无法确认对极端噪声和海量候选项的鲁棒性。
4. **模态覆盖有限**：目前仅处理文本、表格和图像，未涉及视频、音频等模态。
5. **潜在过减风险**：如附录A案例所示，减法操作有时会过于削弱目标证据的语义表示（如将Lons-le-Saunier行排到91位），需要后续迭代恢复。
