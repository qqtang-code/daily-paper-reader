## 摘要
基于文档的大规模工业语料库上的开放域问答（ODQA）的检索增强生成（RAG）面临两个关键瓶颈：定位正确文档的路由失败，以及整合分散信息的证据碎片化。现有方法依赖平面文本块或页面级图像，本质上难以（i）在数千候选文档中精确定位目标文档，以及（ii）在有限的令牌预算内有机连接多模态证据（如表格和图形）。为解决这些挑战，我们提出HiKEY，一种基于层级树的多模态检索框架，将文档层级提升为一等检索信号。不同于简单的分块，HiKEY通过文档层级解析（DHP）重建逻辑异构图，显式编码父子关系。采用由粗到细的层级策略，该框架（1）执行全局路由，利用层级索引快速剪枝搜索空间，以及（2）进行细粒度检索，通过采用捕获最具区分性证据的多模态融合策略对章节进行排序。最后，HiKEY通过混合结构-语义打包策略组装令牌高效的证据子图。在ODQA基准上的实验表明，HiKEY显著优于基于页面和基于块的基线，检索召回率提升高达12.9%，端到端问答性能提升高达6.8%。

## Abstract
Retrieval-augmented generation (RAG) for document-based Open-domain Question Answering (ODQA) on large-scale industrial corpora faces two critical bottlenecks: routing failure in locating the correct document and evidence fragmentation in integrating scattered information. Existing approaches relying on flat text chunks or page-level images inherently struggle to (i) precisely pinpoint the target document among thousands of candidates and (ii) organically connect multimodal evidence, such as tables and figures, within a limited token budget. To address these challenges, we propose HiKEY, a hierarchical tree-based multimodal retrieval framework that elevates document hierarchy to a first-class retrieval signal. Instead of simple chunking, HiKEY reconstructs a logical heterogeneous graph via Document Hierarchical Parsing (DHP), explicitly encoding parent-child relationships. Adopting a hierarchical coarse-to-fine strategy, the framework (1) performs global routing to rapidly prune the search space using hierarchical indexing, and (2) conducts fine-grained retrieval to rank sections by employing a multimodal fusion strategy that captures the most discriminative evidence. Finally, HiKEY assembles a token-efficient evidence subgraph via a hybrid structural-semantic packing strategy. Experiments on ODQA benchmarks demonstrate that HiKEY significantly outperforms page- and chunk-based baselines, improving retrieval recall by up to 12.9% and end-to-end QA performance by up to 6.8%.

- **问题背景**：在金融、法律、工程等专业领域中，大规模工业文档通常包含多页、混杂表格、图表、示意图等多模态元素，且证据可能分散在不同页面或文档中。传统的检索增强生成（RAG）方法采用扁平文本块或整页图像，面临两个关键瓶颈：
  - **路由失败**：系统难以在大型语料库中精准定位包含答案的目标文档。
  - **证据碎片化**：答案涉及的多模态证据（如表格、图表）被割裂，无法在有限的 Token 预算内有效整合。
- **整体意义**：现有方法（文本块 RAG、页面级多模态 RAG、图式 RAG）均未能充分利用文档固有的逻辑层级结构（如章节、小节、标题），导致检索效率和问答质量受限。本文提出将文档层级作为一等检索信号，从根本上解决工业级开放域问答的结构性问题。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **层级树重构**：将每个原始文档通过 **文档层级解析（DHP）** 重建为逻辑层级树，节点为多模态证据单元（段落、表格、图像），边为父子关系（章节路径）。
- **粗到细层级检索**：模仿人类阅读长文的方式——先扫目录确定候选文档（全局路由），再细读相关章节（细粒度检索）。
- **祖先感知子图组装**：在固定 Token 预算下，优先集成锚点单元及其祖先标题、兄弟单元、语义关联单元，形成紧凑且连贯的多模态证据子图。

### 2.2 关键技术细节

#### 离线阶段：层级感知图与索引构建
1. **文档层级解析（DHP）**：使用基于 LVLM 的解析器 M3DocDep 从页面布局块中恢复层级树，每个块被赋予祖先章节路径（如“标题 > 第3节 > 3.1”）。
2. **卡片式索引**：
   - **文档卡（Doc_card）**：包含标题 + 所有章节路径，用于全局路由。
   - **章节卡（Sec_card）**：包含本节内的文本、表格、图像等搜索单元。
3. **上下文丰富**：对缺少标题的视觉单元（表格、图像），通过图遍历获取逻辑前驱文本节点作为**上层上下文**，减少视觉歧义。

#### 在线阶段：粗到细检索
1. **阶段1：层级文档路由**
   - 使用混合评分（稀疏 BM25 + 稠密文本嵌入）对 Doc_card 排序，选出候选文档集合。
   - 公式：`S_doc(d,q) = α·mm(s_lex(x_d, q)) + (1-α)·mm(s_text(x_d, q))`
2. **阶段2：层级章节 MaxSim 评分**
   - 对候选文档内的每个章节，计算其各单元的**类型特定分数**：
     - 文本单元：混合分数（稀疏+稠密）
     - 表格/图像单元：视觉嵌入分数 + 上层上下文文本分数的加权组合
   - 章节分数 = 其单元分数的最大值（MaxSim），再与文档分数融合。
3. **层级子图组装**（混合结构-语义打包）
   - **祖先感知打包**：对每个入选章节，插入锚点单元及其祖先标题，保持逻辑连贯。
   - **混合证据扩展**：
     - **兄弟单元**（结构关联）：优先加入与锚点同父节的表格/图像。
     - **语义关联**（语义关联）：若预算允许，加入与锚点向量相似度高的跨区视觉单元。

## 7. 优点：方法或实验设计上的亮点
- **创新性**：首次将文档层级作为一等检索信号，而非被动元数据；通过 DHP 重建逻辑图，桥接了扁平检索与复杂文档布局之间的鸿沟。
- **设计一致性**：粗到细策略、祖先感知打包、混合结构-语义扩展，均与人类阅读长文档的直觉高度一致。
- **实验严谨性**：
  - 控制 Reader 和 Token 预算不变，隔离检索侧的贡献。

## 8. 不足与局限
- **预处理开销**：DHP 解析需要离线计算，虽然成本摊销，但对频繁更新的语料库不够灵活。
- **上游模块依赖**：依赖 OCR 和布局检测精度，低质扫描件或手写体可能导致层级路径错误，进而影响路由。
- **结构假设限制**：仅适用于具有显式层级结构（章节、标题）的文档，对扁平文本、票据、无结构幻灯片等文档增益有限。
- **后检索差距**：检索召回提升 12.9% 但端到端 QA 仅提升 6.8%，说明后检索环节（OCR 保真度、证据实体化、读者校准）仍有改善空间，但本文未深入探索。
- **读者泛化性**：仅在 Qwen2.5-VL-7B 上验证，未测试其他 LVLM（如 LLaVA、InternVL），推广性需更多验证。
- **实验覆盖**：仅在两个数据集上评测（M3DocVQA 和 FRAMES-PDF），未在真实工业私有语料上测试；跳数分布上 FRAMES 无 1-hop 样本，可能遗漏简单查询场景。
