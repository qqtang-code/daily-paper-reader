## 摘要
长上下文大语言模型（LLM）推理越来越受到键值（KV）缓存的内存占用和解码成本的制约，限制了其在资源受限硬件上的可持续部署。现有的KV缓存驱逐方法通常对基于GQA的LLM中所有注意力头应用启发式令牌评分。这些方法忽略了注意力头的不同功能，导致关键令牌被驱逐，从而降低LLM的性能。为解决这一问题，我们提出了CompressKV，一种针对基于GQA的LLM的资源高效KV缓存压缩框架。CompressKV不聚合所有头的注意力分数，而是识别出语义检索头（SRH），这些头捕获提示语的初始和最终令牌以及语义重要的中间上下文证据，并利用它们选择应保留KV对的令牌。此外，CompressKV根据逐层驱逐误差的离线估计，在各层之间分配缓存预算。在LongBench和Needle-in-a-Haystack上的实验表明，CompressKV在不同内存预算下均一致优于现有KV缓存驱逐方法。值得注意的是，在LongBench问答任务中，仅使用3%的KV缓存即可保持完整缓存性能的97%以上；在Needle-in-a-Haystack中，仅用0.7%的KV存储即可达到90%的准确率。这些结果展示了长上下文LLM推理中资源与性能权衡的改进。我们的代码公开于：https://github.com/TUDa-HWAI/CompressKV

## Abstract
Long-context large language model (LLM) inference is increasingly constrained by the memory footprint and decoding cost of key-value (KV) caches, limiting sustainable deployment on resource-constrained hardware. Existing KV cache eviction methods typically apply heuristic token scoring over all heads in GQA-based LLMs. These methods ignore the different functionalities of attention heads, leading to the eviction of critical tokens and thus degrading the performance of LLMs. To address this issue, we propose CompressKV, a resource-efficient KV-cache compression framework for GQA-based LLMs. Instead of aggregating attention scores from all heads, CompressKV identifies Semantic Retrieval Heads (SRHs) that capture both the initial and final tokens of a prompt and semantically important mid-context evidence, and uses them to select tokens whose KV pairs should be retained. Furthermore, CompressKV allocates cache budgets across layers according to offline estimates of layer-wise eviction error. Experiments on LongBench and Needle-in-a-Haystack show that CompressKV consistently outperforms existing KV-cache eviction methods across memory budgets. Notably, it preserves over 97\% of full-cache performance using only 3\% of the KV cache on LongBench question-answering tasks and achieves 90\% accuracy with just 0.7\% KV storage on Needle-in-a-Haystack. These results demonstrate an improved resource--performance trade-off for long-context LLM inference. Our code is publicly available at: https://github.com/TUDa-HWAI/CompressKV

# CompressKV 总结

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：长上下文大语言模型（LLM）推理中，键值（KV）缓存随上下文长度线性增长，导致巨大的内存占用和解码延迟，成为资源受限硬件上部署的主要瓶颈。
- **现有方法不足**：已有的 KV 缓存驱逐方法（如 SnapKV、PyramidKV、CAKE 等）通常对所有注意力头使用启发式令牌评分，忽略了注意力头的功能异质性。在分组查询注意力（GQA）架构中，某些“流式头”只关注提示的开头和结尾，若它们主导驱逐决策，会错误地丢弃中间上下文的关键令牌，损害模型性能。
- **本文目标**：提出 CompressKV，一种资源高效的 KV 缓存压缩框架，通过识别“语义检索头（SRH）”来引导令牌选择，并基于离线误差进行层自适应预算分配，从而在严格内存约束下实现更好的性能-资源权衡。

## 2. 方法论
**核心思想**：利用具有语义检索能力的注意力头（SRH）来选择重要令牌，避免流式头的错误支配；同时根据各层压缩后的重构误差离线分配缓存预算。

**关键技术细节**：
1. **语义检索头识别**：
   - 使用校准数据集（同文献 [27]），对每个注意力头 \(h\)，在生成正确答案令牌时，计算其注意力权重在答案跨度 \(\mathcal{A}\) 上的总和作为 SRH 分数：
     \[
     S_{\text{SRH}}(h) = \sum_{t=1}^{N} \mathbf{1}\{y_t \in \mathcal{A}\} \sum_{j \in \mathcal{A}} a^{(h)}_{t,j}
     \]
   - 该标准不同于传统检索头（要求 top-1 或 top-k 命中），它通过聚合整个答案跨度的注意力来捕捉语义相关的上下文，即使头从未对单个令牌达到峰值。
2. **令牌选择驱动**：
   - 每层选取 top-\(k\) 个 SRH（实验中取 \(k=4\)）。
   - 将这些头的注意力分数在观察窗口内求和并池化，再取平均，选出得分最高的 \(N\) 个令牌保留其 KV 对。
   - 同一层内所有注意力头共享这组令牌索引，避免流式头主导驱逐。
3. **误差感知的层自适应缓存分配**：
   - 离线计算：在微预算压缩下，测量每层注意力块输出（压缩缓存 vs 完整缓存）的 Frobenius 范数归一化误差：
     \[
     e^{(l)} = \sum_{t=1}^{T} \frac{\|\Delta O_t^{(l)}\|_F}{\|O_{t,\text{full}}^{(l)}\|_F + \epsilon}, \quad \tilde{e}^{(l)} = e^{(l)} / \sum_k e^{(k)}
     \]
   - 分配策略：先给每层最小预算 \(m=32\) 和最大预算 \(M=3\times B_{\text{per-layer}}\)，剩余预算按标准化误差比例分配。
   - 整个过程离线完成，在线推理时无需额外计算。
