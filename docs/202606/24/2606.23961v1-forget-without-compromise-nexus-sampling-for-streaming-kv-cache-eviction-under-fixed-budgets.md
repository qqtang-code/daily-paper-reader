## 摘要
长上下文和智能体LLM工作负载将KV缓存推至固定内存预算之外，迫使推理堆栈在连续推理流的每一步永久驱逐令牌。现有方法都遵循相同的模板：每一步的直接注意力分数后接确定性top-K选择，这会将单个低于阈值的步骤转化为不可逆的裁决，并永久擦除任何直接注意力无法从噪声中区分出来的微妙重要令牌。为应对这一挑战，我们提出Nexus采样，一种无需训练的驱逐方法，将Nexus评分（一种在直接注意力上迭代的行走，揭示桥接令牌）与加权水库采样（以包含概率保留令牌，取代确定性top-K）相结合。理论上，我们证明Nexus采样在微妙重要令牌的长期存活上优于确定性top-K。实验上，在80% KV缓存驱逐下，Nexus采样在LongBench上与密集注意力匹配（差异在1%以内），同时在检索密集型任务上优于top-K基线，每个序列的缓存内存最多减少10倍。

## Abstract
Long-context and agentic LLM workloads push the KV cache past any fixed memory budget, forcing the inference stack to permanently evict tokens at every step of a continuous-inference stream. Existing methods all share the same template, a per-step direct-attention score followed by deterministic top-$K$ selection, which converts a single below-cutoff step into an irreversible verdict and permanently erases any subtly important token that direct attention cannot single out from noise. To address this challenge, we propose Nexus Sampling, a training-free eviction method that pairs Nexus scoring, an iterative walk over direct attention that surfaces bridge tokens, with weighted reservoir sampling, which retains tokens with inclusion probability in place of deterministic top-$K$. Theoretically, we show that Nexus Sampling dominates deterministic top-$K$ in long-run survival of subtly important tokens. Empirically, at 80% KV cache eviction, Nexus Sampling matches dense attention within 1% on LongBench while outperforming top-$K$ baselines on retrieval-heavy tasks, with up to 10x smaller per-sequence cache memory.

- **核心矛盾**：驱逐是流式的、固定预算的问题，而稀疏注意力是单次读取。确定性 top-K 将单步的瞬时边际性误判为永久的不重要性，导致对重要性波动的 token（如桥接 token）不可逆的丢失。
- **论文贡献**：提出 **Nexus Sampling**，一种无需训练的驱逐方法，通过** Nexus 评分**（迭代行走找出桥接 token）和**加权水库采样**（以概率保留 token）替代确定性 top-K，从而在长期流式场景中更好地保留细微重要 token。理论上证明了水库采样的长期存活率随步骤乘积衰减，而 top-K 随最小值衰减；经验上在 LongBench 上接近全注意力，检索任务上显著优于 top-K 基线。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
Nexus Sampling 由两个组件组成，在每个驱逐步骤依次执行：
1. **Nexus 评分**（Upstream）：在直接注意力分数上执行一段短迭代行走（iterative walk），揭示“桥接 token”（bridge tokens）——即那些单次查询不直接强关注，但结构性地连接强相关 token 簇的位置。
2. **加权水库选择**（Downstream）：使用加权无放回的水库采样（weighted reservoir sampling）从所有 token 中抽取 K 个保留 token，替代确定性 top-K。

### 技术细节
- **块粒度**：每个块包含 b = 32 个 token，以块为单位处理。
- **基础分数**：对最近 W 个查询（W = 16）的头平均注意分布进行行平均，得到每个块的直接注意力分数 \(a\)。
- **Bridge 分数**：通过迭代行走计算。从 C=0 开始，对每个窗口查询的行分布 \(a^{(q)}\)，更新：
  \[
  C \leftarrow C + \gamma_q a^{(q)},\quad \gamma_q = 1 + \langle a^{(q)}, C \rangle
  \]
  迭代 H = 3 步后，归一化得到 \(\tilde{c} = C / \|C\|_1\)。
- **组合权重**：\(w_j = a_j + \lambda \tilde{c}_j + \varepsilon_{\text{tie}} r_j\)，其中 \(\lambda=0.5\)，\(\varepsilon_{\text{tie}}=10^{-6}\) 是微小的新近性打破平手因子。
- **水库优先级**：对每个候选块 j，独立抽取 n = 5 个均匀随机变量 \(u_j^{(i)} \sim U(0,1)\)，计算平均优先级：
  \[
  \pi_j = \frac{1}{n} \sum_{i=1}^n \left( u_j^{(i)} \right)^{1/w_j}
  \]
  保留优先级最高的 K 个块（除开强制保留的注意力沉没块和最近窗口块）。
- **理论保证**：Lemma 4.1：确定性 top-K 的长期存活概率是各步骤指示函数的乘积，一旦某步低于阈值即归零；水库采样的存活概率是各步包含概率的乘积，只要每一步权重为正就保持正数。Theorem 4.2：行走迭代能放大枢纽（hub）块（即桥接 token）的期望分数，使其高于直接窗口平均。Proposition 4.3：结合两部分给出端点驱逐质量界。
