## 摘要
自回归（AR）视频生成通过顺序生成潜在块来扩展视频，但扩展到长视频需要重复访问不断增长的历史KV缓存。现有方法通过截断KV缓存或将其压缩为隐式记忆来降低这一成本，但这两种方法都会丢失对查询相关历史细节的显式访问。我们提出OmniMem，一种显式的全范围记忆检索框架，它对历史缓存执行稀疏KV检索。为了使这一方法适用于基于块的AR视频生成，OmniMem解决了两个问题：（i）稀疏KV选择中的局部偏差和（ii）记忆访问中的联合爆炸。自适应窗口排除在具备足够长程历史时从选择候选中移除局部窗口块，从而保留稀疏预算用于信息丰富的长程检索。查询共享KV选择减少了跨查询的多样性，而逐头分散KV访问避免将头部特定选择扩展为大的选定KV缓冲区。这使得每个注意力头能够根据自身的选择模式检索非连续的KV块。在长视频生成上的实验表明，OmniMem将动态度提高了52.3%，并在强基线基础上保持了强一致性，同时保持了可比较的内存使用量。

## Abstract
Autoregressive (AR) video generation extends videos by producing latent chunks sequentially, but scaling to long videos requires repeated access to a growing historical KV cache. Existing methods reduce this cost by truncating the KV cache or compressing it into implicit memory, but both lose explicit access to query-relevant historical details. We propose OmniMem, an explicit full-range memory retrieval framework that performs sparse KV retrieval over the historical cache. To make this practical for chunk-based AR video generation, OmniMem addresses two issues: (i) local bias in sparse KV selection and (ii) Union Explosion in memory access. Adaptive Window Exclusion removes local-window blocks from the selection candidates when sufficient long-range history is available, preserving the sparse budget for informative long-range retrieval. Query-Shared KV Selection reduces cross-query diversity, while Per-Head Scattered KV Access avoids expanding head-specific selections into a large selected KV buffer. This allows each attention head to retrieve non-contiguous KV blocks according to its own selection pattern. Experiments on long-video generation show that OmniMem improves Dynamic Degree by 52.3% and preserves strong consistency over strong baselines, while maintaining comparable memory usage.

## 1. 核心问题与整体含义（研究动机和背景）
- **研究背景**：自回归（AR）视频生成通过逐块生成潜在序列实现长视频扩展，但由于历史KV缓存随生成块数线性增长，导致内存消耗和注意力计算成本急剧上升。
- **现有方法缺陷**：
  - 直接截断KV缓存会丢失长程依赖，造成对象漂移（颜色、形状、轨迹不一致）。
  - 固定锚点（如Sink tokens）虽能保持一致性，但易导致运动重复；辅助模块压缩历史会丢失细粒度细节。
- **核心问题**：如何在长视频生成中实现**显式的全范围历史记忆检索**，同时避免局部偏差和联合爆炸带来的效率问题。
- **整体含义**：OmniMem提出一种可扩展的自适应记忆检索框架，通过稀疏KV检索平衡长程依赖、运动动态与计算效率。

## 2. 方法论：核心思想、关键技术细节与算法流程
### 2.1 核心思想
- 借鉴Native Sparse Attention (NSA) 的稀疏注意力思想，将长距离记忆访问视为显式的块级检索问题。
- 融合三个注意力分支：压缩注意力（CMP，提供粗粒度全局摘要）、选择注意力（SLC，精确检索全分辨率历史KV块）、滑动窗口注意力（SWA，关注近期上下文）。输出通过可学习门控系数融合。

### 2.2 关键技术细节
- **自适应窗口排除 (Adaptive Window Exclusion, AWE)**：
  - 在Top-K选择前，将滑动窗口覆盖的KV块从候选集中移除（条件：当历史缓存中至少有K个块在窗口外）。
  - 避免选择注意力浪费预算在近期冗余内容上，强制检索长程信息。
  - 数学表达（简化）：候选集 \(\Omega^{(n)}_{SLC} = H^{(n)} \setminus W^{(n)}\)（当 \(|H^{(n)} \setminus W^{(n)}| \geq K\)），否则保留全历史。
- **查询共享KV选择 (Query-Shared KV Selection)**：
  - 将当前块的S个查询令牌划分为R个相邻组（每组G<sub>q</sub>个令牌），组内共享同一个Top-K块选择。
  - 通过对组内各查询的压缩注意力分数进行平均池化得到组级分数，再执行Top-K选择。
  - 公式：\(\bar{s}_{h,r,b} = \text{AvgPool}_{q \in G_r} s_{h,q,b}\)， \(I^{(n)}_{h,r} = \text{TopK}_{b \in \Omega^{(n)}_{SLC}} \bar{s}_{h,r,b}\)。
  - 作用：将跨查询选择联合大小从S·K降至R·K，同时保持组间多样性。
- **每头分散KV访问 (Per-Head Scattered KV Access)**：
  - 针对不同注意力头可能选择不同历史区域的问题，不合并成统一大缓冲区，而是为每个头独立维护指针表，直接读取非连续KV块。
  - 实现：全分辨率KV缓存按每个头、每个块独立存储，通过指针表映射选中块索引到物理地址；注意力内核跳过未选中或已换出的块。
  - 作用：内存占用仅与各个头选中块的**总和**成正比，而非跨头并集，避免联合爆炸导致的内存膨胀和冗余PCIe传输。

### 2.3 算法流程（文字说明）
1. **预处理**：对KV缓存进行2D空间邻域平均池化，得到压缩KV块；同时维护全分辨率块级KV缓存（按块、头存储）。
2. **选择候选集确定**：应用自适应窗口排除，从全历史KV块中移除滑动窗口覆盖块（条件满足时）。
3. **查询共享选择**：将当前块查询分组，计算每组内所有查询的压缩注意力分数平均，作为组级分数；对每个头、每组独立执行Top-K选择，得到选中块索引。
4. **指针表构建**：轻量级GPU核将选中块索引映射为对应KV块的GPU内存地址（若块在CPU则等待重载并标记为0）。
5. **稀疏注意力计算**：Triton内核根据指针表读取非连续KV块，进行在线softmax注意力计算，跳过标记为0的块。
6. **分支融合**：将滑动窗口、压缩和选择注意力的输出通过可学习门控系数加权求和，得到最终输出。
