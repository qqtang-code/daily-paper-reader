## 摘要
基于知识的视觉问答（KB-VQA）要求模型将图像理解与外部知识相结合。以往的方法大多采用固定的检索-生成流水线，使用预选的检索器和静态的 top-k 设置，这在推理过程中缺乏自适应性。我们提出了 ProMSA，一种用于 KB-VQA 的渐进式多模态搜索智能体。给定一个图像-问题对，该智能体在明确的工具调用预算下迭代选择图像搜索、文本搜索或停止，并通过去重避免冗余检索。在训练方面，我们首先使用拒绝采样 SFT 学习有效的工具使用格式，然后通过 TN-GSPO（一种序列级 RL 目标函数，通过生成长度和工具交互深度对更新进行归一化）来优化智能体。在 E-VQA 和 InfoSeek 上的实验表明，该方法相对于强大的 RAG 和智能体基线有一致的提升，并提高了检索和端到端准确性。代码开源于 https://github.com/DingWu1021/Promsa。

## Abstract
Knowledge-based Visual Question Answering (KB-VQA) requires models to combine image understanding with external knowledge. Most prior methods use a fixed retrieve-then-generate pipeline with a pre-selected retriever and a static top-k setting, which is not adaptive during reasoning. We propose ProMSA, a progressive multimodal search agent for KB-VQA. Given an image-question pair, the agent iteratively chooses image search, text search, or stop, under explicit tool-call budgets and with deduplication to avoid redundant retrieval. For training, we first use rejection-sampling SFT to learn valid tool-use formats, then optimize the agent with TN-GSPO, a sequence-level RL objective that normalizes updates by both generation length and tool-interaction depth. Experiments on E-VQA and InfoSeek show consistent gains over strong RAG and agent baselines, and improved retrieval and end-to-end accuracy. The code is available at https://github.com/DingWu1021/Promsa.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：基于知识的视觉问答（KB-VQA）要求模型同时理解图像内容并利用外部知识（如Wikipedia）进行推理。现有方法大多采用固定的“检索-生成”流水线：
  - 使用预选的检索器（如EVA-CLIP）和静态的top-k设置，缺乏推理过程中的自适应性。
  - 无法处理实体识别失败后的恢复：一旦首次检索偏离正确实体，后续步骤会在错误页面积累证据，导致偏差难以纠正。
  - 难以支持多跳证据链：静态top-k无法逐步扩展所需知识。
- **研究动机**：KB-VQA不需要更大的top-k或更强的重排序器，而需要一个紧密耦合检索与推理的闭环系统，使模型像一个有预算意识的研究员：不确定时检索，证据不足时继续搜索，证据充足时停止。
- **研究含义**：提出ProMSA（Progressive Multimodal Search Agents），一种渐进式多模态搜索智能体，将KB-VQA形式化为一个预算受限的渐进搜索-推理问题。

## 2. 论文提出的方法论

### 核心思想
- 智能体参数化为单一多模态大模型（MLLM），在推理过程中迭代决策动作：`img_search`（图像搜索）、`text_search`（文本搜索）或`stop`（停止并生成答案）。
- 每个动作有明确的预算约束（`B_img`和`B_text`分别限制图像搜索和文本搜索的最大调用次数），并通过维护访问过的页面集实现去重，避免重复检索。
- 检索到的内容经过摘要压缩（Summarize）后作为观察反馈给模型，更新状态。

### 关键技术细节
1. **动作空间与状态更新**：
   - 动作 `u_t ∈ {img_search, text_search, stop}`。
   - 状态 `s_t` 包含历史生成的推理、工具观察结果等。
   - 结构化输出：`y_t = (z_t, u_t, args_t)`，其中 `z_t` 为推理，`args_t` 指定查询、top-k、去重列表等。

2. **训练两阶段**：
   - **Stage I (Cold Start SFT)**：使用拒绝采样（rejection-sampling）从初始策略采样N条轨迹，过滤出格式正确、可执行且答案正确的轨迹，形成小规模SFT数据集（约3000条），训练模型学会工具调用的基本格式和交互模式。
   - **Stage II (Reinforcement Learning)**：采用序列级策略优化，具体为目标提出了**TN-GSPO（Tool-Normalized Group Sequence Policy Optimization）**。

3. **TN-GSPO 详细设计**：
   - 轨迹的log概率仅计算模型生成的token（排除工具返回的token）：
     \[
     \log \pi_{\theta}(\tau) = \sum_{t \in T_{\text{gen}}} \log \pi_{\theta}(y_t | s_t)
     \]
   - 奖励函数：
     \[
     R(\tau) = r_{\text{ans}}(\tau) + r_{\text{fmt}}(\tau) - \lambda C(\tau), \quad C(\tau) = \frac{\#\text{tool\_calls}(\tau)}{H_{\max}}
     \]
     其中 `r_ans` 为答案正确性，`r_fmt` 为格式合规性，`C` 为工具成本惩罚。
   - 归一化因子 `D(τ)` 同时考虑生成长度 `L(τ)` 和工具交互深度 `H(τ)`：
     \[
     D(\tau) = L(\tau) \cdot (1 + c H(\tau)^{\alpha})
     \]
   - 序列级似然比：
     \[
     r_{\theta}(\tau) = \exp\left( \frac{1}{D(\tau)} \sum_{t \in T_{\text{gen}}} \Delta_t \right)
     \]
     其中 Δ_t = log π_θ(y_t|s_t) - log π_ref(y_t|s_t)。
   - 最终目标函数使用非对称裁剪：
     \[
     \mathcal{L}_{\text{TN-GSPO}}(\theta) = -\mathbb{E}_{\tau \sim \pi_{\theta}} \left[ \min\left( r_{\theta}(\tau) A(\tau), \text{clip}(r_{\theta}(\tau), 1-\epsilon_-, 1+\epsilon_+) A(\tau) \right) \right]
     \]
     其中优势 `A(τ) = R(τ) - b(x)`。

4. **工具实现**：
   - 图像检索：EVA-CLIP，基于输入图像和排除列表检索Wikipedia页面。
   - 文本检索：BGE（BAAI General Embedding），基于模型生成的查询检索Wikipedia章节。
   - 摘要服务：使用Qwen3-VL-8B对检索到的长文本进行问题相关的压缩。

## 6. 论文的主要结论与发现
1. **ProMSA在E-VQA和InfoSeek上均达到SOTA**：
   - E-VQA：All 52.6%（Qwen3-VL-8B），比最强的RAG基线REAL高约7个百分点。
   - InfoSeek：All 53.4%，同样显著领先。
2. **RL阶段带来大幅度提升**：
   - 从Base（平均35.1%）到Stage I SFT（40.7%），再到Stage II RL（53.0%），RL贡献最大。
3. **TN-GSPO优于其他序列级RL方法**：
   - 相较于GRPO*（E-VQA 49.7%，InfoSeek 50.2%）和GSPO*（49.3%/49.6%），TN-GSPO分别提升至52.6%/53.4%。
4. **工具搭配重要性**：同时使用图像搜索和文本搜索显著优于单一工具。
5. **自适应检索行为**：训练后模型学会在首次检索失败时继续搜索，正确检索后倾向于停止，体现了渐进式搜索的有效性。
6. **工具预算和top-k存在最优值**：过度增加预算或top-k会引入噪声，导致性能边际递减。

## 7. 优点（方法或实验设计亮点）
- **创新性**：从固定流水线转向自适应搜索代理，引入去重和预算约束，更贴近实际人类搜索行为。
- **训练策略**：拒绝采样SFT解决冷启动问题，RL阶段使用工具地平线归一化稳定训练，克服了搜索代理中长序列和工具交互带来的方差问题。
- **实验设计严谨**：全面比较了不同训练阶段、RL优化器、工具配置，并分析了检索行为细节，揭示了智能体学习到的策略。
- **效率与性能平衡**：推理时间仅增加1.5倍于EchoSight，准确率却提升至近3倍，展示了良好的实用性。
- **代码开源**：提供完整代码库，便于复现和扩展。

## 8. 不足与局限
- **实验覆盖有限**：
  - 训练数据仅来自E-VQA和InfoSeek，未在更多KB-VQA数据集（如OK-VQA仅在推理阶段测试）上验证泛化性。
  - 只测试了Qwen系列模型，未在LLaVA、BLIP-2等架构上实验，可能限制泛化结论。
- **工具成本惩罚可能过于保守**：
  - 在检索行为分析（表9）中，部分样本在Round-2检索错误后仍选择停止，可能是工具成本惩罚导致的过早停止，影响了收敛。
- **依赖固定摘要模型**：
  - 使用Qwen3-VL-8B作为摘要服务，增加了系统复杂度和推理开销（需要单独部署），且摘要质量可能成为瓶颈。
- **长尾极端情况**：
  - 对于极其罕见的实体，即使多次检索也可能无法召回正确页面，ProMSA仍可能失败（即“Incorrect Retrieval & Continue”下降但“Incorrect Retrieval & Stop”比例仍较高）。
- **未分析泛化到不同知识库**：所有实验基于Wikipedia，若知识库结构不同（如网页、数据库），去重和摘要策略可能需要调整。
