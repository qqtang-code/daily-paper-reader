## 摘要
视觉推理需要整合分布在不同区域、属性和关系中的证据，这使得单链推理容易出现早期感知承诺和幻觉。我们提出视觉并行思考者++（Visual Para-Thinker++），这是一个单策略多智能体框架，其中一个共享的MLLM策略被实例化为具有角色条件的Main、Worker和Summary智能体。Main智能体通过固定分配模式分解任务；Worker智能体在上下文隔离下并行推理；Summary智能体整合完整的Worker推理轨迹，而不是对最终标签进行多数投票。共享策略通过多智能体能力注入和角色解耦的多智能体优化进行训练，这些方法将角色特定的奖励和优势分配给相应的token片段，以减少协作角色间的梯度冲突。原生推理引擎通过共享视觉前缀和KV缓存重用实现高效的多智能体推理。在V*、CountBench、RefCOCO系列和HallusionBench上，视觉并行思考者++始终优于单轨迹和推理时间并行基线，尤其是在对幻觉敏感的视觉推理任务上表现显著提升。

## Abstract
Visual reasoning requires integrating evidence distributed across regions, attributes, and relations, making single-chain reasoning prone to early perceptual commitment and hallucination. We propose Visual Para-Thinker++, a single-policy multi-agent framework in which one shared MLLM policy is instantiated as role-conditioned Main, Worker, and Summary Agents. The Main Agent decomposes the task with fixed allocation patterns; Worker Agents reason in parallel under context isolation; and the Summary Agent reconciles full Worker reasoning traces rather than majority-voting on final labels. The shared policy is trained by Multi-Agent Capability Injection and Role-Decoupled Multi-Agent Optimization, which assign role-specific rewards and advantages to corresponding token segments to reduce gradient conflict among collaborative roles. A native inference engine enables efficient multi-agent rollout through shared visual prefix and KV cache reuse. Across V*, CountBench, the RefCOCO family, and HallusionBench, Visual Para-Thinker++ consistently outperforms single-trajectory and inference-time parallel baselines, with especially strong gains on hallucination-sensitive visual reasoning.

## 2. 论文提出的方法论
### 2.1 核心思想：单策略多智能体框架
- 共享同一个多模态大语言模型（MLLM）策略 `π_θ`，通过**角色条件（role conditioning）** 实例化为三种智能体：
  - **Main Agent**：接收图像和问题，按照固定模式（块状划分或扫描顺序）将任务分解为 K=4 个子任务，分配给 Worker。
  - **Worker Agent**：在上下文隔离（context isolation）下并行执行各自子任务，互不可见对方 token。
  - **Summary Agent**：读取所有 Worker 的完整推理轨迹，进行**轨迹级证据调和（trace-level reconciliation）**，而非简单多数投票，输出最终答案。
- 所有智能体共享相同权重，仅通过角色 token 和可见上下文区分。

### 2.2 关键技术细节
#### （1）Main-Agent 任务分配模式（两种固定模式）
- **Block-based**：将图像划分为 K 个不重叠区域（默认四象限），Worker 收集局部证据，适合细粒度感知、属性验证等。
- **Scan-order**：每个 Worker 保留全局视野，但按不同扫描路径（从左到右、从右到左、从上到下、从下到上）遍历图像，适合计数和全局验证（减少漏计/重复）。
- 默认用 block-based，任务若以计数或全局验证为主则切换为 scan-order。

#### （2）多智能体能力注入（Stage 1：SFT）
- 使用更强的 MLLM 教师合成多智能体轨迹 `τ = [m, s1, ..., sK, u]`，最小化如下损失：
  ```
  L_MAC(θ) = - ∑_{t∈m} log π_θ(m_t | c_m, m_<t)
             - ∑_{i=1}^{K} ∑_{t∈si} log π_θ(si,t | c_i, si,<t)
             - ∑_{t∈u} log π_θ(u_t | c_u, u_<t)
  ```
  其中 c_m=(v,q,r_main), c_i=(v,q,m,r_worker_i), c_u=(v,q,m,s_1:K,r_sum)。
- 使用**上下文隔离掩码**：Worker 只能看到 (v,q,m) 和自己的后缀；Summary 能看到所有 Worker 轨迹。

#### （3）角色解耦的多智能体优化（Stage 2：RL）
- **问题**：Worker 和 Summary 的优化目标不同（局部证据准确性 vs 全局答案正确性），若用单一奖励广播到所有 token，会耦合梯度。
- **方案**：
  - **Worker 奖励** `R_worker^(i)`：基于跨 Worker 多数投票启发式，评价该 Worker 子答案是否与多数一致，只在训练时使用。
  - **结局奖励** `R_out(τ)` ∈ {0,1}：Summary 最终答案是否正确。
  - **角色解耦优势**：分别对两种奖励做组归一化得到 `A_out` 和 `A_worker^(i)`，然后按角色 token 逐位组合：
    ```
    A_t = { A_out, if t∈m∪u; A_out + λ A_worker^(i), if t∈si }
    ```
    其中 λ=0.5。
  - 将此优势代入裁剪的 DAPO 损失进行策略更新。
- **三种组合范式对比**：Naive Sum（所有奖励相加共享）、Conditional Sum（R_out 门控 Worker 奖励后再共享）、Role-decoupled Advantages（本工作）。实验证明角色解耦最优。

- 整体框架流程：图像+问题 → Main Agent 分配 → 四个 Worker 并行推理（上下文隔离）→ Summary Agent 整合 → 最终答案。

## 7. 优点
- **创新性**：首次将单策略多智能体范式系统引入视觉推理，通过角色 token 和上下文隔离实现共享权重的多智能体协作，避免了多模型部署的高成本。
- **训练方法**：角色解耦的优势分配方法有效解决了多角色梯度冲突问题，是 RL 在多智能体优化中的一个巧妙设计。
- **推理效率**：原生推理引擎（基于 vLLM）通过共享视觉前缀和 KV cache 写时复制大大降低了并行推理开销，使得多智能体推理实际可行。
- **实验扎实**：在多个具有不同挑战的视觉基准上评估，消融实验覆盖关键超参数和设计选择，证明了各组件的贡献。

## 8. 不足与局限
- **骨干规模有限**：主要验证在 3B 骨干，7B 只做补充，更大规模（>30B）的行为未探索。
- **任务覆盖不全**：未涉及文档理解、图表推理、视频时序决策等更广泛的多模态场景。
- **Main Agent 固定分配模式**：仅两种模式且由规则选择，未实现自适应、开放式的任务规划，限制了灵活性。
- **Worker 奖励粗糙**：当前使用基于多数投票的启发式，当大多数 Worker 都犯相同错误时会强化错误；缺乏更细粒度的过程监督。
- **Summary Agent 未独立验证**：没有隔离分析 Summary 的轨迹阅读与证据调和两个功能的各自贡献（未来工作可替换为多数投票对比）。
- **计算成本**：虽然推理优化，但增加 Worker 数量（K=8）几乎不提升性能，表明当前设计已达到 trade-off 上限，更大骨干也许会有不同。
- **伦理风险**：改进的推理能力可能被滥用产生更令人信服的错误分析；系统并未消除幻觉，只是减少了，高保真场景仍需人工验证。
