## 摘要
我们引入了一个新工具Express，用于将非因果注意力近似转换为具有匹配近似保证的因果近似。当与最先进的Thinformer近似结合时，Express改进了已知最佳的因果注意力保证，对于长度为n的序列，仅需O(s)内存和O(s^2 log^2(n))压缩开销，即可实现log^{3/2}(n)/s的近似误差。我们将这些进展与高效的I/O感知Triton实现相结合，展示了相对于FlashAttention 2的显著加速，并使用Express克服了语言建模流程中的四个资源瓶颈：长上下文预填充、KV缓存压缩、长形式内存受限解码和长形式计算受限解码。

## Abstract
We introduce a new tool, Express, for converting a non-causal attention approximation into a causal approximation with matching approximation guarantees. When combined with the state-of-the-art Thinformer approximation, Express improves upon the best known causal attention guarantees, delivering $\log^{3/2}(n)/s$ approximation error with only $O(s)$ memory and $O(s^2 \log^2(n))$ compression overhead for a sequence of length $n$. We pair these developments with an efficient I/O-aware Triton implementation, demonstrate substantial speedups over FlashAttention 2, and use Express to overcome four resource bottlenecks in the language modeling pipeline: long-context prefill, KV cache compression, long-form memory-constrained decoding, and long-form compute-constrained decoding.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在语言建模中，自注意力机制的计算和内存开销随序列长度呈二次增长，尤其是因果掩码（causal masking）场景（每个query只关注之前的key-value对），这导致长上下文预填充、KV缓存压缩、长形式解码等环节存在严重资源瓶颈。
- **背景现状**：已有的非因果注意力近似（如Thinformer）具有良好的近似保证，但无法直接用于自回归生成中的因果掩码。同时，HyperAttention、BalanceKV等方法虽然提供了因果近似，但误差衰减慢、对值矩阵依赖大、压缩开销高。
- **研究意义**：本文旨在将非因果近似的优势扩展到因果场景，实现高效、低内存、低误差的因果注意力近似，从而降低语言建模推理中的计算和存储成本。

## 2. 论文提出的方法论

- **核心思想**：提出一个元过程**Express**，能够将任意非因果的薄化（thinning）注意力近似（如子高斯薄化算法）转换为流式因果近似，且保持相同的近似保证。具体地，Express维护一个可更新的加权缓存，通过三个阶段实现高效压缩：
  - **精确阶段**：前\(n_{\text{out}}\)个点直接存入缓存。
  - **薄化阶段**：后续点分批（每批\(2mn_{\text{out}}\)个）先通过均匀子采样（Subsample）再通过压缩2（Compress2）逐步压缩至\(n_{\text{out}}\)个。
  - **Halve阶段**：当缓存大小达到\(4n_{\text{out}}\)时，连续应用两次核减半（KH）算法将缓存减回\(n_{\text{out}}\)，并增大薄化因子\(m\)。
- **关键技术细节**：
  - 结合最优的非因果薄化算法**Thinformer**，并采用**核减半（KH）**实现子高斯薄化，保证\(\log^{3/2}(n)/s\)近似误差，内存仅\(O(s)\)，压缩开销\(O(s^2\log^2(n))\)。
  - 提供理论保证：Express将非因果子高斯常数最多放大\(\sqrt{\frac{16}{15}(16+3m)} \le 2\sqrt{\log_2 n_{\text{out}}}+6\)倍。
  - 实现I/O感知的Triton内核，利用分块、并行化、双指针引用减少高带宽内存读写。
- **算法流程**：输入序列\((x_i)\)，初始化空缓存，按阶段更新；每个query在缓存+当前token上计算加权注意力（WtdAttention）。

## 6. 论文的主要结论与发现

- **理论上**：Express/Thinformer Express提供了首个同时改进误差衰减率（\(n^{-a}\) vs \(n^{-a/6}\)）、减少对值矩阵依赖（\(\|V\|_{2,\infty}\) vs \(\|V\|_{\text{op}}\)或\(\|V\|_F\)）、降低压缩开销（\(O(d n_{\text{out}}^2 \log n_{\text{out}} \log n)\) 对比 BalanceKV的 \(O(d n n_{\text{out}} \log n)\)）的因果注意力保证。
- **实验上**：
  - 预填充：在512K序列上比FlashAttention 2快82倍（h=9），且所有层替换时困惑度仅增加1.06倍。
  - KV缓存压缩：与SnapKV/StreamingLLM/PyramidKV结合，在维持准确率的情况下大幅降低运行时。
  - 内存约束解码：仅用61%缓存即可达到精确注意力准确率。
  - 计算约束解码：仅用56%运行时即可达到精确注意力准确率，且缓存维护开销远低于查询开销。
