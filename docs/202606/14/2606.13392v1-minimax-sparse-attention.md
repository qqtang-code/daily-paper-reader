## 摘要
超长上下文能力正成为前沿大语言模型必不可少的能力：代理工作流、仓库级代码推理和持久记忆都需要模型共同关注数十万到数百万个token，然而softmax注意力的二次方成本使得这种能力在部署规模上难以维持。我们提出了MiniMax稀疏注意力（MSA），一种基于分组查询注意力（GQA）的块状稀疏注意力机制。一个轻量级的索引分支对键值块进行评分，并为每个GQA组独立选择Top-k子集，从而在保持高效块级执行的同时实现组特定的稀疏检索；主分支则仅对所选块执行精确的块稀疏注意力。MSA设计遵循简单性和可扩展性原则，经过精心精简，使其能够在各种GPU上高效部署。为了将稀疏性转化为实际的速度提升，我们与MSA协同设计了一条GPU执行路径，该路径使用无指数Top-k选择和KV-外部稀疏注意力，以在块粒度访问下提高张量核心利用率。在一个具有原生多模态训练的109B参数模型上，MSA的性能与GQA相当，同时在1M上下文下将每个token的注意力计算量减少了28.4倍。结合我们协同设计的内核，MSA在H800上实现了14.2倍的预填充和7.6倍的解码墙钟时间加速。我们的推理内核可在以下地址获取：https://github.com/MiniMax-AI/MSA。基于MSA的生产级原生多模态模型已在以下地址公开发布：https://huggingface.co/MiniMaxAI/MiniMax-M3。

## Abstract
Ultra-long-context capability is becoming indispensable for frontier LLMs: agentic workflows, repository-scale code reasoning, and persistent memory all require the model to jointly attend over hundreds of thousands to millions of tokens, yet the quadratic cost of softmax attention makes this untenable at deployment scale. We introduce MiniMax Sparse Attention (MSA), a blockwise sparse attention built upon Grouped Query Attention (GQA). A lightweight Index Branch scores key-value blocks and independently selects a Top-k subset for each GQA group, enabling group-specific sparse retrieval while maintaining efficient block-level execution; the Main Branch then performs exact block-sparse attention over only the selected blocks. Designed around a principle of simplicity and scalability, MSA is deliberately streamlined, making it straightforward to deploy efficiently across a broad range of GPUs. To translate sparsity into practical speedups, we co-design MSA with a GPU execution path that uses exp-free Top-k selection and KV-outer sparse attention to improve tensor-core utilization under block-granular access. On a 109B-parameter model with native multimodal training, MSA performs on par with GQA while reducing per-token attention compute by 28.4x at 1M context. Paired with our co-designed kernel, MSA achieves 14.2x prefill and 7.6x decoding wall-clock speedups on H800. Our inference kernel is available at: https://github.com/MiniMax-AI/MSA. A production-grade natively multimodal model powered by MSA has been publicly released at: https://huggingface.co/MiniMaxAI/MiniMax-M3.

# MiniMax Sparse Attention (MSA) 论文详细总结

## 1. 核心问题与研究动机
- **背景**：前沿大语言模型（LLM）需要处理超长上下文（数十万到数百万 token），用于 agent 工作流、仓库级代码推理、持久记忆等场景。
- **核心问题**：标准 softmax 注意力的计算复杂度随序列长度二次增长（O(N²)），导致训练和推理在部署规模下难以承受。
- **目标**：设计一种可扩展、高效的稀疏注意力机制，在不显著牺牲模型质量的前提下大幅降低长上下文下的计算开销，并能在多种 GPU 上高效部署。

## 2. 方法论：核心思想与关键技术
### 2.1 整体架构
- MSA 基于 **Grouped Query Attention (GQA)** 构建，包含两个分支：
  - **索引分支 (Index Branch)**：轻量级，为每个 GQA 组独立评分并选择 Top-k 个 KV 块（块大小为 B_k=128，k=16）。
  - **主分支 (Main Branch)**：仅对所选块执行精确的块稀疏 softmax 注意力。
- 始终包含当前查询所在的局部块，以保证训练稳定性。

### 2.2 索引分支设计
- 引入一个索引查询头（每 GQA 组一个）和单个共享索引键头。
- 每个 token 与块内所有可见 key 的分数取块内最大值作为块分数，然后选择 Top-k 个块。
- 计算量：索引分支 FLOPs ≈ H_kv * d_idx * N²（远小于主分支）。

### 2.3 训练机制
- 索引分支的 Top-k 选择不可微，采用 **KL 对齐损失** 训练索引投影（W_q_idx, W_k_idx）。
- 关键技巧：
  - **梯度分离 (Gradient Detach)**：对索引分支输入执行 stop_gradient，防止 KL 损失影响主干网络。
  - **索引预热 (Indexer Warmup)**：初始阶段使用全注意力训练索引器，几轮后再切换到稀疏模式，避免早期随机选择。
  - **局部块强制包含**：始终保留当前 token 所在块。
- 总损失：L = LLM + λ Σ L_KL。

### 2.4 高效 GPU 内核设计
- **无指数 Top-k 选择**：直接对原始分数排序，跳过 softmax 的 max/exp/sum 步骤。
- **KV-外部循环顺序**：外层遍历 KV 块，内层收集选择该块的 query，提高张量核心利用率。
- **预调度块分块**：对热门 KV 块按 query 维度分块，避免原子操作，使用预分配的槽位写入部分结果。
- **两阶段前向**：先计算局部归一化的部分输出和 log-sum-exp，再合并。
- **查询拼接**：将多个 query 拼接到一起进行矩阵乘法，提高 MMA 维度。

## 6. 主要结论与发现
1. **质量无损**：MSA-PT 和 MSA-CPT 在几乎所有 benchmark 上均与 Full Attention 基线持平或略优（部分多模态/长上下文任务有提升）。
2. **大幅计算节省**：1M 上下文时每 token 注意力 FLOPs 减少 **28.4×**（理论），墙钟加速：prefill **14.2×**，decoding **7.6×**。
3. **索引分支有效性**：MSA 能学习到与全注意力一致的稀疏模式（高块召回率和分数召回率），且不同 GQA 组学到的长程关注模式各不相同（非全局一致稀疏）。
4. **训练稳定性**：梯度分离和索引预热是稳定训练的关键；不加梯度分离会导致梯度尖峰和性能退化。
5. **动态选择优于固定窗口**：与相同预算的滑动窗口基线相比，MSA 在代理任务上困惑度更低。
6. **可学习 sink 不必要**：显式可学习注意力 sink 未带来一致提升，且不消除自然出现的首个 token sink 效应。

## 7. 优点
- **设计简洁**：依赖最少的新增参数（仅两个投影矩阵），基于标准 GQA 框架，易于实现和迁移。
- **训练与推理协同**：训练时使用 KL 损失 + 梯度分离，推理时无指数 Top-k + KV-outer 内核，实现理论加速与实际加速的较好匹配。
- **全流程开源**：已开源推理内核和基于 MSA 的生产级多模态模型（MiniMax-M3）。
- **泛化性验证充分**：在 109B 量级 MoE 模型上验证，且支持从全注意力检查点无损转换（MSA-CPT）。
- **长上下文保持良好**：即使每 query 仅关注 2048 个 token，在 128K 上下文下性能与全注意力接近（RULER/HELMET 差值 ≤0.6%）。
