## 摘要
视觉语言模型（VLM）继承了自回归生成范式，并缓存所有先前 token 的键和值（KV）以加速推理，导致内存消耗随上下文长度线性增长。由于视觉模态中存在大量冗余，这一问题在 VLM 中尤为突出。尽管 KV 缓存驱逐方法可以有效减少推理内存，但它们通常会在 VLM 中导致显著的性能下降，因为大多数方法是为语言模型设计的，忽略了文本与视觉之间的固有差距。通过系统分析 VLM 中的模态差距，本文认为视觉信息的重要性应以文本引导为基础，并相应地提出了一种面向 VLM 的文本引导 KV 驱逐方法（TGV-KV）。TGV-KV 包括三个子模块：（1）文本-视觉预算分配（TVB）基于互信息交互为每一层分配预算。（2）文本加权排序（TWR）评估文本的优先级，并基于加权的文本-图像注意力对视觉重要性进行排序。（3）文本优先保留（TPR）策略战略性保留文本 KV 以避免严重的信息损失。我们在五种不同大小和架构的模型上评估了 TGV-KV，结果显示，在 LLaVA-NeXT 上，TGV-KV 在 VizWiz-VQA 任务中保留了 99.2% 的全 KV 准确率，并在极端保留预算为 5% 的情况下将端到端吞吐量提升了 52.6%。代码可在 https://github.com/Danielement321/TGV-KV 获取。

## Abstract
Vision-Language Models (VLMs) inherit the auto-regressive generation paradigm and cache the keys and values (KV) of all previous tokens to accelerate inference, resulting in memory consumption that scales linearly with context length. This issue is particularly pronounced in VLMs due to substantial redundancy in the visual modality. Although KV cache eviction approaches can effectively reduce inference memory, they often incur significant performance degradation in VLMs, as most are designed for language models and overlook the inherent gap between text and vision. By systematically analyzing the modality gap in VLMs in this work, we argue that the importance of visual information should be grounded in textual guidance and accordingly propose a Text-Grounded KV Eviction method for VLMs (TGV-KV). TGV-KV comprises three submodules: (1) Text-Vision Budgeting (TVB) assigns budget to each layer based on the mutual information interaction. (2) Text-Weighted Ranking (TWR) assesses the priority of text and ranks vision importance based on weighted text-image attention. (3) Text-Prioritised Retention (TPR) policy strategically preserves text KV to avoid acute information loss. We evaluate TGV-KV across five models with different sizes and architectures, showing that TGV-KV preserves 99.2% full-KV accuracy on the VizWiz-VQA task with LLaVA-NeXT and boosts end-to-end throughput by 52.6% with an extreme retention budget of 5%. Code is available at https://github.com/Danielement321/TGV-KV.

# TGV-KV: 面向视觉语言模型的文本引导键值驱逐方法 —— 详细总结

## 1. 核心问题与整体含义

- **研究动机**：视觉语言模型（VLM）在推理时缓存所有先前 token 的 Key-Value（KV）对，导致内存消耗随上下文长度线性增长。高分辨率图像或长视频可能涉及上万 token，使 KV 缓存成为严重瓶颈。
- **关键挑战**：现有 KV 驱逐方法多针对纯语言模型设计，未考虑 VLM 中文本与视觉之间的“模态差距”（Modality Gap）。论文通过实验发现三个关键差异：
  - 视觉 token 高度相似，文本 token 多样性高；
  - 文本-视觉注意力区域很弱，造成注意力分布不均；
  - 累积注意力在文-视交界处剧烈变化，导致驱逐不均衡。
- **整体含义**：本文旨在通过“以文本为锚点”来引导视觉 KV 的驱逐，从而在极端压缩下仍保持高精度，实现高效推理。

## 2. 方法论

### 核心思想
- **文本引导视觉驱逐**：视觉信息的重要性应当根据文本指令来判定，而非与文本同等对待。
- 提出 **TGV-KV** 框架，包含三个协同子模块：

### 关键技术细节

#### (1) 文本-视觉预算分配（TVB）
- 原理：不同层对 KV 保留的需求不同，跨模态注意力（text-vision attention）的强度可作为层预算的信号。
- 实现：对每一层，计算 text-vision 注意力矩阵的求和，再按所有层归一化，得到该层应分配的预算比例。公式：
  ```
  b_l = sum(text-vision attention in layer l) / sum of all layers' text-vision attention
  ```
- 特点：逐样本、在线计算，无需预定义规则或校准集。

#### (2) 文本加权排序（TWR）
- 原理：视觉 token 的重要性应由文本 token 的权重来重新衡量；文本 token 中“主导 token”（dominant text tokens，即被后续大量 token 关注的文本）应赋予更高权重。
- 实现：
  - 对每层，计算 text-text 注意力的列和，再除以位置（考虑因果掩码）得到平均文本重要性权重 `w_l`。
  - 对 text-vision 注意力每一行（对应一个文本 token）乘以归一化的文本权重，然后按列求和，得到每个视觉 token 的最终重要性分数。
  - 文本 token 的重要性直接用 self-attention 列和。
- 公式（视觉重要性）：
  ```
  s_{l,j}^{(V)} = sum_{i=1}^{N_t} w_tilde_{l,i} * [A_l^{(TV)}]_{ij}
  ```

#### (3) 文本优先保留（TPR）
- 原理：文本 token 对性能高度敏感，视觉 token 冗余高；因此优先保留所有文本 KV，仅当预算极低时才驱逐文本。
- 实现：
  - 若预算大于文本 token 数：保留所有文本，再从视觉中按重要性挑选剩余预算。
  - 若预算 ≤ 文本 token 数：仅保留最重要的文本 KV，视觉全部驱逐。
- 公式（略，用条件判断）。

#### 整体流程
1. 预填充阶段计算注意力矩阵。
2. TVB 分配每层预算。
3. TWR 计算所有文本和视觉 KV 的重要性分数。
4. TPR 根据预算和重要性决定保留哪些 KV。

## 6. 主要结论与发现

1. **TGV-KV 在极端压缩下仍能保持接近全 KV 的性能**：例如在 LLaVA-NeXT 上，5% 保留率时 VizWiz 保留 99.2% 准确率；在 Qwen3-VL-8B 上 DocVQA 保留 92.5% 性能。
2. **效率提升显著**：16k 上下文时 5% 保留率使吞吐量提升 52.6%，KV 内存从 7.81 GB 降至 0.39 GB。
3. **文本优先保留至关重要**：驱逐少量文本 token 会导致性能崩溃，而视觉 token 可大量压缩。
4. **跨模态注意力（text-vision）是比单模态注意力更好的重要性指标**，结合文本加权后效果最佳。
5. **动态层预算分配优于均匀分配**，text-vision 注意力的强度可反映层需求。
