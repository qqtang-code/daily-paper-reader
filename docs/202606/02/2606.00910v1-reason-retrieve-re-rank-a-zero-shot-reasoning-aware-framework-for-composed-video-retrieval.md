## 摘要
组合视频检索（CoVR）旨在找到对参考视频应用自由形式的文本修改后得到的目标视频。我们针对CVPR 2026 VidLLMs研讨会的“推理感知”CoVR（CoVR-R）挑战，其中检索严格为零样本。我们提出R3-CoVR（推理、检索、重排序），一个完全由冻结基础模型构建的无需训练的流水线。多模态大语言模型（Qwen3-VL-8B）推理编辑所隐含的“后效”——状态转换、动作阶段、场景、镜头和节奏——并生成简洁的编辑后描述；对比式视频-文本编码器（SigLIP-2）对该描述和图库进行嵌入，用于第一阶段检索；最后，约束感知重排序阶段使用相同的多模态模型作为评判者，对每个候选视频根据预期的编辑结果进行评分。在挑战测试集上，R3-CoVR达到91.9%的R@1和98.2%的R@10。两个发现驱动了这些结果：（i）将描述长度与对比编码器的文本窗口匹配，使R@1从67.5提升到72.7；（ii）约束感知重排序器仅对候选列表重新排序，使R@1从72.7提升到91.9——这是最大的单一提升。我们分析了重排序器的行为、检索/重排序混合以及候选列表深度，并发布了一个简洁的三层实现。

## Abstract
Composed Video Retrieval (CoVR) seeks the target video that results from applying a free-form textual modification to a reference video. We address the \emph{Reason-Aware} CoVR (CoVR-R) challenge at the CVPR~2026 VidLLMs workshop, where retrieval is strictly zero-shot. We present \textbf{R3-CoVR} (\emph{Reason, Retrieve, Re-rank}), a training-free pipeline built entirely from frozen foundation models. A multimodal large language model (Qwen3-VL-8B) reasons about the \emph{after-effects} an edit implies -- state transitions, action phases, scene, camera and tempo -- and verbalises a concise post-edit description; a contrastive video--text encoder (SigLIP-2) embeds this description and the gallery for first-stage retrieval; finally a constraint-aware re-ranking stage uses the same multimodal model as a judge that scores each shortlisted candidate against the intended edited result. On the challenge test set, R3-CoVR attains \textbf{91.9\% R@1} and \textbf{98.2\% R@10}. Two findings drive these results: (i)~matching the description length to the contrastive encoder's text window lifts \Rk{1} from $67.5$ to $72.7$; and (ii)~the constraint-aware re-ranker, which reorders only the shortlist, lifts \Rk{1} from $72.7$ to $91.9$ -- the single largest gain. We analyse the re-ranker's behaviour, the retrieve/re-rank blend, and the shortlist depth, and we release a clean three-layer implementation.

#### 2. 方法论
- **核心思想**：完全避免训练，利用冻结的多模态大语言模型（MLLM）和对比编码器，通过三个阶段模拟人类“推理-初步匹配-精筛”的过程。
- **关键技术细节**：
  - **Stage 1 – 推理**：冻结的 Qwen3-VL-8B 观察参考视频（1 fps 采样），被提示输出结构化后效追踪（状态、动作、场景、镜头、节奏），并生成一个**简洁的后编辑描述**（≤40 词，适配编码器窗口）。采用 K=3 次自一致性采样并平均嵌入以稳定化。
  - **Stage 2 – 检索**：冻结的 SigLIP-2 对比编码器将描述嵌入为查询向量，图库视频通过 8 帧均匀采样、均值池化和 L2 归一化得到向量。余弦相似度排序，返回 Top-K 短列表（默认 K=10）。关键发现：描述长度必须匹配编码器的 64 token 窗口（简洁描述比丰富描述提升 +5.2 R@1）。
  - **Stage 3 – 重排序**：同一 MLLM 作为约束评判者，对每个短列表候选视频给出 [0,100] 的整数匹配分数（经 5 次采样平均去噪）。最终分数融合评判分和检索排名分：\(s_{\text{final}}(V) = \lambda s_j(V) + (1-\lambda)\text{rank}_K(V)\)，其中 \(\text{rank}_K(V) = (K-i)/K\)，默认 \(\lambda=0.8\)。重排序仅作用于短列表，控制计算成本。
- **算法流程**：参考视频输入 → 推理生成描述 → 对比检索得到短列表 → 评判打分 → 混合排序输出最终排名。

#### 6. 主要结论与发现
- R3-CoVR 在挑战测试集上达到 **R@1=91.9%，R@10=98.2%**。
- **最大贡献因素**：重排序阶段带来 +19.2 R@1（从 72.7 提升至 91.9）是单一最大增益。
- **描述长度匹配**：将描述限制在 ≤40 词以适配 SigLIP-2 的 64 token 窗口，带来 +5.2 R@1 提升。
- 混合权重 λ=0.8 时最佳，纯评判者 (λ=1) R@1=88.2，纯检索 (λ=0) R@1=72.7，说明结合两者优于任一单独使用。
- 更深短列表（top-20）提升宽泛召回（R@10 升至 98.2），但对 R@1 影响微弱。
- 评判者 5 次采样去噪显著提高得分分辨率，消除排名平局对 R@1 的负面影响。

#### 7. 优点
- **零样本且无需训练**：全冻结模型，避免过拟合，易于迁移至新图库。
- **模块化架构**：三层（数据/模型/运行）实现，可互换骨干（如更强的编码器或更大的 LLM）。
- **解释性强**：Stage 1 的可读推理追踪和 Stage 3 的评判理由可直接用于分析。
- **高效实用**：一次性图库嵌入 + 短列表重排序，在单 GPU 上快速完成。
- **性能显著**：在挑战测试集上接近饱和（R@10 达 98.2%），R@1 远超参考基线。

#### 8. 不足与局限
- **重排序成本随深度线性增长**：K 增大时，每次查询的评判次数增加，较大图库可能需更廉价的首轮过滤或级联评判。
- **评判者继承骨干偏见**：8B 参数的 Qwen3-VL 对细粒度区分不足（少量错误案例中 8B 模型给出相同评分），更大模型（如 72B）可能直接弥合 R@1 与 R@10 的差距。
- **对比编码器文本窗口限制**：64 token 的窗口约束了推理阶段生成的有用信息传递至检索阶段，长上下文或视频原生检索器可望进一步提升短列表召回。
- **计算开销**：自一致性（3×生成）和去噪（5×评分）以推理时间换取稳定性，大规模部署需权衡。
- **外部对比不足**：未在统一基准下与更多现有零样本方法（如 CIReVL、CoTMR 的视频扩展）进行公平比较，实验结果的泛化性有待更多验证。
