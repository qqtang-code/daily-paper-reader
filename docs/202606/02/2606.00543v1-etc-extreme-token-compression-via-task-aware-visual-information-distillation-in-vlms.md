## 摘要
在视觉语言模型中，高分辨率图像会产生大量视觉token，导致推理过程中计算成本高和KV-cache开销大。为解决此问题，我们提出了一种极端Token压缩框架，该框架基于变分信息蒸馏原理，在减少输入token数量时最小化任务损失。具体而言，从信息论角度，我们表明最小化任务损失需要紧凑表示保留用于预测的任务相关视觉信息的指令感知充分统计量。在实践中，ETC利用文本到图像的交叉注意力对原始视觉特征进行加权，以近似潜在的指令感知预测统计量。此外，ETC引入了变分信息蒸馏，使紧凑表示能够保留恢复该预测统计量所需的关键信息。在LLaVA-1.5-7B和Qwen3-VL-2B上的实验表明，即使在单token压缩下，ETC也能保持有效性，大幅降低KV-cache开销同时保留强大的任务性能。

## Abstract
In Vision-Language Models (VLMs), high-resolution images produce a large number of visual tokens, resulting in high computational costs and KV-cache overhead during inference. To address this problem, we propose an Extreme Token Compression (ETC) framework that minimizes task loss when reducing the number of input tokens based on the principle of variational information distillation. Specifically, from an information-theoretic perspective, we show that minimizing task loss requires the compact representation to preserve the instruction-aware sufficient statistic of the task-relevant visual information for prediction. In practice, ETC leverages text-to-image cross-attention to weight the original visual features to approximate the latent instruction-aware predictive statistic. Moreover, ETC introduces a variational information distillation, enabling the compact representation to preserve the essential information to recover this predictive statistic. Experiments on LLaVA-1.5-7B and Qwen3-VL-2B show that ETC remains effective even under single-token compression, substantially reducing KV-cache overhead while retaining strong task performance.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：视觉语言模型（VLM，如LLaVA-1.5、Qwen3-VL）在推理时，高分辨率图像会产生大量视觉token（例如LLaVA-1.5中一张336×336图像生成576个token，Qwen3-VL可超10,000个）。Transformer的自注意力机制计算复杂度与序列长度成二次增长，导致高昂的计算成本和KV-cache存储开销。
- **现有方法局限**：现有视觉token压缩方法分为选择性压缩（如SparseVLM、TopV）和抽象性压缩（如MQT-LLaVA、VoCo-LLaMA）。选择性方法在极端压缩下出现“压缩悬崖”，性能骤降；抽象性方法仅通过下一token预测（NTP）损失间接监督压缩，缺乏对任务相关视觉信息的直接保留，容易过度依赖语言先验。
- **本文目标**：从任务损失最小化的信息论角度出发，提出一种极端Token压缩框架（ETC），在极少量token（甚至1个）下仍能保持强大任务性能，同时大幅降低KV-cache和计算开销。

## 2. 方法论

### 2.1 核心思想
- 基于信息论推导出**任务损失最小化的充要条件**：压缩表示Z应成为原始视觉V关于任务Y的**指令感知充分统计量**，即条件独立性 Y ⊥ V | (Z, T) 成立（T为文本指令）。等价于要求Z保留一个潜在预测统计量 X* = p(Y|V, T)。
- 实践中，X*无法直接获取，因此通过**文本-图像交叉注意力**构造一个近似代理统计量 \(\hat{X}\)，再通过**变分信息蒸馏（VID）** 损失促使Z保留恢复\(\hat{X}\)所需的信息。

### 2.2 关键技术细节

#### 2.2.1 指令感知预测统计量近似
- 利用解码器VLM中文本token对视觉token的交叉注意力分数，计算每个视觉token的平均注意力得分。
- 对得分进行min-max归一化，并与原始视觉特征进行加权融合：\(\hat{X} = X_v \odot (1 - \alpha + \alpha \tilde{S})\)，其中\(\alpha\)控制指令感知权重。
- 该操作定位任务相关的视觉区域，使压缩表示聚焦于指令导向的信息。

#### 2.2.2 变分信息蒸馏（VID）
- 目标是最小化条件熵 H(\(\hat{X}\) | Z, T)，等价于最大化互信息 I(\(\hat{X}\); Z | T)。
- 引入高斯变分分布 q(\(\hat{X}\)|Z, T)，其均值 μ(Z, T) 由MLP解码器从压缩token Z映射回原始视觉特征空间，方差为可学习的同方差参数。
- 最终蒸馏损失：\(L_{ETC} = \frac{1}{B} \sum_b \sum_d \left( \frac{1}{2\sigma_d^2} \sum_n (\hat{X}_{b,n,d} - \mu_{b,n,d})^2 + N \log \sigma_d \right)\)。

#### 2.2.3 压缩架构与训练
- 在LLM输入序列中插入M个可学习压缩token Z（M ≪ Nv），序列为 [V; Z; T]。
- 采用瓶颈注意力掩码：Z可关注V，T只能关注Z，从而将视觉信息蒸馏到Z中。
- 总损失：\(L = L_{CE} + \lambda L_{ETC}\)，其中LCE为自回归交叉熵任务损失。
- 训练后，推理阶段在预填充时将视觉信息压缩到Z中，随后丢弃原始V的KV-cache，解码仅使用Z和T。
