## 摘要
统一多模态建模旨在将视觉理解和生成集成到单一系统中。然而，现有方法通常依赖两个不同的视觉分词器，这分割了表示空间，阻碍了真正的统一建模。我们提出UniAR，一个统一的自回归框架，其中单个离散视觉分词器充当理解与生成之间的关键桥梁，实现共享上下文，使模型能够直接解释自身生成的视觉令牌，无需额外重新编码。UniAR采用预训练的视觉编码器，结合多级特征融合和无查找按位量化方案，在保留高级语义和低级细节的同时，以最小代价扩展有效视觉词汇。基于此，统一自回归模型采用并行按位预测，共同预测空间分组的多级视觉编码，大幅减少视觉序列长度并加速生成。最后，基于扩散的视觉解码器对离散视觉令牌进行操作，解码出高保真图像。通过大规模预训练，随后进行监督微调和强化学习，UniAR在图像生成和图像编辑上达到最先进性能，同时在多模态理解基准上保持竞争力。项目页面：https://sharelab-sii.github.io/uniar-web.

## Abstract
Unified Multimodal Modeling aims to integrate visual understanding and generation within a single system. However, existing approaches typically rely on two disparate visual tokenizers, which splits the representation space and hinders truly unified modeling. We propose UniAR, a unified autoregressive framework where a single discrete visual tokenizer serves as the key bridge between understanding and generation, enabling a shared context in which the model can directly interpret its own generated visual tokens without additional re-encoding. UniAR adapts a pretrained vision encoder with multi-level feature fusion and a lookup-free bitwise quantization scheme, preserving both high-level semantics and low-level details while scaling the effective visual vocabulary at minimal cost. Building on this, the unified autoregressive model adopts parallel-bitwise-prediction to jointly predict spatially grouped, multi-level visual codes, substantially reducing visual sequence length and accelerating generation. Finally, a diffusion-based visual decoder operates on discrete visual tokens to decode high-fidelity images. Through large-scale pre-training, followed by supervised fine-tuning and reinforcement learning, UniAR achieves state-of-the-art performance on image generation and image editing while remaining competitive on multimodal understanding benchmarks. The project page is available at https://sharelab-sii.github.io/uniar-web.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 采用**单个离散视觉分词器**统一视觉理解和生成表示，基于预训练的多模态大模型视觉编码器，通过多级特征融合和免查找按位量化（BSQ）得到离散语义令牌，并利用并行按位预测和扩散视觉解码器实现高效的图像生成。

### 关键技术细节

#### (1) 统一视觉分词器 (Unified Visual Tokenizer)
- **多级特征融合（Multi-level Feature Fusion）**：从 ViT 浅层（低层细节）和深层（高层语义）聚合特征，同时满足生成和理解需求。
- **免查找按位量化（Lookup-free Bitwise Quantization, BSQ）**：将视觉特征映射为 64 位二进制向量（u ∈ {0,1}^64），理论词汇量达 2^64，避免显式码表，以极小代价扩展有效视觉词汇。
- **损失函数**：L = L_CE + λ_BSQ · L_BSQ，其中 L_CE 为原 LMM 的交叉熵损失，L_BSQ 为 BSQ 软熵损失。

#### (2) 统一自回归模型 (Unified Auto-Regressive Model)
- **并行按位预测（Parallel Bitwise Prediction）**：对每个 2×2 空间网格中的多级 BSQ 索引同时预测，实现 32× 视觉压缩比（512×512 图像仅需 256 个令牌）。
- **视觉预测头**：RMSNorm + 线性层，输出维度 d_vis = 2 × d_BSQ × g（g = n_level × n_spatial）。
- **训练损失**：L_AR = L_text + λ_vis · L_vis。
- **随机视觉索引翻转（Random Visual Index Flipping）**：在预训练时随机翻转部分比特，模拟自回归推理时的累积误差，提高生成稳定性。

#### (3) 视觉解码器 (Visual Decoder)
- 基于预训练 DiT（Stable Diffusion 3.5 Medium），接收视觉令牌条件信号 f_v（通过拼接多级特征并投影得到），与噪声隐藏状态逐元素相加，采用条件流匹配（CFM）训练。
- 可选分辨率上采样（2D 双三次插值），使 1024×1024 图像只需预测 256 个令牌。
- 解码器**不接收文本提示**，仅依赖视觉令牌，语义与布局生成全部由自回归模型负责。

#### 训练流程（三阶段）
1. **预训练（Pre-Training）**：约 1T tokens，8K 上下文（800B tokens）→ 32K 上下文（200B tokens），理解与生成数据 1:1，冻结视觉分词器和解码器。
2. **监督微调（SFT）**：约 50B tokens，ChatML 格式，使用公共合成数据。
3. **强化学习（RL）**：仅针对图像生成，使用 GRPO 算法，奖励包括 HPSv2、UnifiedReward、PaddleOCR 编辑距离、基于检测器的指令跟随奖励，两阶段训练（512px 500 steps → 960px 100 steps）。

## 6. 主要结论与发现

1. **共享视觉分词器是实现真正统一多模态建模的关键**：通过单个离散分词器将理解与生成放在同一表示空间，使模型能够直接解读自身生成的视觉内容，无需额外编码。
2. **UniAR 在图像生成上达到 SOTA**：在 GenEval 上总体得分 0.86（带重写），高于 GPT-4o 和 FLUX.1-dev；在文本渲染上优于 GPT-4o 和 Gemini 2.5 Flash Image；图像编辑上竞争力强（3.73 vs 3.71 FLUX.1 Kontext）。
3. **多模态理解保持竞争力**：在 OCR 类任务（OCRBench 833、DocVQA 91.4）和视频理解（MVBench 62.3）上显著优于 Emu3 等早期统一模型，与 LLaVA-OV 相当。
4. **并行按位预测有效加速**：结合解码器上采样，只需预测极少的视觉令牌（256），大幅提升推理速度。
5. **强化学习可有效提升生成质量**：在文本渲染上从 SFT 的 71.1 提升至 RL 后的 87.3。
6. **涌现出多轮理解-生成交互能力**：模型虽未训练，但可回答关于自身生成图像的细节问题。

## 7. 优点（方法或实验设计亮点）

1. **创新性设计**：

3. **实验全面且结果亮眼**：

5. **涌现能力**：展示了模型在没有专门训练的情况下能够进行 interleaved generation-understanding，体现了统一表示空间的优势。

## 8. 不足与局限

3. **资源限制**：未缩放模型参数和数据集规模（论文提到由于资源约束），未来可探索更大规模训练。
4. **实验覆盖局限**：
   - 未评估视频生成（尽管框架理论上可扩展）。
   - 图像编辑仅在一个基准（ImgEdit-Bench）上评估，可能不够全面。
   - 文本渲染仅评估英文子集，未覆盖中文等。
5. **偏差风险**：训练数据主要来源于公开合成数据和重合成数据，可能存在分布偏差；奖励模型（如 HPSv2）可能偏向特定审美。
6. **应用限制**：面向学术研究，未讨论部署效率、安全性、内容过滤等实际应用问题。
