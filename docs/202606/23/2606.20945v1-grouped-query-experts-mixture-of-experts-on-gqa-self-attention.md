## 摘要
自注意力是Transformer性能的核心，在长上下文长度下通常是Transformer中最昂贵的部分，因为其成对token交互的计算量随序列长度呈二次增长。标准的密集注意力对所有token应用相同的注意力头，无论token的难度或信息内容如何。这种均匀激活会浪费计算资源，尤其是当序列变长且注意力成本迅速增加时。我们提出分组查询专家（GQE），这是在分组查询注意力（GQA）之上的一种混合专家层。在每个GQA组内，一个路由器为每个token选择k个查询专家头，而所有键值（KV）头保持密集且不变。因此，GQE保留了GQA的KV缓存优势，仅减少了活跃的查询头计算。在250M参数规模下，固定30B token预算时，GQE在下游任务准确率上与全活跃的GQA基线持平，同时每个token仅激活一半的查询头。

## Abstract
Self-attention is central to Transformer performance and is often the most expensive part of the Transformer at long context lengths because its pairwise token interactions scale quadratically with sequence length. Standard dense attention also applies the same set of attention heads to every token regardless of token difficulty or information content. This uniform activation can waste compute, especially as sequences grow longer and attention cost increases rapidly. We propose Grouped Query Experts (GQE), a mixture-of-experts layer on top of grouped-query attention (GQA). Within each GQA group, a router selects k query-head experts per token while all key-value (KV) heads remain dense and unchanged. Thus, GQE keeps the KV cache benefits of GQA and reduces only the active query-head computation. On a fixed 30B token budget at the 250M parameter scale, GQE matches the all-active GQA baseline in downstream accuracy while activating half the query heads per token.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：Transformer 中的自注意力机制对每个 token 都使用相同数量的注意力头（全密集），这对于不同信息量的 token（如停用词 vs. 内容词）造成了计算浪费；同时，随着序列长度增长，自注意力的计算成本呈二次增长，成为长上下文场景的主要瓶颈。
- **背景**：分组查询注意力（GQA）通过让多个查询头共享一个 KV 头来减少 KV 缓存，但所有查询头仍然全部激活。已有的混合专家（MoE）方法大多应用于前馈网络（FFN），少数应用于注意力头，但要么改变 KV 缓存，要么对全部头进行路由。
- **整体含义**：本文提出将 MoE 思想应用于 GQA 的查询头，实现条件计算：每个 token 只激活需要的查询专家头，从而在保持模型质量的同时降低自注意力计算量，尤其有利于长序列。

## 2. 提出的方法论
### 核心思想
- 在 GQA 每个组内，将查询头看作一组专家（expert），路由器为每个 token 选择 top-k 个专家激活，而该组的 KV 头保持密集不变。
- 引入一个始终激活的共享头（shared head）作为稳定锚点，以及一个重归一化的加权和槽（weighted-sum slot）为路由器提供可微分的学习信号。

### 关键技术细节
- **分组专家设置**：总共有 N 个查询头专家，分为 G 组（G = KV 头数），每组 M = N/G 个专家。每个 token 在每组内选择 k 个专家激活，因此每个 token 激活的查询专家总数为 kG。
- **路由过程**：
  - 对每个 token 和每个组，路由器输出 softmax 概率 p_{i,g}（维度 M），然后选择 top-k 概率对应的专家。
  - 选中的专家输出进行硬拼接（hard concatenation），不包含概率加权。
  - 额外构造一个重归一化的加权和输出：将各组选中的专家输出按概率加权并求和，权重为将该组内概率在所有选中的专家上重新归一化。
  - 最终输出由三部分组成：硬拼接的选中专家输出、加权和输出、共享头输出，再通过输出投影矩阵 W^O 变换。
- **共享头**：始终激活的额外注意力头，提供稳定的基线。
- **辅助损失**：采用负载均衡损失防止路由器坍缩到个别专家。
- **计算量**：以 16 查询头 / 8 KV 头为例，k=1 时每 token 激活 8 个路由专家 + 1 个共享头 = 9/16 ≈ 56% 的查询头计算量。

## 6. 主要结论与发现
- GQE（最终变体）在 30B token 训练后，平均下游准确率（56.04）与全活跃 GQA baseline（55.86）相当（甚至略高），而仅激活 56% 的查询头计算量。
- 稀疏路由成功的关键在于：(i) 路由器需要通过重归一化加权和槽获得正确的学习信号；(ii) 始终激活的共享头为训练提供稳定锚点。缺少任一条件会导致准确率下降。
- 在长序列（4k tokens 以上）上，GQE 实现约 1.67–1.80× 的预填充速度提升，来源于减少的查询头计算。

## 7. 优点
- **简单有效**：直接在 GQA 上添加路由层，不改变 KV 缓存结构，保留 GQA 的 KV 缓存优势。
- **计算成本低**：仅减少活跃查询头计算，不增加额外部署难度（如稀疏 KV 缓存带来的硬件不友好）。
- **消融清晰**：通过逐步添加重归一化槽和共享头，明确证明了设计组件的必要性。
- **吞吐量验证**：在长序列场景下展示了实际的速度提升，符合理论预期。

## 8. 不足与局限
- **规模不足**：仅 250M 参数、30B token 训练，大模型（如 1B、7B 以上）上的效果未验证，结论可能无法直接迁移。
