## 摘要
尽管扩散变换器（DiTs）已经革新了高保真视频生成，但其对3D全注意力的依赖造成了二次计算瓶颈。现有的稀疏方法面临两难：动态剪枝导致高昂的运行时开销和内存碎片化，而静态启发式则无法捕捉细粒度依赖。在本文中，我们提出ScalingAttention，一种无需训练且基于关键归纳偏好的框架：虽然单个激活值依赖于输入，但每个注意力头的高质量区域会迅速收敛到一个稳定、与提示无关的内在稀疏拓扑。该拓扑由权重编码、尺度不变且易于提取。ScalingAttention通过以下方式将拓扑发现与稀疏度控制解耦：（1）WEST（权重编码稀疏拓扑），离线提取鲁棒的块稀疏先验掩码以消除运行时搜索；（2）FAST（保真度感知灵敏度调优），根据扩散保真度需求自适应调整每头的稀疏度。为确保实际加速，我们共同设计了硬件对齐的逐位块稀疏内核。在Wan2.1上的实验显示，端到端加速高达1.90倍，且保真度更优，在现有最先进基线之上建立了新的帕累托前沿。

## Abstract
While Diffusion Transformers (DiTs) have revolutionized high-fidelity video generation, their reliance on 3D full attention creates a quadratic computational bottleneck. Existing sparse methods face a dilemma: dynamic pruning suffers from prohibitive runtime overhead and memory fragmentation, while static heuristics fail to capture fine-grained dependencies. In this work, we propose ScalingAttention, a training-free framework grounded in a key inductive bias: while individual activations are input-dependent, the high-mass attention regions for each head rapidly converge to a stable, prompt-agnostic Intrinsic Sparse Topology. This topology is weight-encoded, scale-invariant, and efficient to extract. ScalingAttention decouples topology discovery from sparsity control via: (1) WEST (Weight-Encoded Sparse Topology), which extracts a robust block-sparse prior mask offline to eliminate runtime search; (2) FAST (Fidelity-Aware Sensitivity Tuning), which adaptively tunes head-wise sparsity based on diffusion fidelity requirements. To ensure practical acceleration, we co-design a hardware-aligned bit-wise block-sparse kernel. Experiments on Wan2.1 show up to 1.90X end-to-end speedup with superior fidelity, establishing a new Pareto frontier over state-of-the-art baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：扩散变换器（DiTs）在高保真视频生成领域取得了革命性成功，但其依赖的3D全注意力机制导致计算复杂度为O(N²)，成为严重性能瓶颈。例如，使用Wan2.1-14B模型生成5秒81帧视频需近30分钟，其中超过半数时间消耗在3D注意力上。
- **核心困境**：现有稀疏注意力方法面临两难：
  - 动态方法（运行时选择重要token）虽准确，但产生高昂计算开销和内存碎片，难以利用GPU Tensor Core；
  - 静态方法（离线固定稀疏模式）虽硬件友好，但使用启发式规则，无法捕捉视频注意力中内容自适应的结构。
- **关键发现**：作者通过实证观察到，视频DiT中每个注意力头的高质量区域（high-mass attention regions）在不同输入下迅速收敛到一个稳定、与提示无关的结构——**内在稀疏拓扑（Intrinsic Sparse Topology）**。该拓扑由权重编码、尺度不变，且可直接从预训练参数中提取。
- **研究含义**：这一发现将稀疏注意力问题分解为两个正交子问题：（1）*何处*允许注意力发生（拓扑）；（2）*多少*稀疏性可被容忍（敏感度）。解耦这两者才能实现高效且保真的加速。

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架：ScalingAttention（无需训练）
由三个紧密协同的组件构成，实现从拓扑发现到高效执行的闭环。

### 2.2 WEST：权重编码稀疏拓扑（Weight-Encoded Sparse Topology）
- **目标**：离线提取一个鲁棒的块稀疏先验掩码，消除运行时搜索。
- **步骤**：
  1. **块级拓扑提取**：将每个头的注意力矩阵Aᵢ划分为B×B块（B=128），通过块内聚合得到块显著性矩阵Sᵢ。
  2. **阈值映射构建**：定义阈值映射T(u,v)，记录每个块在所有校准提示的联合活跃区域中所需的最小稀疏度阈值。该映射编码了完整的稀疏层次。
  3. **推理时使用**：对于任意目标密度ρ，通过轻量比较M_ρ = I(T ≤ ρ)即可即时生成二进制掩码，实现O(1)掩码生成。
- **关键属性**：该拓扑是尺度不变的（低分辨率提取的掩码在高分辨率下Recall>96%），且对纵横比变化鲁棒。

### 2.3 FAST：保真度感知稀疏度调优（Fidelity-Aware Sensitivity Tuning）
- **目标**：根据扩散过程中不同阶段（时间步、层）的保真度需求，自适应调整每头的稀疏度。
- **关键度量**：采用**Hellinger距离**作为保真度指标（优于余弦相似度，避免过早饱和），定义保真度分数F(P, \tilde{P}) = 1 - H(P, \tilde{P})。
- **时空保真度分配函数**：A(l, t) = 1 - [β(1 - (l+ωt)/(N_L+ωN_T))^γ + (1-β)]，根据层l和时间步t动态确定可接受的保真度阈值。
- **高效处理**：采用**分层采样**（仅采样约0.05%的行）即可准确估计Hellinger距离，大幅降低离线分析成本。
- **流程**：离线构建每头的保真度-稀疏度曲线 → 推理前根据全局密度目标查找最优阈值p* → 生成静态掩码。

### 2.4 CRM内核：硬件对齐的逐位块稀疏执行
- **问题**：传统索引格式（如CSR）或稠密布尔掩码带来元数据开销和内存碎片。
- **方案**：采用**压缩行掩码（CRM）** 表示，每行用对齐的uint32数组存储位掩码（每位表示一个KV块是否激活）。利用GPU原生位操作（如bit scan）实现常量时间推进，无缝替代FlashAttention中的稠密块迭代器。
- **优势**：即使序列长度达256k，每行仅需64次掩码加载，开销低于10%；随稀疏度增加，延迟线性降低（50%稀疏度时降低40-60%）。

## 6. 主要结论与发现

- **内在稀疏拓扑**：视频DiT中每个注意力头存在一个权重编码、尺度不变、与提示无关的稳定稀疏拓扑，这是模型内在结构约束的体现。
- **ScalingAttention实现1.90倍端到端加速**（Wan2.1-14B），同时保持高保真度（PSNR 24.15 dB），在所有对比方法中建立新帕累托前沿。
- **核心解耦设计**：将稀疏注意力分解为拓扑发现（WEST）与敏感性调制（FAST），避免了动态方法的运行时开销和静态方法的精度损失。
- **硬件对齐**：CRM内核将理论FLOPs减少转化为实际加速，在50%稀疏度下延迟降低40-60%。

## 7. 优点

1. **新颖的理论洞察**：首次揭示视频DiT注意力中存在与提示无关的内在稀疏拓扑，为无训练加速提供理论基础。
2. **高效的解耦设计**：WEST离线提取拓扑，FAST自适应控制稀疏度，两者正交且协同，使方法无需训练即可应用。
3. **硬件-算法协同优化**：CRM内核利用位掩码和GPU原生操作，实现低开销、可扩展的块稀疏执行。
4. **尺度不变性与鲁棒性**：低分辨率提取的拓扑可复用于高分辨率，且对不同纵横比稳定，实用性强。
5. **与正交技术兼容**：能与蒸馏、token重排序等方法结合，进一步提升性能。
6. **全面的实验验证**：在三个代表性大模型上覆盖效率、质量、消融、敏感度等多维度评估。

## 8. 不足与局限

1. **粒度瓶颈**：为最大化Tensor Core利用率，采用128×128块大小，限制了细粒度剪枝能力。在极端稀疏度（<40%）下性能下降明显。
2. **静态包络冗余**：WEST提取所有活跃区域的并集，相对于动态方法会保留更多token（尤其是局部运动场景），造成一定冗余。未来可通过token排列机制缓解。
3. **基线覆盖不足**：未与更多近期稀疏注意力方法（如AdaSpa、Radial、SpargeAttention）对比，评估全面性略有瑕疵。
4. **校准依赖**：需要通过少量提示（N_p=27）离线预计算，虽然成本可分摊，但对模型首次部署仍有一定门槛。
5. **硬件局限**：crm kernel依赖GPU位操作，在CPU或其他加速器上移植性可能受限。
6. **保真度度量依赖**：FAST基于Hellinger距离，虽然理论合理，但可能无法完全反映人类感知质量，需与下游感知指标联合验证。
