## 摘要
多模态大语言模型（MLLMs）实现了强大的视觉-语言推理，但长视觉上下文会扩大KV缓存并增加解码延迟。现有的压缩方法依赖于观察窗口注意力来进行稳定的令牌重要性估计，但这种聚合可能会稀释稀疏的视觉证据，并在激进压缩下丢弃关键答案令牌。因此，我们识别出最后查询注意力作为恢复此类证据的补充来源，但其与答案无关的信号可能会误导保留。我们提出了BACON，一种即插即用方法，通过最后查询证据校准观察窗口注意力，并通过层内一致性和层间持续性抑制孤立噪声。在不同的基准、模型、预算和压缩方法下，BACON在最激进的预算下平均提升了7.5%的多模态KV压缩性能，最高提升达30.9%。我们的项目页面可在https://ryu1ion.github.io/official_BACON/找到。

## Abstract
Multimodal Large Language Models (MLLMs) achieve strong vision-language reasoning, but long visual contexts enlarge the KV cache and increase decoding latency. Existing compression methods rely on observation window attention for stable token-importance estimation, yet this aggregation can dilute sparse visual evidence and discard answer-critical tokens under aggressive compression. Therefore, we identify last-query attention as a complementary source for recovering such evidence, but its answer-irrelevant signals can mislead retention. We propose BACON, a plug-and-play method that calibrates observation window attention with last-query evidence and suppresses isolated noise via intra-layer coherence and inter-layer persistence. Across diverse benchmarks, models, budgets, and compression methods, BACON improves multimodal KV compression by 7.5% on average under the most aggressive budget, with gains up to 30.9%. Our project page is available at https://ryu1ion.github.io/official_BACON/

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：多模态大语言模型（MLLMs）在处理长视觉上下文时，KV 缓存急剧膨胀，导致解码延迟显著增加。现有 KV 缓存压缩方法普遍依赖“观察窗口注意力”（observation window attention）来估计 token 重要性（即取最后若干 prompt 查询的平均注意力）。但这种聚合操作会稀释稀疏的视觉证据，特别是在激进压缩（低预算）下，容易丢弃与答案高度相关的关键视觉 token，造成性能严重下降。
- **动机**：研究者发现“最后查询注意力”（last-query attention）比观察窗口注意力更敏感地捕捉到边界出现的视觉证据（boundary-emergent evidence），但直接使用 final query 会引入约 70% 的与答案无关的高注意力噪声。因此，需要一种校准机制，既利用 last-query 补充证据，又抑制其噪声。
- **贡献**：提出 **BACON**（Boundary Attention CalibratiON），一种即插即用的 token 保留校准方法，在不改变原有压缩流程的前提下，显著提升多模态 KV 缓存压缩效果。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：保持观察窗口注意力作为稳定的基础得分，用最后查询注意力中正向的“边界残差”来校准，并通过**层内一致性**（intra-layer coherence）和**层间持续性**（inter-layer persistence）过滤噪声，最终输出校准后的保留得分。
- **关键步骤**（文字描述）：
    1. **计算基础得分**：对每个 layer-head 对，计算观察窗口平均注意力 \(B^l_h_i\)。
    2. **计算边界残差**：取最后 query 的注意力 \(Q^l_h_i\)，定义残差 \(E^l_h_i = \max(Q^l_h_i - B^l_h_i, 0)\)，表示 last-query 比窗口更关注的 tokens。
    3. **结构校准**：
        - **层内一致性**：对每个 token i，取其半径 r=5 的邻域内的残差平均值，得到 \(L^l_h_i\)。
        - **层间持续性**：对当前层 l，向前取 m=4 层中同位置残差的平均值，得到 \(T^l_h_i\)。
        - 组合：\(V^l_h_i = E^l_h_i + L^l_h_i + T^l_h_i\)。
    4. **方差匹配**：通过求解一个最小二乘问题获得校准系数 \(\lambda\)，使 \(\lambda V\) 的方差与窗口得分 \(B\) 的局部方差对齐（控制校准强度 \(\gamma\)）。
    5. **最终得分**：\(S^l_h_i = B^l_h_i + \lambda V^l_h_i\)，然后按 \(S\) 排序保留 Top-K tokens。
- **流程**：BACON 仅在预填充（prefill）阶段计算，不改变解码过程，因此不增加推理延迟。

- **主要结论**：BACON 在几乎所有模型、基准和极端压缩预算下，一致地提升了现有 KV 缓存压缩方法的性能。在激进预算（如 64）下，平均提升 7.5%，最高达 30.9%（例如 Qwen2-VL-7B 在 DocVQA 上使用 PyramidKV 提升 +18.7 点）。

    - BACON 对文本长上下文任务同样有效，证明其通用性。
    - 不增加额外推理成本（仅预填充阶段增加微小计算）。

## 7. 优点
- **创新性**：首次系统分析最后查询注意力在 KV 压缩中的价值，并提出结构校准机制，兼顾窗口稳定性和边界敏感性。
- **即插即用**：无需修改模型、无需训练、无需改变预算分配，可直接集成到现有任意基于注意力的压缩方法中。
- **强大兼容性**：在多种模型架构（稠密、MoE）、多种压缩范式（均匀/自适应/头稀疏）下均有显著提升。
- **实验全面**：覆盖图像、视频、GUI、长文本，多预算、多基线，并包含详细的消融和敏感性分析，论证充分。

## 8. 不足与局限
- **训练信号未利用**：BACON 是训练-free 方法，不包含任何可学习参数，因此无法利用样本级特征或任务先验进一步优化保留策略。作者指出将其融入训练可能带来更自适应的压缩。
- **高精度或安全关键场景风险**：极端压缩下仍可能丢弃关键证据，在低容错率应用中（如医疗、自动驾驶）需谨慎部署，建议配合预算约束或置信度检查。
- **参数通用性**：邻域半径 r、跨层深度 m 等超参数在实验中表现鲁棒，但未测试在所有模型或任务上最优值是否一致。虽然作者提供了默认值并验证了敏感性，但理论指导不足。
- **全局偏差**：仅依赖注意力权重，未考虑 token 语义多样性（MixKV 部分弥补了这一点，但 BACON 本身未纳入多样性约束）。
