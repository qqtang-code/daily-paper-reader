# PolyKV: Heterogeneous Retention and Allocation for KV Cache Compression 

Chao Fei Panos Kalnis King Abdullah University of Science and Technology {chao.fei, panos.kalnis}@kaust.edu.sa 

## Abstract 

KV cache compression is essential for reducing the memory cost of long-context large language model inference. Existing approaches, however, typ-ically apply a single compression policy and a uniform cache budget across all transformer layers. This uniform design ignores the fact that different layers can play different roles during prefill and decoding, and may there-fore require different eviction strategies and cache capacities. We present PolyKV, a layer-wise KV cache optimization framework that considers de-sign space with method selection and budget allocation. PolyKV routes each layer to a suitable KV compression policy based on layer-level sig-nals, while assigning non-uniform budgets under a fixed total budget. This formulation enables heterogeneous compositions of existing KV cache meth-ods. Experiments on LLaMA-3.1-8B and Qwen3-8B show that, under the same 512-token average KV budget, PolyKV recovers 54.5% and 25.7% of the LongBench performance gap between the strongest single-policy base-line and FullKV, respectively. Across 128–1024 budget sweep, PolyKV con-sistently improves over the strongest baseline by 1.7%–6.4%, corresponding to 40.0%–54.5% recovery of the FullKV gap. 

## 1 Introduction 

Long-context LLM inference is becoming increasingly important with the rise of agentic applications, retrieval-augmented generation, and other workloads that require models to process extended contexts. However, longer contexts substantially increase the size of the KV cache, creating a major memory bottleneck during inference. The growing KV cache can exhaust HBM capacity, reduce achievable batch size, and slow down token generation. KV cache eviction mitigates this bottleneck by reducing cache memory usage while preserv-ing long context quality. Existing methods have introduced a range of effective eviction patterns. Heavy hitter and persistence based methods retain tokens that receive large or stable attention mass (Zhang et al., 2023; Liu et al., 2023). Attention sink methods pre-serve sink tokens together with recent tokens (Xiao et al., 2024b). Prompt observation methods identify salient prompt tokens before generation (Li et al., 2024). Together, these approaches show that cache eviction pattern is a key factor in the trade off between quality and memory usage during long context inference. However, canonical KV eviction methods primarily define useful eviction patterns, and are often instantiated as model-wide strate-gies. This uniform treatment overlooks the heterogeneity of transformer layers, implicitly assuming that different layers should follow the same eviction rule and receive comparable cache capacity. Transformer layers are not homogeneous computation blocks. Prior work has shown that layers and heads differ in their attention patterns, represented abstractions, contextual roles, and sensitivity to pruning (Tenney et al., 2019; Jawahar et al., 2019; Clark et al., 2019; Voita et al., 2019; Michel et al., 2019). This heterogeneity suggests that a global KV eviction rule is a coarse abstraction. Global policy imposes the same eviction bias and comparable cache capacity across all layers. We therefore examine KV eviction as a per layer design space. This design space exposes two coupled choices. The first is what to evict, since different 1

## 2 Related Work 

2.1 KV Cache Eviction and Transformer Heterogeneity KV cache eviction reduces inference memory by retaining only a subset of historical key and value states. Existing methods can be understood as distinct token eviction patterns, each driven by a different eviction pattern. Cumulative attention methods score historical tokens using accumulated attention scores and protect a recent window (Zhang et al., 2023; 2024; Wan et al., 2025). Recent query window methods instead score historical tokens using attention from the most recent query window, optionally applying pooling or smoothing, and retain the top k historical tokens together with recent tokens (Li et al., 2024; Cai et al., 2024). Persistence based methods identify tokens that repeatedly exceed an attention threshold across recent query steps, thereby favoring tokens whose importance remains stable over time (Liu et al., 2023). StreamingLLM represents another eviction pattern. Rather than relying on an attention based salience score, it preserves fixed initial sink tokens and the most recent tokens according to their positions (Xiao et al., 2024b). TOVA uses attention from the current last query token to select the global top k cached tokens, making eviction depend on the immediate decoding state (Oren et al., 2024). These methods rely on different eviction patterns, yet they all apply the same strategy to every layer in the model. This uniform treatment is coarse because transformer blocks are heterogeneous. Prior analyses show that LLM layers differ in represented abstractions, attention patterns, contextual roles, and sensitivity to pruning (Tenney et al., 2019; Jawahar et al., 2019; Clark et al., 2019; Voita et al., 2019; Michel et al., 2019). For KV cache eviction, this suggests that different layers may prefer different eviction biases and different eviction cache budgets. Layer heterogeneity therefore turns eviction into two related design questions. The first is which eviction pattern each layer should use, and the second is how much cache budget each layer should receive. 2.2 Adaptive and Non-uniform KV Cache Eviction Adaptive and non-uniform KV cache eviction methods, which challenge the assumption that all layers should have the same cache budget. Some methods allocate different cache budgets across model depths, assigning descending or heterogeneous budgets to different layers (Ge et al., 2024; Cai et al., 2024; Feng et al., 2025; Fu et al., 2025). Other methods adapt cache budgets using heuristic metrics derived from the prompt, task, or generation time statistics (Zhou et al., 2025; Shen et al., 2025; Tang et al., 2025; Xiao et al., 2024a). Together, these works show that a uniform cache budget is often suboptimal and that tailored budgets can improve performance. These methods explored the importance of heterogeneous budgets, however their design spaces are still limited. Many adaptive methods primarily adjust how much cache budget is assigned, while ignoring which eviction pattern should be assigned to each layer. In conventional model-wide eviction methods, all layers have the same eviction pattern and the same cache budget. Adaptive eviction methods have different cache budgets, but they still follow a common eviction pattern. PolyKV enlarges the design space by allowing both the eviction pattern and the cache budget to vary across layers. To instantiate this design space, we incorporate a set of representative KV eviction baselines as eviction patterns within the PolyKV framework. The goal of PolyKV is to explore whether existing eviction patterns can be used more effectively when they are assigned in a layer-wise manner with different cache budgets. 3.2 Offline Calibration The design space defined above is too large for exhaustive enumeration. With |R| candidate patterns, L layers, and two phases, the number of possible eviction assignments is |R| 2L even before considering budget allocation. PolyKV therefore uses offline calibration signals to construct a fixed strategy directly, rather than performing online policy search or exhaustive configuration search. Given a small calibration set Dcal , PolyKV first runs fullKV inference to obtain reference activations. It then profiles baselines with the same token prefixes and records activations for comparison. Calibration estimates two layer-specific signals: • eviction preference signal, which measures which eviction parttern best preserves each layer’s fullKV computation. • Cache budget sensitivity signal, which measures how strongly each layer is affected as its KV budget is reduced. The calibration set is disjoint from evaluation data. The resulting strategy is fixed for a model and target budget, and is reused during deployment without online policy search. Calibration metrics. For eviction preference, PolyKV uses these cosine similarity and PPL sensitivity as layer-wise signals. Our survey of existing methods shows that, compared with standard FullKV inference, most methods inject additional logic at a few common points in the gener-ation pipeline, as illustrated in Figure 2(b). PolyKV exposes these points as programmable hooks. The offline calibration stage produces eviction strategies. As shown in Figure 2(a), PolyKV supports YAML-based configurations, which makes large-scale experiments practical and reproducible. During inference, PolyKV loads the calibrated strategy and attaches the corresponding logic to the model generation process through lightweight hooks. Each eviction pattern is implemented as a policy composed of several hook handlers. If a policy does not implement a particular hook, PolyKV falls back to the original model execution path. This design decouples eviction logic from model specific code and allows different eviction methods to be incorporated under a unified interface. ## 4 Evaluation 

We evaluate PolyKV under fixed total KV cache budgets, comparing against representative KV cache eviction baselines. For all methods, the total cache budget is equivalent. 4.1 Experimental Setup We evaluate PolyKV on LLaMA-3.1-8B and Qwen3-8B. Experiments are conducted on two long-context benchmarks, LongBench and RULER. LongBench evaluates realistic long-context understanding across multiple task families, while RULER provides controlled stress tests for long-context retrieval and tracking. We instantiate the candidate eviction set in PolyKV with representative KV cache com-pression methods, including H2O, TOVA, CAM, PyramidKV, SnapKV, D2O, CAKE, Scis-sorHands, and StreamingLLM. 4.2 Main Results under a Fixed KV Budget Table 1 reports the main comparison results. On LongBench, PolyKV improves over the strongest baseline for both model families. For LLaMA, the best baseline reaches 36.35, whereas PolyKV reaches 37.79, narrowing the compression-induced gap to FullKV by ap-proximately 54.5%. For Qwen, PolyKV improves the best baseline from 47.01 to 47.54, recovering approximately 25.7% of the corresponding gap to FullKV. The improvement is not restricted to a single configuration. The top five PolyKV configurations fall within narrow ranges, and all remain above the strongest baseline. This suggests that, on realistic long-context workloads, heterogeneous policies expose a stable high performance region that is not reached by a single model-wide policy.

Across all tested budgets, PolyKV consis-tently outperforms the strongest baseline. Notably, the strongest baseline is not fixed across budgets. TOVA performs best under 128-token setting, while CAKE becomes stronger at larger budgets. The advantage of PolyKV is largest under tighter budgets and gradually narrows as the budget increases, which is expected because all compressed methods move closer to the FullKV and the remaining recoverable gap becomes smaller. A more direct indication of cache efficiency comes from cross-budget comparison. Using only half the cache budget, PolyKV outperforms the best 512-token baseline by up to 0.49 points at a 256-token budget, and the best 1024-token baseline by up to 0.22 points at a 512-token budget. These results indicate that PolyKV can reach a quality level that a global eviction rule requires substantially more cache to approach. The sweep also helps separate the effects of the two layer-wise design choices. The main performance gain comes from assigning different eviction rules to different layers. Budget allocation has a more conditional effect. Under the smallest budgets, uniform allocation is more reliable, likely because every layer needs a minimum amount of cache to avoid severe information loss. Under this budget, redistributing an already limited budget can make some layers too constrained. Once the average budget reaches 512 tokens or above, heterogeneous allocation begins to outperform uniform allocation, and the advantage re-mains consistent at larger budgets. This suggests that budget heterogeneity is most useful when there is sufficient capacity to reallocate without starving sensitive layers. ## 5 Conclusion 

This paper revisits KV cache eviction from the perspective of transformer layer-wise hetero-geneity, showing that a single global eviction pattern is a coarse abstraction for long context inference. We introduced PolyKV, a phase-aware layer-wise framework that jointly assigns eviction pattern and KV cache budgets under a fixed total memory constraint. PolyKV uses offline calibration to estimate each layer’s eviction pattern preference and cache sensitivity, enabling static heterogeneous compositions of existing KV eviction methods. Experiments on LLaMA-3.1-8B and Qwen3-8B show that, on LongBench under the same 512-token aver-age budget, PolyKV improves over the strongest single-policy baselines and recovers 54.5% and 25.7% of the gap to FullKV, respectively. Across a 128–1024 budget sweep on LLaMA-3.1-8B, PolyKV consistently outperforms the best baseline, with the largest gains under tighter cache budgets. Ablations indicate that layer-wise retention routing, particularly during prefill, is the main source of improvement, while non-uniform budget allocation pro-vides additional but more model- and budget-dependent benefits. At the same time, weaker results on exact retrieval-style RULER tasks and code-like workloads indicate that current calibration signals do not always preserve sparse but decisive evidence tokens. Overall, PolyKV suggests that KV cache eviction should be treated as a compositional, layer-aware optimization problem rather than as a single global retention policy, opening a path toward more reliable long-context eviction strategies.
