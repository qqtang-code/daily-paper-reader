## 摘要
多模态大语言模型（MLLMs）近期通过下一标记预测直接建模查询-文档相关性，展现出作为逐点重排序器的强大潜力。然而，逐点重排序在跨查询-文档对时存在大量重复计算，而Transformer的因果结构仅允许通过预缓存重用前缀片段。为解决现有查询优先和文档优先格式与VQA风格提示及计算感知重用之间的错位问题，我们提出了一种视觉优先的公式，既提高了缓存重用效率，又提升了重排序性能。然而，剩余成本仍然可观，主要来源于三个方面：（1）模型深度，我们通过早期退出减少活跃参数；（2）跨段注意力，我们将其限制在少数层上的窄交互带内；（3）视觉标记，我们通过嵌入器引导的剪枝减少标记数量。这些设计共同构成了miniReranker，在单个查询的高重用设置下，其重排序运行时间降至密集实现的<1%，同时保留了>96%的密集模型性能。

## Abstract
Multimodal large language models (MLLMs) have recently shown strong potential as point-wise rerankers by directly modeling query--document relevance through next-token prediction. However, point-wise reranking suffers from substantial repeated computation across query--document pairs, while the causal structure of transformers allows only prefix segments to be reused via pre-caching. To address the misalignment of existing query-first and document-first formats with both VQA-style prompting and computation-aware reuse, we propose a $\textit{vision-first}$ formulation that improves both cache reuse efficiency and reranking performance. However, the remaining cost is still considerable and stems from three main sources: (1) $\textit{model depth}$, for which we reduce active parameters via early exit; (2) $\textit{cross-segment attention}$, which we restrict to a narrow interaction band across a few layers; and (3) $\textit{visual tokens}$, where we reduce the number of tokens via embedder-guided pruning. Together, these designs form miniReranker, which reduces reranking runtime to <1% of the dense implementation under high-reuse settings for a single query, while preserving >96% of the dense model performance.

- **核心问题**：多模态大语言模型（MLLMs）作为逐点重排序器（point-wise reranker）时，需要对每个查询-文档对独立进行前向计算，导致大量重复计算。同时，Transformer的因果自回归结构仅允许重用前缀片段的键值缓存（KV cache），而现有的查询优先（query-first）或文档优先（document-first）固定序列格式与MLLM的VQA风格预训练对齐不佳，且无法最大化视觉计算的复用效率。
- **整体含义**：该工作旨在解决多模态检索系统中重排序阶段的计算瓶颈，提出一种既保留MLLM细粒度交互能力、又大幅降低计算开销的通用框架，使得高效部署成为可能。

## 2. 论文提出的方法论
### 核心思想
通过**视觉优先（vision-first）提示重格式化**与**三级模型级压缩**（深度、交互密度、序列长度）协同降低冗余计算。

### 关键技术细节
1. **视觉优先提示重格式化**：
   - 动态调整输入顺序：**视觉输入始终置于文本输入之前**。
   - 文本→视觉（T→V）：文档优先 [inst, d, q] → 视觉文档可离线缓存。
   - 视觉→文本（V→T）：查询优先 [inst, q, d] → 视觉查询可在线广播。
   - 视觉→视觉（V→V）：文档优先，文档可全局预缓存。
   - 该策略同时恢复VQA预训练对齐、最大化KV缓存重用，并提升重排序准确率（表1）。

2. **早期退出（Early Exit）**：
   - 通过logit透镜分析发现重排序信号在中间层已饱和（约60%深度达到95%最终性能）。
   - 在预设中间层截断前向传播，减少活跃参数至原模型的58%（2B: 16/28层；4B/8B: 21/36层）。

3. **交互带（Interaction Band）**：
   - 通过注意力掩码分析，确定有效跨段（查询-文档）信息交换仅发生在少数中间层。
   - 只在该窄层范围内启用跨段注意力，其余层阻断查询-文档交互，大幅减少二次注意计算。
   - 例如2B模型交互带设为8-16层，4B/8B设为8-21层。

4. **嵌入器引导的视觉标记剪枝（Embedder-guided Token Pruning）**：
   - 利用上游检索阶段嵌入器（如Qwen3-VL-Embedding）已有的注意力权重，无需额外前向计算。
   - 聚合所有层的最后标记对视觉标记的注意力分数，选择Top α%视觉标记保留，丢弃其余50%视觉标记。
   - 相比单层启发式方法，层聚合策略更稳定，基本恢复密集基线性能（表2）。
