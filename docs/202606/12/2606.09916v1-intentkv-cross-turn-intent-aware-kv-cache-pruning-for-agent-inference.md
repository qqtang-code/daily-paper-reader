## 摘要
多轮LLM代理将短查询扩展为工具调用、搜索结果和中间推理的长轨迹。KV内存和KV读取带宽在单个轨迹中增长数个数量级，使得键值（KV）缓存（而非参数计算）成为长期代理的主要服务瓶颈。我们提出IntentKV，一种学习型KV剪枝方法，保持基础LLM冻结。IntentKV维护跨轮次意图的会话级QueryMemory，通过记忆-注意力规则对历史token进行评分，并添加一个零初始化的残差头，该残差头与当前查询的K向量进行交叉注意力。为了保持与前缀缓存的组合性，驱逐通过插槽映射重定向实现：被丢弃的位置路由到一个哨兵死亡插槽，而幸存的K/V行、RoPE相位和插槽标识保持不变。在严格的KV预算下，IntentKV与无剪枝的全缓存基线匹配，几乎没有精度下降：在8k KV预算下，Qwen3-8B的平均峰值请求token降低23.9%，Qwen2.5-14B降低30.7%。在Qwen2.5-14B上所有方法都能完成的100个最长BCP查询中，IntentKV-8k进一步将最坏情况下的峰值请求token从92.3k减少到20.5k，减少了77.8%，最坏情况下的原始KV读取从4.11亿减少到3100万，减少了92.6%。

## Abstract
Multi-turn LLM agents fan short queries into long trajectories of tool calls, search results, and intermediate reasoning. Both KV memory and KV read bandwidth grow by orders of magnitude across a single trajectory, making the key-value (KV) cache, not parameter compute, the dominant serving bottleneck for long-horizon agents. We introduce IntentKV, learned KV pruning that keeps the base LLM frozen. IntentKV maintains a session-level QueryMemory of cross-turn intent, scores live history tokens with a memory-attention rule, and adds a zero-initialized residual head with cross-attention over current-query K-vectors. To stay composable with prefix caches, eviction is a slot-map redirection: dropped positions route to a sentinel dead slot while surviving K/V rows, RoPE phases, and slot identities stay in place. IntentKV matches the no-pruning full-cache baseline with almost no accuracy drop under tight KV budgets: at an 8k KV budget, mean peak request tokens drop 23.9% on Qwen3-8B and 30.7% on Qwen2.5-14B. On the 100 longest BCP queries that all methods complete on Qwen2.5-14B, IntentKV-8k further cuts worst-case peak request tokens from 92.3k to 20.5k, a 77.8% reduction, and worst-case raw KV reads from 411M to 31M, a 92.6% reduction.

# 论文《IntentKV》: 面向Agent推理的跨轮次意图感知KV缓存剪枝

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多轮LLM Agent（如浏览搜索、深度研究、工具调用）中，一个短用户查询会展开为包含大量工具调用、检索结果、中间推理步骤的长轨迹。这导致KV缓存的内存和读取带宽随轨迹增长数个数量级，成为比参数计算更显著的服务瓶颈。
- **现有方法缺陷**：现有KV剪枝方法（如StreamingLLM、SnapKV、H2O）大多针对单轮提示压缩设计，存在两个关键问题：
  - (1) 重要性评分基于当前单轮提示，后续轮次的新查询、工具输出可能使早期历史重新变得有用，单轮评分会错误丢弃未来有用的证据。
  - (2) 压缩通常将幸存K/V行重新排列连续物理位置并重新旋转RoPE，这会破坏前缀/基数缓存的身份，导致后续轮次无法复用共享前缀。
- **整体含义**：需要一种既能感知跨轮意图演化、又能与现有前缀缓存系统（如SGLang、vLLM）兼容的KV缓存剪枝方法，以在长时Agent情境下维持精度同时大幅降低KV资源开销。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **保留策略与布局策略分离**：保留策略使用会话级QueryMemory跟踪跨轮意图，对历史token评分；布局策略采用“槽映射重定向”驱逐被丢弃的物理槽，而不移动幸存K/V行，从而保持逻辑位置和缓存标识不变，使剪枝与基数前缀缓存兼容。
- **保持基础LLM冻结**：不修改原模型参数，仅添加一个轻量可学习残差头。

### 2.2 关键技术细节

#### 2.2.1 跨轮查询记忆（QueryMemory）
- 为每个会话维护一个记忆状态 \(M_t \in \mathbb{R}^{L \times H_q \times D}\)（形状等于一个后RoPE查询行）。
- 更新规则：\(M_t = e^{-\lambda} M_{t-1} + \text{Enc}(q_t)\)，其中\(q_t\)是当前可操作查询跨度，\(\text{Enc}\)取该跨度内Q行平均；\(\lambda\)固定为0.5。
- 更新后对\(M_t\)沿头维度投影为单位范数，防止范数线性增长。
- 规则得分：对历史位置j，计算其对所有层和头与\(M_t\)的softmax注意力得分之和作为rule_j。

#### 2.2.2 残差可学习剪枝器
- 在规则得分基础上添加可学习残差：\(s_j = \text{rule}_j + \alpha \cdot \text{MLP}([\phi_j; c_j])\)，其中\(\alpha\)为可学习的裁剪后标量（范围[-5,5]）。
- 每个token特征向量\(\phi_j\)包含：K向量均值、M均值、元素积、rule_j，聚合所有层和头。
- 交叉注意力\(c_j\)：使用当前查询的K向量作为键和值，特征\(\phi_j\)投影为查询，4头、宽度128，读出当前查询中哪些token与候选位置匹配。
- 初始化：MLP输出投影全零，\(\alpha=1\)，使得初始时完全恢复规则得分，梯度可流。

#### 2.2.3 槽映射驱逐（Slot-Map Eviction）
- 驱逐时不压缩K/V行，而是将被丢弃位置的槽映射重定向到一个哨兵死槽\(s^\dagger\)。
- 哨兵槽的K设为\(-10^4\)（使softmax分数极低），V设为0向量，效果等同硬掩码但不修改注意力内核。
- 释放物理槽时需检查三条件：(a) 位置不在受保护的基数前缀内，(b) 槽不是哨兵，(c) 该槽未被其他幸存位置或同一基数分支的兄弟请求共享，避免腐败。

### 2.3 训练细节
- 训练冻结模型的特性提取器（后RoPE的K和Q），使用ToolBench多轮轨迹。
- 标签：扫描未来5次工具调用，标记历史中匹配工具参数子串的位置为正例，当前查询内和模板token标记为-100忽略，无匹配行的样本丢弃。
- 损失函数：四项组合：BCE损失（自适应正类权重）、pairwise排序损失、正则化项防止无界得分、单位范数约束项。
- 超参数：学习率1e-3，权重衰减0.01，梯度累积8，2个epoch，每epoch采样3000-5000例。

## 3. 实验设计

### 3.1 数据集/场景
- **主基准**：BrowseComp-Plus (BCP)，830个查询的深度研究套件，包含约10万文档，检索器冻结以避免检索漂移。
- **交叉验证**：FRAMES（824个多跳问答，经改编适配多轮Agent场景，保持原始问题和文档URL）。

### 3.2 模型与设置
- **主干模型**：Qwen3-8B-Instruct和Qwen2.5-14B-Instruct（Neox-RoPE GQA架构）。作者报告在Llama-3.1、Mistral等7个非Qwen模型上因Agent行为不足无法有效评估。
- **服务系统**：SGLang，启用基数前缀缓存，确定性解码（T=0），最多32工具轮。
- **预算**：C ∈ {8192, 16384} token，应用于每次剪枝事件的可压缩历史。

### 3.3 对比方法
- Full-cache（无剪枝上限）
- StreamingLLM（sink=4+近期窗口）
- SnapKV（核7, W=32, max-pool）
- H2O（比率模式+近期下限）
- TrimKV（位置时间衰减先验）

### 3.4 指标
- **精度**：完成率（Compl）、正确率（Raw）、真实准确率（True Acc = Raw×Compl，未完成计为错误）。
- **系统成本**：峰值请求token（PT）、有效存活的KV（Eff. Live KV）、原始KV读取（Raw KV Reads）、有效KV读取（Eff. KV Reads）、端到端延迟（Wall）、前缀命中率。

## 6. 主要结论与发现

- **8k紧张预算下**：IntentKV-Phase-2在Qwen3-8B上达到14.10 True Acc，超过StreamingLLM 2.29点、TrimKV 3.14点、H2O 4.34点、SnapKV 4.94点；在Qwen2.5-14B上达到18.55 True Acc，超过最强启发式StreamingLLM（8.19）达10.36点。完成率显著优于所有基线（84.58% vs 45-65%）。
- **16k较宽松预算下**：接近无剪枝上限（14.94 vs 15.06），仍优于所有基线。
- **效率提升**：Qwen3-8B上，8k预算下Wall时间140.4s（比H2O减少43%），原始KV读取31.6M（比H2O/SnapKV减少59%）。20.7%前缀命中率，而压缩基线降至0-3%。
- **最坏情况压力**：100个最长BCP查询中，8k预算下最坏峰值请求token从92.3k降至20.5k（↓77.8%），最坏原始KV读取从4.11亿降至3100万（↓92.6%）。
- **记忆是主导信号**：去掉跨轮记忆导致True Acc下降3.13点（16k），残差和交叉注意力各自贡献不到1点。
- **布局迁移收益**：对SnapKV/H2O改用死槽布局可获44-46%墙钟加速和39-47%原始KV读取降低，精度也有提升。

## 7. 优点

1. **方法创新性**：明确分离保留策略和布局策略，解决了跨轮意图漂移和前缀缓存复用的双重矛盾。
2. **模块化设计**：规则评分+可学习残差，零初始化保证无额外训练时也能直接使用启发式；驱逐方法可独立替换任何基于评分的剪枝器。
3. **系统兼容性**：不修改注意力内核，仅操作槽映射，可直接集成到现有服务系统（SGLang/vLLM）中。
4. **轻量训练**：仅需数万参数和短时训练，不更新基础模型，易于部署。
5. **实证全面**：在主流Agent基准（BCP）和跨域基准（FRAMES）上验证，包含精度、系统成本、最坏情况分析、消融、布局迁移等多维度评估。

## 8. 不足与局限

1. **无自主预算选择**：使用全局固定预算C，对所有查询统一处理，短查询承受不必要开销，超长轨迹被迫压缩；未来可探索自适应预算控制器。
2. **主干验证有限**：仅验证了Qwen3-8B和Qwen2.5-14B两个模型。论文试图测试Llama、Mistral、Hermes等7个模型，但均因缺乏BCP所需自主多步工具调用行为而无法对比。这意味着方法在非Qwen家族上的泛化性尚未证实，可能存在训练数据分布偏差。
3. **标签信号依赖子串匹配**：训练残差头时使用的未来动作标签基于工具参数文本子串匹配，可能存在标签不完整或不精确的情况，且仅在tool-use场景上训练；对非工具类多轮Agent场景的迁移能力未验证。
4. **哨兵死槽需要fp16/bf16 KV池**：哨兵处的超大负值在整数精度下可能无法有效实现数值掩码效果。
5. **与现有工作对比缺失部分基线**：如量化方法（KIVI、GEAR）未对比（但作者指出正交性），另一些学习型方法（如SideQuest、Activation Beacon）因修改基础模型而未比较。
6. **性能上限与真实部署差距**：实验在A100上进行，但多租户并发场景下的实际缓存复用率等未深入讨论。论文提到在≥70B模型上留待未来工作，大模型上的效果未知。
