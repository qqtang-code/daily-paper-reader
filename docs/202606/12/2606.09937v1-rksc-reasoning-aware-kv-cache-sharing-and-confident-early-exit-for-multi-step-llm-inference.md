## 摘要
我们提出RKSC（推理感知KV缓存共享），一种无需训练的推理框架，消除了多分支LLM推理流水线中的两种结构性冗余。ASKS（注意力相似性KV共享）通过隐状态余弦相似度计算一次前缀KV缓存并将其广播到所有语义相似的分支，严格泛化了vLLM和SGLang使用的令牌精确前缀缓存。CGEE（置信门控早退）应用两种互补的退出机制：（1）当跨分支的生成置信度具有决定性时，完全跳过验证前向传播；（2）当逐层熵稳定时，在中间层终止验证传播，使用轻量级钩子作用于Transformer骨干网络。RSBCM（推理选择性块缓存管理器）通过注意力加权深度优先驱逐防止缓存无界增长。在五个模型系列（7B-10B）、四个基准测试和1000个评估问题上，RKSC相对于无KV基线实现了平均3.008倍的加速（峰值3.990倍），相对于等效于vLLM的前缀缓存实现了1.66倍的平均改进，CGEE导致的错误率仅为0.37%（1616次验证调用中6次错误）。无需微调或架构更改。代码可在https://github.com/AnirudhSekar/RKSC获取。

## Abstract
We introduce RKSC (Reasoning-Aware KV Cache Sharing), a training-free inference framework that eliminates two structural redundancies in multi-branch LLM reasoning pipelines. ASKS (Attention-Similarity KV Sharing) computes the prefix KV cache once and broadcasts it to all semantically similar branches via hidden-state cosine similarity, strictly generalising the token-exact prefix caching used by vLLM and SGLang. CGEE (Confidence-Gated Early Exit) applies two complementary exit mechanisms: (1) it skips the verification forward pass entirely when generation confidence is decisive across branches, and (2) it terminates the verification pass at an intermediate layer when per-layer entropy stabilises, using lightweight hooks on the transformer backbone. RSBCM (Reasoning-Selective Block Cache Manager) prevents unbounded cache growth via attention-weighted depth-priority eviction. Across five model families (7B-10B), four benchmarks, and 1,000 evaluated problems, RKSC achieves a mean speedup of 3.008x over the No-KV baseline (peak 3.990x), a 1.66x mean improvement over vLLM-equivalent prefix caching, with a CGEE-induced error rate of only 0.37% (6 errors out of 1,616 verify calls). No fine-tuning or architecture changes are required. Code is available at https://github.com/AnirudhSekar/RKSC.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前推理时扩展系统（如DeepSeek-R1、Qwen3等）生成多条推理轨迹，但存在两种结构性冗余：①**跨分支KV缓存浪费**：并行分支共享大部分前缀KV计算，却独立重算；现有方案（vLLM、SGLang）仅基于精确令牌匹配重用缓存，忽略语义相似性。②**验证深度过深**：过程奖励验证执行完整深度前向传播，即使模型已足够自信；缺乏利用逐层熵塌陷在单次验证内提前退出的机制。
- **整体目标**：提出一个无需训练的推理加速框架RKSC，消除上述冗余，同时不改变推理算法、不增加可学习参数、不依赖架构修改。

## 2. 论文提出的方法论

- **核心思想**：通过三个互补机制协同加速：
  1. **ASKS（Attention-Similarity KV Sharing）**：利用隐状态余弦相似度，将一次性计算的前缀KV缓存广播给所有语义一致的分支，严格泛化令牌精确前缀缓存。
  2. **CGEE（Confidence-Gated Early Exit）**：双层退出机制——（a）当生成置信度跨分支具有决定性时，跳过整个验证前向传播（Level 1）；（b）当逐层对数熵稳定时，在中间层提前终止验证（Level 2）。
  3. **RSBCM（Reasoning-Selective Block Cache Manager）**：通过注意力加权深度优先驱逐策略，防止深层树搜索中缓存无界增长。
- **关键技术细节**：
  - **ASKS**：单次前缀前向传播后，分支隐藏状态与根隐藏状态计算加权余弦相似度（指数加权，α=1.5），σ_b ≥ τ则共享缓存；否则独立重算。严格包含令牌精确匹配（σ=1）。
  - **CGEE**：解码时记录每步top-1概率平均值作为生成置信度p(b)。若max p(b)≥τ_conf且相对间隙≥r_gap，则跳过验证（Level 1）；否则验证时，每层计算最后一个token的logit熵，当深度≥l_min、熵<θ且连续层变化<ε时，提前退出（Level 2）。
  - **RSBCM**：重要性得分ω = 分支得分 / (深度+1)，按ω升序驱逐，优先保留浅层高置信度分支的块。

## 6. 论文的主要结论与发现

- **总体加速**：平均加速3.008×（峰值3.990×），比vLLM等效前缀缓存提升1.66×。
- **CGEE错误率**：仅0.37%（1616次验证调用中6次错误），平均准确率下降-0.2%。
- **KV共享架构不变性**：vLLM等效增益在5个模型间几乎恒定（±0.03×），证明KV共享增益不依赖架构。
- **CGEE难度自适应**：跳过率从GPQA的42%到MMLU-STEM的92%，与任务难度单调相关。
- **ASKS超越令牌精确缓存**：在多样化措辞场景下实现28.6%额外重用，而令牌精确缓存为0%。
- **层级别退出有效性**：在Qwen2.5-7B上平均退出层为18.4/28，节省约34%验证成本。

## 7. 优点

- **无需训练或架构修改**：纯推理时加速，可直接作为包装器用于任何多步推理流水线。
- **机制可独立开关**：每个组件均可单独启用或禁用，无依赖。
- **严格泛化现有方案**：ASKS是令牌精确前缀缓存的超集；CGEE与推测解码等方法互补。
- **自校准机制**：KV共享通过运行时探针自动判断是否受益；CGEE阈值基于校准集统计量设定，保守且有效。
- **低错误率**：双层门控（绝对置信度+相对间隙）有效防止误跳过。
- **公开代码**：提供完整可复现代码。

## 8. 不足与局限

- **样本量有限**：每数据集N=50，导致CGEE跳过率置信区间宽；KV共享部分不受影响。
- **模型规模范围窄**：仅评估7B-10B模型，更大模型（14B+）和多GPU张量并行场景未验证；理论预测更大模型受益更多，但缺乏实证。
- **领域覆盖有限**：仅覆盖科学和数学；CGEE熵阈值在GPQA上校准，可能不适用于其他熵分布不同的领域。
- **CGEE非零错误**：6次错误均发生在Level 1（跳过验证），Level 2保留完整验证结果；要求零错误的应用应禁用Level 1。
- **ASKS广播内存开销**：repeat_interleave扩展为O(BL d_k)，在大B（≥64）长前缀时可能显著。
- **钩子依赖架构**：Level 2依赖register_forward_hook和存储的解嵌入矩阵；对MoE或融合层架构可能需要适配。
- **RSBCM仅单求解器验证**：多租户并发场景未经验证。
- **无真实树搜索设置评估**：主实验均为单深度；RSBCM仅在模拟压力测试中验证，未在完整树搜索中端到端评估。
