## 摘要
大型语言模型通过扩展的思维链推理提高了最终答案的准确性，但往往存在令牌使用效率低下、推理过程缺乏控制的问题。现有的高效推理方法通过缩短、提前终止或压缩推理轨迹来控制思考长度，但隐含了对模型思考方式的假设。本文提出智能思维链引导（Agentic Chain-of-Thought Steering, ACTS），将推理引导形式化为一个马尔可夫决策过程，其中控制器代理在推理过程中自适应地引导冻结的推理器。每一步中，控制器观察推理轨迹和剩余思考预算，然后发出一个引导动作，由推理策略和引导短语组成，以启动推理器的下一步。这实现了预算感知的策略控制，既保证了高效推理，又保持了推理器生成的连续性。我们从构建的合成引导轨迹出发，通过多预算增强初始化控制器代理，并通过基于预算条件的奖励塑形的强化学习进一步优化。多个基准实验表明，ACTS在显著节省令牌的情况下达到了与完全思考相当的性能，并能在不同推理器和任务之间实现可控的精度-效率权衡。代码可在https://github.com/Andree-9/ACTS获取。

## Abstract
Large language models improve final-answer accuracy through extended chain-of-thought reasoning, but often spend tokens inefficiently and offer little inference-time control. Existing efficient reasoning methods control thinking length by shortening, early-stopping, or compressing traces, leaving how the model thinks implicit. In this paper, we propose Agentic Chain-of-Thought Steering (ACTS), which formulates reasoning steering as a Markov decision process where a controller agent adaptively steers a frozen reasoner during inference. At each step, the controller observes the reasoning trace and remaining thinking budget, then issues a steering action consisting of a reasoning strategy and a steering phrase that initiates the next reasoner step. This enables budget-aware strategy control for efficient reasoning while preserving the reasoner's generation continuity. We initialize the controller agent from our constructed synthetic steering trajectories with multi-budget augmentation, and further optimize it via reinforcement learning with budget-conditioned reward shaping. Experiments across multiple benchmarks show that ACTS matches full-thinking performance with substantial token savings, and enables controllable accuracy-efficiency trade-offs across different reasoners and tasks. The code is available at https://github.com/Andree-9/ACTS.

## 2. 论文提出的方法论

- **核心思想**：将推理引导形式化为一个 **马尔可夫决策过程（MDP）**，其中 **控制器代理** 在推理过程中 **自适应地引导一个冻结的推理器**。
- **关键技术细节**：
  - **状态与动作**：控制器在每一步观察完整的引导历史（状态），并输出一个引导动作，包含：
    - **推理策略**（如理解、规划、执行、探索、检查、总结、结论）
    - **自然语言引导短语**（如“等一等，让我验证一下”）
  - **过渡**：推理器接收问题、当前推理轨迹和控制器的引导短语，生成下一步内容。剩余预算根据推理器生成的 token 数量更新。
  - **终止条件**：控制器发出“结论”动作、推理器输出结束标记，或达到最大步数。
  - **预算条件奖励塑形**：奖励函数同时惩罚 **过度思考**（超预算但正确）和 **过早终止**（预算未用尽但回答错误）。
- **训练流程**：
  1. **行为初始化**：从 DeepSeek-R1 的专家推理轨迹中，通过分段、标注策略和提取引导短语，构建合成引导轨迹，并加入 **多预算增强**（使轨迹在不同剩余预算下终止），然后进行 **监督微调**。
  2. **强化学习优化**：使用 **分组相对策略优化（GRPO）** 和上述预算条件奖励，进一步优化控制器。

## 3. 实验设计

- **匹配全推理性能并显著节省 token**：ACTS 在多个基准上达到与 Vanilla 相当或更高的准确性，同时平均 token 使用量大幅减少（如 DeepSeek-R1-Distill-Qwen-7B 在 MATH-500 上节省 57% token）。

- **控制器泛化性好**：在 DeepSeek-R1 推理轨迹上训练的控制器能成功转移到不同模型族（Qwen3-8B）和不同领域（科学问答 GPQA Diamond），并带来显著提升。
- **token 节省可解释**：节省主要来自两类机制：**拯救**（将错误回答转为正确）和 **缩短**（正确回答的冗余验证步骤被剪掉），而非盲目截断。
- **异步推理开销可忽略**：异步两服务器流水线使吞吐量与 Vanilla 相当，低于 DEER。

## 7. 优点

- **方法创新性强**：将推理控制从简单的长度控制提升到 **策略级别**，通过显式的推理策略和自然语言引导短语实现细粒度指导。
- **两阶段训练策略合理**：先通过合成轨迹 SFT 获得合理的行为基础，再通过 RL 优化预算感知和正确性目标，避免从头训练的不稳定性。
- **推理器无关设计**：控制器独立于推理器，可即插即用地应用于不同推理模型。
- **推理开销低**：异步服务器设计解决了多轮交互带来的延迟问题，使得实际部署可行。
- **实验分析深入**：不仅报告平均指标，还进行了 token 节省分解、预算扫描、泛化性测试等，增强了结果的说服力。
