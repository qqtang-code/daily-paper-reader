## 摘要
迭代式检索-推理智能体近来在多模态长文档问答中显示出潜力。然而，现有大多数系统维护一个单一增长的上下文，混合了检索轨迹、观察结果和中间推理。随着交互的累积，关键证据变得分散和稀释，使多跳推理产生噪声。我们提出MARDoc，一种记忆感知精炼智能体框架，将长文档问答解耦为三个专用智能体：探索者（Explorer）用于多粒度多模态检索，精炼者（Refiner）用于将交互轨迹提炼为结构化证据和推理记忆，以及反思者（Reflector）用于检查证据充分性并提供有针对性的反馈。在迭代过程中，智能体依赖于动态更新的结构化记忆，而非完整的累积交互历史。这种设计减少了上下文噪声，同时保留了答案关键事实及其逻辑依赖关系。在MMLongBench-Doc和DocBench上的实验表明，MARDoc取得了强结果，优于同骨干网络基线，证明了结构化记忆对于智能体文档问答的有效性。

## Abstract
Iterative retrieval-reasoning agents have recently shown promise for multimodal long-document question answering. However, most existing systems maintain a single growing context that mixes retrieval traces, observations, and intermediate reasoning. As interactions accumulate, key evidence becomes scattered and diluted, making multi-hop reasoning noisy. We propose MARDoc, a Memory-Aware Refinement Agent framework that decouples long-document QA into three specialized agents: an Explorer for multi-granularity multimodal retrieval, a Refiner for distilling interaction traces into structured evidence and reasoning memories, and a Reflector for checking evidence sufficiency and providing targeted feedback. Across iterations, the agents rely on a dynamically updated structured memory rather than a full accumulated interaction history. This design reduces context noise while preserving answer-critical facts and their logical dependencies. Experiments on MMLongBench-Doc and DocBench show that MARDoc achieves strong results, outperforming same-backbone baselines and demonstrating the effectiveness of structured memory for agentic document QA.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：多模态长文档问答中，现有迭代检索-推理智能体（agent）普遍采用“单一增长上下文流”范式，即每次交互后简单地将新的检索结果、中间推理追加到已有上下文中，导致随着交互轮次增加，关键证据被分散和稀释，多跳推理的噪声累积严重，且错误在早期引入后会持续传播并加剧幻觉。
- **背景与动机**：真实应用（如财报分析、科学文献、企业文档搜索）要求模型对齐文本、布局、表格、图形等多模态信息，且证据常稀疏分布于多页，现有方法（端到端OCR模型、检索增强生成、智能体方法）均存在局限性——前两者难以处理长文档和多跳推理，后者则受困于上下文污染和缺乏记忆驱动的质量控制。
- **研究意义**：提出一种可控的、记忆感知的精炼框架，通过结构化记忆替代杂乱的历史上下文，旨在提升长文档多跳问答的准确性和鲁棒性。

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将长文档问答解耦为三个专门智能体（Explorer、Refiner、Reflector），通过一个动态更新的结构化记忆（Structured Memory）进行通信，使推理状态保持紧凑，同时保留关键事实及其逻辑依赖。
- **关键组件**：
  1. **文档处理与大纲构建**（Document Processing）：使用 MinerU2.5 精确解析文档，生成细粒度语义节点（保留布局结构）；利用 Qwen3-VL-235B 为视觉元素生成描述，构建可作为检索索引的结构化大纲（Outline）。
  2. **探索者（Explorer）**：基于 ReAct 框架，接收查询 Q、大纲 O、当前记忆 M_{t-1} 和反思指令 R_{t-1}，通过多粒度工具集（search、get_section、get_paragraph、get_page_images、get_image、get_table_image、search_metadata）进行文本和视觉模态的检索，生成推理链 T_t 和候选答案。
  3. **精炼者（Refiner）**：模拟人类专家推理——仅保留关键信息而非所有细节。输入查询 Q、当前推理链 T_t 和上一轮记忆 M_{t-1}，输出更新后的结构化记忆 M_t，包含两个部分：
     - **证据记忆（Evidence Memory, M_E）**：与问题最相关的事实性信息，附带来源，去除冗余，将图像/表格浓缩为证据节点。
     - **推理记忆（Reasoning Memory, M_R）**：从原始上下文提取的清晰推理路径，每个推理节点标注所依赖的证据节点，解耦证据与逻辑。
  4. **反思者（Reflector）**：基于当前压缩后的结构化记忆 M_t 进行评估：
     - 如果证据充分且推理正确，则直接输出最终答案 A。
     - 如果不充分或存在错误，则生成针对性反馈指令 R_t（例如“缺少某个总数”、“问题关于 ChatGPT-4o 而非 BERT”），引导下一轮探索。

- **算法流程（文字说明）**：初始化记忆 M_0 和指令 R_0 为空。每轮迭代 t：
  1. Explorer 根据 Q, O, M_{t-1}, R_{t-1} 进行多轮工具调用，生成推理链 T_t 和候选答案。
  2. Refiner 利用 Q, T_t, M_{t-1} 蒸馏出结构化的 M_t = (M_E, M_R)，覆盖旧记忆。
  3. Reflector 检查 M_t：若充分则输出答案；否则生成 R_t 并进入下一轮迭代。重复至多 K 次（实验中 K=3）。

- **关键公式描述**：
  - 推理链 T = (τ₀, a₀, ε₀, …, τₙ, aₙ)
  - Explorer 输出：T_t = M_Exp(Q, O, M_{t-1}, R_{t-1})
  - Refiner 输出：M_E, M_R = M_Ref(Q, T_t, M_{t-1})
  - Reflector 评估：若充分则 A = M_Ref(M_t, Q)；否则 R_t = M_Ref(M_t, Q)

### 6. 论文的主要结论与发现

- **核心结论**：MARDoc 通过动态结构化记忆代替单一增长上下文，显著提升了多模态长文档问答的准确性。在相同骨干（Qwen3-30B）下，MARDoc 比 DocAgent 高出 8.8 个 ACC 点（57.1% vs. 48.3%），且性能接近 DocAgent 使用 Claude 3.5 Sonnet 的水平（57.3%）。
- **关键发现**：
  - Refiner 和 Reflector 必须协同工作：单独保留 Reflector 而无 Refiner 会导致性能下降，因为嘈杂的原始上下文干扰了反思质量。
  - 证据记忆（M_E）保证事实忠实，降低幻觉；推理记忆（M_R）维持多跳推理连贯性；二者互补。
  - 无结构化的纯文本压缩效果最差，确认结构化记忆的必要性。
  - 反思者提供针对性指令（而非仅判断充分性）可显著提升单跳和多跳性能，但对不可答问题略有负面影响。
  - 迭代次数 K=3 为最优；超过后冗余信息可能稀释记忆。

### 7. 优点：方法或实验设计上的亮点

- **方法论亮点**：
  - **解耦设计**：将检索、精炼、反思三个角色分离，各司其职，避免单一智能体负担过重。
  - **结构化记忆**：Evidence Memory 与 Reasoning Memory 的显式分离支持错误溯源（证据错误 vs. 逻辑错误）。
  - **无需微调**（training-free），仅通过 prompt 工程实现，对社区友好且易于复现。
  - **多粒度工具集**：覆盖文本、布局、表格、图表等多种模态的检索，粒度从关键词搜索到段落/图像提取。
- **实验设计亮点**：
  - 大量消融实验系统验证每个组件和记忆类型的贡献。
  - 跨文档类型、证据页数、问题类型的细粒度分析，揭示模型在不同难度下的表现差异。
  - 成本与性能的权衡分析，表明额外计算开销带来的性能提升是合理的。
