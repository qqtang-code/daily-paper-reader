## 摘要
大型推理模型通过生成扩展的思维链推理来提升性能，但这种行为在应用于LLM智能体时变得低效。当前的LLM智能体通常在每个决策步骤生成冗长的文本推理，并且几乎均匀地在各轮次分配推理努力，导致多轮智能轨迹中的显著低效。我们提出自适应潜在智能推理（ALAR），一种双模式框架，在常规轮次中使用紧凑的潜在推理，并在需要更深思考时选择性地升级为显式思维链。ALAR通过使用智能体的动作作为监督锚点来学习潜在推理，并进一步优化，使得在潜在推理足以完成任务时使用它，而将显式思维链保留给更困难的决策。在智能搜索和工具使用基准上的实验表明，ALAR在保持相当或更好的任务准确性的同时，大幅减少了生成的令牌数量，在搜索中最多减少43.6%，在工具使用中最多减少84.6%。这些结果表明，ALAR通过减少不必要的文本推理，同时为更困难的决策步骤保留显式思考，改善了LLM智能体的准确率-效率权衡。

## Abstract
Large reasoning models improve performance by generating extended chain-of-thought (CoT) reasoning, but this behavior becomes inefficient when applied to LLM agents. Current LLM agents often generate verbose textual reasoning at every decision step and allocate reasoning effort nearly uniformly across turns, leading to substantial inefficiency in multi-turn agentic trajectories. We propose Adaptive Latent Agentic Reasoning (ALAR), a dual-mode framework that uses compact latent reasoning for routine turns and selectively escalates to explicit chain-of-thought when deeper deliberation is needed. ALAR learns latent reasoning by using the agent's actions as supervision anchors and is further optimized to use latent reasoning when it is sufficient for task success and reserve explicit CoT for harder decisions. Experiments on agentic search and tool-use benchmarks show that ALAR maintains comparable or better task accuracy while substantially reducing generated tokens by up to 43.6% in search and 84.6% in tool use. These results demonstrate that ALAR improves the accuracy-efficiency trade-off of LLM agents by reducing unnecessary textual reasoning while preserving explicit deliberation for harder decision steps.

-   **核心问题**：当前大型语言模型（LLM）智能体（Agents）在进行多轮交互（如搜索、工具调用）时，继承了大型推理模型（LRM）在每个决策步骤都生成冗长“思维链”（Chain-of-Thought, CoT）文本的习惯。这导致了两个严重问题：
    -   **不必要的冗长**：许多常规的、依赖环境观察的决策步骤（如工具调用后的信息读取）并不需要完整的文本推理。
    -   **推理分配不均**：模型在各轮次间均匀分配推理努力，而不同轮次的推理难度差异巨大。
-   **研究动机**：为了解决LLM智能体在多轮交互中的效率瓶颈，探索一种超越“压缩文本推理”的更根本性方法，即在不需要显式思考时，彻底避免生成文本推理。
-   **整体含义**：论文旨在改善LLM智能体的“准确率-效率权衡”（accuracy-efficiency trade-off），在保持或提升任务性能的同时，大幅降低计算和延迟成本。

#### 2. 论文提出的方法论

-   **核心思想**：提出一个双模式（Dual-Mode）推理框架——**自适应潜在智能推理（ALAR）**。智能体默认使用**紧凑的潜在推理（Latent Reasoning）** 进行内部计算，只有在遇到需要深度思考的困难决策时，才**升级（Escalates）** 为效率较低但功能更强大的**显式CoT**。决定使用哪种模式的决策本身是由同一策略模型（Policy）在每轮做出的。

-   **关键技术细节**：
    1.  **潜在智能推理（Latent Agentic Reasoning）**：
        -   在每轮开始时，智能体生成一个固定长度（K=4）的连续潜向量序列（Continuous Thoughts）来代替文本CoT。
        -   这些潜向量在模型的隐状态空间中生成，不消耗任何文本token，从而实现了极致的效率。
    2.  **自适应模式选择（Adaptive Mode Selection）**：
        -   模型在每个决策步骤前，会从策略中采样一个模式标记 `mt`（`<LAT>` 或`<THINK>`），以决定使用潜在推理还是显式CoT。这使得模式选择成为智能体自身决策的一部分。
    3.  **训练流程（两阶段优化）**：
        -   **第一阶段：动作锚定自蒸馏（AASD）**：用于训练潜在的推理模式。
            -   **挑战**：潜向量没有文本标签作为直接监督。
            -   **解决方案**：让同一个模型扮演“教师”（使用显式CoT）和“学生”（使用潜在推理）。将教师轨迹中的显式CoT替换为潜在块，然后训练学生模型去预测教师的**下一动作（Action）**。因为“动作”是模型与环境的交互点，其正确性有明确标准（正确答案或正确工具调用），可以作为天然的监督锚点。这就绕开了对齐潜向量与文本CoT的难题。
        -   **第二阶段：自适应推理GRPO（AR-GRPO）**：用于学习何时使用哪种模式。
            -   **目标**：鼓励在能成功完成任务的情况下多使用潜在推理，在需要深入思考时则使用显式CoT。
            -   **奖励函数设计**：这是一个关键创新。
                -   **格式奖励（Format Reward）**：`rfmt(τ) = (1 + αf(τ))` 当任务成功(EM=1)，`-αf(τ)` 当任务失败。其中 `f(τ)` 是轨迹中潜在推理轮次的比例。这使得成功的轨迹中，使用潜在推理越多，奖励越高；失败的轨迹中，使用潜在推理越多，惩罚越重。
                -   **多样性奖励（Diversity Bonus）**：在训练早期鼓励模型探索不同模式组合。
                -   **长度缩放因子（Length Scaling）**：惩罚过长的显式CoT。
            -   **优化**：使用群体相对策略优化（GRPO）对奖励进行标准化，并优化策略。

#### 7. 优点

-   **方法创新性强**：首次将“潜在推理”这一概念系统性地引入智能体（Agent）场景，并提出了完整的训练框架（AASD + AR-GRPO），而非简单地对显式CoT进行压缩。
-   **双模式自适应架构设计巧妙**：模型自主决定何时需要“深思”，何时只需“灵光一闪”，这比外部规则或固定预算更灵活、更贴近实际需求。
-   **激励机制设计合理**：AR-GRPO的奖励函数核心思想——“成功时鼓励效率，失败时惩罚效率”非常直观且有效，是模型能学会自适应选择的关键。
-   **实验全面且结果显著**：在两个代表性场景、多个数据集上的实验结果一致且显著，尤其在工具使用场景下实现了“又快又好”。

#### 8. 不足与局限

-   **领域覆盖有限**：未在数学、编程等更依赖纯推理、环境交互较少的任务上评估。这些任务中，更多的推理token可能直接提升最终答案的准确性，ALAR的效果可能不同。
-   **训练依赖教师轨迹**：AASD阶段完全依赖于基座模型在显式CoT模式下成功产生的轨迹，这会继承教师的潜在偏见和错误模式。
-   **固定潜在块长度**：使用固定的长度 K=4，缺乏细粒度控制。未来工作可以探索根据任务难度动态调整潜在块的长度。
-   **可解释性降低**：潜在推理的内部计算过程是“黑盒”，不像显式CoT那样可以审计或理解模型思考过程。在高风险场景（如金融、医疗）中，这可能是一个缺点。
-   **标记符号开销**：虽然潜在推理本身不产生token，但引入的模式标签（`<LAT>` 和 `<THINK>`）本身也消耗少量token，虽然可以忽略不计。
