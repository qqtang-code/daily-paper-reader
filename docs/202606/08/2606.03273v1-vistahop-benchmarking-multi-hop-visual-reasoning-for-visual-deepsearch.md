## 摘要
视觉深度搜索（Visual DeepSearch）要求多模态大型推理模型（MLRM）代理通过反复检查图像区域、将中间推理过程锚定于视觉证据、并在长推理链中连接细粒度线索，来回答复杂的视觉查询。然而，现有基准主要关注单步视觉理解或静态图像问答，对迭代图像检查、视觉锚点定位和多跳证据整合的评估有限。本文提出了VistaHop，一个用于评估视觉深度搜索中视觉中心搜索和多跳视觉推理的基准。VistaHop包含300张高分辨率图像、25个视觉搜索场景和350个多跳问答任务，要求模型遵循从视觉锚点出发的证据链，或融合跨多个图像基础推理路径的信息。我们进一步开发了VistaArena，一个统一的评估环境，支持文本搜索、图像搜索、图像裁剪和基于证据的答案验证等工具增强推理。对七个代表性MLRM的实验表明，当前模型远未解决VistaHop：最佳模型SenseNova-MARS-32B仅达到24.31%的Pass@1。这些结果揭示了在视觉定位、证据重访、长链推理和多锚点信息融合方面的持续局限性，凸显了对更强基准和训练方法的需求，以推动视觉深度搜索的发展。

## Abstract
Visual DeepSearch requires multimodal large reasoning model (MLRM) agents to answer complex visual queries by repeatedly inspecting image regions, grounding intermediate reasoning in visual evidence, and connecting fine-grained clues across long reasoning chains. However, existing benchmarks mainly focus on single-step visual understanding or static image-question answering, offering limited evaluation of iterative image inspection, visual-anchor grounding, and multi-hop evidence integration. In this work, we introduce VistaHop, a benchmark for evaluating vision-centric search and multi-hop visual reasoning in Visual DeepSearch. VistaHop contains 300 high-resolution images, 25 visual search scenarios, and 350 multi-hop QA tasks that require models to follow evidence chains from visual anchors or fuse information across multiple image-grounded reasoning paths. We further develop VistaArena, a unified evaluation environment that supports tool-augmented reasoning with text search, image search, image cropping, and evidence-based answer validation. Experiments on seven representative MLRMs show that current models remain far from solving VistaHop: the best model, SenseNova-MARS-32B, achieves only 24.31% Pass@1. These results reveal persistent limitations in visual grounding, evidence revisiting, long-chain reasoning, and multi-anchor information fusion, highlighting the need for stronger benchmarks and training methods for Visual DeepSearch.

    3. **证据链构建**：从种子出发，通过 Wikidata 进行关系扩展，要求链长 k≥5，并应用关系验证（NLI 置信度≥0.9）、熟悉度过滤（分数<0.4）、多样性评分 $D(c)=0.6\cdot\min(1.0,|V_c|/|c|)+0.4\cdot\min(1.0,|R_c|/|c|)$。通过深度优先扩展至目标深度。

### 6. 论文的主要结论与发现

1. **现有 MLRM 在 Visual DeepSearch 上能力严重不足**：最佳模型 SenseNova-MARS-32B 在 Search+Crop 下仅达 **24.31% Pass@1**，其他通用模型更低（如 Qwen3-VL-30B 仅 16.60%）。
2. **工具使用显著提升性能**：从 No-Tool（平均 7.72%）到 Search（20.11%），再到 Search+Crop（略提升），说明文本搜索和图像裁剪对多跳视觉推理至关重要。
3. **链长度和融合复杂度是关键瓶颈**：L3 难度（≥10 跳或多链）的表现远低于 L2（5-9 跳），平均 Pass@1 从 24.40% 降至 15.03%。
4. **最佳轮次 N=10**，更大预算会导致冗余和噪声，性能饱和甚至下降。
5. **主要失败模式**：视觉锚点错误（27.4%）、检索漂移（24.1%）、长链规划崩溃（21.5%）、多锚点融合计算错误（16.8%）、答案归一化边界问题（10.2%）。

### 7. 优点

- **自动化构建流程**：几乎全自动，从图像采集、实体提取、证据链构建到防泄漏验证，可扩展且可重复（但依赖强模型和手册，见局限）。
- **强防泄漏设计**：多重机制（文本 Solver 检测、条件消融、人工检查）确保问题必须依赖图像才能回答，减少了文本捷径。
- **多链融合任务**：创新地组合多个单独证据链，要求模型具备分解、并行推理和算术/条件推理能力，填补了现有多跳推理基准的空白。
- **统一评估框架 VistaArena**：支持多种工具交互，提供标准化的搜索、裁剪、验证接口，便于未来对比。
- **人工验证保证质量**：三名标注者逐条检查，确保任务正确、唯一、无歧义。
- **开放式发布**：代码和数据集已公开，利于后续研究。

### 8. 不足与局限

- **构建依赖性**：大量依赖商业 API（Gemini-3.1-Pro）和开源大模型（Qwen-3 系列）进行实体提取、关系验证和问题生成，成本高且可重复性受 API 服务变动影响。作者承认未来可改用更强开源模型和确定性模块来缓解。
- **语言和场景限制**：仅英文；图像来源以 Wikimedia 等开放资源为主，可能存在域偏差（偏向文化、科技常见实体）。
- **静态答案假设**：未包含需要实时网络搜索才能获取动态信息（如当日天气、股价）的任务，虽然论文提及时间敏感性问题，但实际构建中未引入。
- **模型训练视角缺失**：论文仅评估，未探索通过数据训练提升模型策略（如使用 Pipeline 合成数据微调）。未给出在 VistaHop 上训练模型的基线，限制了基准对训练方法的指导作用。
- **人工验证的局限**：虽然有三名标注者，但样本量相对有限（350 个任务全部被验证，但人类评估仅采样 210 个输出），可能存在标注偏差。
- **未见实际搜索环境**：VistaArena 使用预设工具（爬虫、反向搜索），未在真实复杂网页或实时动态环境中测试，与实际应用仍有差距。
