## 摘要
随着推理模型生成长达数万tokens的思维链，KV缓存日益成为部署瓶颈。现有的缓存驱逐方法通过注意力权重对token进行排序，但在长推理轨迹中，注意力权重是一个噪声较大的重要性代理，并且迫使模型物化注意力矩阵，从而阻止在生产推理中使用融合内核。在这项工作中，我们转而使用一种称为“顿悟分数”的度量标准对token进行评分：该分数直接从前向传播中读取模型内部表示的变化，无需注意力矩阵且额外状态可忽略不计。由此产生的缓存驱逐方法EpiKV无需训练、分类器或自定义内核，可直接在FlashAttention推理栈中未修改地使用——其可行上下文长度比基于注意力评分的缓存方法长16倍。我们发现了一些有趣的模式（低层贡献积极，中高层贡献消极），并通过因果滚动z分数消除了位置趋势。在4096-token缓存下，EpiKV在MATH-500上达到72%，与最强的基于注意力的基线（ThinKV 71%, H2O 67%）相当；一种延迟归一化的KV变体在AIME-2024上以8192 tokens达到37%，优于最佳基线（33%），且速度提升高达2.8倍。

## Abstract
As reasoning models emit chains of thought tens of thousands of tokens long, KV cache increasingly becomes a deployment bottleneck. Existing cache eviction methods rank tokens by attention weight, which is a noisy importance proxy in long reasoning traces, and prohibits the use of fused kernels in production inference by forcing the model to materialize the attention matrix. In this work, we instead score tokens with a metric we term the epiphany score: the change in the model's internal representation, read directly from the forward pass with no attention matrix and negligible extra state. Our resulting cache eviction method, EpiKV, requires no training, classifier, or custom kernel, and can be used directly in FlashAttention inference stacks unchanged -- scaling to a 16x longer feasible context than attention-based scoring. upper-mid layers negatively) and remove a positional trend with a causal rolling z-score. ## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：长链推理模型（如 DeepSeek-R1）生成数万 tokens 的思维链，导致 KV 缓存线性增长，成为部署中的内存瓶颈。现有 KV 缓存淘汰方法依赖**注意力权重**来评估 token 重要性，但在长推理轨迹中存在严重缺陷：
    - 注意力权重是噪声信号：注意力汇聚（attention sinks）会吸收大量权重而不反映内容重要性，填充 token 也容易获得高权重却无关紧要。
    - 架构代价高：读取注意力权重需要物化完整的 n×n 注意力矩阵，这迫使模型使用 eager 模式而非 FlashAttention 等融合内核，导致内存爆炸（80GB GPU 在 8192 tokens 时耗尽）。
- **动机**：提出一种无需注意力矩阵的 token 重要性评分方法，使缓存淘汰能与 FlashAttention 无缝兼容，从而支持更长的上下文（16 倍扩展）并提升推理速度。

## 2. 论文提出的方法论

### 核心思想
- 使用**顿悟分数（epiphany score）** 替代注意力权重：通过前向传播中隐藏状态的变化幅度来衡量 token 的重要性，该变化反映了模型内部表征的“顿悟式”转换（如推理步骤的结论、关键洞察的浮现）。
- 发现两层带解剖结构：在 32 层的模型中，**Band A（7-13 层）** 的隐藏状态变化与 token 重要性正相关（大变化代表重要 token），**Band B（18-25 层）** 的变化与重要性负相关（大变化代表可丢弃的填充 token）。最终得分 s(t) = z₁₀(t) - z₂₁(t)，其中 z 为因果滚动 z 分数。
- 因果滚动 z 分数：消除原始信号中随时间单调递减/递增的位置趋势，使评分仅反映局部偏差而非绝对位置。

### 技术细节
1. **隐藏状态差分**：定义 gₗ(t) = ||hₗ(t) - hₗ(t-1)||₂，即第 l 层在生成 token t 后的 L2 变化。
2. **因果滚动平均**：对 gₗ 保持长度为 w=64 的因果滑动窗口，计算均值 ¯gₗ(t)。
3. **滚动 z 分数**：zₗ(t) = (gₗ(t) - μₗ(t)) / (σₗ(t) + ε)，其中 μ 和 σ 来自滑动窗口。
4. **双带组合**：s(t) = z₁₀(t) - z₂₁(t)（分别对应 Band A 和 Band B 的代表层）。
5. **淘汰策略**：保留所有预填充 token 和一个近期窗口（min(128, K/4)），然后从剩余 token 中选择得分最高的 K 个。评分在 token 生成时一次性计算并冻结。

### 算法流程（文字说明）
- 对每个生成的 token t：
    - 执行前向传播，收集各层隐藏状态 hₗ(t)（利用 `output_hidden_states` 接口，无需 attention 矩阵）。
    - 计算 gₗ(t)，更新滑动统计量。
    - 计算 z₁₀(t) 和 z₂₁(t)，得到 s(t)。
    - 维护候选 token 列表（排除预填充和近期窗口），保留得分最高的 K 个 token 的 KV 缓存。
- 模型继续解码时，仅使用保留的 KV 缓存进行注意力计算（仍可结合 FlashAttention）。

- **注意力自由可行**：EpiKV 在 4096 缓存下 MAT-500 准确率达 72%，与最强注意力基线 ThinKV（71%）和 H2O（67%）相当，且无需注意力矩阵。

- **内存扩展**：注意力矩阵使 80GB GPU 在 8192 tokens 时 OOM，而 FlashAttention 可支撑至 65536 tokens（16 倍）。
- **两层带解剖**：低层（7-13）正相关，中高层（18-25）负相关，且该现象随任务难度变化（GSM8K 时正相关移至早期层），表明信号追踪推理计算发生的真实位置。
- **位置趋势消除**：原始隐藏状态差分信号受位置漂移污染，因果滚动 z 分数有效去除趋势，提升排名质量。

## 7. 优点
- **无需注意力矩阵**：首次在推理缓存淘汰中完全避免注意力矩阵，直接兼容 FlashAttention，具有实际部署优势。
- **零额外训练/校准**：无需离线训练、分类器或自定义内核，可直接插入 vLLM、TGI 等推理栈。
- **信号设计巧妙**：基于残差流变化与机制可解释性工作的发现（ROMEMIT 定位事实检索层），提供可解释的 token 重要性度量。
- **位置趋势校正**：因果滚动 z 分数是一种通用廉价的方法，可应用于其他残差流信号。
- **广泛消融与验证**：通过反事实遮挡标签验证了层解剖结构，并提供标签资源。

## 8. 不足与局限
- **实验覆盖有限**：仅测试一个模型（DeepSeek-R1-Distill-LLaMA-8B），未验证跨架构（如 LLaMA-70B、Mistral、Qwen）或更大规模的泛化性。
- **统计力度弱**：AIME-2024 仅 30 题，差异可能不显著；GSM8K 仅用于探针，未做准确率 head-to-head。
