## 摘要
交错多模态推理通过在多步生成过程中重新访问视觉证据来改进视觉定位，但现有方法通常依赖令牌回放，重复转发选定的视觉令牌。一个自然的捷径是直接重用历史视觉键值（KV）缓存。然而，我们发现了该策略的一个关键失败模式：缓存的视觉键已经绑定到其原始位置上下文。这种过时的位置绑定会在后续解码上下文中扭曲注意力，并可能触发严重的自回归解码崩溃。这种失败表明，有效的缓存复用需要在与当前解码状态兼容的位置下重建视觉证据，而不是直接复制绑定位置的缓存条目。为此，我们提出位置重绑定缓存复用（PRCR），一种用于无回放视觉重访的缓存级框架。PRCR将原始视觉KV缓存及其原始空间坐标一起存储，然后重新分配与位置兼容的坐标以选择条目，并在将重建的缓存注入到活动解码器缓存之前重新绑定它们的键。这种设计在保留文本位置连续性和相对视觉结构的同时重用了历史视觉证据。在多个多模态推理基准上的实验表明，PRCR达到了与回放相当或更好的性能，平均准确率提高了5%，并将视觉重访计算减少了多达数万倍。

## Abstract
Interleaved multimodal reasoning improves visual grounding by revisiting visual evidence during multi-step generation, yet existing methods typically rely on token replay, repeatedly forwarding selected visual tokens. A natural shortcut is to reuse the historical visual key-value (KV) cache directly. However, we identify a critical failure mode of this strategy: cached visual keys are already bound to their original positional context. Such stale positional binding distorts attention under later decoding contexts and can trigger severe autoregressive decoding collapse. This failure suggests that effective cache reuse requires reconstructing visual evidence under positions compatible with the current decoding state, rather than directly copying position-bound historical cache entries. To this end, we propose Position Rebinding Cache Reuse (PRCR), a cache-level framework for replay-free visual revisiting. PRCR stores raw visual KV cache together with their original spatial coordinates, then reassigns position-compatible coordinates to select entries and rebinds their keys before injecting the reconstructed cache into the active decoder cache. This design reuses historical visual evidence while preserving textual positional continuity and relative visual structure. Experiments across multiple multimodal reasoning benchmarks show that PRCR achieves replay-level or better performance, improving average accuracy by 5 percent and reducing visual-revisiting computation by up to tens of thousands of times.

- **核心问题**：一个自然的替代方案是直接复用历史视觉 **KV 缓存**（Direct KV Cache Reuse），但作者发现该策略会导致灾难性的自回归解码崩溃（accuracy 从 66.4% 骤降至 23.5%）。崩溃根源在于：缓存的视觉 **key** 已通过 RoPE 与原始 2D 空间坐标绑定，当被直接插入到后续的解码上下文时，**过时的位置绑定** 会扭曲注意力分布，进而引发反复的重复 token 循环（stuck decoding）。
- **整体意义**：本文揭示了视觉 KV 缓存直接复用的失败模式，并提出了一种无回放的缓存级框架 **PRCR**，通过位置重绑定（Position Rebinding）实现高效的视觉证据回顾，在保持推理质量的同时大幅降低计算开销。

## 2. 论文提出的方法论

### 核心思想
PRCR 的核心是 **避免直接复制已绑定位置的缓存**，而是存储原始（pre‑RoPE）的视觉 KV 缓存及其原始空间坐标，在回顾时根据当前解码状态重新分配与位置兼容的坐标，并重新对 key 进行 RoPE 编码，从而重构出位置正确的缓存条目。

### 关键技术细节
PRCR 包含三个模块（如图 4 所示）：

1. **Raw Visual Evidence Memory (RVEM)**
   - 在预填充阶段，存储每个视觉 token 的 **原始 key/value 投影**（即 RoPE 之前的 \(k^{\text{raw}}, v^{\text{raw}}\)），以及该 token 的原始 2D 空间坐标 \(\mathbf{p}_m = (h_m, w_m)\)。
   - 构建记忆库 \(\mathcal{M} = \{ \{ (k^{\text{raw}}_{m,\ell}, v^{\text{raw}}_{m,\ell}) \}_{\ell\in\mathcal{L}}, \mathbf{p}_m \}_{m=1}^M\)。

2. **Position Reassignment for Cache Reinsertion**
   - 当 ADS（Attention-Driven Selection）在解码步 \(\tau\) 选中一组视觉索引 \(\mathcal{I}_\tau\) 时，PRCR 为其分配新的坐标。作者提出了 **Position-Consistent Reinsertion (PCR)** 策略：
     - 保持下一个文本位置 \(p^{\text{txt}}_\tau + 1\) 不变；
     - 向左扩展插入区间为 \((p^{\text{txt}}_\tau - L, p^{\text{txt}}_\tau + 1)\)（默认 \(L=2\)）；
     - 对每个空间轴 \(a \in \{h, w\}\)，计算选中子集的范围 \(p^{(a)}_{\tau,\min}, p^{(a)}_{\tau,\max}\)；
     - 归一化得到相对坐标 \(u^{(a)}_{\tau,j} = (p^{(a)}_{m_j} - p^{(a)}_{\tau,\min})/(p^{(a)}_{\tau,\max} - p^{(a)}_{\tau,\min})\)；
     - 重锚定坐标：\(\hat{p}^{(a)}_{\tau,j} = (p^{\text{txt}}_\tau - L) + (L+1) \cdot \frac{1 + (K_\tau - 1)u^{(a)}_{\tau,j}}{K_\tau + 1}\)。
   - 该策略保证了文本位置连续性，并保留了选中视觉子集的相对空间布局。

3. **Replay-Free Cache Decoding**
   - 对于每个 transformer 层 \(\ell\)，从 RVEM 中取出选中索引的原始 key/value：\(\mathbf{K}^{\text{raw}}_{\tau,\ell}, \mathbf{V}^{\text{raw}}_{\tau,\ell}\)；
   - 使用新分配的坐标 \(\hat{\mathbf{P}}_\tau\) 对 key 重新应用 RoPE：\(\hat{\mathbf{K}}^{\text{vis}}_{\tau,\ell} = \text{RoPE}(\mathbf{K}^{\text{raw}}_{\tau,\ell}, \hat{\mathbf{P}}_\tau)\)，value 保持不变；
   - 将重构的视觉缓存 \(\hat{\mathbf{C}}^{\text{vis}}_{\tau,\ell} = (\hat{\mathbf{K}}^{\text{vis}}_{\tau,\ell}, \mathbf{V}^{\text{raw}}_{\tau,\ell})\) 拼接到现有的解码器缓存 \(\mathbf{C}^{\text{ctx}}_{\tau,\ell}\) 中，形成活跃缓存 \(\mathbf{C}^{\text{act}}_{\tau,\ell}\)；
   - 后续解码按标准自回归进行（Algorithm 1）。

## 6. 论文的主要结论与发现

1. **直接复用历史视觉 KV 缓存会导致严重解码崩溃**（accuracy 从 66.4% 降至 23.5%），崩溃表现为重复 token 循环，根源在于 stale positional binding 扰乱注意力。
2. **PRCR 彻底避免了这一崩溃**，且性能达到甚至超越 Token-Replay（例如在 Qwen3-VL-8B 上 M3CoT 总体准确率 68.68% vs Token-Replay 68.46%）。
3. **计算效率大幅提升**：在 32B 模型上，PRCR 视觉重访 FLOPs 仅为 Token-Replay 的约 1/3300（如 \(K=128\) 时 125.83M vs 7.75T FLOPs），额外 GPU 内存开销仅 0.3%–0.64%。
4. **位置重绑定与相对结构保持是关键**：消融实验表明，仅重绑定（无位置保持）仍有 7.93% stuck rate，而 PCR 完全消除 stuck，accuracy 最高。

## 7. 优点

- **高效无回放**：完全在缓存层面操作，避免了重复前向计算，适合需要频繁回顾的复杂推理任务。
- **无需训练**：可直接与现有 MLLM 和 ADS 模块结合，兼容性强。
- **保持推理连续性**：PCR 策略不改变后续文本位置，保留文本位置连续性；同时通过归一化相对坐标保持视觉子集的原始空间结构。
- **注意力稳定**：重新计算后的注意力模式与 Token-Replay 相似，无注意力扰动（见 Figure 8）。
- **消融彻底**：对每个设计选择（重插入策略、扩展长度、锚定方向）均有实验验证。

## 8. 不足与局限

- **无法处理动态分辨率调整或局部缩放**：PRCR 依赖预填充阶段生成的原始 KV 缓存。若某些方法（如 ICoT 中的局部放大）对选中区域进行缩放后重新编码，则缩放后的特征没有对应的预填充缓存，PRCR 无法复用。这会限制其在需要动态视觉调整场景下的应用。
- **未来方向**：作者提出两个扩展思路——多尺度 KV 预填充（生成多个分辨率下的缓存）和学习型尺度感知重绑定（通过映射网络适配不同分辨率），但尚未实现。
- **实验覆盖**：虽然模型和基准较全，但所有方法均基于“ADS 选择相同证据”的设定，未测试其他证据选择策略（如基于区域裁剪、生成式图像）。此外，未在真实世界复杂场景（如自动驾驶视频推理）上验证。
- **内存开销**：尽管额外内存占比很小，但对于超大规模模型（如 70B+）或极多视觉 token 的情况，存储原始缓存可能需要更多显存优化。
