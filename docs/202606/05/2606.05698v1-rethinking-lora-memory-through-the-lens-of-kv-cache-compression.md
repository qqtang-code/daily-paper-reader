## 摘要
参数化检索增强将文档信息编码为轻量级的、文档特定的模块（如LoRA适配器），从而减少了将所有证据作为输入上下文的需求。然而，这种参数侧内存与存储在KV缓存中的上下文侧内存如何相互作用仍不清楚。我们通过逐步驱逐文档键值状态并测量文档LoRA在保留上下文之外的贡献，在文档级问答中研究了这种相互作用。我们发现，当KV缓存基本完整时，文档LoRA贡献很小，但在激进压缩下变得日益有用——当没有文档上下文保留时，可恢复13-21个ROUGE-L点。当基础模型编码文档且仅在答案生成期间应用适配器时，增益最大，这表明文档LoRA更应被理解为解码时的参数化内存，而非文档编码器。最后，QA风格的监督比原始上下文的下一个标记预测产生了明显更强的适配器。这些结果将文档LoRA定位为一种互补的内存通道，其价值恰好在上下文侧证据稀缺时显现。

## Abstract
Parametric retrieval augmentation encodes document information into lightweight, document-specific modules such as LoRA adapters, reducing the need to include all evidence as input context. However, it remains unclear how this parameter-side memory interacts with context-side memory stored in the KV cache. We study this interaction in document-level question answering by progressively evicting document key-value states and measuring when a document LoRA contributes beyond the retained context. We find that document LoRA adds little when the KV cache is largely intact, but becomes increasingly useful under aggressive compression, recovering 13-21 ROUGE-L points when no document context remains. The gain is largest when the base model encodes the document, and the adapter is applied only during answer generation, suggesting that document LoRA is better understood as decoding-time parametric memory than as a document encoder. Finally, QA-style supervision produces substantially stronger adapters than raw-context next-token-prediction. These results position document LoRA as a complementary memory channel whose value emerges precisely when context-side evidence is scarce.

## 1. 论文的核心问题与整体含义（研究动机和背景）

检索增强生成（RAG）通常将检索到的文档直接拼接到输入上下文中，这会导致 KV 缓存膨胀和计算开销。两种主流缓解思路是：(a) **KV 缓存压缩**——通过驱逐或合并保留重要令牌，减少上下文侧内存占用；(b) **参数化检索增强**——将文档信息编码到轻量级文档特定模块（如 LoRA 适配器）中，作为参数侧内存。然而，这两类方法通常被孤立研究，其**相互作用**尚不明确。本文旨在探究：文档 LoRA 与 KV 缓存压缩如何互补？在不同的压缩程度下，何时 LoRA 真正发挥作用？LoRA 应作用于推理的哪个阶段？何种训练目标能产生最强的参数侧记忆？

整体含义：文档 LoRA 不是 KV 缓存的替代品，而是一种**互补记忆通道**，其价值恰好在上下文侧证据稀缺时（激进压缩时）才凸显出来。

## 2. 论文提出的方法论

### 核心思想
为每个文档训练一个单独的 LoRA 适配器，然后通过**逐步压缩文档部分的 KV 缓存**（保留率从 100% 降至 0%），测量有/无 LoRA 时的 ROUGE-L 差异，从而诊断两种记忆源的相对贡献。

### 关键技术细节

- **文档 LoRA 训练**：
  - 将文档拆分为长度为 64 词的块。
  - 使用**合成 QA 对**作为默认监督信号（用语言模型对每个块生成问答对）。
  - 训练时冻结基模型参数，仅更新 LoRA 参数（DoRA 使能，rank=16，alpha=32，dropout=0.1，仅作用于 MLP 投影层）。
  - 优化：AdamW，lr=1e-4，batch_size=4，weight_decay=0.001，4 epochs。

- **KV 缓存压缩**：
  - 使用 **Compactor** 算法（结合非因果注意力和近似 leverage 分数）为每个文档令牌评分，保留得分最高的前 `rN` 个令牌（r=1-ρ，ρ 为压缩率）。
  - 压缩率 ρ 取值为：0%, 80%, 90%, 95%, 99%, 99.5%, 100%（100% 表示完全移除文档上下文）。
  - 压缩仅作用在文档部分，用户前缀不变。

- **推理阶段控制**（用于诊断 LoRA 最佳使用时机）：
  - 将推理分为三个阶段：文档预填充（Prefill）、压缩评分（Score）、答案解码（Decode）。
  - 比较四种模式：
    - Base：全程无 LoRA。
    - Adapter score + adapter prefill（默认）：LoRA 全程启用。
    - Base KV + adapter decode：基模型执行预填充和评分，仅在解码时加载 LoRA。
    - Base score + adapter prefill：LoRA 执行预填充和解码，评分用基模型。

- **训练格式控制**（用于诊断最佳监督信号）：
  - Synthetic QA（默认）：仅问答对，损失仅作用于答案令牌。
  - Raw Context：原始文档文本，损失作用于所有令牌。
  - Chunk → Next：相邻块预测，损失作用于目标块。
  - Context + QA：源上下文 + 问题，损失仍只作用于答案。

## 6. 论文的主要结论与发现

1. **文档 LoRA 在激进 KV 压缩时贡献显著**：当 KV 缓存保留大部分文档时，LoRA 增益很小；当压缩率达到 99%-100% 时，LoRA 可恢复 13-21 ROUGE-L 点（无上下文时）。这证明了记忆的互补性。
2. **LoRA 更适合作为解码时记忆，而非文档编码器**：基模型预填充和评分 + 仅解码时启用 LoRA（Base KV + adapter decode）通常是最高效的方案，优于全程使用 LoRA。
3. **QA 式监督产生最强的文档 LoRA**：合成 QA 训练（仅问答对）远优于原始文本 NTP、块续写或上下文+QA 格式；仅仅“接触”文档文本不足以形成有效的问答记忆。

## 7. 优点

- **诊断性视角新颖**：首次系统研究 LoRA（参数侧记忆）与 KV 缓存压缩（上下文侧记忆）的交互，并揭示了互补性。
- **方法论控制精细**：通过 7 级压缩曲线、4 种推理阶段模式、4 种训练格式的交叉对比，准确找到了 LoRA 的“有效区间”和最佳使用方式。
- **消融实验充分**：覆盖压缩算法、LoRA 目标模块、超参数等，增强了结论的鲁棒性。
- **跨模型验证**：使用 4B 和 8B 两个规模的指令模型，结果一致，避免了单一模型偏差。
- **结论具有实用指导意义**：建议在 KV 缓存预算紧张时采用“基模型预填充 + 解码时 LoRA”的方案，并推荐使用 QA 监督训练适配器。
