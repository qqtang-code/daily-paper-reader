## 摘要
视觉基础模型通常被训练为静态特征提取器，将任务适应的负担转移给大型下游模型。我们提出一种替代范式：不是仅仅将视觉特征馈入语言模型，而是使用语言本身动态地指导视觉编码器。我们的方法，即语言指导的视觉嵌入（LIVE），利用语言作为高层指导，在推理时生成以任务为中心的嵌入，消除了对特定任务重新训练的需求。这使得编码器能够聚焦于输入中与上下文相关的方面，产生更可控和更可泛化的表示。实验上，LIVE减少了视觉幻觉（在MMVP上提升34个百分点），在视觉问答上超越了参数多出几个数量级的视觉-语言模型，并且能够泛化到未见过的指令和任务——为自适应、指令驱动的视觉智能提供了直接路径。

## Abstract
Vision foundation models are typically trained as static feature extractors, placing the burden of task adaptation onto large downstream models. We propose an alternative paradigm: instead of solely feeding visual features into language models, we use language itself to dynamically guide the vision encoder. Our method, Language-Instructed Vision Embeddings (LIVE), leverages language as high-level guidance to produce task-centric embeddings at inference time, removing the need for task-specific retraining. This enables the encoder to focus on contextually relevant aspects of the input, yielding more controllable and generalizable representations. Empirically, LIVE reduces visual hallucinations (+34 points on MMVP), surpasses vision-language models with orders of magnitude more parameters on visual question answering, and generalizes to unseen instructions and tasks -- offering a direct path toward adaptive, instruction-driven visual intelligence.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题：** 现有视觉基础模型（如 CLIP、SigLIP）通常作为静态特征提取器，其表示是“通用”且与下游任务无关的。用户需要借助大型下游语言模型（LLM）进行任务适配，不仅计算成本高昂，而且静态编码器容易忽略细粒度细节，导致视觉幻觉（hallucination）。
- **整体含义：** 本文提出一种新范式——**语言指导的视觉嵌入（LIVE）**——将语言直接注入视觉编码器内部，使其在推理时动态聚焦于任务相关的图像区域，从而生成**可控、可泛化**的视觉表示。这避免了为每个任务重新训练或依赖庞大 LLM 解码器，为轻量级、指令驱动的视觉智能提供了直接路径。

## 2. 方法论
- **核心思想：** 使用文本指令（query）作为额外输入，通过学习一个语言条件的视觉编码器 \(E_{\text{live}}(x, T(q))\)，使视觉嵌入根据指令动态调整，突出任务相关特征、抑制无关特征。
- **关键技术细节：**
  1. **模型架构：** 基于两塔架构（预训练图像编码器 + 文本编码器）。文本编码器保持冻结，将文本指令嵌入通过一个线性投影层注入视觉 Transformer 的输入序列（作为额外的 token）。
  2. **训练目标：** 采用 sigmoid 对比损失（类似 SigLIP），最小化匹配的图像-指令-答案三元组 (\(x, q, a\)) 的视觉嵌入与答案文本嵌入之间的距离。
  3. **训练数据生成：** 为解决缺少大规模三元组数据的问题，使用 Gemini 2.0 Flash 作为知识源，在 ImageNet 图像上自动生成多样化的问题-答案对，共获得约 1640 万个三元组。
  4. **推理：** 仅运行轻量级视觉编码器 \(E_{\text{live}}\)，无需 LLM 解码器，可预计算和缓存文本嵌入以进一步降低计算成本。

  - **GQA**：评估视觉问答和场景图推理。
  - **ImageNet†, Caltech101†, SUN†, RefCOCO†**：利用 Gemini 生成测试指令和答案，用于测量对 LLM 知识源（Gemini，作为 100% 准确 Oracle）的逼近程度（零样本）。
  - **ImageNet OCR 测试**：评估文本识别能力。
  - **ImageNet 对抗样例（印刷攻击）**：评估对文本干扰的鲁棒性。

- **评估指标：** Top-1 检索准确率。

## 4. 资源与算力
- **训练算力：** 使用 256 个 TPUv3 核心，批次大小 8192，训练步数 122k 步。学习率为 0.001。

- **数据生成：** 使用 Gemini 2.0 Flash API 离线生成，无在线 LLM 开销。

## 5. 实验数量与充分性
- **实验组数：** 非常充分。涵盖 7 个以上不同基准（MMVP、GQA、4 个 Gemini 标注数据集、OCR、对抗鲁棒性），并进行了多项消融研究。
  - 训练数据来源消融（Open Images vs WebLI vs CC3M vs 本工作生成数据）。
  - 视觉编码器规模消融（ViT-T → B → SO400M）。
  - 语言注入深度消融（Layer 1/4/8）。

## 6. 主要结论与发现
- **指令控制有效：** LIVE 在 MMVP 上比所有基线（包括 SOTA 集成模型 BRAVE）提升 34 个百分点（76.3% vs 42.0%）；在 GQA 上比 LLaVa 高 7.9 个百分点（71.2% vs 63.3%），但参数不到后者的 10%。
- **泛化性强：** 在完全未见过的指令和图像领域（Caltech101、SUN、RefCOCO）上，LIVE 显著优于所有静态嵌入基线，接近 Gemini Oracle 的准确率（缩小差距最多 49 个百分点）。
- **训练数据关键：** 使用 Gemini 生成的大规模多样化三元组数据至关重要，现有图像语言数据集（如 CC3M-VQA）不足以训练出高精度的指令感知编码器。
- **语言注入位置影响：** 早期注入（第 1 层）有利于减少幻觉（MMVP），晚期注入（第 8 层）有利于关系推理（GQA）。
- **指令特异性驱动增益：** 换成中性指令或仅用类名作为监督，性能急剧下降，证明增益源于语言指令对视觉表示的显式引导。

## 7. 优点
- **创新性范式：** 将语言知识反向蒸馏到视觉编码器，使视觉编码器自身具备指令跟随能力，颠覆了“静态视觉+大 LLM 解码器”的传统路线。
- **高效轻量：** 推理时仅需视觉编码器，无 LLM 开销，可部署于资源受限设备；参数仅增加极少量。
- **可控性与可解释性：** 注意力热图显示模型能根据指令精准定位相关区域（如“水果” vs “文字”），提供了可解释的控制机制。
- **强泛化：** 在多种未见图像域和指令类型下仍表现优异，展示了良好的零样本迁移能力。
- **实验全面：** 覆盖幻觉、VQA、跨域泛化、OCR、对抗鲁棒性等多个维度，消融分析深入，有力地支持了结论。

## 8. 不足与局限
- **指令格式依赖：** 模型在面对复杂、组合式查询（如多步推理或否定）时效果可能下降，因预训练文本编码器能力有限。
- **指令设计需手动优化：** 有效指令的措辞、细粒度仍需针对具体任务调试，缺乏自动发现最优指令的机制。
- **潜在滥用风险：** 指令可控性可被恶意利用（如引入偏见或生成有害内容），目前没有内置的指令审查机制。
- **训练数据来源单一：** 目前训练数据仅基于 ImageNet（带 Gemini 生成的问题），虽然泛化到其他域，但可能存在数据分布偏差（如缺失真实世界复杂场景）。
- **与 LLM 知识源仍有差距：** 在零样本设置下，LIVE 与 Gemini Oracle 仍有 23-41 个百分点的差距，尤其是在细粒度推理任务上。
