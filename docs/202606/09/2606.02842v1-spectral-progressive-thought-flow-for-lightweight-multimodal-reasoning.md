## 摘要
多模态空间推理通常依赖于长链的中间文本和视觉思维，其中积累的视觉标记和密集的跨模态注意力会导致大量的计算和内存开销。为了解决这一挑战，我们提出了谱渐进思维流（SpecFlow），一种新颖的轻量级多模态空间推理框架，该框架在固定大小的离散余弦空间中表示中间视觉思维。通过利用强大的能量压缩，SpecFlow保留了全局布局和关系结构，同时仅在需要增加空间精度时引入高频细节。为了使视觉状态演化与语言意图对齐，无分类器引导使得自回归文本思维能够指导基于流的视觉工作空间/状态更新，而无需扩展上下文。因此，SpecFlow维护了一个有界的视觉工作空间，其更新仅依赖于当前的视觉状态和累积的文本轨迹，从而实现了长时域推理，且延迟和内存使用稳定，与推理深度无关。实验结果表明，SpecFlow在实现具有竞争力或更优推理性能的同时，将计算和KV缓存成本降低了多达2.1倍。

## Abstract
Multimodal spatial reasoning often relies on long chains of intermediate textual and visual thoughts, where accumulating visual tokens and dense cross-modal attention incur substantial computation and memory overhead. To address this challenge, we propose Spectral-Progressive Thought Flow (SpecFlow), a novel lightweight multimodal spatial reasoning framework that represents intermediate visual thoughts in a fixed-size discrete cosine space. By exploiting strong energy compaction, SpecFlow preserves global layout and relational structure while introducing high-frequency details only when increased spatial precision is required. To align visual state evolution with linguistic intent, classifier-free guidance enables autoregressive textual thoughts to steer flow-based updates of the visual workspace/state without expanding the context. As a result, SpecFlow maintains a bounded visual workspace whose updates depend only on the current visual state and accumulated textual trace, enabling long-horizon inference with stable latency and memory usage independent of reasoning depth. Empirical results show that SpecFlow achieves competitive or superior reasoning performance while reducing computation and KV cache costs by up to 2.1 times.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：多模态空间推理（Multimodal Spatial Reasoning, MSR）常需要长链的中间文本与视觉思考（如MVoT范式），但视觉token的重复生成和累积会导致上下文长度和KV缓存急剧增长，尤其在多跳推理中，计算和内存开销随推理深度线性增加，最终限制可用上下文窗口、加剧内存带宽压力并导致高延迟。
- **现有方法的不足**：
  - 明确token剪枝方法（如FastV、SparseVLM）依赖局部启发式重要性准则，未考虑多跳推理中空间线索的复用需求，容易欠剪枝或过剪枝。
  - 隐式潜在推理方法（如Heima、CODI）将中间思考内化为连续潜在状态，但缺乏显式监督和可控性，在复杂空间推理任务上表现不佳。
- **SpecFlow的动机**：中间视觉思考在空间推理中通常只需承载稀疏的抽象线索（如全局布局和几何关系），而非像素级细节。直接在像素空间生成思考会引入不必要的自由度和成本。因此，作者提出将中间视觉思考表示为固定大小、频率受限的离散余弦空间状态，实现轻量级、内存稳定的多跳推理。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：用固定大小的连续视觉工作空间（visual workspace）替代累积增长的视觉token序列；在离散余弦域（DCT）中维护该空间，利用能量压缩特性保留低频全局结构，仅当需要时逐步激活高频细节；通过流匹配（Flow Matching）在余弦空间中执行确定性状态更新，并利用无分类器引导（Classifier-Free Guidance, CFG）使文本思考指导视觉状态演化。
- **关键技术细节**：
  - **谱渐进频率分配（Spectral-Progressive Frequency Allocation）**：将视觉状态通过块余弦投影（块大小 \(b \times b\)）变换到余弦域；定义一个随时间 \(t\) 扩大的二进制掩码 \(M(t)\)，只激活低频至中高频的系数。早期状态只需少量低频系数即可重建全局布局，后期逐步引入细粒度信息。
  - **余弦空间流匹配（Cosine-Space Flow Matching）**：在余弦系数空间定义连续时间流，用可学习速度场 \(u_\theta\) 控制演化。训练时，对数据样本和噪声样本在系数空间线性插值，回归常值目标速度 \(X_0 - X_1\)，损失函数为 \(L_{FM} = \mathbb{E}[||u_\theta(M(t)\odot X_t, t, c) - (X_0-X_1)||^2]\)。推理时，每次跳从当前视觉状态出发，用固定步数（如5步）的欧拉求解器进行确定性子采样。
  - **文本引导与CFG**：将当前文本轨迹和原始查询组合成条件 \(c_i\)，通过CFG在速度层面增强文本对齐：\(u_{\text{guid}} = u_\theta(\cdot, t, \emptyset) + w (u_\theta(\cdot, t, c_i) - u_\theta(\cdot, t, \emptyset))\)。引导后的速度驱动余弦系数演化，得到更新后的视觉状态 \(v_{i+1}\)，随后自回归生成下一文本思考 \(t_{i+1}\)。
  - **整体推理流程**（如Algorithm 1）：每个hop依次进行条件构建、余弦投影、迭代更新（应用掩码和CFG速度）、逆余弦变换得到新视觉状态、文本生成。由于视觉工作空间被覆盖，之前的状态可丢弃，内存仅依赖文本轨迹长度和固定尺寸的视觉状态。

## 7. 优点
- **内存稳定**：固定大小的视觉工作空间，推理深度不增加KV缓存和上下文长度，适用于长程推理。
- **高效轻量**：余弦投影是固定可逆变换，无需学习潜在瓶颈；流匹配确定性采样，仅需少量步骤。
- **可控性**：CFG速度级引导使文本意图直接控制视觉更新，避免视觉-文本不对齐。
- **谱渐进机制**：粗到细的进度与多跳推理自然匹配，早期快速建立全局结构，后续逐步精化，平衡效率与精度。
- **理论支撑**：附录证明低频限制可降低速度场的Lipschitz常数，从而允许更大求解步长，解释其轻量性。

## 8. 不足与局限
- **任务特定训练**：每个任务类别训练独立的FM模型，限制了泛化性（尽管作者声明这是为了公平比较）。
- **仅适用空间推理**：方法专为多跳空间推理设计，对需要高保真图像细节的任务（如细粒度图像识别）可能不适用。
- **实验覆盖**：虽然基准多样，但未涵盖真实世界机器人导航或3D空间推理等更复杂场景。
- **潜在偏差**：所有实验基于Qwen系列模型，结果是否能迁移到其他基础模型（如LLaVA、InternVL）尚未验证。
- **计算开销**：虽然推理比MVoT轻量，但训练仍需较高算力（16张H100训练数十小时），且每次跳仍需执行多次流匹配步骤（T=5），对于特低延迟场景可能仍有优化空间。
