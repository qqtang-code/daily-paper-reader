## 摘要
传统大语言模型在解码过程中始终加载完整的KV缓存，导致超长上下文服务面临严重的GPU内存瓶颈。本报告提出基于DeepSeek-V4架构构建的神经记忆索引器驱动的新型推理范式——前瞻稀疏注意力（LSA）。LSA并非被动关注所有历史令牌，而是主动预测未来上下文需求，仅将查询关键性KV块保留在GPU内存中。关键之处在于，我们通过无骨干的解耦训练策略实例化该架构：将索引器表述为标准双编码器架构，使用标准检索训练框架独立训练，无需将庞大的骨干模型加载至GPU内存。我们证明，这种“少即是多”的范式在最大化服务效率的同时，能在依赖长期全局记忆的任务中充当有效的注意力去噪器。在主流长上下文评估套件（如LongBench-v2、LongMemEval和RULER）中，FM-DS-V4将物理KV缓存平均占用压缩至全上下文基线的仅13.5%，同时始终保持或略微提升下游准确率（平均绝对边际+0.6%）。关键的是，在极端500K规模下，FlashMemory将物理KV缓存开销降低超过90%，且不破坏骨干模型的核心推理能力。

## Abstract
Conventional LLMs keep the full KV cache loaded during decoding, causing a severe GPU memory bottleneck for ultra-long context serving. In this report, we propose Lookahead Sparse Attention (LSA), a novel inference paradigm powered by a Neural Memory Indexer built upon the DeepSeek-V4 architecture. Rather than passively attending to all historical tokens, LSA proactively predicts future context demands and preserves only the query-critical KV chunks in the GPU memory. Crucially, we instantiate this architecture via a backbone-free decoupled training strategy. By formulating the indexer as a standard dual-encoder architecture, we train it independently using standard retrieval training frameworks without ever loading the massive backbone model into GPU memory.   We demonstrate that this "less is more" paradigm significantly maximizes serving efficiency while acting as an effective attention denoiser in tasks that rely on long-term global memory. Across primary long-context evaluation suites (e.g., LongBench-v2, LongMemEval, and RULER), FM-DS-V4 compresses the average physical KV cache footprint down to merely 13.5% of the full-context baseline, while consistently preserving or slightly elevating downstream accuracy (+0.6% absolute margin on average). Crucially, at extreme 500K scales, FlashMemory suppresses the physical KV cache overhead by over 90% without destabilizing the backbone's core reasoning capacities.

- **整体含义**：作者提出了一种“少即是多”的推理范式——前瞻稀疏注意力（Lookahead Sparse Attention, LSA），通过神经记忆索引器主动预测未来解码步骤所需的 KV 块，仅将关键块加载至 GPU，从而大幅降低内存消耗，同时借助注意力去噪效应维持甚至提升下游精度。该方法在 DeepSeek-V4 架构上实现，平均 KV 缓存压缩至全上下文的 13.5%，精度平均提升 0.6%。

## 2. 方法论

### 核心思想
- **LSA 范式**：在解码过程中，每隔固定 τ 步（τ=64）触发一次 Memory Indexer，根据当前查询隐状态动态评估所有历史压缩 KV 条目，仅拉回那些预测为“查询关键”的条目（Sigmoid 分数 ≥ 0.5），其余历史块仍留在 CPU冷池中。
- **架构**：Memory Indexer 复用 DeepSeek-V4 中 Lightning Indexer 的结构，但将激活函数从 ReLU 改为 Sigmoid（输出 (0,1)），并将固定 Top-k 选择替换为阈值选择（0.5）。它由一个双编码器构成：查询编码器（低秩投影 + 多头）与固定的压缩索引键。

### 关键技术细节
1. **标签构建（Lookahead Dataset）**：基于冻结的 DS-V4-Flash 骨干，对每个解码 token i 的未来窗口 [t, t+τ-1]，提取所有 CSA 层的原始索引器分数。经过三步去噪得到黄金标签：
   - Softmax 归一化 → Top-p 筛选（p=0.6）→ 跨层多数投票（投票数 ≥ θ=3 的条目为正标签）。
2. **训练**：将索引器视为标准检索模型，仅训练查询编码器（低秩矩阵 W_DQ, W_IU_Q, W_w），使用 Focal Loss（γ=2）进行点式二分类，负采样比 3:1。历史压缩键 KI_Comp 完全冻结，训练过程无需加载主模型。
3. **最优配置**：通过约 500 次 Pareto 搜索确定在层 10、12、20 上各放置一个独立索引器，推理时采用 OR 模式（任意一层预测分数≥0.5 即拉回该块）。

## 6. 主要结论与发现

- FM-DS-V4 平均 GPU 内存占用降至 DS-V4-Flash 的 **13.5%**（降低 86.5%），同时平均准确率 **提升 0.6%**（77.5% vs 76.9%）。
- 在 500K 超长上下文中，内存节省达 **90%**（7.79 GB → 0.77 GB）。
- LSA 充当注意力去噪器，过滤无关历史块，在 LongBench-v2-L 上提升 1.9%。
- 然而，在需要密集全局记忆的任务（如 MRCR）上出现严重性能下降（48% vs 76%），因为索引器采用浅层点积匹配，无法捕获密集依赖。
- 长度泛化仅达训练长度的 2 倍（例如训练 512K，安全泛化至约 1M 但超过即崩溃）。
- 对抗性无上下文场景下，内存并未严格恒定（仍有泄漏），因为 Sigmoid 概率在巨大候选池中累积假阳性。
