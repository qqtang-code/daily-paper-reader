## 摘要
随着推理模型生成长达数万token的思维链，KV缓存日益成为部署瓶颈。现有缓存驱逐方法通过注意力权重对token进行排序，但在长推理轨迹中，注意力权重是噪声较大的重要性代理，并且通过强制模型显式化注意力矩阵，阻碍了生产推理中使用融合内核。本文中，我们使用一种称为“顿悟分数”的度量来对token进行评分：该分数是模型内部表示的变化，可直接从前向传播中读取，无需注意力矩阵且额外状态可忽略。由此产生的缓存驱逐方法EpiKV无需训练、分类器或自定义内核，可直接在FlashAttention推理栈中原样使用——相比基于注意力的评分，其可扩展至16倍以上的可行上下文长度。我们通过分析发现，早期层（bottom层且特别是中间层）具有正面顿悟趋同效应对齐，中上层具有负面效应，并通过因果滚动z分数消除了位置趋势。在4096 token缓存下，EpiKV在MATH-500上达到72%，与最强的基于注意力的基线（ThinKV 71%，H2O 67%）相当；归一化延迟KV变体在AIME-2024上以8192 token达到37%，优于最佳基线（33%），速度提升高达2.8倍。

## Abstract
As reasoning models emit chains of thought tens of thousands of tokens long, KV cache increasingly becomes a deployment bottleneck. Existing cache eviction methods rank tokens by attention weight, which is a noisy importance proxy in long reasoning traces, and prohibits the use of fused kernels in production inference by forcing the model to materialize the attention matrix. In this work, we instead score tokens with a metric we term the epiphany score: the change in the model's internal representation, read directly from the forward pass with no attention matrix and negligible extra state. Our resulting cache eviction method, EpiKV, requires no training, classifier, or custom kernel, and can be used directly in FlashAttention inference stacks unchanged -- scaling to a 16x longer feasible context than attention-based scoring. upper-mid layers negatively) and remove a positional trend with a causal rolling z-score. ## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：推理模型（如DeepSeek-R1）生成数万token的思维链，KV缓存线性增长成为内存瓶颈。现有解码时缓存驱逐方法（如H2O、ThinKV、RaaS）均依赖注意力权重进行token重要性排序，但注意力权重存在两个关键缺陷：
  - 噪声大：注意力汇聚（attention sinks）和填充token会吸引权重，与实际内容重要性无关。
  - 架构开销大：读取注意力权重需要显式化注意力矩阵，迫使模型使用`output_attentions=True`，从而放弃FlashAttention等融合内核，导致显存和计算开销剧增（图1显示，单次前向传播在80GB A100上，注意力矩阵模式在8192 token时OOM，而FlashAttention模式可扩展到65536 token）。
- **含义**：亟需一种无需注意力矩阵、可直接从标准前向传播（FlashAttention兼容）中获取重要性信号的方法，以实现高效、可部署的长上下文KV缓存驱逐。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 提出 **EpiKV**，使用“顿悟分数”（epiphany score）评估token重要性。该分数定义为模型内部表示（残差流隐藏状态）的变化幅度，而非注意力权重。
- 进一步提出归一化延迟KV变体（Lag-KV），基于KV向量统计的滞后归一化。

### 关键技术细节
1. **信号定义**：
   - 在解码步骤t，对于层l，计算隐藏状态L2变化：
     \[
     g_l(t) = \| h_l(t) - h_l(t-1) \|_2
     \]
     大的\(g_l(t)\)表示生成token t时模型内部状态发生了显著变化，对应视为“顿悟”token（如推理步骤完成、关键结论等）。

2. **两层解剖（Two-Band Anatomy）**：
   - 通过反事实遮挡实验生成重要性标签（见§3.4），发现32层推理模型中存在两个功能带：
     - **Band A（层7-13）**：\(g_l\)与重要性正相关（Spearman ρ > 0）。
     - **Band B（层18-25）**：\(g_l\)与重要性负相关（Spearman ρ < 0）。
   - 因此，组合成原始分数：\(s(t) = \bar{g}_{10}(t) - \bar{g}_{21}(t)\)，其中\(\bar{g}_l\)是滑动窗口（窗口大小w=64）的滚动均值。

3. **时序趋势校正**：
   - 发现原始信号在推理轨迹内存在单调位置趋势（Band A衰减、Band B增加），导致位置干扰内容。
   - 使用因果滚动z-score消除趋势：
     \[
     z_l(t) = \frac{g_l(t) - \mu_l(t)}{\sigma_l(t) + \varepsilon}
     \]
     最终分数为\(z_{10}(t) - z_{21}(t)\)。该变体称为EpiKV。

4. **其他变体**：
   - **KV向量家族**：直接利用已缓存的key/value向量统计（滚动方差）作为重要性信号，无需隐藏状态。包括KV-key、KV-val、Lag-KV（对逐通道范围进行滞后归一化）。
   - **混合信号**（用于消融）：注意力权重×隐藏状态z分数，或ThinKV分段+隐藏状态排名。

5. **FlashAttention兼容**：只需`output_hidden_states=True`（或KV cache本身），绝不请求`output_attentions`，无需显式注意力矩阵，可直接在FlashAttention推理栈中运行。

### 算法流程（文字说明）
- 在每个解码步骤t，进行一次标准前向传播，获取所有层的隐藏状态（或直接从KV cache读取key/value向量）。
- 计算Band A和Band B对应层的滚动z分数，组合得最终重要性分数。
- 将新生成的token的分数固定，并累加到历史缓存中。
- 保留提示（prefill）token、近期窗口（recent window）以及分数最高的K-token，其余驱逐。

## 6. 主要结论与发现

1. **EpiKV实现精度持平或超越注意力基线**：
   - 4096缓存下，MATH-500准确率72%，与ThinKV（71%）相当，优于H2O（67%）。

2. **无需注意力矩阵带来工程优势**：
   - 速度提升：在AIME-2024 8192缓存下，Lag-KV每问题440s，较ThinKV（721s）快1.6倍，较RaaS（1239s）快2.8倍。

3. **两层解剖的发现**：
   - Band A（层7-13）正相关、Band B（层18-25）负相关，该模式在MATH和AIME上一致，但在简单任务GSM8K上移位（Band A移至更浅层，Band B覆盖更广）。说明信号追踪任务难度相关的负载计算位置。

4. **因果滚动z-score有效消除位置趋势**，显著提高排名质量（Spearman ρ提升32-57%）。

## 7. 优点

- **无需训练/分类器/自定义内核**：直接利用标准前向传播的隐藏状态或缓存KV向量，即插即用。
- **FlashAttention兼容**：无需回退至eager模式，避免O(n²)显存，可直接集成到vLLM、TGI、SGLang等生产推理栈。
- **完全因果在线评分**：每个token的分数在生成时立即确定，无需未来信息，无需刷新窗口（如ThinKV的τ=128延迟）。
- **提供反事实重要性标签**：作为验证资源发布，促进后续研究。
- **探索了信号来源的层级机制**：与ROME/MEMIT等可解释性工作对齐，提供了理论支撑。
