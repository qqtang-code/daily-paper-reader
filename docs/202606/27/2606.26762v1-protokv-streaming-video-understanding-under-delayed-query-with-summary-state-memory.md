## 摘要
流式视频理解（SVU）必须在视觉令牌连续流式传输的同时，在严格的GPU内存和查询时间延迟预算下，回答异步到达的查询。一个关键挑战是延迟查询：决定性线索可能短暂出现，但在查询到达之前会发生许多后续更新，这增加了在有限内存下这些线索被驱逐或稀释的风险。我们提出了ProtoKV，一种恒定占用的SVU内存，它将远历史表示为固定容量的摘要状态，而不是保留令牌实例。ProtoKV保留一个精确的近窗口KV缓存，并将较旧的内容聚合到一个具有残差统计量的语义-空间原型库中。在查询时，每个原型通过一个与标准注意力兼容的有界伪令牌接口暴露出来。在匹配的预算和可比的查询时间成本下，ProtoKV在长延迟场景下的SVU基准测试中，准确率比令牌保留基线提高了多达12.5个百分点，且收益随着查询延迟的增加而增长。

## Abstract
Streaming video understanding (SVU) must answer queries that arrive asynchronously while visual tokens stream continuously under strict GPU-memory and query-time latency budgets. A key challenge is delayed query: decisive cues may appear briefly, yet many subsequent updates occur before the query arrives, increasing the risk that those cues are evicted or diluted under bounded memory. We propose ProtoKV, a constant-footprint SVU memory that represents far history as a fixed-capacity summary state rather than retaining token instances. ProtoKV keeps an exact near-window KV cache and aggregates older content into a semantic-spatial prototype bank with residual statistics. At query time, each prototype is exposed through a bounded pseudo-token interface that is drop-in compatible with standard attention. Under matched budgets and comparable query-time cost, ProtoKV improves accuracy by up to 12.5 points over token-retention baselines on SVU benchmarks in the long-delay regime, with gains that grow as query delay increases.

## 6. 主要结论与发现
- ProtoKV在标准SVU基准上一致优于SWA和InfiniPot-V（例如RVS-Ego 59.4% vs 58.4%/58.9%）。
- **长延迟鲁棒性突出**：随着延迟Δ增大，ProtoKV的准确率下降更平缓；在最长延迟下，相比SWA最高提升20.4个百分点，相比InfiniPot-V最高提升12.5个百分点。收益随延迟增加而扩大。
- 消融显示：远记忆本身是关键（移去后长延迟准确率从54.3%降至40.5%）；残差统计和质量偏置各贡献约3-4个点；连续性分配防止碎片化（贡献约6个点）。
- 在离线场景（VideoMME/MLVU）下，ProtoKV在紧预算下与专用OVU方法竞争力相当。
- 查询时间TTFT低于ReKV等卸载方法，在线更新开销可被帧间间隔吸收。

## 7. 优点
- **创新性**：将远历史表示为**摘要状态**（原型+残差统计），而非token留存，从根本上缓解了长延迟下token竞争和碎片化问题。
- **实用性**：训练无关、查询时间复杂度固定、与标准注意力无缝兼容，易于部署到现有VLM。
- **设计细致**：连续性感知分配（融合key相似性、空间一致性、时效）有效防止跨对象原型切换；质量感知偏置合理模拟了多token效果；残差直方图+束搜索提供了高效解码。
- **实验扎实**：覆盖多数据集、多骨干、多延迟梯度，并提供了系统级验证（延迟、吞吐）、定性案例分析（成功与失败案例），论证充分。

## 8. 不足与局限
- **实验覆盖**：未在更长视频（如数小时）或更高帧率场景下测试；未与所有现有SVU方法（如Flash-VStream、VideoLLM-MoD）直接对比；延迟扫描仅基于数据集时间戳代理，而非真实证据最后出现时间。
- **偏差风险**：伪token合成假设key和value残差独立解码，可能丢失联合相关性；质量偏置为log线性，可能不如学习型偏置灵活；RoPE锚点仅用最近时间，对时间顺序敏感任务（如动作排序）可能出错（如案例分析所示）。
- **应用限制**：原型容量固定，对大量相似重复事件（如多次切南瓜）无法区分计数，因同一原型内边界被模糊化；在线更新增加约30%的逐帧开销（虽可隐藏，但在超低延迟场景仍可能成瓶颈）。
- **超参**：多个超参（λ_sp, λ_idle, T_idle, α, β, γ, S, G, C等）需手动设定，虽当前扰动影响小，但不同视频域可能需重新调整。
