## 摘要
延迟交互视觉语言检索器将每个文档页面表示为多个视觉标记嵌入，并通过MaxSim对查询进行评分。在ColPali、ColQwen、ColNomic和Nemotron ColEmbed等系统中，文档嵌入是在未见查询的情况下生成的，因此同一页面在表格查找、图表问题以及布局敏感的证据请求中会被相同地表示。我们引入了\textbf{Argus}，这是一个基于Qwen3.5-VL构建的查询条件延迟交互检索器系列。Argus增加了一个区域感知的混合专家模块：查询编码器同时生成检索嵌入和一个紧凑的上下文向量，文档页面被池化为空间区域，在MaxSim之前，一个查询感知路由器为每个区域选择潜在专家。输出仍然是一个与ColPali风格检索兼容的多向量索引，但文档表示现在依赖于查询（即$\mathbf{D}(q)$）。所有Argus模型都使用1024维的检索头，而近期最先进系统使用的则是2560维和4096维的头，并且只使用了约9%的可用公开监督数据进行训练，而非全部数据。9B模型在ViDoRe V1上达到\textbf{92.67} NDCG@5，在V1+V2组合排行榜上达到\textbf{86.0} NDCG@5，这是公开的延迟交互模型在组合排行榜上报告的最高值。在ViDoRe V3上，将Argus-9B嵌入到基于Qwen3.6-27B的智能体检索流程中，其在公开任务上的NDCG@10从60.28进一步提升至\textbf{64.80}，表明同一检索器既可作为强大的独立系统，也可作为迭代LLM智能体的搜索原语。

## Abstract
Late-interaction vision-language retrievers represent each document page as many visual token embeddings and score queries with MaxSim. In systems such as ColPali, ColQwen, ColNomic, and Nemotron ColEmbed, the document embeddings are produced without seeing the query, so the same page is represented identically for a table lookup, a chart question, and a layout-sensitive evidence request. We introduce \textbf{Argus}, a family of query-conditioned late-interaction retrievers built on Qwen3.5-VL. Argus adds a region-aware Mixture-of-Experts module: the query encoder produces both retrieval embeddings and a compact context vector, the document page is pooled into spatial regions, and a query-aware router selects latent experts per region before MaxSim. The output remains a multi-vector index compatible with ColPali-style retrieval, but the document representation is now dependent on the query (i.e., $\mathbf{D}(q)$). All Argus models use a 1024-dimensional retrieval head, compared with the 2560-dimensional and 4096-dimensional heads of recent state-of-the-art systems, and are trained on roughly 9\% of the available public supervision rather than the full pool. The 9B model reaches \textbf{92.67} NDCG@5 on ViDoRe V1 and \textbf{86.0} NDCG@5 on the combined V1+V2 leaderboard, the highest reported value for an open late-interaction model on the combined leaderboard. Wrapped in a Qwen3.6-27B agentic retrieval pipeline on ViDoRe V3, Argus-9B further improves its NDCG@10 from 60.28 to \textbf{64.80} over public tasks, showing that the same retriever serves both as a strong standalone system and as a search primitive for iterative LLM agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的视觉文档延迟交互检索器（如 ColPali、ColQwen、ColNomic、Nemotron ColEmbed）在离线阶段对每个文档页面生成固定表示，**不依赖查询**。这意味着同一页面对于表格查找、图表问题、布局敏感的证据请求等不同查询，会被编码为完全相同的多向量索引。这种缺乏适应性的表示限制了检索精度，尤其当查询类型差异大或文档分布与训练数据不同时（域外迁移）。
- **研究动机**：希望在不牺牲延迟交互优势（如细粒度匹配、可索引性）的前提下，让文档表示能够**根据查询动态调整**，从而获得一部分交叉编码器的适应性，同时保持高效的离线/在线分离。
- **整体含义**：Argus 提出了首个**查询条件化的延迟交互视觉文档检索器**，通过区域感知的混合专家（MoE）模块，使文档的多向量嵌入随每个查询变化，在标准排行榜上取得开源模型最佳结果，且嵌入维度更小、训练数据更少。

## 2. 论文提出的方法论

### 核心思想
- 在 ColPali 风格的晚期交互框架（MaxSim 评分）基础上，引入**查询条件化的文档编码**：文档侧不再固定，而是通过一个**查询感知路由器**，为每个空间区域选择不同的潜在专家（MoE），生成依赖于查询的文档多向量索引 $\mathbf{D}(q)$。
- 保持输出仍为多向量索引，与 MaxSim 兼容；推理时文档侧的大部分计算可离线缓存，仅查询侧和路由融合阶段在查询时运行。

### 关键技术细节（结合图2）
1. **共享骨干**：使用 Qwen3.5-VL（2B/4B/9B）作为图像和文本编码器。
2. **查询分支**：对文本查询编码得到令牌嵌入 $Q \in \mathbb{R}^{m \times d}$（d=1024），并通过对令牌池化 + L2 归一化得到查询上下文向量 $z_q \in \mathbb{R}^d$。
3. **文档分支**：
   - 从骨干网络两个深度提取视觉特征：$H^{(L-5)}$（用于路由）和 $H^{(L)}$（用于检索）。
   - 将 $H^{(L-5)}$ 重塑为 2D 网格，按区域大小 $r=4$ 平均池化得到 $N$ 个区域令牌 $R \in \mathbb{R}^{N \times h}$，并记录每个区域的坐标 $c \in \mathbb{R}^{N \times 4}$。
4. **查询感知路由器**：
   - 每个区域的输入 $u_r = R_r + W_{coord}c_r + W_{query}z_q$（区域内容 + 位置嵌入 + 查询上下文）。
   - 通过两层 MLP 得到路由 logits，使用 top-2 稀疏 softmax（训练时加噪声）得到每个区域对 $K=4$ 个潜在专家的权重 $g_r$。
5. **专家库与共享专家**：
   - 4 个潜在专家（小型 FFN）和 1 个一直激活的共享专家（FFN）。
   - 路由权重从区域广播到其内的每个 patch，形成加权和 $S_{ij} = \sum_{k=1}^K g_{r(i,j),k} E_k(G_{final})$。
6. **门控残差融合**：
   - 最终网格 $F = G_{final} + \sigma(\alpha) S_{shared} + \sigma(\beta) S$，其中 $\alpha, \beta$ 是可学习标量，初始化为 0.5 使其接近基线。
   - 融合后投影到 1024 维，得到 $\mathbf{D}(q)$，与 $Q$ 进行 MaxSim 评分。
7. **损失函数**：
   - 联合训练：$\mathcal{L} = \mathcal{L}_{NCE} + \lambda_{bal}\mathcal{L}_{bal}$，其中 $\mathcal{L}_{bal}$ 是负载均衡损失（Fedus et al.）。
   - 两阶段训练：先进行路由器预热（仅路由和平衡损失，无检索信号），再联合训练。

### 推理流程
- 离线：每个文档页面通过图像编码器一次，缓存早期网格、最终网格、区域特征、坐标等**与查询无关**的特征。
- 在线：编码查询得到 $Q$ 和 $z_q$，对候选页面运行路由器、专家融合、投影、MaxSim 评分。文档编码器不重复运行。

## 6. 主要结论与发现

- **SOTA 表现**：Argus-9B 在 ViDoRe V1+V2 联合排行榜上以 **86.0 NDCG@5** 创下开源延迟交互模型新高，超过 Nemotron-8B-v2（84.7）和 Ops-Colqwen3-4B（84.6）。最大增益在域外 V2 任务（+4.3）。
- **高效性**：以 1024 维检索头（对比 2560/4096）和仅 9.3% 训练数据达到最佳，且无需种子平均或模型合并。
- **查询条件化的有效性**：消融证明查询上下文 $z_q$ 是核心贡献之一，去掉后性能显著下降。
- **区域感知 MoE 的作用**：专家数量 K=4、路由深度 L-5 为最优配置，路由器能根据查询动态调整区域分配。
- **作为智能体基元**：Argus-9B 可进一步提升（60.28 → 64.80 NDCG@10）当被嵌入迭代搜索代理时。
- **存储-精度权衡**：Argus-9B 每页仅 4.2 MB（bf16），是 Nemotron-8B-v2 的 1/4.5，但精度更高。
- **推理速度优势**：离线编码比 Nemotron-8B-v2 快 13.6×，在线查询快 2×（因固定令牌预算和较小维度）。

## 7. 优点

1. **创新性**：首次在视觉文档的延迟交互检索中实现**查询条件化的文档表示**，结合区域感知 MoE，设计简洁且有效。
2. **实用导向**：保持与 ColPali 索引的兼容性，支持离线-在线分离；使用较小检索头降低存储和计算成本；训练数据需求少。
3. **全面消融**：从组件、参数（专家数、路由深度、单向量对比）到效率（延迟、存储），验证充分，支撑设计选择。
4. **多维度评估**：涵盖英文核心基准、多语言 MIRACL-Vision、智能体扩展，展示鲁棒性和灵活性。
5. **开源友好**：提供代码和数据（GitHub 仓库），便于复现和后续研究。

## 8. 不足与局限
