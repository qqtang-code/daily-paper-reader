## 摘要
尽管取得了快速进展，多模态大语言模型（MLLMs）在人类能轻松完成的任务上仍然失败，例如从屏幕截图导航网格迷宫或选择正确的拼图块。我们不重新训练模型，而是提出一个互补的问题：仅通过改进其周围的执行框架，能从冻结的MLLM中激发出多少能力？我们引入MUSE，一个多模态统一的执行框架，它用可组合的模块包装任何现成的MLLM，这些模块用于任务表示、视觉处理、感知工具使用、结构化解析、确定性验证和验证器引导的修复，无需任何模型重新训练。我们在多个跨越视觉空间规划、视觉感知、多模态推理和细粒度视觉辨别的基准测试上，使用多个最先进的MLLM评估MUSE。MUSE在所有设置中都比裸模型带来一致的提升，在具有挑战性的实例上提升最大。进一步分析表明，许多MLLM的失败源于框架层面的缺陷，而非模型本身的不足，并且可以通过验证器引导的修复来解决，而无需接触模型。这些发现凸显了智能体多模态框架作为一个关键但尚未充分探索的设计维度，为超越以模型为中心的优化提供了改进MLLM的正交途径。

## Abstract
Despite rapid progress, multimodal large language models (MLLMs) still fail on tasks that humans solve effortlessly, such as navigating a grid maze from a screenshot or selecting the correct puzzle piece. Rather than retraining the model, we ask a complementary question: how much capability can be elicited from a frozen MLLM purely by improving the execution scaffold around it? We introduce MUSE, a multimodal unified structured execution harness that wraps any off-the-shelf MLLM with composable modules for task representation, visual processing, perception tool use, structured parsing, deterministic verification, and verifier-guided repair, without any model retraining. We evaluate MUSE across diverse benchmarks spanning visual spatial planning, visual perception, multimodal reasoning, and fine-grained visual discrimination, using multiple state-of-the-art MLLMs. MUSE delivers consistent gains over the bare model in all settings, with the largest jumps on challenging instances. Further analysis reveals that many MLLM failures arise from harness-level shortcomings rather than fundamental model deficits, and can be addressed through verifier-guided repair without touching the model. These findings highlight the agentic multimodal harness as a critical yet underexplored design dimension, offering an orthogonal avenue for improving MLLMs beyond model-centric optimization.

## 2. 方法论：核心思想、关键技术细节、算法流程
### 核心思想
- 将执行框架（harness）视为**一等优化目标**，而非静态执行层。在固定MLLM情况下，通过优化框架H最大化期望奖励：  
  `H* = argmax E[r(τ,x)]`，约束执行成本≤预算κ。  
  框架空间H不仅要包含文本操作，还需包含图像预处理、感知工具调用、图像接地验证等多模态组件。
- 采用**设计驱动方法**：通过系统分析已有基准（VSP-Grid、BLINK-Jigsaw、CoMT、TIR-Bench）中观察到的失败模式，构建一个统一的框架H，而非通过外层循环搜索。

### 关键技术细节（核心组件）
1. **统一任务表示（Unified Task Representation）**  
   - 每个数据集通过适配器（adapter）标准化为统一记录schema，包含任务类型、共享指令、图像路径、结构化输入、可选标注。执行器（executor）遍历统一记录，无需数据集特定分支。
2. **统一任务钩子（Unified Task Hooks）**  
   - 为每个任务类型定义不可变钩子记录，包含：答案提取（将原始输出映射到结构化候选）、指令初始化（可选覆盖首轮提示）、修复策略（将验证器失败转化为反馈）、工具需求（声明该任务可能需要调用的感知工具）。解耦任务特定逻辑与核心执行循环。
3. **指令接口（Instruction Interface）**  
   - 构建初始用户消息和后续修复提示，组合数据集文本、可选工具输出、预处理视觉工件（按任务特定模板）。修复轮次中追加历史输出及验证反馈的排序记录（按修复策略格式化）。
4. **结果策略（Outcome Policy / 验证器）**  
   - 外部化任务语义到框架，而非模型。验证器为一系列任务特定函数，编码正确性概念，并可结合工具辅助检查。每次拒绝按粗粒度分类（格式错误、任务逻辑错误、证据缺口），修复策略据此选择反馈措辞。
   - **仿真器支持的验证**：适用于可执行环境的任务（如导航网格模拟移动序列、拼图缝像素L1差异）。
5. **感知工具注册表（Perception Tool Registry）**  
   - 可扩展的插件式工具注册表，每个工具是从图像到结构化负载（文本、嵌入或图像）的纯函数。执行器仅运行当前任务声明的工具。内置工具包括几何变换、OCR、嵌入工具。
6. **修复循环（Repair Loop）**  
   - 验证器拒绝初始候选后，进入修复循环：每轮重新调用πθ，将原始上下文附加失败记录（按顺序索引、模型输出引用、仿真器反馈），重新解析和验证，直到接受或达到最大修复轮次K。超过K后返回失败。

### 算法流程（文字描述）
```
输入：冻结MLLM πθ，任务实例x，最大修复次数K≥0
1. 从任务钩子获取该任务的工具需求r
2. 运行r中声明的感知工具，得到工具输出T（如OCR结果）
3. 构建初始提示m0（包含x和T）
4. 调用πθ获得初始输出o0
5. 解析o0得到结构化候选p0
6. 调用验证器得到结果v0（包含ok标志和失败细节）
7. 若v0.ok，返回成功结果；否则进入修复循环
8. 初始化失败记录F=[(o0, v0.detail)]
9. for i=1 to K:
   - 构建修复提示mi（原消息+失败记录+压缩摘要）
   - 调用πθ获得oi，解析得到pi，验证得到vi
   - 若vi.ok，返回成功；否则追加(oi, vi.detail)到F
10. 若所有修复都失败，返回失败结果
```

## 6. 论文的主要结论与发现
1. **MUSE一致提升冻结MLLM性能**：在所有四个基准和四个模型上，MUSE相对BASE均取得正向增益，特别是困难实例（如Word Search上GPT-4o从3%提升到21%，GPT-5.4从30%提升到62%）。
2. **许多失败源于框架而非模型**：失败模式分析表明，多数初始错误可通过验证引导的修复纠正（例如VSP-Grid中61%的失败被修正），说明模型本身能力足够，但单次生成受限于输出格式、感知误差或推理链不完整。
3. **修复循环的有效性**：增加修复预算K从2到3带来最大增益，但继续增加至5反而导致性能下降（过修复），建议默认K=3。
4. **计算匹配控制证明增益非来自额外调用**：自一致性多数投票几乎无改善甚至下降，而MUSE以更少调用实现大幅提升。
5. **感知工具与框架协同**：Word Search任务中，OCR工具提供粗略位置证据，结合验证引导修复显著提升精细判别能力（甚至对初始性能极低的模型）。
6. **瓶颈转移**：随着难度增加，失败主导因素从高层规划错误转向低层感知错误（如危险单元格定位），提示感知工具和更鲁棒视觉表示仍是挑战。

## 7. 优点
- **无需模型重训练**：完全黑箱，适用于任何闭源/开源MLLM，适应模型快速演进。
- **模块化设计**：任务表示、解析、验证、修复、工具注册等组件解耦，易于扩展新任务（仅需注册适配器和任务钩子）。
- **验证器多样化**：包括确定性验证（如仿真器、像素匹配、字符串匹配）、可选工具辅助，避免了自验证的“自我验证差距”。
- **失败分类与针对性修复**：修复提示中对错误类型进行显式分类（格式错误、逻辑错误、证据缺口），引导模型更有效地修正。
- **广泛的实验验证**：覆盖四种不同类型视觉任务，多个前沿模型，消融实验控制计算量，提供定性案例支持。
- **开源实现**：论文提供GitHub代码链接，促进可复现性。

## 8. 不足与局限
- **无法解决根本感知缺陷**：对于持续性的感知失败（如复杂场景中的危险定位错误），仅靠框架修复难以完全纠正，最终仍需更强的视觉表示。
- **依赖任务特定验证器**：对于开放生成任务（如自由文本描述、创意设计），难以定义确定性验证信号，MUSE的应用受限。
- **额外计算成本**：修复循环增加MLLM调用次数（最多K+1次），需要在质量和成本间权衡。虽然计算匹配实验表明框架效率高于多次独立采样，但仍在某些场景下增加数倍调用。
- **手动构建模块**：当前任务钩子、修复策略、验证器均需手工设计，自动化框架设计与适应是未来方向。
- **实验覆盖广度有限**：仅评估了四个基准，未涉及视频理解、具身控制、对话等更复杂场景；模型仅来自OpenAI和Anthropic两家，未测试开源模型（如LLaVA、Qwen-VL）。
- **未提供严格的理论保证**：验证器的正确性未形式化证明，修复循环的收敛性也未分析。
- **数据集规模较小**：每个基准实例数从100到200不等，统计显著性可能不足，大样本验证有待进行。
