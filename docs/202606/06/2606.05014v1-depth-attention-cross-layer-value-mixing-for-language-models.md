## 摘要
自注意力机制可以在序列中自由选择信息，但在深度维度上，Transformer 仅将每层的输出添加到残差流中，因此后续层无法有选择地重用前面层的表示。最近的跨层方法改善了这种流动，但在注意力之外的隐藏状态上操作，在推理时增加了键值缓存之外的状态——随着现代大语言模型通过分组查询和多头潜在注意力压缩缓存，这一成本变得越来越突出。我们引入了深度注意力（Depth-Attention），它在注意力模块内部执行这种选择：在一层对序列进行注意力之前，其查询会关注同一 token 位置处前面层的键，并将它们的值混合到自注意力随后读取的值中。由于深度注意力重用了标准的注意力查询、键和值缓存槽，将深度混合的值存储在原始值的位置，因此它不增加参数，也不引入超出标准键值缓存的持久推理状态——与普通解码器相同的缓存大小，且少于基于隐藏状态的跨层方法。在 1.5B 和 3B 参数的 Qwen3 风格解码器上，深度注意力获得了最低的困惑度和最高的平均下游准确率，相比普通 Transformer 提升了最多 2.3 个准确率点，并在困惑度和平均准确率上超越了强跨层基线，同时额外计算量低于 0.01% 的算术 FLOPs，且无额外的持久推理状态。这些收益在 360M 到 3B 参数范围内保持，并扩展到循环 Transformer。

## Abstract
Self-attention selects information freely across the sequence, but across depth, Transformers merely add each layer's output to the residual stream, so later layers cannot selectively reuse earlier-layer representations. Recent cross-layer methods improve this flow but operate on hidden states outside attention, adding state beyond the key-value cache at inference--a cost that becomes increasingly salient as modern LLMs compress the cache with grouped-query and multi-head latent attention. We introduce Depth-Attention, which performs this selection inside the attention module itself: before a layer attends over the sequence, its query attends over the keys of earlier layers at the same token position and mixes their values into the value that self-attention then reads. Because Depth-Attention reuses the standard attention queries, keys, and value-cache slots, storing depth-mixed values in place of the original values, it adds no parameters and introduces no persistent inference state beyond the standard key-value cache--the same cache size as a vanilla decoder and less than hidden-state-based cross-layer methods. On Qwen3-style decoders at 1.5B and 3B parameters, Depth-Attention attains the lowest perplexity and the highest average downstream accuracy, improving over the vanilla Transformer by up to 2.3 accuracy points and surpassing strong cross-layer baselines in perplexity and average accuracy, while adding under 0.01% extra arithmetic FLOPs and no additional persistent inference state. The gains hold from 360M to 3B parameters and extend to looped Transformers.

# 论文总结：Depth-Attention: Cross-Layer Value Mixing for Language Models

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：标准 Transformer 解码器在**序列维度**上通过自注意力机制实现了灵活的信息选择，但在**深度（层）维度**上，信息流动机制非常有限。信息仅通过简单的“残差流”相加来传递，导致后续层无法有选择地、动态地重用或检索前期特定层的表示，这限制了模型深度方向的表达能力。
- **研究动机**：现有的一些跨层信息流动方法（如 DenseFormer, Hyper-Connections, Attention Residuals）虽然尝试改善此问题，但它们通常基于**隐藏状态（hidden states）**进行操作。这意味着推理时，模型需要额外存储或维护这些隐藏状态，从而在标准的**键值（KV）缓存**之外增加了推理状态和内存开销。随着现代 LLM 使用 GQA 或 MLA 等技术极力压缩 KV 缓存，这种额外开销变得愈发显著。
- **整体含义**：论文提出了一种名为 **Depth-Attention** 的全新方法，旨在**在注意力模块内部**实现高效的、无需额外推理状态的跨层信息选择。它将跨层交互视为与序列注意力平行的、互补的操作，从而在不增加任何参数和持久化推理状态的前提下，显著提升了模型性能。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将标准自注意力中的“跨序列位置”信息混合思想，**移植到“跨模型深度（层）”**。具体来说，对于当前层（Layer ℓ）的一个 token，它的查询（Query）不仅关注序列中的其他 token（标准自注意力），还会“向上看”，关注同一 token 位置、来自之前各层的键（Key），并用这种“深度注意力”的权重来混合之前各层的值（Value），形成一个“深度混合后的值（depth-mixed value）”。这个新值随后被用于标准的、跨序列位置的因果自注意力。

- **关键技术细节与算法流程（以文字说明）**：
    1.  **输入**：标准 Transformer 层 ℓ，输入为查询 $Q^ℓ$、键 $K^ℓ$ 和值 $V^ℓ$。
    2.  **深度注意力操作**（这是核心创新）：
        - 对于序列中位置 *t* 的 token，当前层的查询 $q_t^ℓ$ **沿着深度方向**，去关注同一位置 *t* 处的、当前层和之前所有层的键 $\{k_t^1, k_t^2, ..., k_t^ℓ\}$。
        - 计算深度注意力权重 $\alpha_t^{ℓ,j}$（即一个 softmax 函数作用在当前层查询 $q_t^ℓ$ 与每个源层键 $k_t^j$ 的点积上）。
        - **递归地**形成一个“深度混合后的值” $\tilde{v}_t^ℓ$。这个 $\tilde{v}_t^ℓ$ 是当前层的值 $v_t^ℓ$ 与之前各层**已经混合好的值** $\tilde{v}_t^j$ ($j < ℓ$) 的加权和，权重就是上一步计算的 $\alpha_t^{ℓ,j}$。
    3.  **替代标准值**：该层最终使用这个“深度混合后的值” $\tilde{V}^ℓ$ 来**替代原始的值 $V^ℓ$**。
    4.  **标准自注意力**：随后，该层执行标准因果自注意力，但注意力计算中读取的值是 $\tilde{V}^ℓ$，而查询和键保持不变：$O^ℓ = CausalAttn(Q^ℓ, K^ℓ, \tilde{V}^ℓ)$。
    5.  **稀疏性与效率**：为了防止深度源集合随层数线性增长，论文采用了**步长（stride）**策略。默认使用步长 $s = L/2$（L 为总层数），即每层只关注自己、以及隔 s 层的更早层。这使得计算复杂度从 $O(L)$ 降至 $O(L/s)$，相对于 $O(T^2)$ 的序列注意力可忽略不计。
    6.  **无缝集成 GQA**：在分组查询注意力（GQA）设置下，将组内查询平均为一个查询后进行深度注意力计算，进一步降低开销。
