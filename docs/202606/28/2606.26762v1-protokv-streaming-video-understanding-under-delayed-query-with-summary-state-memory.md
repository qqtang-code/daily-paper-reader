## 摘要
流式视频理解必须在视觉令牌持续流式输入且受严格GPU内存和查询时间延迟预算限制下，回答异步到达的查询。一个关键挑战是延迟查询：决定性线索可能短暂出现，但查询到达前会发生许多后续更新，增加了这些线索在有界内存下被驱逐或稀释的风险。我们提出ProtoKV，一种恒定足迹的流式视频理解内存，将远历史表示为固定容量的摘要状态，而不是保留令牌实例。ProtoKV保持一个精确的近窗口KV缓存，并将较旧的内容聚合到具有残差统计的语义空间原型库中。在查询时，每个原型通过一个有界的伪令牌接口暴露，该接口与标准注意力机制兼容。在匹配的预算和可比的查询时间成本下，ProtoKV在长延迟场景下的流式视频理解基准测试中，准确率比令牌保留基线高出最多12.5个百分点，且增益随查询延迟增加而增长。

## Abstract
Streaming video understanding (SVU) must answer queries that arrive asynchronously while visual tokens stream continuously under strict GPU-memory and query-time latency budgets. A key challenge is delayed query: decisive cues may appear briefly, yet many subsequent updates occur before the query arrives, increasing the risk that those cues are evicted or diluted under bounded memory. We propose ProtoKV, a constant-footprint SVU memory that represents far history as a fixed-capacity summary state rather than retaining token instances. ProtoKV keeps an exact near-window KV cache and aggregates older content into a semantic-spatial prototype bank with residual statistics. At query time, each prototype is exposed through a bounded pseudo-token interface that is drop-in compatible with standard attention. Under matched budgets and comparable query-time cost, ProtoKV improves accuracy by up to 12.5 points over token-retention baselines on SVU benchmarks in the long-delay regime, with gains that grow as query delay increases.

## 2. 方法论
### 核心思想
- 采用**两级内存架构**：
  - **近窗口（Near Window）**：保留最近W个token的精确KV缓存，保证近期细节无损。
  - **远历史摘要（Far Summary State）**：将超出近窗口的历史token通过在线连续性感知分配聚合到一个固定容量（Kmax）的原型库中，每个原型存储中心、质量（mass）、残差统计、空间位置信息和时间戳，而非保留单个token实例。
- **伪令牌接口（Pseudo-Token Interface）**：查询时，从每个原型的残差统计中解码S个伪KV令牌，并附带质量感知偏置（log nk），与近窗口令牌拼接后输入标准注意力，实现固定查询上下文长度（W + Kmax·S）。

### 关键技术细节
- **连续性感知原型分配**（公式1）：结合键相似性、空间连续性（马氏距离）、和空闲惩罚，将驱逐的令牌分配给最合适的原型。

- **质量感知偏置**（公式2）：在注意力logit上加log nk，使原型在注意中的权重与其吸收的令牌数成比例。
- **原型维护**（算法2）：空闲原型质量衰减、合并冗余原型、回收零质量槽位，确保银行长期稳定。

- **在线更新**（算法1）：新令牌先加入近窗口；若超出，驱逐最旧令牌，先尝试填入空原型槽，否则进行连续性分配并更新原型统计（EMA更新中心、质量、空间状态、残差直方图），之后进行维护操作。

## 6. 主要结论与发现
- **ProtoKV在长延迟下显著优于token保留方法**：在RVS-Movie上，最大Δ下比SWA高20.4个百分点，比InfiniPot-V高12.5个百分点；增益随延迟增加而增大。
- **摘要状态比token实例更鲁棒**：原型库通过连续性分配和质量衰减，有效保留稀疏决定性线索，避免token竞争导致的突出丢失。
- **残差统计和质量偏置至关重要**：消融显示移除它们会导致长期延迟性能下降（-7.5、-7.3个百分点）。
- **与offloading基线相比**：ProtoKV在保持相近准确率的同时，TTFT和端到端延迟分别降低26%，更适合低延迟交互场景。
- **在离线场景也有效**：在VideoMME和MLVU上，与训练过的OVU基线相比具有竞争力。

## 7. 优点
- **核心创新**：将远历史表示为**摘要状态**（原型+残差）而非token实例，从根本上避免了token保留方法在固定预算下竞争和丢失信息的缺陷。
- **兼容性**：伪令牌接口与标准注意力机制完全兼容，无需修改模型架构或进行额外训练，易于部署。
- **恒定复杂度**：内存和查询时间与流长度无关，保证可预测的延迟和预算。
- **连续感知分配**：结合键相似性和空间连续性，有效减少原型切换和碎片化，提升长期稳定性。
- **质量感知偏置**：使得原型在注意力中的权重与其累积证据成比例，模拟多个实例的效果而无需实例化。
- **实验设计严谨**：提出了延迟查询协议和有效查询过滤方法，确保延迟比较公平；匹配内存预算和查询时间成本进行比较。

## 8. 不足与局限
- **计数类任务失败**：当多个视觉相似的事件重复出现时，它们会被合并到同一个原型中，导致无法区分事件边界（例如Action Count任务，ProtoKV输出1次而非5次）。
- **残差统计的独立性假设**：键和值残差独立编码（独立PQ直方图），可能丢失键-值联合相关性；联合直方图成本更高，但可能提升表现。
- **仅覆盖7B模型**：实验仅在7B参数模型上验证，更大模型（如13B、70B）上的表现未知。
- **视频采样率固定**：通常采用0.5 fps，可能错过快速动作。高频采样下在线更新开销可能更显著。
- **延迟协议基于基准时间戳**：缺乏真实的每个查询的证据时间戳（te），使用t0作为近似，可能引入噪声。
- **在线更新开销**：每帧33.8ms虽可部分与帧间隔重叠，但对高帧率或严格实时系统仍构成挑战。
- **英文基准**：所有实验基于英文视频源和问答，多语言泛化未验证。
- **超参数固定**：所有基准使用同一组超参数（表5），尽管敏感性分析显示变化不敏感，但特定场景下最优参数可能不同。
