## 摘要
随着实际应用越来越多地需要处理10万+词元的输入，上下文长度与推理效率之间的差距已成为关键瓶颈。上下文压缩提供了一种降低预填充成本同时保持任务准确性的方法。然而，现有的无训练注意力方法在代码推理等要求严苛的长上下文任务中仍存在显著差距。我们提出了LongAttnComp，这是AttnComp的长上下文适配版本，它微调了一个轻量级交叉注意力评分层，并引入了词元级分块、词元预算top-p算法、位置重排序以及格式无关的查询解析器。我们进一步为压缩器设计了一个两阶段微调方案：第一阶段从NIAH风格的数据构建通用的检索基础，第二阶段通过多跳和推理数据扩展第一阶段，以覆盖更广泛的长上下文任务。在InfiniteBench Code-Debug上，LongAttnComp达到或超越了全上下文准确率，显著优于无训练基线，并在来自三个族的四个目标模型上实现了迁移。在LongBench v2上，两阶段方案在很大程度上缩小了第一阶段在多文档推理上的差距，同时保持了Code-Debug的性能。

## Abstract
As real-world applications increasingly require processing inputs of 100k+ tokens, the gap between context length and inference efficiency has become a critical bottleneck. Context compression offers a way to reduce prefill costs while preserving task accuracy. However, existing training-free attention-based methods leave substantial gaps in demanding long-context tasks such as code reasoning. We present LongAttnComp, a long-context adaptation of AttnComp that fine-tunes a lightweight cross-attention scoring layer and introduces tokenlevel chunking, a token-budget top-p algorithm, positional reordering, and a formatagnostic query parser. We further design a two-stage fine-tuning recipe for the compressor: Stage 1 builds a general retrieval foundation from NIAH-style data, and Stage 2 extends it with multi-hop and reasoning data for broader long-context task coverage. On InfiniteBench Code-Debug, LongAttnComp matches or exceeds full-context accuracy, substantially outperforms training-free baselines, and transfers across four target models from three families. On LongBench v2, the two-stage recipe largely closes the Stage 1 gap on multi-document reasoning while preserving Code-Debug performance.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：基于AttnComp框架（微调交叉注意力层进行文档评分），进行长上下文适配，并设计两阶段微调策略，使压缩器具备通用检索和复杂推理能力。
- **技术细节**：
  - **架构适应**：
    - **Token级分块**：将输入切分为固定大小的token块（而非文档），便于处理无自然边界的输入，且块大小成为可调超参数。
    - **Token预算Top-p算法**：改进AttnComp的得分阈值top-p选择：用内容token预算B替代最小得分阈值ϵ，选择停止条件为累积得分超过p或保留token数达到B；同时支持**预算-only模式**（禁用累积得分终止，直接按得分选满B个token）。
    - **位置重排序**：保留的块按原始顺序排列，维持语篇连贯性。
    - **格式无关查询解析器**：支持两种解析策略——精确解析（针对特定任务格式）和任意解析（取最后N个token作为查询），后一种在Code-Debug上仅损失约1个百分点，表明鲁棒性。
  - **两阶段微调方案**：
    - **阶段1（通用检索基础）**：在NIAH风格数据上训练，数据来自SQuAD（单事实）和HotpotQA（基础多跳），共32k样本，每个样本100个候选文档，25%为全负样本。
    - **阶段2（扩展多跳与推理）**：从阶段1检查点继续训练，加入MuSiQue（8k）和2WikiMultiHopQA（4k）数据，并混合阶段1数据的回放（SQuAD和HotpotQA各4k），防止灾难性遗忘。同时探索两种查询构造变体：`subq`（含数据集提供的子问题分解）和`nosubq`（仅原始多跳问题）。
  - **压缩器架构**：冻结Llama-3.1-8B-Instruct的前13层，附加可训练的交叉注意力层（仅更新约0.5%参数）。训练损失为文档级二分类交叉熵加上指令级辅助损失（λ=0.8）。

### 6. 论文的主要结论与发现

- **主要结论**：LongAttnComp是一种有效的基于微调的长上下文压缩方法，可模块化地作为目标无关的预处理步骤，跨模型族迁移且无需重新训练。
- **具体发现**：
  - **Code-Debug**：阶段1即超过全上下文（75.38% vs 74.37%），阶段2 subq进一步提升至76.90%；显著优于Speculative Prefill（62.44%）。

  - 两阶段微调策略有效扩展任务覆盖，同时利用回放保留前期能力。
  - 格式无关查询解析降低部署依赖。
