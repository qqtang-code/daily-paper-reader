## 摘要
推理多模态大语言模型（MLLMs）的出现——即在生成答案前先产生显式的思维链（CoT）推理——为知识编辑带来了新挑战：在传统评估指标（教师强制准确率高达100%）下看似成功的方法，在检查模型推理过程时可能会严重失败（基础成功率低至0%）。我们识别出三种失败模式：（1）结构崩塌，即权重修改方法破坏了CoT格式；（2）认知失调，即模型的推理链基于视觉证据主动拒绝注入的编辑事实；（3）浅层内化，即方法在精确查询上成功，但在改写或多跳变体上失败。在推理MLLMs上，这些模式相互交织：具有泛化能力的方法（FT、LoRA）会触发格式崩塌，而无深层修改的方法无法泛化。为了揭示这些失败，我们提出了一种CoT感知的评估协议，并构建了ReasonEdit-Bench，其中包含冲突分层、多级探测和多跳可移植性测试。我们提出了CRANE，一种无需每次编辑修改参数的检索增强框架。CRANE将模态感知的双库检索系统与两阶段训练策略相结合：监督微调（SFT）用于结构初始化，随后采用GRPO以及认知路由奖励，训练模型在视觉先验和注入的编辑事实之间进行仲裁。在ReasonEdit-Bench上，CRANE在冲突场景中实现了96.9%的基础成功率，多跳链中96.9%的中间实体使用率，以及97.6%的文本局部性和68.1%的图像局部性编辑独立性。在分布外MMEVOKE基准上，CRANE在黄金检索下达到87.0%。

## Abstract
The emergence of reasoning multimodal large language models (MLLMs), which generate explicit chain-of-thought (CoT) reasoning before producing answers, has introduced a new challenge for knowledge editing: methods that appear successful under traditional metrics (teacher-forcing accuracy up to 100%) can fail severely when the model's reasoning process is examined (Grounded Success as low as 0%). We identify three failure modes: (1) Structural Collapse, where weight-modifying methods destroy the CoT format; (2) Cognitive Dissonance, where the model's reasoning chain actively rejects the injected edit fact based on visual evidence; and (3) Shallow Internalization, where methods succeed on exact queries but fail on rephrase or multi-hop variants. On reasoning MLLMs, these modes interact: methods that generalize (FT, LoRA) trigger format collapse, while methods without deep modification cannot generalize. To expose these failures, we propose a CoT-aware evaluation protocol and construct ReasonEdit-Bench, with conflict stratification, multi-level probes, and multi-hop portability tests.   We propose CRANE, a retrieval-augmented framework that requires no per-edit parameter modification. CRANE combines a modality-aware dual-library retrieval system with a two-phase training strategy: Supervised Fine-Tuning (SFT) for structural initialization, followed by GRPO with a Cognitive Routing Reward that trains the model to arbitrate between visual priors and injected edit facts. On ReasonEdit-Bench, CRANE achieves 96.9% Grounded Success on conflict scenarios and 96.9% intermediate entity usage in multi-hop chains, with 97.6% text-locality and 68.1% image-locality Edit Independence. On the out-of-distribution MMEVOKE benchmark, CRANE reaches 87.0% under gold retrieval.

- **背景**：新一代推理型多模态大语言模型（MLLMs，如 DeepSeek-R1、Vision-R1）会在生成答案前产生显式的思维链（CoT）推理（位于 `<think>...</think>` 标签内）。这种架构转向给知识编辑带来了根本性挑战：传统编辑方法（权重修改、参数隔离、上下文编辑）在教师强制（Teacher-Forcing）指标下看似成功（准确率可达100%），但在检查模型的自由生成推理链时会发现严重失败（基础成功率 GS 可低至0%）。
- **核心问题**：论文识别出三种先前未被系统研究的失败模式：
  1. **结构崩塌**：参数修改或路由重写方法破坏了 CoT 的 `<think>...<answer>` 格式完整性，导致格式率为0%。
  2. **认知失调**：当编辑事实与输入图像视觉内容矛盾时，模型的 CoT 推理链会“看到”视觉证据并主动拒绝编辑事实，即使编辑被正确存储。
  3. **浅层内化**：存储基于输入键值的方法（如 GRACE）在精确查询上成功，但无法泛化到改写（rephrase）或多跳变体。  
    这三种模式相互作用形成一个困境：深层修改参数的方法（FT、LoRA）能泛化但触发结构崩塌，而不修改参数的方法（GRACE、IKE）无法泛化。
- **研究动机**：需要一种新的评估协议来暴露这些失败，并设计专门针对推理 MLLMs 的知识编辑框架。

## 2. 方法论：CRANE 框架

### 核心思想
CRANE（Counterfactual Reasoning Arbitration for Multimodal kNowledge Editing）是一个**检索增强**的编辑框架，**无需每次编辑修改模型参数**，从而从根本上避免结构崩塌。它结合了：
- **模态感知双库检索系统**：独立维护图像索引（SigLIP嵌入）和文本索引，避免跨模态干扰。
- **对比投影头**：使用对称 InfoNCE 损失训练线性投影头（1152维→256维），提升检索命中率（hit@1从69.7%提升至90.4%）。
- **置信阈值防御**：设θ=0.55，过滤掉低相似度的假阳性检索，保持局部性。
- **两阶段训练策略**：
  - **阶段1：监督微调（SFT）**：在6000个高质量CoT样本上训练（2个epoch，冻结ViT和对齐器，仅训练LM），学习元技能（如何引用编辑事实、如何处理视觉冲突、如何识别无关性），不注入具体事实。
  - **阶段2：GRPO 强化学习**：使用组相对策略优化（GRPO），G=8个候选，ϵ=0.2，β=0.05。总奖励 R_total = R_accuracy + R_format + R_reasoning。
    - R_accuracy：软匹配（F1或包含判断）。
    - R_format：二元奖励，是否包含有效 CoT 格式。
    - **认知路由奖励 R_reasoning**：关键创新，根据样本类型（正常/冲突/局部性）应用不同子奖励：
      - 正常样本→R_align：奖励在<think>中显式引用编辑事实。
      - 冲突样本→R_override：奖励“承认视觉证据→转折词（however/but/despite）→承诺编辑事实”的三段式模式（完全奖励1.0，部分0.5，否则0）。
      - 局部性样本→R_isolate：奖励识别图像或文本与编辑事实无关，惩罚幻觉。

### 技术流程（文字说明）
1. 输入查询（图像+问题）→检索系统：双库检索（图像查询走图像索引，文本查询走文本索引），经投影头和阈值过滤后，若命中则拼接编辑事实到输入；若低于阈值则使用“无编辑事实”提示。
2. SFT 初始化后的模型接收输入，产生 CoT 推理和答案。
3. GRPO 阶段：针对不同场景样本，应用对应的认知路由奖励，强化模型在冲突时覆盖视觉先验，在局部性时忽略无关编辑事实。

## 3. 实验设计

### 数据集与 Benchmark
- **ReasonEdit-Bench**：基于 VLKEB 经过三管道处理构建：
  - Pipeline 1：视觉相关性过滤和改写（使用 Gemini-2.5-Flash 自动标注，>95%人类一致性）。
  - Pipeline 2：冲突严重性分类（OCR冲突/类别冲突 → conflict；其余 normal）。
  - Pipeline 3：自由形式知识扩展（将编辑目标扩展为两句话描述，并生成二级探测问题 Rel-1 实体识别、Rel-2 属性推理）。  
  最终包含 filter split（1,623 normal + 1,070 conflict + 对应多跳子集）和 freeform split。
- **MMEVOKE**：分布外基准，包含新闻和维基百科图像，测试泛化能力。

### 对比方法
- 参数编辑：FT、LoRA、WISE、GRACE、LiveEdit（单编辑模式）。
- 上下文编辑：IKE（直接注入编辑事实）。
- **基线模型**：Qwen2.5-VL-7B-Instruct（CRANE 的基座模型）、Vision-R1-7B（推理型）、LLaVA-1.5-7B（非推理型）。

### 评估指标
- **Schema Integrity**：格式率（CoT 格式有效比例）。
- **Grounded Success (GS)**：答案匹配编辑事实且推理链显式引用该事实（LLM-as-Judge 使用 Gemini-2.5-Flash）。
- **Edit Independence (EI)**：局部性指标，推理链和答案不被无关编辑事实污染。
- **Intermediate Entity Used (IU) / Used Visual Instead (UVI)**：多跳指标，是否正确传播编辑实体 / 是否回退到视觉先验。

## 6. 主要结论与发现
1. **CRANE 有效解决所有三种失败模式**：
   - 无参数修改 → 格式完整率 99.9%（避免结构崩塌）。
   - 认知路由奖励 → 冲突场景 GS 96.9%（高于正常场景的90.5%），表明模型学会覆盖视觉先验。
   - SFT+GRPO → 多跳 IU 96.9%（对比基线多数低于1%），实现深度内化。
2. **高可靠性与局部性并非不可兼得**：CRANE 在冲突 GS 96.9% 的同时，文本 EI 达到97.6%；图像 EI 68.1% 部分受检索假阳性影响，通过阈值可提升至75.4%。
3. **检索是当前部署瓶颈**：在分布外 MMEVOKE 上，黄金检索时 CRANE 达87.0%，但融合检索仅69.8%（+14.6pp vs. 基础模型55.2%），提示需改进域自适应检索。
4. **SFT 是结构基础**，GRPO 主要改善局部性：无 SFT 时冲突 GS 下降17.8pp，多跳 IU 崩溃至35.9%；移除推理奖励仅影响图像局部性（-9.7pp），可靠性不变。
5. **传统评估存在系统性误报**：TF-Acc 与 GS 的差距在推理 MLLMs 冲突场景高达99pp，必须检查完整推理链。

## 7. 优点
- **问题诊断新颖**：首次系统识别并分类推理 MLLMs 上的三种知识编辑失败模式，并提出 CoT 感知评估协议，对后续研究有重要指导意义。
- **方法设计巧妙**：检索增强避免了参数修改带来的格式崩塌；认知路由奖励按场景差异化设计，直接针对认知失调和浅层内化；SFT 学习元技能而非具体事实，保证了泛化能力。
- **实验全面且严谨**：覆盖多种模型、方法、子集、消融、分布外泛化，并采用人类验证的 LLM-as-Judge 评估，结果可信度高。

## 8. 不足与局限
- **图像局部性较低**：图像 EI 68.1% 明显低于文本 EI 97.6%，主要由于检索系统在无关图像上偶尔返回假阳性编辑事实（假阳率18.5%），导致模型被误导。虽然阈值可部分改善，但本质上仍是检索级问题。
- **检索是当前部署瓶颈**：分布外泛化实验显示，检索系统性能（融合检索 CEM 69.8%）远低于黄金检索（87%），限制了 CRANE 在实际动态知识更新场景中的可用性。
- **模型规模有限**：仅在 7B 参数模型上验证，更大规模模型（如 72B）上的表现未知，框架的可扩展性尚待检验。
- **评估依赖 LLM as Judge**：使用 Gemini-2.5-Flash 作为判断器，可能存在自身偏好或误差（尽管人类一致性>95%）。不同 LLM 作为判断器的影响未探讨。
- **数据分布偏差**：ReasonEdit-Bench 基于 VLKEB，VLKEB 的图像和实体分布可能偏向特定领域（如地标、名人），在其他视觉域（如医学、遥感）的泛化性未知。
- **未探究多编辑场景**：论文仅评估单编辑模式（每次重置模型）；连续或多个编辑同时存在时的性能（如 WISE 的多编辑能力）未被讨论。
