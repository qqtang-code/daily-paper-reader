## 摘要
多模态大语言模型（MLLMs）面临显著的推理瓶颈，原因在于长视觉token序列上的自注意力计算成本呈二次增长。然而，我们发现了当前架构中的一个关键低效问题：视觉注意力饱和。我们的分析表明，视觉token在早期层中迅速建立其空间结构和模态内关系，使得深层中的视觉-视觉自注意力在计算上变得冗余。相反，这些层中的前馈网络（FFN）对于将视觉特征投影到不断演化的文本语义空间中仍然至关重要。基于这一洞察，我们提出了Visual-Skip（V-Skip），一种无需训练的推理范式，将空间交互与语义演化解耦。V-Skip不是丢弃token，而是通过选择性地跳过饱和的视觉自注意力模块，施加块级结构化稀疏性。此外，考虑到不同的下游任务需要不同的推理深度，V-Skip采用轻量级的少样本校准，动态路由任务最优的稀疏路径。大量实验表明，V-Skip有效地跳过了冗余的视觉注意力以实现块级稀疏性，在多种MLLMs上保持了94.16%至100.31%的性能保留。最终，我们证明，为了更有效地推理，模型不需要丢弃它们所看到的内容——它们只需在正确的深度“少看”即可。

## Abstract
Multimodal Large Language Models (MLLMs) face a significant inference bottleneck due to the quadratic computational cost of self-attention over long visual token sequences. However, we identify a critical inefficiency in current architectures: Visual Attention Saturation. Our analysis reveals that visual tokens rapidly establish their spatial structure and intra-modal relationships in early layers, rendering visual-to-visual self-attention in deeper layers computationally redundant. Conversely, Feed-Forward Networks (FFNs) in these layers remain essential for projecting visual features into the evolving textual semantic space. Leveraging this insight, we present Visual-Skip (V-Skip), a training-free inference paradigm that decouples spatial interaction from semantic evolution. Rather than discarding tokens, V-Skip imposes block-wise structured sparsity by selectively bypassing saturated visual self-attention modules. Furthermore, recognizing that varying downstream tasks demand distinct reasoning depths, V-Skip employs a lightweight, few-shot calibration to dynamically route the task-optimal sparsity path. Extensive experiments demonstrate that V-Skip effectively bypasses redundant vision attention to achieve block-wise sparsity, maintaining a 94.16% to 100.31% performance retention across diverse MLLMs. Ultimately, we prove that to reason more effectively, models do not need to discard what they see -- they simply need to "look less" at the right depth.

# 论文详细总结

## 1. 论文的核心问题与整体含义

- **研究动机**：多模态大语言模型（MLLMs）在处理长视觉 token 序列时，自注意力机制的二次复杂度（$O(N_v^2)$）导致推理延迟和内存消耗急剧增长。为缓解该瓶颈，现有方法（如 token 剪枝、层跳过）通常破坏性地丢弃 token 或冻结整层，虽降低计算量，但不可逆地损害视觉完整性、引发物体幻觉或语义错位。
- **核心问题**：能否在保留完整视觉上下文的前提下，仅消除冗余计算，同时维持模型推理能力？
- **整体含义**：论文提出“少看，多想”范式——模型无需丢弃视觉信息，只需在合适深度跳过冗余的视觉自注意力，同时保持前馈网络（FFN）的语义演化，从而高效且准确地推理。

## 2. 方法论

- **核心思想**：发现 **视觉注意力饱和** 现象——视觉 token 在早期层快速建立空间结构，深层视觉自注意力计算冗余，但 FFN 对语义对齐仍至关重要。据此提出 **V-Skip**，一种无需训练的推理范式，通过 **块级结构化稀疏** 选择性地跳过饱和层的视觉自注意力，保持 FFN 完整。
- **关键技术细节**：
  - **Visual Information Gain (VIG)**：度量跳过某层视觉自注意力（保留 FFN）后模型预测分布与原始分布的 KL 散度，量化该层注意力对最终输出的功能重要性。
  - **Few-shot 校准**：从下游任务随机采样 20 个图像-文本对，计算各层 VIG，选择 Top-N 个最低 VIG 层作为跳过目标。
  - **解耦注意力掩码**：在跳过层中，视觉 token 的 query 不参与自注意力（视觉绕过），文本 token 仍可关注所有 token；所有 token 均经过 FFN 更新。
- **算法流程**（文字说明）：
  1. 对目标任务采样少量样本，计算各层 VIG。
  2. 按 VIG 升序排序，选取卡号 N 的最低 VIG 层（如 N=19 代表跳过约 60% 层）。
  3. 推理时，在选定层实施视觉绕过：`H'_V ← LN(H_V) + H_V`（跳过注意力），文本注意力正常；FFN 对全体 token 执行 `H_{l+1} = FFN(LN(H'_l)) + H'_l`。

## 3. 实验设计

- **数据集 / Benchmark**：使用 9 个多模态基准：MME（通用评估）、MMMU（专家级知识）、MMBench（通用）、GQA（视觉推理）、POPE（物体幻觉）、ScienceQA（科学推理）、AI2D（图表理解）、VizWiz（野外 VQA）、OCRBench（文本识别）。
- **对比方法**：
  - 基于 token 剪枝：VTW（在 K 层后丢弃所有视觉 token）、FastV（剪枝比例 0.5）。
  - 基于层跳过：ShortV（跳过整层，包含 FFN）。
  - 变体：FFN-Skip（跳过 FFN 保留注意力）、随机/CosSim 识别策略。
- **模型架构**：LLaVA-1.5（7B/13B）、LLaVA-NeXT（7B/13B，高分辨率达 2880 tokens）、Qwen2.5-VL-7B。

## 6. 主要结论与发现

- V-Skip 在 LLaVA 系列上保持 **98.42%~100.31%** 的性能保留，在 Qwen2.5-VL 上保持 **94.16%**，显著优于 VTW（最低 88.71%）和 ShortV（最低 87.11%）。
- 跳过视觉注意力而非 FFN 是关键：FFN-Skip 在深度跳过时性能急剧下降（N=32 时平均仅 71.59%）。
- 任务特定的 few-shot 校准优于统一策略：通用校准（MME→全局）平均 99.55%，但细粒度任务（如 OCRBench）下降。
- V-Skip 可与其他加速方法（如 FastV）互补，进一步降低计算量并恢复性能。
- 表明“视觉注意力饱和”是 MLLM 的内在机制，深层自注意力可安全跳过，FFN 必须保留。

## 7. 优点

- **无需训练**：即插即用，零额外训练成本，仅需离线 few-shot 校准（约 2 分钟）。
- **非破坏性**：保留完整视觉 token 序列，避免信息丢失，减轻物体幻觉。
- **理论创新**：首次揭示“视觉注意力饱和”现象并据此解耦空间交互与语义演化。
- **高效校准**：20 样本即可稳定识别饱和层，对采样种子不敏感。
- **灵活可扩展**：可通过调整稀疏预算 N 平衡精度与速度，且可集成到现有剪枝方法中。

## 8. 不足与局限

- **依赖原始权重**：模型在密集注意力假设下预训练，推理时跳过注意力属于结构偏移，可能达不到原生稀疏训练的最优性（论文在局限部分承认）。
- **任务特定校准**：虽开销小，但仍需为每个新任务离线采样，应用时需额外步骤；未来可探索将 V-Skip 融入训练实现端到端稀疏路由。
- **架构适配性**：在 Qwen2.5-VL 上性能保留较低（94.16%），需调整预算（如 N=4 可恢复至 98.72%），说明不同架构饱和形态不同，需要个别调参。
- **实验覆盖**：主要基于 LLaVA 系列和 Qwen2.5-VL，未测试更大模型（如 70B+）或更多样化的 MLLM 架构（如基于 ViT-LLaMA 的变体）。尚未在生成式任务（如描述、对话）中全面评估影响。
- **偏差风险**：few-shot 校准依赖于随机子集，尽管论文验证了种子稳定性，但在极端异常样本下可能偶有偏差；论文通过 5 次随机实验展示均值与方差（图5），但未提供标准差的具体数值。
