## 摘要
超长上下文能力正成为前沿大语言模型不可或缺的特性：智能体工作流、仓库级代码推理和持久记忆都要求模型能同时关注数十万到数百万个token，然而软注意力（softmax attention）的二次方成本使其在部署规模下难以维系。我们提出MiniMax稀疏注意力（MSA），一种基于分组查询注意力（GQA）的块级稀疏注意力。轻量级索引分支对键值块进行评分，并为每个GQA组独立选择Top-k子集，从而在保持高效块级执行的同时实现组特定的稀疏检索；主分支随后仅对选中的块执行精确的块稀疏注意力。MSA的设计遵循简单和可扩展原则，刻意精简，使其易于在各种GPU上高效部署。为了将稀疏性转化为实际加速，我们与MSA协同设计了GPU执行路径，采用无指数Top-k选择和KV外部稀疏注意力，以在块粒度访问下提升张量核心利用率。在采用原生多模态训练的109B参数模型上，MSA性能与GQA相当，同时在1M上下文下将每token注意力计算量减少28.4倍。结合我们协同设计的内核，MSA在H800上实现了14.2倍的预填充（prefill）和7.6倍的解码（decoding）墙钟加速。我们的推理内核已发布于：https://github.com/MiniMax-AI/MSA。一个由MSA驱动的生产级原生多模态模型已公开发布于：https://huggingface.co/MiniMaxAI/MiniMax-M3。

## Abstract
Ultra-long-context capability is becoming indispensable for frontier LLMs: agentic workflows, repository-scale code reasoning, and persistent memory all require the model to jointly attend over hundreds of thousands to millions of tokens, yet the quadratic cost of softmax attention makes this untenable at deployment scale. We introduce MiniMax Sparse Attention (MSA), a blockwise sparse attention built upon Grouped Query Attention (GQA). A lightweight Index Branch scores key-value blocks and independently selects a Top-k subset for each GQA group, enabling group-specific sparse retrieval while maintaining efficient block-level execution; the Main Branch then performs exact block-sparse attention over only the selected blocks. Designed around a principle of simplicity and scalability, MSA is deliberately streamlined, making it straightforward to deploy efficiently across a broad range of GPUs. To translate sparsity into practical speedups, we co-design MSA with a GPU execution path that uses exp-free Top-k selection and KV-outer sparse attention to improve tensor-core utilization under block-granular access. On a 109B-parameter model with native multimodal training, MSA performs on par with GQA while reducing per-token attention compute by 28.4x at 1M context. Paired with our co-designed kernel, MSA achieves 14.2x prefill and 7.6x decoding wall-clock speedups on H800. Our inference kernel is available at: https://github.com/MiniMax-AI/MSA. A production-grade natively multimodal model powered by MSA has been publicly released at: https://huggingface.co/MiniMaxAI/MiniMax-M3.

# MiniMax Sparse Attention 论文总结

## 1. 核心问题与整体含义

- **研究动机**：前沿大语言模型（LLM）正从短文本交互转向超长上下文场景（代理工作流、仓库级代码推理、持久记忆），标准 softmax 注意力的 O(N²) 计算与显存开销成为部署瓶颈。如何在保持模型质量的同时大幅降低注意力计算量，是实现百万级上下文推理的关键挑战。
- **整体意义**：提出一种简洁、可扩展的块稀疏注意力机制（MSA），通过轻量索引分支实现每 GQA 组的独立 Top-k 块选择，将计算复杂度从 O(N²) 降至 O(kBₖ)，在 109B 多模态模型上实现 28.4× 理论加速和 10 倍以上的实际墙钟加速，同时保持与全注意力相当的性能，为超长上下文的工业部署提供了可行方案。

## 2. 方法论：核心思想与关键技术

- **核心思想**：采用“索引-主分支”双阶段结构。索引分支对每个 GQA 组独立选择 k 个关键块（块大小 Bₖ），主分支仅计算这些块内的精确 softmax 注意力。设计遵循奥卡姆剃刀原则，仅保留必要组件，便于高效部署。
- **关键技术细节**：
  - **索引分支**：引入一个索引查询头（每 GQA 组）和一个共享索引键头，通过点积评分并聚合为块级分数（使用 max-pooling），然后执行无 softmax 的 Top-k 选择（因 softmax 保序）。始终强制包含查询 token 所在局部块。
  - **主分支**：使用标准缩放点积注意力，但仅限于选中的块内的可见 token。
  - **训练策略**：
    - **KL 对齐损失**：让索引分支的分布（块内 token 归一化）与主分支在选中支持集上的平均分布（跨组内查询头）对齐，并使用 stop-gradient 隔离梯度，仅更新索引投影。
    - **梯度分离**：索引输入梯度从主干分离，防止 KL 损失反向传播影响主干。
    - **索引器预热**：前 40B token 使用全注意力训练索引分支（仅通过 KL 损失），之后才启用稀疏选择。
    - **强制局部块**：始终包含查询所在块，避免退化。
  - **计算复杂度**：GQA 为 O(2H_q d_h N²)，MSA 为 O(H_kv d_idx N² + 4H_q d_h NkBₖ)。当 kBₖ ≪ N 且 H_kv d_idx ≪ H_q d_h 时，复杂度大幅降低。
- **GPU 内核协同设计**：
  - **Top-k 选择**：设计无指数内核，小 k 下使用每线程寄存器最小堆 + shuffle 合并，显著快于 torch.topk 和 TileLang。
  - **稀疏注意力前向**：采用 KV-outer 迭代顺序（外循环遍历 KV 块，收集选择该块的查询），通过预分块（pre-scheduled chunking）处理热点块，两阶段 combine 避免原子操作，并利用查询拼接提高 MMA 利用率（打包 ⌈128/G⌉ 个查询）。
  - **稀疏 KL 损失**：主分支前向时直接将 LSE 写入全局内存，避免单独前向；反向时使用持久内核动态负载均衡。

## 6. 主要结论与发现

- MSA 在 109B 模型上匹配 GQA 全注意力性能，同时在 1M 上下文下实现理论 28.4× FLOPs 减少、实际 prefill 14.2× 加速、decode 7.6× 加速。
- 从头训练的 MSA-PT 在数学、多模态、长上下文中表现更强；从全注意力继续训练的 MSA-CPT 更保守，但保持接近基线。
- 动态稀疏显著优于 FLOPs 匹配的滑动窗口固定模式。
- 索引分布可视化显示：不同 GQA 组学会不同的长程稀疏模式（共享局部对角和 sink 块，但长程条纹不同）；注意力 sink 现象自然涌现。
- 训练稳定性关键：梯度分离、索引器预热、强制局部块缺一不可。

## 7. 优点

- **方法简洁高效**：仅增加两个投影矩阵，无复杂结构；与现有 GQA 架构无缝兼容，可从全注意力检查点零损伤转换。
- **GPU友好设计**：块级粒度、KV-outer 迭代、预分块和查询拼接最大化张量核心利用率；无指数 Top-k 内核专门优化小 k 场景。
- **训练稳定性强**：通过梯度分离、KL 对齐、预热等机制避免稀疏训练不稳定，LM loss 与全注意力几乎重叠。
- **验证充分**：在 109B 原生多模态模型上从零训练 + 继续预训练，覆盖 30+ 基准，且开源模型和推理代码，可复现。

## 8. 不足与局限

- **长上下文检索仍有残余差距**：MSA-CPT 在 HELMET-128K 上比全注意力低 0.6 分（绝对），RULER 上高 0.12 分但部分子集仍差 1-2 分。论文承认“残余 gap”。
