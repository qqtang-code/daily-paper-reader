## 摘要
扩展的链式思维（CoT）轨迹提升了LLM的推理能力，但带来了巨大的计算和内存成本。现有的CoT压缩方法通过记忆令牌将思维步骤压缩为紧凑表示，并在推理时仅保留这些表示来缓解这一问题，但细粒度信息的丢失使得后续步骤更容易出错。为了解决这个问题，我们提出了HybridThinker，除了保留这些表示外，还临时保留思维步骤以提供细粒度细节。然而，我们观察到，在训练过程中简单地将思维步骤暴露给后续步骤会让模型绕过记忆令牌，直接从这些步骤中检索信息，导致模型通过记忆令牌压缩和检索信息的能力训练不足。因此，我们引入了一种混合训练方案，其中只有部分思维步骤通过注意力直接对后续步骤可访问，而其他思维步骤被屏蔽，迫使模型使用记忆令牌进行压缩和检索。在4个推理基准测试中，HybridThinker与未压缩的基线性能相当，在相似的推理时间内将CoT压缩的最新技术平均准确率提高了5.8个百分点。消融研究证实，临时思维步骤保留和混合训练方案共同促进了这些提升。

## Abstract
Extended chain-of-thought (CoT) traces improve LLM reasoning but incur substantial computational and memory costs. While existing CoT compression methods mitigate this by condensing thought steps into compact representations via memory tokens and retaining only these representations at inference time, the loss of fine-grained information makes subsequent steps more error-prone. To alleviate this, we propose \textbf{HybridThinker}, where in addition to preserved these representations, thought steps are also temporarily retained to provide fine-grained details. However, we observe that naively keeping thought steps accessible to subsequent steps \emph{during training} lets the model bypass memory tokens by retrieving information directly from these steps, leaving the model's ability to compress and retrieve information through memory tokens insufficiently trained. We therefore introduce a hybrid training scheme, in which only some thought steps are directly accessible through attention to subsequent steps, while the other thought steps are masked, forcing the model to use memory tokens for compression and retrieval. Across 4 reasoning benchmarks, HybridThinker matches the uncompressed baseline, advancing the state of the art in CoT compression by 5.8 points on average accuracy with similar inference time. Ablation studies confirm that both temporary thought-step retention and the hybrid training scheme contribute to these gains.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：扩展的链式思维（Chain-of-Thought, CoT）能显著提升大型语言模型（LLM）的推理能力，但长序列推理会导致自注意力计算量呈平方增长，KV 缓存线性增长，造成巨大的计算和内存开销。
- **现有方法局限**：现有 CoT 压缩方法（如 LightThinker）通过记忆令牌（memory tokens）将每个思维步骤压缩为紧凑表示，并在压缩后立即丢弃原始思维步骤的 KV 缓存。这虽然减少了内存使用，但丢失了细粒度局部信息（如精确数值、严格约束、中间结论），使得后续步骤容易产生错误，且错误会级联传播。
- **本文目标**：在不显著增加推理开销的前提下，缓解细粒度信息丢失问题，使压缩推理的准确率接近未压缩的基线。

## 2. 论文提出的方法论

### 核心思想
- **HybridThinker**：保留现有 CoT 压缩方法中的记忆令牌机制（用于压缩和长期访问），但额外**临时保留**每个原始思维步骤的 KV 缓存，使其在后续若干步骤内仍然可见，以提供细粒度细节。当临时保留期结束后才丢弃该思维步骤的缓存。
- 实现了一种“临时思维步骤保留 + 压缩记忆”的混合推理范式，在高效性和准确性之间取得更好平衡。

### 关键技术细节

#### 推理过程（以符号表示）
- 设思维步骤编号 \( t = 1, 2, \dots, k \)，临时保留窗口大小为 \( w \)（例如 w=4，表示一个思维步骤对后续最多 w-1 个步骤可见）。
- 每个步骤 St 生成后，通过前向传播计算其记忆令牌 KV 缓存 \( \tilde{M}_t \)。
- 更新 KV 缓存容器：保留问题缓存 \( \tilde{Q} \)、所有历史记忆令牌缓存 \( \tilde{M}_1,\dots,\tilde{M}_t \)，同时临时保留最近 w 个原始思维步骤的 KV 缓存（当 t≥w 时，丢弃最旧的 \( \tilde{S}_{t-w+1} \)）。
- 生成下一步 St+1 时，模型可以同时关注记忆令牌（压缩信息）和临时保留的原始步骤（细粒度信息）。

#### 训练方案（关键创新：混合注意力）
- 直接复制推理时的注意力模式（即 Shortcut Attention）进行训练会导致**捷径学习**：模型更倾向于从原始文本中直接读取信息，而不学习通过记忆令牌进行压缩和检索，导致压缩通路训练不足。
- 提出 **Hybrid Attention**：在同一训练序列中随机将一部分思维步骤设置为 Shortcut Attention（让后续几步可以直接看到该步骤），另一部分设置为 **Bottleneck Attention**（屏蔽直接通路，强制信息仅通过记忆令牌传递）。
- 两种注意力模式混合训练，使得压缩通路和直接通路都得到充分训练，同时避免训练-推理不匹配。

#### 数据重构
- 将 CoT 输出按分隔符（`\n\n`）拆分为思维步骤 S1…Sk，在每个中间步骤后插入记忆令牌序列 Mt，形成训练序列：`X = Q ⊕ S1 ⊕ M1 ⊕ S2 ⊕ M2 ⊕ … ⊕ Sk`。

## 7. 优点

- **方法创新**：提出“临时保留原始思维步骤 + 压缩记忆”的混合范式，在不显著增加缓存的前提下保留细粒度信息，解决了现有压缩方法信息丢失问题。
