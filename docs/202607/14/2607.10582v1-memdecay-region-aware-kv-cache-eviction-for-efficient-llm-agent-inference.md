## 摘要
大语言模型（LLM）Agent会累积异构上下文，包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理，其键值（KV）缓存可能成为主要的内存瓶颈。现有的逐出策略通常对所有词元应用相同的基于注意力或基于最近使用的规则，忽略了Agent编排器已有的语义结构。我们提出MemDecay，一种无需训练、区域感知的KV缓存逐出策略。MemDecay为词元分配区域特定的基础优先级和衰减率，当词元获得注意力时刷新保留分数，并在固定缓存预算下逐出得分最低的页面，同时允许关键区域被固定。我们还提供了一种从测量的注意力寿命中校准衰减率的方法。我们使用Qwen2.5-1.5B和3B模型，在约450和1700个词元的上下文中评估MemDecay。在所有设置中，不同区域的注意力寿命相差一个数量级：系统词元的半衰期范围为148到189个解码步，而草稿板词元仅为14到16个步。固定策略在所有设置中以全缓存精度保留了系统区域的事实，而所有基线最多只能保留24个中的13个。随着上下文增长，区域感知的保留仍然有效，而基于最近使用的保留则崩溃。然而，累积注意力保留在未固定内容上表现更好，消融实验表明注意力分数归一化是当前公式的主要限制。这些结果确立了语义提示结构作为KV缓存管理的稳健信号，同时阐明了如何将其与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

### 2. 论文提出的方法论
- **核心思想**：每个token被分配一个区域标签 \( r(i) \in \{\text{system, plan, user, tool\_in, tool\_out, retrieval, scratchpad}\} \)。每个区域配置基础优先级 \( b_r \) 和衰减率 \( \lambda_r \)。保留得分结合结构先验和注意力重要性：
  \[
  s_i(t) = b_{r(i)} \exp(-\lambda_{r(i)} \Delta t_i(t)) + \alpha a_i(t)
  \]
  其中 \( \Delta t_i \) 是从最后一次被注意力刷新（注意力超过阈值 \( \tau \)）以来的步数，\( a_i \) 是通过指数加权移动平均（EWMA）更新的注意力重要性。
- **关键技术细节**：
  - 注意力重要性仅在观测步（每k步）采样，采用平均注意力质量。
  - 刷新机制：当 token 获得的注意力超过阈值 \( \tau \) 时，其衰减时钟重置，但结构先验保持不变。
  - 固定区域（如系统指令）可以规避逐出；其他区域按页面（16 token）计平均分，在预算约束下逐出得分最低的页面。
  - 衰减率通过测量每个区域在不同年龄下的平均注意力质量，拟合指数衰减模型 \( \log \bar{A}_r(\Delta t) \approx \log c_r - \hat{\lambda}_r \Delta t \) 得到。
- **算法流程**（文字描述）：
  1. 每个观测步更新所有 token 的注意力重要性 \( a_i \)。
  2. 根据注意力是否超过阈值更新刷新时钟。
  3. 计算所有非固定 token 的保留得分。
  4. 计算每个页面的平均得分。
  5. 如果缓存大小超过预算，则从最小堆中逐出得分最低的非固定页面，直到满足预算。

### 6. 论文的主要结论与发现
- **区域寿命层次稳定**：系统 token 半衰期 148–189 步，scratchpad 仅 14–16 步，差距一个数量级；检索区域在所有设置中最长久的可移动区域。

- **基于最近使用的策略随上下文增长崩溃**：Streaming 在短上下文有竞争力，但在长上下文降至接近零。
- **累积注意力（H2O）在未固定内容上胜出**：在未固定区域召回上，H2O 优于 MemDecay，且随模型规模增强。

### 7. 优点
- **无需训练**：仅需轻量级统计校准（拟合衰减率），不更新模型权重。
- **利用已有语义结构**：直接使用配置器提供的区域标签，与现有服务堆栈（如 PagedAttention）的页面粒度兼容。
- **注意力刷新机制**：允许活跃 token 重新获得保护，且区域先验独立。
- **固定区域提供确定性保护**：在内存紧张时保证关键指令不丢失。
- **方法可解释**：每个 token 的保留得分由明确的基本优先级、衰减和注意力组成。
- **测量协议严谨**：通过拟合 log-linear 模型得到衰减率，并进行 leave-one-scenario-out 交叉验证，避免数据泄露。

### 8. 不足与局限
- **未固定内容召回不足**：相比 H2O，MemDecay 在用户等旧区域召回较差，原因是注意力项幅度过小。论文指出需要通过归一化修复，未给出最终解决方案。
- **实验规模较小**：仅测试了 1.5B/3B 模型、<2k token 上下文，未在生产级长上下文（如 32k+）和更大模型上验证。
- **位置混杂**：system、plan、scratchpad 区域位置固定，其寿命估计可能受固定位置影响，无法完全分离。
- **不可逆逐出**：一旦逐出无法恢复，可能错过后期重新相关的 token（如用户问题在早期 evict 后又被引用的场景），尽管论文提出分级降级扩展。
- **标签质量依赖**：依赖编排器提供准确标签，未来可能需训练分类器，且尚未测试标签噪声鲁棒性。
- **跨区域信息流**：工具输出被引用到 scratchpad 时，两者都衰减得快，可能导致信息丢失。
- **开销**：当前必须使用 eager attention 读取注意力权重，增加解码延迟；生产部署需抽样或复用。
- **结论普适性**：区域寿命层次可能高度依赖具体工作负载，论文中检索区域衰减慢于直觉，表明需要每工作负载校准，而非固定默认值。
