## 摘要
我们研究了在硬性键值（KV）缓存内存预算下，批量大语言模型（LLM）推理的非预见性调度问题。每个请求具有已知的提示长度但未知的响应长度，其内存占用包括固定的提示部分和随每个解码令牌增长的响应部分。在每个解码轮次中，调度器选择一个可行的活跃请求批次；驱逐一个请求会丢弃其累积的缓存状态，浪费先前的计算。目标是针对已知所有响应长度的最优预见性调度，最小化总完成时间。我们提出了首个在任意提示长度和任意响应长度下无需额外假设的常数竞争比算法。我们的算法并非依赖单一的通用调度策略，而是构建于一种新颖的基于模式感知路由框架之上。专用子调度器处理不同的内存增长几何特征，而元调度器在它们之间时间共享内存预算，并根据每个作业执行过程中逐渐揭示的行为动态路由该作业。该框架还在在线到达情况下为完工时间和总完成时间提供了常数竞争比保证。

## Abstract
We study non-clairvoyant scheduling for batched Large Language Model (LLM) inference under a hard Key-Value (KV) cache memory budget. Each request has a known prompt length but an unknown response length, and its memory footprint comprises a fixed prompt component together with a response component that grows with each decoded token. At each decoding round, the scheduler chooses a feasible batch of active requests; evicting a request discards its accumulated cache states, wasting prior computation. The goal is to minimize total completion time against the optimal clairvoyant schedule that knows all response lengths.   We present the first constant-competitive algorithm for arbitrary prompt lengths and arbitrary response lengths with no additional assumptions. Rather than relying on a single universal scheduling policy, our algorithm is built on a novel regime-aware routing framework. Specialized sub-schedulers handle different memory-growth geometries, while a meta-scheduler time-shares the memory budget across them and dynamically routes each job as its execution progressively reveals its behavior. This framework also yields constant-competitive guarantees for makespan and for total completion time under online arrivals.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用**模式感知路由（Regime-Aware Routing）**框架，将作业分为三种几何增长模式（regime），每种模式由专门的子调度器处理，并通过一个元调度器（meta-scheduler）在模式间时分共享内存预算并动态路由作业。
  - **三种模式**（基于提示长度阈值 \(\Lambda = M/4\)）：
    1. **大作业（Large jobs）**：\(s_i > \Lambda\)，几乎占满整个内存。
    2. **提示重型小作业（Small prompt-heavy jobs）**：\(s_i \le \Lambda\) 且 \(o_i \le s_i\)，内存增长受限（不超过2倍提示长度）。
    3. **响应重型小作业（Small response-heavy jobs）**：\(s_i \le \Lambda\) 且 \(o_i > s_i\)，内存增长远大于提示长度，面积（area）由响应长度主导。
- **关键子调度器**：
  - **矩形条调度器（Rect）**：处理大作业和提示重型小作业。将每个作业抽象为固定宽度 \(w_i\)、未知处理时间 \(p_i\) 的矩形条。使用**预算翻倍贪婪算法**，维护一个按面积预算排序的待处理队列，确保队列单调性，从而保证高内存利用率和接近面积顺序的完成顺序，证明其竞争比为36（当 \(\Lambda = M/4\)）。
  - **响应型调度器（Response）**：处理响应重型小作业。利用响应长度与面积成平方关系的特点，采用**几何切片**方法，将未完成作业按响应长度几何增长的阶段（cap）划分，在每个阶段内用SPS（Staggered Pipeline Scheduling）子程序调度替代作业（共享代理提示长度），保证高内存利用率，并证明竞争比约为78.67。
- **元调度器（Meta-scheduler）**：
  - **黑盒元调度器（BlackBoxMeta）**：若已知作业分区，则在几何增长的阶段中轮换运行各子调度器，损失常数因子（论文证明为 \(4k-1\) 倍，k为子调度器个数）。
  - **路由元调度器（Route）**：非先知场景下，初始将小作业分配到提示重型分支，若运行到 \(s_i\) 步仍未完成，则认证为响应重型并路由到响应重型分支；大作业直接到大作业分支。时间分片与黑盒版本相同。该路由机制保证了认证和完成均在常数因子内。
- **关键引理**：
  - **面积下界（Lemma 1）**：最优总完成时间下界为 \(\frac{1}{M}\sum_{r}(n-r+1)A_{\pi(r)}\)，其中 \(A_i = s_i o_i + o_i(o_i+1)/2\)。

  - **路由损失（Lemma 11）**：响应重型作业在Route中的完成时间不超过 \(8C^P_i + C^R_i\) 或 \(9C^R_i\)。
