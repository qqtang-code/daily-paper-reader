## 摘要
固定状态序列模型将无界过去压缩为有界状态，这将其关联召回上限大致限制在状态维度；注意力通过为每个令牌保留键值条目来逃避上限，但代价是二次计算和随序列增长的缓存。我们研究中间地带：一种稀疏缓存，仅在输入新颖时分配一个槽位，因此其大小跟踪不同项目的数量而非令牌数量。分配规则是DP-means聚类规则，即狄利克雷过程混合的小方差极限，不用于潜在变量推断，而是作为深度递归主干的键值记忆运算符。我们以两种形式开发它：具有固定浓度的静态缓存和其浓度遵循近期新颖率的惊喜自适应变体。在具有冗余的受控关联召回基准上，我们展示了该缓存匹配全注意力召回，同时仅存储不同项目；它在召回与大小前沿上支配固定预算驱逐缓存；并且在状态空间骨干上，它以所有测试模型中最低的记忆回答了召回查询和长程聚合查询。分配是可端到端学习的：一个仅在任务损失上训练的双参数新颖阈值门精确恢复了该规则，而过参数化门则失败，因此操作成分是归纳偏置而非容量。证据是在适度规模上进行的一系列受控机制研究，在四个真实流（推荐、系统日志、临床事件和保险索赔）上确认了不同项目属性；在同伴研究中进行了真实骨干、真实语料语言验证。

## Abstract
Fixed-state sequence models compress an unbounded past into a bounded state, which caps their associative recall at roughly the state dimension; attention escapes the cap by keeping a key-value entry for every token, at quadratic compute and a cache that grows with the sequence. We study the middle ground: a sparse cache that allocates a slot only when an input is novel, so its size tracks the number of distinct items rather than the number of tokens. The allocation rule is the DP-means clustering rule, the small-variance limit of a Dirichlet-process mixture, used not as latent-variable inference but as the key-value memory operator for a deep recurrent backbone. We develop it in two forms, a static cache with a fixed concentration and a surprise-adaptive variant whose concentration follows the recent novelty rate. On a controlled associative-recall benchmark with redundancy we show that the cache matches full-attention recall while storing only the distinct items, that it dominates a fixed-budget eviction cache on the recall-versus-size frontier, and that on a state-space backbone it answers both a recall query and a long-range aggregate at the lowest memory of any model tested. The allocation is learnable end to end: a two-parameter novelty-threshold gate trained on the task loss alone recovers the rule exactly, whereas an over-parameterized gate fails, so the operative ingredient is the inductive bias rather than capacity. The evidence is a family of controlled mechanism studies at modest scale, with the distinct-items property confirmed on four real streams (recommendation, systems logs, clinical events, and insurance claims); a real-backbone, real-corpus language validation is pursued in a companion study.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：序列模型通常采用两种极端方案来记忆历史信息：
  - **固定状态模型**（如结构化状态空间模型 S4、Mamba）将整个过去压缩为有界状态，计算线性、常数推理，但其关联召回（associative recall）能力受限于状态维度，无法区分超出维度的不同事件。
  - **全注意力机制**（Transformer 等）为每个 token 保留独立的键值条目，召回能力强，但缓存随序列线性增长、计算呈二次复杂度。
- **中间地带需求**：真实序列中包含大量重复或冗余 token，关键信息由**不同（distinct）项目**承载，而非每个 token。因此理想记忆应只在新异项目出现时分配槽位，否则合并到已有槽位，使缓存大小与不同项目数成正比。
- **现有方法不足**：
  - 聚类注意力（Clustered/Routing Attention）在固定窗口内近似全注意力，但簇数是超参数，不针对不同项目。
  - 增长型记忆（Memorizing Transformers、kNN-LM 等）仍然存储每个 token，不按新颖性分配。
  - 固定预算稀疏缓存（StreamingLLM、H2O、SnapKV）按启发式规则驱逐，不按新颖性增长。
  - 惊奇门控记忆（Titans、EM-LLM）通过可学习写入门控增长，但未采用 DP-means 分配。
- **本文贡献**：首次将 DP-means（Dirichlet-process 混合的小方差极限）作为序列深度模型的键值记忆算子，使缓存大小自适应地与不同项目数对齐，并验证其可端到端学习。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 在线 DP-means 分配规则：输入一个新键值对时，计算其与已有槽位的最大余弦相似度。若最大相似度小于阈值 τ（即足够新颖），则分配新槽；否则合并到最相似槽位。
- 该规则本质是 Dirichlet-process 混合模型的最大后验/小方差近似，可无缝集成到深度递归骨干（如 SSM）中。

### 2.2 两种形式
- **Approach I：静态 DP 缓存（Dirichlet-process Cache）**
  - 固定阈值 τ，缓存大小单调增长到不同项目数。
  - 在新颖性低于阈值时合并，槽键保持第一次出现或运行质心。
  - 查询时通过 attention 在所有槽位上读取（softmax 加权求和）。
- **Approach II：惊喜自适应 DP 缓存（Surprise-adaptive Dirichlet-process Cache）**
  - 阈值 τ 由温度信号自适应调节：温度 T_t = (1-η)T_{t-1} + η o_t，其中 o_t 表示是否分配新槽。
  - 有效预算 M_t = M_0 + β T_t，当槽数超过 M_t 时根据使用计数驱逐（最小使用槽位）。
  - 适合非平稳需求：在高新异阶段增加容量，在平稳阶段合并释放槽位。

### 2.3 可学习门控
- 提出一个可微分两参数门控：g_t = σ(a (nov_t - b))，其中 nov_t 是新颖性得分（1 - 与之前 token 最大相似度）。
- 训练时用软掩码（叠加 log g_t 到 attention logits），损失函数包含预算惩罚项 (∑ g_t - M)^2。
- 推理时保留 top-M 作为硬缓存。
- 对比实验表明：两参数门控可精确恢复规则，而过参数化 MLP 门控失败，证明归纳偏置（新颖性特征和阈值结构）是关键。

## 6. 主要结论与发现

1. **缓存大小匹配不同项目数**：在合成冗余流上，DP 缓存（静态）仅存储 K 个不同项目，达到完美召回（1.00），而全注意力需要 L 个槽位，在 r=4 时节省 4 倍。
2. **召回-大小前沿最优**：静态 DP 缓存前沿完全支配固定预算驱逐策略（包括 heavy-hitter、recency、StreamingLLM），在相同槽位数下召回更高。
3. **在真实任务上同样有效**：MovieLens 流派召回任务，DP 缓存使用 ~1370 槽位达到 0.99 召回，而 heavy-hitter 在 1600 槽位仅 0.61。归因于对重尾分布的罕见项覆盖。
4. **分配可端到端学习**：两参数新颖阈值门可从任务损失学习，无需手调，且必须要有新颖性归纳偏置；过参数化门失败。
5. **与 SSM 结合效果最佳**：SSM+DP缓存既完成召回（perfect）又完成长程聚合（R²=0.99），总槽位仅为不同项目数，低于任何单独模型。
6. **自适应浓度在非平稳需求中有效**：当需求变化时，自适应缓存以相同平均槽位数取得更高召回；当需求平稳时，与固定预算持平。
7. **真实流验证**：在推荐、日志、临床、保险四个领域，分配规则自动将槽位数对齐到不同实体数（60-140 倍压缩），任务是端任务或统计验证。
8. **阈值鲁棒**：在 μ 值 [0.5, 0.8] 范围内所有真实流表现一致；低于该阈值欠分配，高于则良性多存储重复。

## 7. 优点（方法或实验设计上的亮点）

- **概念清晰**：精确定义了“不同项目”作为缓存大小的自然单位，从信息理论角度优于基于 token 的缓存。
- **合成控制设计**：通过冗余度 r 有意识分离 token 数与不同项目数，直接检验核心声称。

- **消融完整**：通过可学习门控消融证明归纳偏置的重要性，而非仅是容量。
