## 摘要
视觉Transformer在获得高精度的同时推理延迟较高。半结构化N:M稀疏性可降低算术成本，但其理论节省在现代GPU上往往无法转化为实际的端到端加速。这种不匹配源于部署延迟不仅取决于算术缩减，还取决于稀疏条件下的执行规律性和硬件调度。因此，实现实际加速需要跨稀疏执行和稀疏配置的协同设计。为此，我们提出一种用于N:M稀疏ViT推理的软硬件协同设计框架。在硬件方面，我们设计了MD-SpMM，这是一个N:M稀疏CUDA内核，它将稀疏GEMM重组为微密集的张量核心对齐数据流，并使用推理感知的自适应并行性来维持利用率。在软件方面，我们在明确的端到端延迟预算下执行逐层稀疏搜索，采用三阶段启发式搜索与约束松弛，以避免过早收敛并实现部署感知的稀疏分配。在多个ViT/Swin模型和GPU平台上的实验表明，该框架实现了超过2.2倍的延迟加速，同时保持相当的精度，并在相同延迟约束下提供更优的精度。源代码已在https://github.com/liuganhuo/realizable-nm-sparse-transformer公开。

## Abstract
Vision Transformers (ViTs) achieve strong accuracy but incur high inference latency. Semi-structured N:M sparsity can reduce arithmetic cost, yet its theoretical savings often fail to translate into proportional end-to-end speedups on modern GPUs. This mismatch arises because deployment latency depends not only on arithmetic reduction but also on execution regularity and hardware scheduling under sparsity. Achieving practical acceleration, therefore, requires coordinated design across sparse execution and sparsity configuration. To this end, we propose a hardware-software co-design framework for N:M sparse ViT inference. On the hardware side, we design MD-SpMM, an N:M sparse CUDA kernel that reorganizes sparse GEMM into micro-dense, Tensor-Core-aligned dataflow and uses inference-aware adaptive parallelism to sustain utilization. On the software side, we perform layer-wise sparsity search under explicit end-to-end latency budgets using a three-stage heuristic search with constraint relaxation to avoid premature convergence and enable deployment-aware sparsity allocation. Experiments on multiple ViT/Swin models and GPU platforms show that the framework achieves over 2.2x latency speedup while maintaining comparable accuracy and delivering superior accuracy under the same latency constraint. The source code is publicly available at https://github.com/liuganhuo/realizable-nm-sparse-transformer.

- **核心问题**：视觉Transformer（ViT）推理延迟高，半结构化N:M稀疏性理论上可以减少计算量，但实际端到端加速往往远低于理论预期。原因在于部署延迟不仅取决于算术运算减少，更依赖于执行规律性和硬件调度行为。

- **背景**：N:M稀疏在CNN、ViT等模型中已被广泛采用，但INT8推理下，密级GEMM已经高效，稀疏带来的不规则存取和元数据开销会破坏Tensor Core数据流，抵消算术节省。同时，统一应用相同N:M比例无法适配不同层对精度和延迟的差异化影响。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出一个搜索-内核协同设计框架，包含两个层面：
  - **执行可实现性（Execution Realizability）**：设计MD-SpMM内核，将稀疏计算转换为Tensor Core友好的规则数据流。
  - **配置可实现性（Configuration Realizability）**：在显式延迟约束下进行逐层N:M稀疏配置搜索，使用延迟-感知的轻量级预测器和可行区域感知的进化搜索。

- **关键技术细节**：
  - **MD-SpMM内核**：
    - **Weight Packing**：对权重进行编码友好的打包表示（B_pack），将非零值和位置信息联合编码，减少额外索引加载。
    - **Micro-Dense Execution**：将N:M稀疏窗口中的N个非零值物化为固定大小的稠密块（与Tensor Core的MMA片对齐），使Tensor Core仍在规则稠密片段上运算，稀疏处理仅限加载和解码阶段。
    - **Adaptive Parallelism（Split-K）**：将归约维度K分割成多个部分，由多个线程块并行计算部分和，增加并行度（从M×N扩展到M×N×splitK），提升SM占用率；同时通过规则驱动策略选择splitK值，控制归约开销。
  - **延迟-感知的逐层稀疏搜索**：
    - **LUT-based延迟预测器**：对每个算子形状和N:M比例预先profile得到查找表（LUT），然后通过非负最小二乘拟合线性模型预测端到端延迟，误差<5%。
    - **可行区域感知进化搜索（三点式）**：
      - 初始化：从均匀稀疏的可行配置中选择初始父代。
      - **Overshoot**：随机对一小部分层进行“稠密化”（增加非零比例），主动违反延迟预算，避免在边界附近信号消失。
      - **Repair**：随机对部分层增加稀疏度，直到满足延迟约束，避免贪婪方向偏见。
      - **Refine**：在可行边界附近逐步稠密化，只接受不降低适应度的合法更新。
    - 适应度函数：满足延迟预算时使用继承精度，否则为负无穷。

- **公式/算法流程**：延迟预测模型为 \( T_{\text{pred}} = \sum_{\ell=1}^{L} \gamma_\ell t_\ell(s_\ell) + b \)，其中 \( t_\ell \) 为LUT查表值，\( \gamma_\ell, b \) 由NNLS校准。搜索算法为(1+λ)进化算法，具体流程见Algorithm 1（见论文第8页）。
