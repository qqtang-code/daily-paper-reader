## 摘要
随着大型语言模型处理越来越长的提示，计算和KV缓存内存成本已成为推理系统的主要瓶颈。现有的输入级提示压缩方法试图解决这一问题，但仅通过标量相关性分数对每个句子进行排序，将文档视为单词和句子的非结构化集合。在预算紧张的情况下，这会导致主题崩溃，即文档的主导主题消耗了预算，丢弃了较少出现但与任务相关的主题。为了保持主题覆盖，需要跨重复出现的主题分配预算，而不是孤立地对句子评分。为此，我们提出SALT，一种模型无关的抽取式框架，它将每个句子的关键词组织成一个按句子频率排序的树，句子频率是文档主题结构的轻量级可重用代理。这种基于树的结构平滑了内存分配，并防止主导主题垄断预算。多锚点检索可激活由查询关键词标记的任意深度的树节点，并且该树在对话轮次中持续存在，支持多轮使用而无需重新编码文档。通过保留文档主题，SATP减少了长上下文提示的预填充计算和内存成本，同时与针对解码延迟和内存的KV缓存方法兼容。

## Abstract
As large language models (LLMs) process increasingly longer prompts, computation and KV-cache memory costs have emerged as major bottlenecks in inference systems. Existing input-level prompt compression methods address this, but rank each sentence by a scalar relevance score, treating the document as an unstructured pool of words and sentences. Under tight budgets, this causes theme collapse, where the dominant theme(s) of a document consumes the budget, discarding less-frequent yet task-relevant themes. Preserving thematic coverage instead requires allocating the budget across recurring themes rather than scoring sentences in isolation. To this end, we propose SALT, a model-agnostic extractive framework that organizes per-sentence keywords into a trie ordered by sentence frequency (SF), a lightweight, reusable proxy for document thematic structure. This trie-based organization smooths memory allocation and prevents dominant themes from monopolizing the budget. Multi-anchor retrieval activates trie nodes labeled by query keywords at any depth, and the trie persists across dialogue turns, supporting multi-turn use without re-encoding the document. By preserving document themes, SALT reduces the prefill computation and memory cost of long-context prompts while remaining composable with KV-cache methods that target decoding-time latency and memory.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）处理长上下文提示时，预填充阶段的计算和KV缓存内存成为主要瓶颈。现有的输入级提示压缩方法（如RECOMP、EXIT、CPC等）对每个句子或段落进行标量相关性评分，然后按得分高低填充固定预算。这种方式将文档视为同质化的句子集合，忽略了文档内在的**主题结构**，在紧预算下易导致 **“主题崩溃（theme collapse）”** ：主导主题占用大部分预算，使得低频但任务相关的次要主题被丢弃。
- **意义**：SALT 旨在解决主题崩溃问题，通过显式感知文档的主题结构来分配压缩预算，从而在降低推理计算量和内存占用的同时，**保留文档的完整主题覆盖**，特别是保护低频但有价值的主题。

### 2. 论文提出的方法论

- **核心思想**：将文档的提取式压缩重新定义为**按主题分配预算**的问题，而非全局标量排序。提出 **SALT（Salience-Aware Lexical Trie）** 框架，轻量级构建文档的词汇主题树，并按树的支系分配预算，再在各支系内选择句子。
- **关键技术细节（两个阶段）**：
  - **索引阶段**：
    1. **关键词提取**：使用轻量编码器 BGE-small-en-v1.5 对文档中的每个句子，基于[CLS]注意力机制选出内容词，并通过Kneedle算法自动确定保留词数（上限40%），获得句子关键词集 $K_i$。
    2. **计算句子频率（SF）**：统计每个关键词出现在多少个句子的关键词集中，即 $SF(w) = |\{i : w \in K_i\}|$。
    3. **构建显著性集合 $S$**：保留句子频率高于 $p$ 分位数（默认0.9）的关键词，并归一化得到权重 $cSF(w) = SF(w)/\max_{u \in S} SF(u)$。
    4. **组织成字典树（Trie）**：对每个句子，取其关键词与 $S$ 的交集，按 $SF$ 降序排序，作为一条从根到叶子（句子ID）的路径插入树中。这样，具有相同高频关键词的句子共享前缀，形成主题分支。
  - **选择阶段**：
    - **摘要模式**（无查询）：从根开始，为每个深度-1分支（即顶层主题）分配预算配额：固定底数 + 比例部分（基于分支的未覆盖质量 $(M_b + \epsilon)^\alpha$，$0<\alpha<1$）。然后在各分支内根据句子对未覆盖质量的减少 $\Delta_b(s_i|R)$ 计算得分（包括句子长度奖励和位置/词法先验），贪心选择句子，达到配额为止。剩余的预算用于全局填充（GlobalFill）。
    - **查询模式**：提取查询关键词，激活树中匹配（包括原始$S$中被修剪但出现于句子中的关键词）的节点，形成激活区域。先执行锚点阶段：选择与查询词法重叠大且嵌入相似度高的节点，获得部分预算 $\beta_q B$。剩余预算同摘要模式分配。最后GlobalFill。
- **多轮对话支持**：索引阶段一次性构建，后续查询只需遍历树并更新激活节点，无需重新编码文档。

### 6. 论文的主要结论与发现

- SALT 在 LongBench 上以20%预算取得预处理方法中的**最佳平均准确率（43.51%）**，接近全基线（50.19%），优于RECOMP（40.22%）等。
- 在 QuALITY 多轮设置下，SALT 准确率（72.6%）接近基线（74.1%）和 FastKV（73.9%），远高于 RECOMP（61.5%）；同时累积计算成本最低（0.91s vs 基线3.71s），实现 **4倍端到端加速**。
- 在 NIAH 任务中，SALT 在不同长度下保持与全基线一致的检索准确率，且能在基线因超长而OOM时仍运行。
- 效率上，SALT 在128k/256k时延迟和内存均低于大多数预处理和KV缓存方法，且**硬件可移植性突出**：V100 (2017) 可处理256k输入，而FastKV等依赖FlashAttention的KV缓存方法在V100上不可运行。
- 主要发现：按主题分配预算能有效防止主题崩溃，保护低频但重要的信息，使压缩在紧预算下仍保持更高的语义覆盖。

### 7. 优点

- **模型无关性**：输出纯文本，可与任何下游LLM和KV缓存优化方法组合。
- **主题感知分配**：通过句子频率字典树显式建模文档主题结构，预算先按主题分配，再在主题内贪心选择，从根本上避免主题崩溃。
- **轻量且可重用**：索引阶段仅需一次轻量编码（BGE-small），构建的字典树跨查询和多轮对话复用，每轮仅需增加极少的激活和选择计算。
- **多锚点检索**：查询模式通过激活树中多个相关节点，能捕获查询关键词以任何深度出现在句子中的情况，增强召回。
- **硬件兼容性高**：不依赖自定义CUDA内核或FlashAttention，可在老式GPU（V100）上运行极长上下文。

### 8. 不足与局限

- **代码任务表现弱**：句子级别操作无法保留代码的行级语义，导致在LongBench代码子任务上显著低于KV缓存方法和全基线。这是所有句子/段落级预处理方法的通病。
- **句子级别粒度限制**：SALT 选择的最小单元是句子，无法在更细粒度（token/子句）上压缩，导致在任务相关信息集中在较小跨度时可能引入噪声或遗漏细节。参考文献中设定20%作为保守下限。
- **关键词提取依赖BGE-small**：虽然轻量，但[CLS]注意力对代码、非英语文本可能不够准确，且未评估多语言场景。
- **未与最新大模型（如GPT-4、Claude）配合测试**：所有实验限于8B参数开源模型，规模差异可能影响结论。
- **无训练机制**：无法通过下游任务信号调整关键词提取或预算分配，灵活性略低于可学习的方法。
- **未涵盖实时流式或极低延迟场景**：虽然效率好，但索引阶段（190ms）在多轮中仍需一次性开销。
