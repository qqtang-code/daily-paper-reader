## 摘要
最近的统一多模态模型表明，单一架构可以同时执行视觉/语言理解和图像生成/编辑。然而，它们反复将所有历史视觉和文本输入馈送到共享的上下文窗口中，由于视觉令牌爆炸和不可靠的跨轮次引用，限制了长时程多模态对话。我们提出了一种认知结构多模态Agent，它将视觉信息外化到情景视觉记忆中，并在推理过程中选择性地重新激活相关情景。该Agent由用于结构化视觉抽象的感知抽象引擎、用于跨轮次记忆检索的认知检索引擎以及用于自主任务推理和动作规划的多模态执行控制器组成。为了解决现有数据集中缺乏轮次级检索监督的问题，我们开发了一个统一的场景引擎，该引擎通过编程生成具有细粒度检索注释的结构化多轮对话，使强化学习能够优化抽象和检索策略。我们还构建了一个按难度分层的长时程视觉对话基准，以评估情景视觉回忆。我们的8B Agent在20轮会话中实现了91.4%的检索准确率，超过了32B基线模型8.2%，同时每轮推理时间几乎减半（23.1秒→12.7秒）。我们进一步提出了认知结构多模态Agent工具包（CMA-Harness），这是一种增强工具的相同认知结构部署，集成了持久化多模态记忆、网络访问、图像生成/编辑/合成工具以及与OpenAI兼容的服务。与单一参数扩展相比，结构化记忆和模块化决策为长时程多模态Agent提供了一种更具可扩展性和效率的范式。代码：https://github.com/caseclose/cma-harness；项目页面：https://caseclose.github.io/cma-harness/

## Abstract
Recent unified multimodal models show a single architecture can jointly perform vision/language understanding and image generation/editing. However, they repeatedly feed all historical visual and textual inputs into a shared context window, limiting long-horizon multimodal dialogue due to visual token explosion and unreliable cross-turn referencing. We propose a Cognitive-structured Multimodal Agent that externalizes visual information into an Episodic Visual Memory and selectively reactivates relevant episodes during reasoning. The agent consists of a Perceptual Abstraction Engine for structured visual abstraction, a Cognitive Retrieval Engine for cross-turn memory retrieval, and a Multimodal Executive Controller for autonomous task inference and action planning. To address the lack of turn-level retrieval supervision in existing datasets, we develop a Unified Scenario Engine that programmatically generates structured multi-turn conversations with fine-grained retrieval annotations, enabling reinforcement learning to optimize abstraction and retrieval policies. We also construct a long-horizon visual-dialogue benchmark stratified by difficulty to evaluate episodic visual recall. Our 8B agent achieves 91.4% retrieval accuracy over 20-turn sessions, surpassing 32B baselines by +8.2% while nearly halving per-turn inference time (23.1s -> 12.7s). We further present the Cognitive-structured Multimodal Agent Harness (CMA-Harness), a tool-augmented deployment of the same cognitive structure integrating persistent multimodal memory, web access, image generation/editing/composition tools, and OpenAI-compatible serving. Structured memory and modular decision-making offer a more scalable, efficient paradigm for long-horizon multimodal agents than monolithic parameter scaling. ## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前统一多模态模型（如BAGEL、Chameleon等）在单一架构内联合处理视觉/语言理解和图像生成/编辑，但在长时程多模态对话中存在两个关键瓶颈：
  - **视觉令牌爆炸**：每轮对话都需反复注入所有历史图像的视觉令牌，随着轮次增加，令牌数量线性增长，严重占用上下文预算。
  - **跨轮引用不可靠**：模型依赖隐式注意力处理长历史上下文，导致视觉引用错误和语义漂移，尤其在需要回溯多轮之前图像的场景中表现极差（如最困难轮次检索准确率低于3%）。
- **整体含义**：单纯扩大模型参数无法解决长程交互中的记忆管理问题，需要显式的结构化记忆和模块化决策机制，将视觉信息外化并选择性激活，才能实现可扩展、高效的长时多模态智能体。

## 2. 方法论：核心思想、关键技术细节与流程

### 2.1 核心思想
- **外化视觉记忆**：不再将所有历史图像作为连续令牌存储在上下文窗口中，而是通过感知抽象引擎（PAE）将每张图像压缩为结构化记忆条目（标签、描述、缩略图），存储在外部情景视觉记忆（EVM）中。
- **选择性回忆**：认知检索引擎（CoRE）根据当前查询和对话状态，从EVM中检索出最相关的视觉片段，仅将少量图像送入后续处理，避免冗余。
- **模块化决策**：多模态执行控制器（MEC）进行任务分类（生成、编辑、理解、纯聊天等），并协调检索和生成流程，确保每个模块可独立训练和升级。

### 2.2 关键技术细节

#### 感知抽象引擎 (PAE)
- 对每张输入的图像生成结构化JSON条目，包含：
  - `tags`：至少5个语义标签（如“sunset, sports car, highway”）。
  - `description`：至少100词的详细文本描述。
  - `thumbnail`：压缩缩略图，用于后续多模态检索。
- 写提示确保输出符合JSON格式，便于机器解析。

#### 认知检索引擎 (CoRE)
- 输入：候选缩略图列表、对应记忆条目（标签+描述）、当前用户查询。
- 输出：图像索引列表（空列表表示无需检索）。
- 使用多模态信息（缩略图+文本）进行跨模态推理，而非仅基于文本。

#### 多模态执行控制器 (MEC)
- 任务分类：将用户请求分为5种模式：生成（generate）、编辑（edit）、合成（composite）、理解（understand）、纯聊天（pure chat）。
- 动作规划：
  - 生成/编辑：将指令重写为详细视觉提示，送入文本到图像模型（Qwen-Image）或图像编辑模型（Qwen-Image-Edit）。
  - 理解/聊天：直接利用检索到的图像和对话历史生成文本应答。
- 通过控制每轮仅处理检索到的少量图像，大幅降低视觉令牌消耗。

#### 训练优化：分阶段SFT+RL
1. **SFT on CoRE**：使用统一场景引擎生成的多轮对话数据监督微调检索器，学习匹配查询与相关记忆。
2. **RL on CoRE (DAPO)**：使用难度加权的Jaccard相似度作为奖励，进一步优化检索策略。奖励函数：\( r = \frac{|\hat{R} \cap R^*|}{|\hat{R} \cup R^*|} \cdot w_d \)，其中\( w_d \)对困难/非常困难轮次设为1.2，普通轮次为1.0；正确弃检（预测空集且真实空集）奖励1，仅一方为空的奖励0。
3. **RL on PAE (DAPO)**：冻结CoRE，以检索准确率作为PAE的奖励，确保记忆表示直接服务于下游检索任务，而非传统字幕质量指标。

### 2.3 数据生成：统一场景引擎 (Unified Scenario Engine)
- 为解决现有数据集缺乏轮次级检索标注的问题，构建自动生成多轮对话的闭环流水线：
  - **用户模拟器（Gemini）**：采样主题，生成查询、真实检索集及难度标签。
  - **零样本多智能体系统（Qwen-VL + Qwen-Image-Edit）**：产生候选回答。
  - **GT-justify验证**：检查检索正确性并修正错误。
  - 已验证的回答反馈给模拟器，生成下一轮。
- 数据规模：270会话×20轮（SFT训练）+ 30会话（SFT验证）+ 强化学习采样 + 100会话（M2CA-Bench测试）。
- 会话结构：\( S = \{(u_i, \tau_i, R^*_i, d_i, f_i)\}_{i=1}^T \)，包含用户输入、任务类型、真实检索集、难度、约束标签。
- 55个主题覆盖8个领域（商业、工业、教育、公共服务、酒店、自然景观、科学、空间）。
- 难度分级（表1）：
  - Easy：同主题顺序生成/编辑。
  - Medium：高相似度变体、简单主题切换。
  - Hard：跨主题检索、长程回调（≥8轮）。
  - Very Hard：多图像比较、融合编辑、模糊引用。
- 防捷径设计：高相似度混淆图像（近重复）和负样本（无需检索的查询）。

## 6. 主要结论与发现

1. **统一模型在长程检索上失效**：BAGEL在Full子集仅23.3%准确率，Hard子集低于3%，表明无显式记忆时无法可靠引用历史图像。
2. **结构化记忆筛选优于全上下文输入**：Multi-Agent Baseline（带PAE+CoRE但未训练）已显著超越All-Context基线，说明过滤无关记忆是长对话的关键。
3. **任务特定训练超越参数缩放**：本文8B优化模型在Full子集上91.4%准确率，超过32B All-Context基线8.2%，且差距在Hard子集扩大至18.4%。
4. **检索准确率直接影响生成质量**：生成质量排名与检索准确率一致，错误检索导致编辑和理解任务质量下降。
5. **分阶段SFT+RL逐步提升**：SFT带来7.3%提升，RL on CoRE再提升6.5%，RL on PAE再提升4.6%，说明优化每模块都是关键的。
6. **效率优势显著**：相比32B全上下文基线，推理时间从23.1s降至12.7s（约1.8倍加速），同时生成质量提升。
7. **文本记忆不足以支撑细粒度视觉回忆**：纯文本记忆在Hard子集上仅24.4%-39.0%，全文EVM（缩略图+文本）达到82.0%。

## 7. 优点

- **方法创新**：将人类认知层次结构（感知、情景记忆、检索、执行控制）引入多模态Agent设计，首次在长程对话中实现视觉信息外化和选择性回忆，突破了统一模型在长上下文下的令牌瓶颈。
- **数据生成机制新颖**：统一场景引擎自动生成带密集检索标注的多轮对话，解决监督信号缺失问题，且设计了难度分层和防混淆机制，提高训练数据和评测基准的质量。
- **强化学习闭环优化**：直接以检索准确率作为PAE和CoRE的奖励，使记忆抽象和检索策略联合优化，优于传统字幕质量指标。
- **高效实用**：8B模型达到超参数32B模型的性能，推理时间减半，证明结构化方法在资源效率上的巨大优势。
- **部署系统CMA-Harness**：将同一架构扩展为增强工具的工程部署框架，集成持久化记忆、网络搜索、多个视觉工具，具备实际应用价值。
