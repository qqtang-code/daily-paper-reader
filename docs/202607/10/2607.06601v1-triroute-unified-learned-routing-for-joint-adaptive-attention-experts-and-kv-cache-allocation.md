## 摘要
条件计算可以将语言模型质量与每令牌推理成本解耦，但领先技术孤立地作用于单一轴线上：专家混合（MoE）稀疏化FFN，深度混合（MoD）跳过整个Transformer块，KV缓存量化压缩注意力内存。我们认为这三个决策（注意力分辨率、专家选择和缓存位宽）是强耦合的，应联合做出：一个稀有到值得全注意力的令牌，无论由哪个专家处理，可能也需要高精度缓存。我们提出TriRoute，一个在三个轴线上共享的轻量级控制器，对于每层每个令牌，发出协调策略：（i）注意力模式（跳过/局部/全局），（ii）稀疏的FFN专家集（使用空专家恢复MoD），以及（iii）KV缓存位宽。该控制器通过异质松弛（用于分类决策的Gumbel-Softmax与直通估计，以及用于专家的负载均衡top-k门控）在拉格朗日预算约束下进行端到端训练，将平均计算和内存成本转换为可控旋钮。我们识别了朴素联合训练中的跨轴线路由崩溃级联，其中一条轴线的崩溃会传播到其他轴线，并通过每轴线归一化和耦合感知的平衡损失来解决。在160M到1.3B参数的计算最优令牌计数的仅解码器模型上，TriRoute在匹配推理FLOPs和内存下帕累托支配了最佳的独立MoD+MoE+KV量化组合，同时更好地保留了稀有实体、代码和算术上的尾案例鲁棒性，而纯困惑度优化会侵蚀这些。事后分析揭示了可解释的结构：控制器将全注意力和高精度缓存分配给句首位置、稀有子词和命名实体，同时廉价地路由功能词。

## Abstract
Conditional computation can decouple language model quality from per-token inference cost, yet leading techniques act on a single axis in isolation: Mixture-of-Experts (MoE) sparsifies the FFN, Mixture-of-Depths (MoD) skips whole transformer blocks, and KV-cache quantization compresses attention memory. We argue these three decisions (attention resolution, expert selection, and cache bit-width) are strongly coupled and should be made jointly: a token rare enough to warrant full attention may also need high-precision caching regardless of which expert processes it. We introduce TriRoute, a single lightweight controller shared across all three axes that, for every token at every layer, emits a coordinated policy: (i) an attention mode (skip/local/full), (ii) a sparse set of FFN experts (with a null expert recovering MoD), and (iii) a KV-cache bit-width. The controller trains end-to-end via a heterogeneous relaxation (Gumbel-Softmax with straight-through estimation for categorical decisions and load-balanced top-k gating for experts) under a Lagrangian budget constraint that turns the average compute and memory cost into a controllable knob. We identify a cross-axis routing-collapse cascade in naive joint training, where collapse on one axis propagates to the others, and address it with per-axis normalization and a coupling-aware balancing loss. On decoder-only models from 160M to 1.3B parameters at compute-optimal token counts, TriRoute Pareto-dominates the best independent MoD+MoE+KV-quantization combination at matched inference FLOPs and memory, while better preserving tail-case robustness on rare entities, code, and arithmetic that pure perplexity optimization erodes. Post-hoc analysis reveals interpretable structure: the controller allocates full attention and high-precision cache to sentence-initial positions, rare subwords, and named entities, while cheaply routing function words.

- **核心问题**：现有条件计算方法（MoE、MoD、KV量化）各自针对Transformer的不同轴线独立优化，但这三个决策（注意力分辨率、专家选择、KV缓存位宽）是**强耦合的**。例如，一个稀有令牌可能需要全注意力与高精度缓存，却可能跳过FFN变换；而独立机制无法表达这种联合模式，导致计算和内存资源分配欠优。

## 2. 论文提出的方法论

### 核心思想
- **统一控制器**：每个令牌在每个层通过共享表示，输出联合策略：注意力模式（skip / local / full）、FFN专家选择（含空专家，实现MoD样式跳过）、KV缓存位宽（2/4/8/16）。
- **端到端训练**：使用异质松弛方法（Gumbel-Softmax + 直通估计处理分类决策；Top-k软门控处理专家选择），并在拉格朗日预算约束下同时优化语言模型损失和成本惩罚。

### 关键技术细节
- **控制器结构**：两层MLP作为共享主干，生成128维表示，然后分为三个线性头（注意头、专家头、位宽头）。输入包含归一化残差状态和廉价侧特征（位置、边界、预测熵、前一层决策）。

- **耦合感知平衡损失**：为防止“跨轴崩溃级联”（一条轴崩溃导致共享表示方差降低，进而引发其他轴崩溃），提出每轴特征白化（去均值除标准差）和边际熵下限（当使用熵低于最大熵的ζ倍时惩罚）。

## 6. 论文的主要结论与发现

1. **联合路由帕累托主导独立组合**：在匹配推理FLOPs和内存下，TriRoute的Pareto前沿始终位于独立组合之下（更好质量），差距在激进预算区域最大。
2. **接近全模型质量**：TriRoute恢复密集模型96-99%的下游准确率，在约一半推理成本下实现。
3. **尾案例鲁棒性显著优于独立组合**：独立组合在稀有实体（困惑度+2.1）、代码、数学、GSM8K上严重退化；TriRoute仅退化0.3-0.4及格点，表明联合路由能够保护重要但罕见的令牌。
4. **路由策略可解释**：
   - 注意力和位宽高度相关，分配给稀有子词、句首令牌、命名实体；功能词获得低资源。
   - FFN计算与注意力解耦：稀有实体通常跳过FFN但仍保留全注意力与高精度缓存（MoD无法实现）。
   - 层间模式：早期层注意力高，后期层选择性跳过；专家使用随深度增加。
5. **跨轴耦合非平凡**：注意力和位宽互信息≈0.31 nats，FFN与其他轴<0.08 nats，支持共享主干+分离头的设计。

## 7. 优点

- **统一路由框架**：将三个独立机制整合为一个端到端学习的控制器，无需手工调参多个超参数。
- **耦合感知设计**：识别并解决跨轴崩溃级联问题，使联合训练可行。
- **单预算控制**：通过Lagrange乘子将FLOPs和内存约束合并为一个标量旋钮，方便实际使用。
- **可解释性强**：学习到的策略与语言学特征对齐，提供了计算节省的机制解释。
- **尾案例鲁棒性**：相较于独立组合，更好地保护了稀有实体、代码、数学等关键输入，降低了部署风险。

## 8. 不足与局限

- **规模限制**：实验仅到1.3B参数，未在≥10B模型上验证，可能无法保证在大规模下的协调优势依然成立。

- **训练复杂性**：联合训练超参数（α, β, γ, ζ, ρ_λ）较多，虽在实验中转移，但可能在新场景下需要调优。
- **动作空间简化**：仅使用固定三种注意力模式和四种位宽，未探索连续或更丰富的变体（如自适应跨度、检索式稀疏注意力、学习组大小等）。
