## 摘要
视频扩散变换器处理长时空序列，使得自注意力成为高分辨率视频生成的主要瓶颈。免训练稀疏注意力降低了这一成本，但自适应Top-p路由在多GPU序列并行下会导致每头工作负载不均衡。由此产生的工作负载异质性将稀疏注意力转变为层级拖累问题。我们提出\method{}，一种免训练的稀疏注意力系统，可在多GPU序列并行下提升自适应稀疏注意力的分布式执行效率。\method{}采用Top-p路由、Top-k安全下限和视频感知的块组织作为稀疏路由前端，然后在运行时修复物化掩码。运行时负载均衡通过点对点通信迁移少量重负载头，以缩短当前关键路径。松弛感知稀疏增强用额外高价值块填充残差非关键秩的松弛，而重叠将调度和迁移开销隐藏在现有计算之后。在步骤蒸馏的Wan2.2 I2V上，\method{}将平均负载不均衡从1.34降至1.08，相对于FlashAttention实现了4.41倍的注意力加速，同时实现了2.02–2.11倍的DiT推理加速，并保持了有竞争力的视频质量。

## Abstract
Video Diffusion Transformers process long spatio-temporal sequences, making self-attention the main bottleneck in high-resolution video generation. Training-free sparse attention reduces this cost, but adaptive Top-$p$ routing creates uneven per-head workloads under multi-GPU sequence parallelism. The resulting workload heterogeneity turns sparse attention into a rank-level straggler problem. We present \method{}, a training-free sparse-attention system that improves the distributed execution efficiency of adaptive sparse attention under multi-GPU sequence parallelism. \method{} uses Top-$p$ routing, a Top-$k$ safety floor, and video-aware block organization as the sparse-routing frontend, then repairs the materialized mask at runtime. Runtime Load Balancing migrates a small number of heavy heads via P2P communication to shorten the current critical path. Slack-Aware Sparse Augmentation fills residual non-critical-rank slack with additional high-value blocks, while overlap hides scheduling and migration overhead behind existing computation. On step-distilled Wan2.2 I2V, \method{} reduces average load imbalance from 1.34 to 1.08 and delivers a $4.41\times$ attention speedup over FlashAttention, while achieving a $2.02$--$2.11\times$ DiT inference speedup with competitive video quality.

# FVAttn: 自适应稀疏注意力与运行时负载均衡——论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：视频扩散变换器（Video DiTs）在生成高分辨率长视频时，自注意力计算量随序列长度呈二次增长，成为主要的推理瓶颈。免训练稀疏注意力通过跳过低价值的查询-键交互来加速，其中自适应 Top‑p 路由能够在保持保真度的同时灵活分配计算预算。
- **核心问题**：Top‑p 路由导致不同注意力头保留的关键块数量差异巨大，在采用 Ulysses 序列并行的多 GPU 环境中，这种每头的负载不均匀会通过全到全（All‑to‑All）映射到不同 GPU 秩（rank）上，造成严重的秩级拖累（straggler）：最慢的秩决定同步注意力的完成时间，其他秩则产生空闲气泡，抵消了稀疏带来的计算节省。
- **整体含义**：现有方法（如 db‑SP、S‑HPLB）依赖历史或离线画像进行预调度，但由于少步蒸馏设置下每步稀疏模式变化剧烈（最高相邻步变化达 97%），预调度方案效果不佳。因此，有必要在掩码物化后根据实际负载进行轻量级运行时修复，以恢复分布式效率。

## 2. 方法论：核心思想与关键技术细节

- **整体系统**：FVAttn 是一个免训练的稀疏注意力系统，包含稀疏路由前端、运行时负载均衡（RLB）和松弛感知稀疏增强（SASA）三个模块，并通过重叠技术隐藏开销。
- **稀疏路由前端**：
  - 使用视频感知的块组织（Hilbert 曲线重排）使相邻 Token 倾向落在同一块。
  - 主要采用 Top‑p 阈值（累积归一化重要性达到 p）自适应决定保留的块数，同时加入 Top‑k 安全底（至少保留 k 个块）防止过度稀疏。
- **运行时负载均衡（RLB）**：
  - 掩码物化后，每头的实际工作负载 \(L_h = \text{mean}_i \sum_j M_{h,i,j}\) 完全可观察。
  - 观察到负载不均衡高度集中在少数重负载头（heavy heads），因此采用约束的点对点（P2P）头迁移：每个秩最多发送一个本地头，接收一个头（默认 8 GPU 下约 20% 的本地头）。
  - 迁移计划通过轻量级 all‑gather 获得全局密度信息，每个 GPU 独立求解最小化最大负载的排列 \(\sigma\)（目标函数：先最小化 \(\max_r L_r^{\text{new}}\)，再最小化方差）。
  - 通信开销极小，且与当前计算可重叠。
- **松弛感知稀疏增强（SASA）**：
  - RLB 后非关键秩仍有残差松弛 \(\Delta L_r = L_{\max}^{\text{new}} - L_r^{\text{new}}\)。
  - 对满足松弛阈值 \(n_2\)（默认 0.07）的秩，分配额外稀疏预算 \(B_r = n_1 \Delta L_r\)（\(n_1=0.8\)），利用路由阶段已排序的块重要性顺序添加高价值块，不延长关键路径。
  - 触发条件：当负载不均衡因子 \(\rho = \max_r L_r / \text{mean}(L_r)\) 在 [1.05, 1.10) 时仅启用 SASA，≥ 1.10 时启用 RLB+SASA。
- **重叠与优化**：
  - CPU‑GPU 重叠：平衡计划搜索和 SASA 预算计算在 GPU 执行 V 量化时完成。
  - 计算‑通信重叠：P2P 迁移与非迁移头的计算和 SASA 增强同时进行。
  - 最终使调度、迁移、增强的可见开销降至注意力的 1.87%。

## 6. 主要结论与发现

1. **负载均衡有效**：RLB 将平均负载不均衡因子从 1.34 降至 1.08，注意力延迟减少约 10%。
2. **松弛利用提高质量**：SASA 在几乎不增加延迟的情况下提升 PSNR 达 0.46 dB，SSIM 提升 1.7%，LPIPS 降低 13.9%。
3. **总体加速显著**：完整 FVAttn 在 I2V 上实现 4.41× 注意力加速和 2.02–2.11× DiT 推理加速，同时 VBench 得分优于或持平密集基线。
4. **优于历史调度**：与 db‑SP（基于历史）相比，RLB 在当前掩码物化后修复更有效，在不均衡较高时优势更大。
5. **开销低**：所有运行时组件的可见开销仅占注意力延迟的 1.87%，通过重叠几乎完全隐藏。

## 7. 优点

- **问题发现精准**：指出了自适应 Top‑p 路由与多 GPU 序列并行之间的根本矛盾，并定位到“少部分重负载头”导致的关键路径。
- **方案轻量且实用**：RLB 采用约束 P2P 迁移（每秩只动一个头）而非全局重分区，通信和搜索代价极低，适用于生产部署。
- **利用松弛双重获利**：SASA 将等待时间转化为质量提升，巧妙结合了系统调度与算法保真度。
- **硬件无关性思考**：虽然实验在 H20 上，但给出了可调的触发阈值和增强系数，便于适配不同通信/计算比硬件。
- **实验设计公平**：在同一硬件、相同输入种子下比较，且对基线（如 db‑SP）使用其原始实现。

## 8. 不足与局限

- **适用范围有限**：主要针对长视频生成（长时空序列）。对于短序列或弱稀疏性任务（如图像生成），稀疏注意力节省和负载均衡收益都会显著降低。
- **硬件依赖**：NVLink 的高带宽是 P2P 迁移低开销的关键。在 PCIe 连接或通信带宽较低的环境下，迁移可能会带来更大的可见开销，需要重新校准参数。
- **并行策略单一**：仅在 Ulysses 风格序列并行下验证，未探索 Ring Attention、USP 等其他序列并行范式。
- **少步蒸馏设定**：论文聚焦 4 步蒸馏模型，稀疏模式随步变化剧烈；对于更多步数（如 50 步）或未蒸馏模型，预调度可能更有效，FVAttn 的优势可能减小。
- **Animate 数据集小**：仅 20 例，统计显著性一般。VBench 测试集虽然大，但仅供度量视频质量，未提供人类评估。
