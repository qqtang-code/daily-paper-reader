## 摘要
参数高效微调减少了模型和优化器的内存占用，但密集注意力仍然使得长训练序列成本高昂。我们将层次化全局注意力（HGA）与分段反向传播和分层KV存储相结合。只有当前活跃段在显存中保持可微；较早的KV被分离到内存或NVMe中，HGA为每个查询块加载一组有界的精确历史令牌。在采用4位QLoRA的Qwen3-8B模型和PG19数据集上，对16 GB Quadro RTX 5000进行密集训练时，可容纳2048个令牌，但在4096个令牌时失败，而HGA在峰值显存15.28 GB下可达16384个令牌。在评估阶段，相同的适配器在该显卡上运行131072个令牌；显存并非恒定不变，而是随着驻留的分块摘要缓慢增长，因此内存和NVMe容量设定了超出这些长度的实际限制。在共享的2K训练长度下，HGA训练和密集训练的适配器在相同的密集注意力读取下分别获得2.7405和2.7383 nat，而原始模型获得2.9541 nat。在此边界上，HGA训练已经略快（217.75 vs. 207.02令牌/秒），并且HGA与密集的吞吐量比从1K提升到2K；由于HGA使每个令牌关注的历史集大致恒定，而每个令牌的密集工作量随上下文增长，我们预期这一优势将随着上下文增加而扩大。主要质量和检索比较中使用密集注意力，以便它们衡量学习到的权重并保持与标准生成框架兼容。HGA也可用于检索和生成；一个优化的生产级服务实现正在开发中。

## Abstract
Parameter-efficient fine-tuning reduces model and optimizer memory, but dense attention still makes long training sequences expensive. We combine Hierarchical Global Attention (HGA) with segment-wise backpropagation and tiered KV storage. Only the active segment remains differentiable in VRAM; older KV is detached into RAM or NVMe, and HGA loads a bounded set of exact historical tokens for each query block. On Qwen3-8B with 4-bit QLoRA and PG19, dense training on a 16 GB Quadro RTX 5000 fits 2,048 tokens but fails at 4,096, whereas HGA reaches 16,384 tokens with 15.28 GB peak VRAM. Under evaluation the same adapter runs through 131,072 tokens on this card; VRAM is not constant but grows gently with the resident chunk summaries, so RAM and NVMe capacity set the practical limit beyond these lengths. At the shared 2K training length, HGA-trained and dense-trained adapters obtain 2.7405 and 2.7383 nat under the same dense-attention readout, while the stock model obtains 2.9541. At this boundary HGA training is already marginally faster (217.75 vs. 207.02 tokens/s), and the HGA-to-dense throughput ratio improves from 1K to 2K; because HGA keeps the attended historical set per token approximately constant while dense work per token grows, we expect this lead to widen as context grows. Dense attention is used for the main quality and retrieval comparisons so that they measure the learned weights and remain compatible with standard generation frameworks. HGA can also be used for retrieval and generation; an optimized production-grade serving implementation is under development.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型的长上下文微调受限于 GPU 显存（VRAM），因为密集注意力（dense attention）需要为每个 token 处理所有历史 token，导致显存消耗随序列长度线性增长。即使采用参数高效微调（如 QLoRA），密集注意力的计算和存储开销仍成为瓶颈。

## 2. 论文提出的方法论

### 核心思想
- 利用 HGA 进行精确 token 的层次化路由，仅加载当前查询块所需的有界历史 token 到 VRAM，而非全部历史。
- 采用截断反向传播（TBPTT）将长序列分割为固定长度段，每个段独立进行前向和反向传播，段间历史 KV 被分离并卸载到外部存储（RAM 或 NVMe）。
- 设计分层 KV 存储架构：紧凑的块摘要常驻 VRAM，组摘要部分缓存，精确 token KV 按需从 RAM/NVMe 加载。

### 关键技术细节
- **两级路由**：
  1. **块选择**：将上下文分成 64-token 块，每个块生成紧凑 key 摘要常驻 VRAM。查询块对所有块摘要评分，选择最相关的 k 个块（加上 sink 和局部窗口）。
  2. **组选择**：对被选中的块，进一步分为 8-token 组，其摘要部分缓存于 VRAM（缓存未命中时从 RAM/NVMe 获取），路由器选择最佳组。
  3. **精确 token 注意力**：仅加载选中组的精确 token K/V 到工作集，计算标准注意力。路由选择不追踪梯度，但 K/V 收集和注意力计算保持可微。
- **无需额外可学习参数**：HGA 复用预训练的 Q/K/V/O 投影，无新增路由器权重，梯度仅更新原有投影层，与密集训练完全一致。
- **分段反向传播（TBPTT）**：序列长度 L 分为段长 S 的若干段。每段计算损失后立即反向传播，段边界处分离历史 KV，梯度不跨段传播。
- **分层 KV 存储**：
  - VRAM：量化模型 + 适配器/优化器 + 当前可微段 + 路由精确 K/V + 所有块摘要 + 组摘要缓存。
  - RAM：已分离的历史 K/V 与非驻留组摘要。
  - NVMe：完整冷记录（可选溢出层）。
  - 峰值显存公式近似：$$M_{\text{GPU}} \approx M_{\text{model}} + M_{\text{adapters/optimizer}} + O(S) + O(B_{\text{route}})$$ 其中 S 为段长，$B_{\text{route}}$ 为路由预算，外部历史记录随 L 线性增长但不占用 VRAM。
- **算法流程**（文字描述）：
  1. 将长序列分割为段（如 2048 token）。
  2. 对当前段，计算其查询块。
  3. 通过 HGA 路由：使用常驻块摘要选择 top-k 块，再打开块按组摘要选择 top 组，从外部存储加载精确 token K/V。
  4. 在 routed working set 上计算注意力，得到输出并进行前向传播。
  5. 计算语言模型损失，立即运行反向传播，更新梯度。
  6. 在段边界处，将当前段的 K/V 作为历史分离并存储到 RAM/NVMe，清空可微状态，前进到下一段。

## 6. 论文的主要结论与发现

- **训练上下文扩展**：在 16 GB GPU 上，HGA + 分段反向传播将可行训练序列长度从密集的 2K tokens 提升到 16K tokens（显存峰值 15.28 GB），且可进一步扩展至 32K（通过流式加载）甚至更长（受限于 RAM/NVMe 容量）。
- **质量保持**：在共享的 2K 训练长度下，HGA 训练和密集训练的适配器在密集注意力评估下的损失仅差 0.0022 nat（2.7405 vs 2.7383），困惑度差异约 0.03 PPL。在 4K 下差异也不超过 0.04 PPL。HGA 训练不损害模型质量。
- **效率优势**：在 2K 下 HGA 训练已略快于密集训练（217.75 vs 207.02 tokens/s）。理论分析表明，因 HGA 每 token 关注的历史集大小约为常数，而密集注意力每 token 工作量随上下文线性增长，因此随上下文增加，HGA 的速度优势将进一步扩大。
- **评估长文本**：HGA 评估可在同一硬件上处理 131,072 tokens，显存仅随块摘要数缓慢增长。
- **检索性能**：在 RULER 风格检索任务中，HGA 训练的适配器在密集评估下达到与密集训练适配器一致的召回率（除一次微小差异外），表明 HGA 训练不影响模型的长上下文检索能力。
- **路由稀疏性**：默认 8 块路由已接近精度饱和；减少到 2 块可大幅节省 KV 交互（81.2%）但代价是困惑度增加 0.54 PPL。缓存命中率高（87.7%~95.3%）。

## 7. 优点

1. **内存效率突破**：将 GPU 工作集与历史长度解耦，使得在极有限显存（16 GB）下实现数万 token 的训练，具有很强的实用价值。
2. **质量无损**：在可比较的上下文下，HGA 训练的适配器与密集训练几乎无差异，说明层次路由未引入有害偏差，且训练过程等价于常规参数更新。
3. **向后兼容**：训练得到的适配器可直接使用标准密集注意力进行推理，兼容现有推理框架（如 vLLM、SGLang），无需特殊部署。
4. **理论速度优势**：由于路由预算固定，HGA 在长序列下理论上必然优于密集注意力，且实验已验证在 2K 时已开始超越，为长上下文训练提供了可扩展的途径。
5. **分层存储设计**：清晰地将 VRAM、RAM、NVMe 分级利用，适应不同访问速度和容量，NVMe 支持无限历史但以页面缓存实现。
6. **无额外可训练参数**：避免引入新参数影响梯度更新一致性，简化实现。
7. **消融实验充分**：对路由稀疏性、段长等进行了消融，验证了默认参数选择合理性。

## 8. 不足与局限

1. **因果泄漏风险**：当前实现中，同一块内 token 共享路由决策，导致早期 token 可能从后期 token 的选择中获取未来信息。实验表明在超过约 100~200M 训练 token 后，模型会利用此泄漏改进预测，因此不适合从零预训练，仅适用于有限步数的微调。作者提及严格因果路由变体可解决此问题，但尚未验证。
2. **实验规模有限**：仅单模型（Qwen3-8B）、单 GPU、100 步训练，缺乏跨模型大小（如 70B）、跨硬件（如 A100）和更大训练步数的验证。实验结果泛化能力存疑。
3. **缺乏长上下文密集基线**：在 8K~16K 等更长上下文下，无法与密集训练直接比较质量，仅能报告 HGA 评估的损失，无法证明 HGA 训练在长上下文上是否仍保持与密集训练等价（只能与自身低上下文结果对比，但上下文不同导致数据截断不同）。
4. **NVMe 加持未测试**：虽然实现了 NVMe 后存储，但论文中未对其进行性能基准测试，实际应用中的速度与延迟未知。
5. **检索实验细节不足**：仅报告了召回率，且结果接近完美，未报告置信区间或统计显著性，可能受任务本身难度低或样本量小的影响。
6. **生产服务未完善**：HGA 推理引擎仍为研究原型，尚不能直接用于生产环境，与现有框架的集成仍在开发中。
7. **梯度截断影响**：尽管 TBPTT 让梯度不跨段传播，但分段边界处历史仍被阅读（通过精确 K/V），理论上损失了跨段的时间依赖性学习能力，但实验表明在微调中影响不大。长期训练或预训练下可能受限。
8. **缺少与其他稀疏注意力方法的比较**：如 LongLoRA、FlashAttention 等，论文未在实验部分与之对比，仅提及在 Related Work 中，缺乏公平的基准对比。
