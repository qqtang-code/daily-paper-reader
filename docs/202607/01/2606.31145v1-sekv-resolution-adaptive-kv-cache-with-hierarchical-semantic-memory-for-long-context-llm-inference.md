## 摘要
大语言模型越来越多地处理长上下文场景，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且需在整个解码过程中保留，若不进行压缩，完整的GPU缓存将过于昂贵。现有KV缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。令牌驱逐会丢弃信息，而语义分组则在预填充阶段固定压缩决策；两者都无法在生成过程中从压缩片段中恢复因相关而需要使用的令牌级细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，将上下文组织为熵引导的语义片段，并将其存储在GPU-CPU内存层级中而不丢弃信息。每个片段在GPU上保留轻量级摘要向量用于粗粒度路由，在CPU上保留低秩SVD基用于按需令牌级重建。一个训练的缩放机制在解码期间选择性地扩展与查询相关的片段，实现了精确检索而无需在GPU上实例化完整KV缓存。SeKV实现了自适应令牌级重建，同时保持基础LLM完全冻结，仅增加不到0.05%的可训练参数。在四个基准测试中，SeKV相比最强的语义压缩基线平均提升5.9%，同时相比128K上下文下的完整KV缓存减少53.3%的GPU内存。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

# SeKV 论文详细中文总结

## 1. 核心问题与研究动机

- **背景**：大语言模型（LLM）在长上下文任务（如文档分析、多轮对话、智能体规划）中广泛应用，但其 KV 缓存随序列长度线性增长，且必须在整个解码过程中保留，导致 GPU 内存成为主要瓶颈。全量 KV 缓存在大上下文下费用高昂，难以部署。
- **问题**：现有 KV 缓存压缩方法无法同时兼顾效率与上下文保真度：
  - **令牌驱逐方法**（如 StreamingLLM、H2O）永久丢弃信息，导致无法恢复被压缩的令牌级细节。
  - **语义分组方法**（如 ChunkKV、SemantiCache、SentenceKV）在预填充阶段固定压缩决策，无法在生成过程中因查询相关而动态恢复令牌级细节。
  - 当早期看似边缘的证据在后续成为关键逻辑锚点时，这两种方法都会失败。
- **核心挑战**：需要一个**分辨率自适应**的 KV 缓存，能根据查询相关性动态调整细节级别，同时避免信息不可逆丢失。

## 2. 方法论：核心思想与技术细节

### 2.1 核心思想
SeKV 将上下文组织为**分层语义记忆**，构建 GPU–CPU 内存层次结构：
- GPU 端：保留**锚点令牌**（高信息密度边界令牌）的完整 KV、轻量**摘要向量**（用于粗粒度路由）、以及局部窗口令牌。
- CPU 端：存储每个语义跨度的**低秩 SVD 基**，用于按需令牌级重建。
- 训练一个**缩放机制（zoom-in）**：在解码时动态选择哪些跨度需要扩展至令牌级分辨率，触发异步 CPU→GPU 传输。

### 2.2 关键技术细节

#### （1）熵引导的跨度分割
- 利用预填充阶段的令牌**惊喜度（surprisal）**：\( H_t = -\log p(x_t | x_{<t}) \)。
- 将 \( H_t > \mu + \alpha\sigma \) 的位置作为跨度边界，边界令牌保留为**锚点令牌**（全分辨率在 GPU 上）。
- 相邻锚点之间的令牌构成一个语义跨度 \( S \)。内容自适应：信息密集区域生成更短跨度。

#### （2）双分辨率跨度表示
- **摘要向量（GPU）**：对于每个令牌 \( t \in S \)，计算归一化权重 \( w_t = H_t / \sum_{k \in S} H_k \)。对每层 \( l \) 和每个注意力头 \( h \)，通过可学习的投影 \( W^{(l,h)} \in \mathbb{R}^{d' \times d_h} \) 将键向量投影到低维空间 \( d'=32 \)，然后计算加权平均归一化摘要 \( \bar{K}_S^{(l,h)} \)。同时保留原始空间中的加权平均键 \( \bar{k}_S^{(l,h)} \) 和值 \( \bar{v}_S^{(l,h)} \) 作为未扩展跨度的粗略贡献。
- **低秩 SVD 基（CPU）**：对跨度的键矩阵 \( K_S^{(l,h)} \) 进行截断 SVD（最大秩 \( R=32 \)）。一个轻量预测器 \( g_\phi \) 为每个奇异分量生成软门控 \( m_{S,i}^{(l,h)} \)，控制有效秩 \( r_S^{(l,h)} \)。重建公式：\( \hat{K}_S^{(l,h)} = \sum_{i=1}^{R} m_{S,i}^{(l,h)} \sigma_{S,i}^{(l,h)} u_{S,i}^{(l,h)} (v_{S,i}^{(l,h)})^\top \)。对值矩阵做同样处理。有效存储成本 \( r_S (|S| + d_h + 1) \ll |S| d_h \)。

#### （3）训练的缩放机制
- 每个解码步骤分两阶段：
  - **阶段 1：路由**：计算查询与所有跨度摘要的相似度得到路由概率 \( \tilde{\alpha}_S^{(l,h)} \)，并与可学习阈值 \( \tau^{(l,h)} \) 比较决定是否扩展：\( z_S^{(l,h)} = \mathbb{1}[\tilde{\alpha}_S^{(l,h)} > \tau^{(l,h)}] \)。
  - **阶段 2：细粒度注意力**：扩展的跨度从 CPU 获取 SVD 基并重建令牌级 KV；未扩展的跨度贡献一个粗略条目（均值和 log|S| 校正）。锚点令牌和局部窗口令牌直接参与注意力。所有条目在同一个 softmax 中。
- **训练损失**（四项之和）：
  - \( \mathcal{L}_{\text{distill}} \)：蒸馏损失（KL 散度），对齐全 KV 教师输出。
  - \( \mathcal{L}_{\text{zoom}} \)：缩放监督（加权二元交叉熵），目标来自教师注意力分布中达到累积覆盖比例 \( \rho \) 的最小跨度集。
  - \( \mathcal{L}_{\text{recon}} \)：重建损失，监督门控 SVD 的近似误差。
  - \( \mathcal{L}_{\text{budget}} \)：预算损失，惩罚扩展跨度数量和总秩。
- **可学习参数**：仅路由投影 \( \{W^{(l,h)}\} \)、阈值 \( \{\tau^{(l,h)}\} \)、秩门控预测器 \( g_\phi \) 共约 4.3M 参数（少于基础模型的 0.05%）。

## 6. 主要结论与发现

1. **SeKV 在所有压缩方法中表现最佳**：在 20 个骨干–基准设置中均取得第一；平均提升 5.9% 对 SentenceKV。

## 7. 优点

1. **不丢弃信息**：通过 GPU–CPU 层次结构避免永久令牌驱逐，所有信息以可恢复的紧凑形式保存。
2. **自适应分辨率**：根据查询相关性动态调整缓存细节，实现“存储即服务”的更优资源分配。
3. **轻量级训练**：仅训练约 0.05% 参数的扩展组件，基础模型完全冻结，便于部署。
4. **高效传输**：传输压缩 SVD 基而非完整令牌 KV，显著降低 CPU–GPU 带宽需求。
5. **内容自适应分割**：熵引导分割无需外部解析器，自动适应文本复杂度。
6. **鲁棒性**：多个骨干和预算下性能一致，超参数敏感度低。

## 8. 不足与局限

1. **依赖语义跨度质量**：熵引导分割在碎片化文本、表格、代码或异常格式下可能产生次优边界，导致内存分配不佳。
2. **超参数较多**：包括熵阈值、锚点数量、低秩秩、预算等，新模型/任务可能需要验证调整。
3. **CPU–GPU 带宽依赖**：在慢速传输通道或攻击性查询反复激活远距离跨度时，延迟优势可能下降。
4. **静态建模能力**：冻结基础模型限制了完全适应压缩内存结构的能力；联合训练可能进一步提升。
5. **实验覆盖**：虽然覆盖多个基准，但未涉及提示注入、多轮对话等动态场景；长上下文推理的泛化性仍需更多验证。
6. **资源披露部分**：训练数据来源为 RedPajama 特定子集，未说明其与测试集的重叠风险（虽声明不相交）。
7. **SVD 近似误差**：低秩重建本质上是损失压缩，在极端低预算或信息密度极高的跨度上可能影响保真度。
