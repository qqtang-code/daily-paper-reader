## 摘要
审查核监管文件需要在数万页文档中进行多跳推理，判断依据跨多个章节收集的证据。我们将此任务构建为规划问题：基于LLM的智能体观察已收集的证据，选择下一个要检查的文档片段，并在证据足够时停止。智能体在无向量的文档树上操作，使用浏览、阅读和搜索工具，并维护一个动态知识图谱作为状态。在NuScale最终安全分析报告文档的200个问题基准测试中，该系统达到81.5%的准确率，RAGAS忠实度为0.93。主导性能因素是规划：与使用相同文档树但无状态条件动作选择的PageIndex相比，差距为+38.0个百分点（43.5%对81.5%，p<0.001）。该系统还优于LightRAG（73.0%，p<0.05）、HippoRAG（70.5%，p<0.01）和GraphRAG（49.5%，p<0.001），并且在无离线索引情况下与RAPTOR（75.5%，p=0.11）相当。边缘推理增加了2.8倍成本但未提高准确率；我们将其保留为可追溯性模块。在7,391条推断边中，有3条违反边（0.04%）标记了范围边界（Q058）和部分一致性（Q176）作为类型化注释，供人工审查员审核。

## Abstract
Reviewing nuclear regulatory documents requires multi-hop reasoning across tens of thousands of pages, where judgments depend on evidence assembled across multiple chapters. We frame this task as planning: an LLM-based agent observes the evidence collected so far, picks the next document fragment to inspect, and stops when the evidence is sufficient. The agent operates over a vectorless document tree using browse, read, and search tools, and maintains a dynamic knowledge graph (KG) as state. On a 200-question benchmark over NuScale Final Safety Analysis Report (FSAR) documents, the system reaches 81.5% accuracy with a RAGAS Faithfulness of 0.93. The dominant performance factor is planning: against PageIndex, which uses the same document tree without state-conditioned action selection, the gap is +38.0pp (43.5% to 81.5%, p<0.001). The system also outperforms LightRAG (73.0%, p<0.05), HippoRAG (70.5%, p<0.01), and GraphRAG (49.5%, p<0.001), and matches RAPTOR (75.5%, p=0.11) without offline indexing. Edge inference adds 2.8x cost without raising accuracy; we retain it as a traceability module. Of 7,391 inferred edges, 3 Violates edges (0.04%) flag scope boundaries (Q058) and partial conformance (Q176) as typed annotations that a human reviewer can audit.

## 2. 方法论

### 核心思想
- 将文档构建为**无向量的文本环境**，agent 通过 **browse、read、search** 三种工具与文档树交互，维护一个**动态知识子图（Dynamic Sub-KG）** 作为状态，采用**状态条件动作选择**的闭环规划循环，并在证据充分时**动态终止**。

### 关键技术细节
- **文档树环境**：保持原生文档结构（章节→节→段落），不进行分块或嵌入。支持多模态引用（图表元数据通过 `references` 字段附着）。
- **状态表示（短期记忆）**：动态知识图 \( G_t = (V_t, E_t) \)，节点为文档片段，边由**双层本体**驱动：
  - **Tier 1（结构边）**：REFERENCES（交叉引用）、SPECIFIES（上级详述下级）。
  - **Tier 2（语义边）**：SATISFIES/VIOLATES（符合/违反要求）、SUPPORTS/CONTRADICTS（支持/反驳）、LEADS_TO（因果链）、IS_PREREQUISITE_OF（概念前提）。
- **动作规划**：
  - 每步 agent 观察当前 KG，选择动作（browse/read/search），执行后更新状态。
  - 强制 browse-first 模式（Hop 1 注入目录），BM25Okapi 搜索（标题权重×3），伪相关反馈（RM3）扩展查询，维护搜索历史去重。
  - 动态终止：从 Hop 2 开始，每次动作前 LLM 判断当前 KG 是否足够回答，若是则提前终止（33% 查询提前终止）。
- **后检索边推断（可选）**：
  - Stage 1：LLM 生成自然语句描述节点间关系。
  - Stage 2：将描述映射到本体（无映射时保留为 SEMANTIC）。
- **视觉增强最终答案**：仅在最终答案生成时对图表所在 PDF 页面进行视觉处理（GPT-4.1 VLM），中间步骤保持纯文本，避免高成本视觉调用。

### 算法流程（文字概述）
1. 初始化空 KG，注入目录结构（Hop 0）。
2. 循环至多 \( T \) 步：
   - 基于当前 KG 和查询，LLM 规划下一步动作（browse/read/search）。
   - 执行动作，整合新节点。
   - 可选：边推断。
   - 如果从 Hop 1 起满足证据充分条件，则提前终止。
3. 最终使用 VLM 结合 KG 上下文和图表图像生成答案。

## 7. 优点

1. **新颖的问题框架**：将核监管审查形式化为规划问题，符合实际人类审查流程。
2. **无向量文档树设计**：保留原生结构，避免分块导致的上下文断裂；无需嵌入或索引，适合频繁更新的文档。
3. **状态条件闭环规划**：相比于单次检索或静态索引方法，能逐步积累证据并动态终止。
4. **有效的动态终止**：33% 查询提前终止，降低成本且准确率不降（甚至略高）。
5. **可追溯性模块**：通过双层边本体提供类型化注释（VIOLATES 等），满足监管审计要求。
6. **多模态处理策略**：仅在最终答案生成时使用视觉 API，中间步骤纯文本，平衡成本和性能。
7. **双评估框架**：综合 RAGAS 和 LLM-as-Judge，减少单一评估偏见。

## 8. 不足与局限

1. **性能局限**：在纯文本问题上略低于 RAPTOR（76.2% vs 80.0%，-3.8pp）；与 RAPTOR 的整体差距未达统计显著（p=0.11）。
2. **成本较高**：每次查询 93 秒/$0.215，比 RAPTOR 贵约 42 倍。虽在监管场景可接受，但限制了普适性。
3. **基准偏见**：
   - 判断型问题 98% 答案为“是”（FSAR 为已认证合规设计）。
   - 66% 问题实际为 2 跳，深度有限。
   - 文档覆盖不均衡（第 01 章仅使用 19% 页面）。

4. **缺乏按引用工具**：系统未实现直接导航“见图 5.1-1”类指针的功能。
5. **最大跳数 4**：67% 查询用完 4 跳预算，更高跳数的影响未探索。

7. **实验样本量限制**：10 问消融仅为指示性；主要对比与 RAPTOR 的差异在 n=200 时未达显著，需要更大规模验证。
8. **Factual Correctness 分数低**（~0.42）并非检索失败，而是单一参考答案对措辞敏感，反映基准局限性。
