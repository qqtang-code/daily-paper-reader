## 摘要
层次全局注意力（HGA）是预训练长上下文Transformer中密集因果注意力的即插即用替代方案。HGA保留了原始检查点参数：预训练的$W_Q$、$W_K$、$W_V$和$W_O$投影保持不变，不引入校准参数，也不需要重新训练。应用于单个RTX 5090（32GB）上的Qwen3-30B-A3B-Instruct-2507-FP8时，修补后的模型可在64K令牌上下文中开箱即用，而在此硬件上，令牌级的K/V存储是不可行的。与先前的稀疏注意力方法不同，HGA执行层次化的两级路由。它首先使用紧凑的RoPE感知摘要检索相关块，然后通过仅路由最相关的组来细化选择，最后执行精确的令牌级注意力。这种层次化检索显著减少了获取的令牌数量，同时保留了对检索令牌集的精确注意力，使得基于RAM和NVMe的存储变得实用。完整的歷史令牌K/V驻留在主机RAM或NVMe存储中，而注意力期间仅将小的路由工作集传输到GPU内存。因此，GPU内存消耗主要取决于模型权重和路由工作集，而非总上下文长度。在所有测试的上下文长度（4K-64K令牌）中，路由注意力与密集注意力的差异保持在约$0.01$-$0.02$ nats内，而使用的稀疏度仅为约3%。这些结果表明，层次路由引入的近似误差很小，剩余的质量差距可能主要由长上下文位置编码而非路由算法本身主导。

## Abstract
Hierarchical Global Attention (HGA) is a drop-in replacement for dense causal attention in pretrained long-context transformers. HGA preserves the original checkpoint parameters: the pretrained $W_Q$, $W_K$, $W_V$, and $W_O$ projections remain unchanged, no calibration parameters are introduced, and no retraining is required.   Applied to Qwen3-30B-A3B-Instruct-2507-FP8 on a single RTX~5090 (32GB), the patched model runs out of the box at a 64K-token context, where token-level K/V storage is not feasible on this hardware.   Unlike previous sparse-attention methods, HGA performs hierarchical two-level routing. It first retrieves relevant chunks using compact RoPE-aware summaries and then refines the selection by routing only the most relevant groups before performing exact token-level attention. This hierarchical retrieval significantly reduces the number of fetched tokens while preserving exact attention over the retrieved token set, making RAM- and NVMe-backed storage practical.   The full historical token K/V resides in host RAM or NVMe storage, while only a small routed working set is transferred to GPU memory during attention. Consequently, GPU memory consumption depends primarily on model weights and the routed working set rather than on the total context length.   Across all tested context lengths (4K - 64K tokens), routed attention remains within approximately $0.01$--$0.02$ nats of dense attention while the sparsity used is just about 3%. These results suggest that the approximation introduced by hierarchical routing is small, and that the remaining quality gap is likely dominated by long-context positional encoding rather than by the routing algorithm itself.

# 论文总结：Hierarchical Global Attention (HGA)

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：长上下文Transformer在处理超长序列（如64K tokens以上）时，密集因果注意力的计算复杂度为$O(n^2)$，且K/V缓存必须完全驻留GPU显存，导致显存瓶颈，尤其对于量化大模型（如Qwen3-30B-A3B，FP8权重已接近32GB显存上限），无法在消费级GPU（如RTX 5090）上运行长上下文。
- **整体含义**：提出一种即插即用（drop-in replacement）的稀疏注意力方法，无需重新训练或修改预训练模型参数，通过层次化两级路由，仅将少量相关token的K/V传输到GPU，完整历史K/V存储于主机RAM/NVMe，从而在有限显存下实现超长上下文处理，同时保持接近密集注意力的精度。

## 2. 论文提出的方法论
### 核心思想
- 将长距离检索从token级别提升到chunk/group级别：先检索相关块（chunk），再精炼到块内组（group），最后仅对选中的token执行精确注意力。路由决策基于预训练模型自身的key投影构建的RoPE感知摘要，不引入额外可学习参数。

### 关键技术细节
- **序列划分**：固定chunk大小$C=64$，每个chunk再分为group（$gs=16$）。历史序列分为固定可见的“注意力沉没”块（前几个chunk）、近期局部块（最后几个chunk）以及中间内容路由块。
- **混合RoPE摘要**：针对高/低频RoPE维度采用不同策略——高频对每个token旋转后平均，低频先在原始key空间平均再以chunk/group中间位置旋转。摘要是投影key的求和/平均，不作为value使用。
- **两级路由**：
  1. 第一级：根据当前chunk的查询，对中间所有历史chunk的摘要计算相似度，取top-K个chunk。
  2. 第二级（可选）：在选中的chunk内，对group摘要评分，取top-Kg个group，打开对应token K/V。
- **确定性可见性**：当前chunk（因果掩码）、固定数量的起始chunk（注意力沉没）、固定数量的最近chunk（局部上下文）始终可见。
- **精确token注意力**：将路由选中的token K/V与确定性token K/V拼接，执行标准softmax注意力，不使用摘要作为value。
- **分层存储**：热数据（摘要、固定窗口）驻留GPU，暖数据（LRU缓存块）在GPU与CPU间动态交换，冷数据（全部历史token K/V）驻留主机RAM。

### 公式与算法流程（文字说明）
1. 输入序列划分为chunk和group。
2. 对当前chunk的每个查询，构建候选集 = 固定可见chunks + 路由选中的中间chunks/group。
3. 路由过程：当前chunk的查询与所有中间chunk摘要计算相似度 → 选top-K chunk；若启用group路由，则对选中chunk内的group摘要评分选top-Kg group。
4. 将候选集中所有token的K/V从存储中获取（优先从GPU缓存、否则从CPU传输）。
5. 对候选集执行标准因果softmax注意力（只对当前chunk及其之前token）。
6. 更新LRU缓存，将当前chunk写回。

## 6. 论文的主要结论与发现
- HGA作为密集注意力的即插即用替代，在无需重训练、无额外参数下，可在单卡RTX 5090上运行Qwen3-30B-A3B的64K上下文。
- 与密集注意力相比，路由注意力损失差距保持在0.01~0.02 nats内，稀疏度仅约3%（即只检索约1K个token/步）。
- 路由算法本身引入的近似误差较小，剩余差距可能更主要源于长上下文位置编码（如YaRN等扩展方法）与稀疏路由的交互。
- 两级路由（chunk→group）进一步降低路由预算，同时保持检索精度（64K上下文下1.9%稀疏度，100%针检索率）。
- 存储层次解耦GPU显存与上下文长度，使主机RAM+NVMe实用化。
- 微调后路由注意力仍保持低损失（差距~0.015 nats），且路由模型与密集模型在“互换注意力”后性能接近，表明路由训练不会损害模型质量。

## 7. 优点
1. **即插即用**：无需修改预训练模型权重、无需校准数据、无需重新训练，直接替换注意力模块即可。
2. **内存效率创新**：通过层级路由+存储层次（Hot/Warm/Cold），将GPU显存消耗从$O(n)$降为$O(\text{固定窗口})$，显著扩展长上下文可用性。
3. **精确注意力保证**：路由仅决定检索区域，最终输出基于精确token K/V的softmax，无近似value或合成token，保证了输出质量。
4. **混合RoPE摘要设计**巧妙处理高频/低频分量，兼容预训练RoPE。
5. **实验设计严谨**：copy-only测试明确隔离路由引入的误差，微调实验验证路由不影响模型底层表示，正确性检查验证数值精度。
6. **工程实用性**：提供开源实现，支持Triton融合加速，展示实际硬件可行案例。

## 8. 不足与局限
1. **实验覆盖不足**：
   - 下游任务评估缺失：未在LongBench、RULER、MT-Bench等标准长上下文基准上评估。
   - 与同类稀疏方法（如MInference、StreamingLLM、H₂O等）缺乏定量比较，仅定性讨论。
   - Needle-in-a-Haystack测试量少（3个深度、1种长度），不充分。
   - 速度数据仅在小模型上取得，大模型推理性能未量化。
2. **路由算法的潜在偏差**：top-k路由可能遗漏关键远程token，尤其是在需要“针”在罕见位置的场景。虽损失差距小，但未测试长文档QA等复杂任务召回。
3. **上下文长度限制**：当前实验止于64K，模型原生能力262K（Qwen3-30B），未验证128K/256K/512K下的路由行为与开销。
4. **对位置编码的依赖**：剩余质量差距与长上下文位置编码密切相关，HGA本身无法解决位置编码的外推问题，需配合YaRN等扩展方法，且存在交互不确定性。
5. **性能瓶颈**：大模型实现中TTFT较长，受限于PyTorch顺序逐块预填和Triton回退，未利用DeepGEMM等优化；RAM传输延迟可能影响解码速度。
6. **非对称性**：固定路由预算对于不同任务（如代码理解 vs 对话）可能不是最优，缺少自适应路由机制。
7. **硬件依赖**：演示仅限单卡32GB消费级GPU，未测试多卡场景或更大显存环境下的扩展性。
