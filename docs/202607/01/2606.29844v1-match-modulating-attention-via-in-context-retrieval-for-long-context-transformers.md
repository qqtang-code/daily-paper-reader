## 摘要
传统注意力机制的二次计算成本对大型语言模型（LLM）的可扩展性和实际部署构成了主要瓶颈，尤其是在长上下文场景中。为了提高效率，现有方法通常强制施加刚性结构约束，例如局部注意力窗口。然而，这些策略通常会导致在需要精确长距离召回的任务上性能显著下降。在本工作中，我们提出了MATCH，一个可扩展且高效的框架，通过高效的检索系统将稀疏化注意力机制与动态集成的上下文内信息相结合。实验结果表明，MATCH显著提升了稀疏注意力模型在合成和真实自然语言任务上的性能。这些发现凸显了MATCH作为一种通用方法的多样性，能够在保持稀疏注意力架构效率优势的同时，增强上下文内检索能力。

## Abstract
The quadratic computational cost of traditional attention mechanisms poses a major bottleneck to the scalability and practical deployment of large language models (LLMs), particularly in long-context scenarios. To improve efficiency, existing approaches often enforce rigid structural constraints such as local attention windows. However, these strategies typically lead to substantial performance degradation on tasks requiring precise long-range recall. In this work, we propose MATCH, a scalable and efficient framework that augments sparsified attention mechanisms with dynamically integrated in-context information through an efficient retrieval system. Empirical results show that MATCH significantly improves the performance of sparse-attention models on both synthetic and real-world natural-language tasks. These findings highlight the versatility of MATCH as a general approach for enhancing in-context retrieval capabilities while maintaining the efficiency benefits of sparse attention architectures.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：大型语言模型（LLM）在处理长上下文时，标准注意力机制的二次计算复杂度（O(N²)）成为主要瓶颈，导致内存和时间开销急剧增长。现有稀疏注意力方法（如滑动窗口注意力 SWA）虽然将复杂度降至线性或亚线性，但强制施加的局部窗口模式严重限制了模型对长距离信息的召回能力，在需要精确回忆上下文细节的任务（如复制型任务、多跳问答）中性能显著下降。
- **研究动机与含义**：旨在平衡稀疏注意力的效率优势与长距离召回能力，通过外部检索系统动态补充稀疏注意力缺失的全局信息，而非修改模型架构或增加计算预算。作者提出 MATCH 框架，将输入上下文视为动态数据存储，利用轻量级检索模块为稀疏注意力注入相关的远程上下文，从而在不牺牲效率的前提下恢复模型的上下文理解能力。

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
- **将稀疏注意力的局部模式与外部检索增强的远程上下文相结合**：通过一个高效的外部检索系统（bi-encoder + cross-encoder reranker），为每个查询 token 动态选择最相关的历史 chunk，并在注意力计算中通过自定义 mask 允许这些 token 参与加权。
- **即插即用**：适用于预训练时已使用稀疏注意力的模型（前稀疏化）和全注意力模型后转换为稀疏注意力的模型（后稀疏化），仅需少量持续训练即可适应。

### 关键技术细节
1. **上下文分块与检索**：
   - 将输入序列划分为固定大小的 chunk（长度 U，如 128）。
   - 对每个查询位置，使用其前若干 token 构成查询 chunk，通过 Sentence-BERT 等 bi-encoder 编码，与所有历史 chunk 向量计算点积相似度，选择 top-K（如 K=64）候选。
   - 在解码阶段，额外使用 cross-encoder reranker 将候选减少为 top-k（如 k=4 或 8），提升检索精度。
   - 检索仅在预填充阶段执行一次，解码阶段按间隔重复，避免每步都检索。

2. **注意力计算中的掩码设计**：
   - 每个 token 的注意力范围 = 因果 mask + 滑动窗口 mask + 检索 mask。
   - 检索 mask 允许 token 访问被检索到的历史 chunk 中的所有 token，而不仅仅是局部窗口内的。

3. **解码阶段的 KV 重计算**：
   - 在每步解码时，将检索到的原始 token 作为辅助输入，与当前 token 一起通过模型，只在该步（或每 chunk 的第一步）重计算这些检索 token 的 KV 值，并缓存供后续 token 复用，避免逐层重复计算。
   - 公式：注意力输出由当前 query、缓存的滑动窗口 KV 和重计算的检索 KV 共同计算。

4. **持续训练适应**：
   - 由于检索 token 可能位于滑动窗口之外，原始模型参数未见过此类模式，因此需要进行两阶段持续训练：
     - 第一阶段：50B tokens（Cosmopedia + Fineweb-edu + Pro-Long 数据集）。
     - 第二阶段：1.5B Prolong short-mix + 1.5B NIAH-like 合成数据 + 1B Narrative QA。
