## 摘要
长上下文语言模型推理日益受到存储键值缓存所需的内存带宽和容量的限制，然而现有的压缩方法通常在各层或令牌之间应用均匀预算，并且在词汇线索和语义状态需要不同保留时会降低检索质量。我们提出DepthWeave-KV，一种令牌自适应的缓存压缩方法，该方法使用共享的低秩通道基对相邻Transformer层之间的键和值状态进行分解，同时在注意力行为敏感的令牌处保留轻量级的令牌特定残差。DepthWeave-KV将跨深度残差分解与令牌条件深度路由器相结合，为携带指令和检索关键的令牌分配更高的重建秩，并使用来自注意力输出探针的免校准在线误差跟踪，在生成过程中自适应压缩，无需重新训练基础模型。一个融合的CUDA实现联合执行基查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、Needle-in-a-Haystack、L-Eval以及长格式问答和摘要基准测试中，DepthWeave-KV以显著更低的内存使用实现了接近全缓存的任务质量，与先前的压缩缓存相比，提高了平均得分和检索准确性，同时在64K上下文中实现了8.3倍的KV内存减少和每秒72.8个令牌的吞吐量。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：长上下文语言模型推理时，KV（Key-Value）缓存成为主要的内存瓶颈，限制批处理大小、上下文长度和服务吞吐量。现有压缩方法（如令牌驱逐、合并、量化、低秩注意力和层间缓存共享）大多对各层或所有令牌应用均匀的压缩预算，忽视了不同令牌在语义上的重要性差异（例如，指令分隔符、实体提及、检索关键跨度 vs. 流畅续写令牌），以及不同Transformer层（浅层词汇锚点 vs. 深层语义抽象）所需保真度的不同。这导致在需要精确检索的任务（如多跳问答、长文档摘要）中出现脆弱的失败，即使平均困惑度保持良好。
- **整体含义**：亟需一种同时考虑深度维度冗余和令牌自适应性的压缩方法，能够根据在线注意力行为证据动态调整每个令牌在每个深度窗口的重建保真度，从而在不牺牲检索鲁棒性的前提下大幅降低内存占用。

## 2. 方法论

### 2.1 核心思想
DepthWeave-KV 提出**跨层残差因子分解**：将相邻Transformer层的KV缓存共享一组低秩通道基（basis），同时保留轻量级的令牌特定残差（residual）以恢复注意力敏感的信息。该方法结合了**令牌条件深度路由器**和**免校准在线误差跟踪**，以动态分配重建秩（rank）并适应生成过程中的变化。

### 2.2 关键技术细节
1. **跨深度残差因子分解**：将层划分为重叠深度窗口。在每个窗口和注意力头内，维护共享基矩阵 **B**∈R^{r_b×d}（r_b为基秩）。每个令牌在窗口内某层的键/值状态重建为：
   \[
   \hat{X}_{ℓ,h,t} = A_{ℓ,h,t} B_{m,h} + g_{ℓ,h,t} R_{ℓ,h,t}
   \]
   其中 **A** 是紧凑系数向量（8-bit块量化存储），**R** 是量化后的残差（4-bit组量化），**g** 是轻量残差门控。未分配残差秩的令牌仅由共享基表示。

2. **令牌条件深度路由器**：为每个令牌计算一个得分，基于四个在线特征：累积注意力质量、最大注意力尖峰、分隔符/指令边界指示符、最近探测间隔的注意力输出重建误差。路由后的残差秩为：
   \[
   \rho = \rho_{\min} + (\rho_{\max}-\rho_{\min}) \cdot \mathbb{1}[\sigma(\mathbf{u}^\top \phi) > \tau_m]
   \]
   实际使用三级秩（0、2、4或8）。路由器参数 **u** 预先在小规模无标注长文本上训练一次（冻结教师缓存目标），部署后无需任务特定校准。

3. **在线误差跟踪**：每p个解码步，使用一小部分头和查询位置，计算压缩重建后的注意力输出与临时高保真缓存注意力输出之间的相对误差。若误差超过窗口特定容差，则降低阈值τ_m，为更多令牌分配更高残差秩；反之则增加阈值，减少残差存储。该反馈直接针对影响下一隐藏状态的量，比最小化所有缓存向量的均方误差更可靠。

4. **融合CUDA实现**：在解码时，一个融合内核同时执行基查找、残差反量化、系数乘法和注意力投影，重构后的KV张量不写回全局内存，从而减少内存带宽消耗。

### 2.3 无需修改基础模型
所有可学习组件（残差门控和路由器参数）在部署前训练；基础模型权重保持不变。
