## 摘要
参数高效微调减少了模型和优化器的内存占用，但密集注意力仍然使得长序列训练成本高昂。本文将层次化全局注意力（HGA）与分段反向传播和层级KV存储相结合。只有当前活跃的分段在显存中保持可微；较早的KV被分离到内存或NVMe中，HGA为每个查询块加载一组有界的精确历史令牌。在配备4位QLoRA和PG19的Qwen3-8B上，使用16 GB Quadro RTX 5000进行密集训练时，只能处理2,048个令牌，而4,096个令牌则失败，但HGA在15.28 GB峰值显存下可处理16,384个令牌。在评估中，相同的适配器在此显卡上可处理131,072个令牌；显存占用并非恒定，而是随驻留的分块摘要缓慢增长，因此在实际应用中，内存和NVMe容量设定了超过这些长度的实际限制。在相同的2K训练长度下，使用相同的密集注意力读取方式，HGA训练和密集训练的适配器分别获得2.7405和2.7383 nat，而原始模型获得2.9541 nat。在此边界下，HGA训练已略快（217.75 vs. 207.02 tokens/s），且HGA与密集注意力的吞吐率比从1K提升至2K；由于HGA保持每个令牌所关注的历史集大致恒定，而密集注意力的每个令牌计算量随上下文增长，我们预计这一优势将随上下文扩大而进一步拉大。主要质量和检索比较均采用密集注意力，以便衡量学习到的权重，并与标准生成框架保持兼容。HGA也可用于检索和生成；优化的生产级服务实现正在开发中。

## Abstract
Parameter-efficient fine-tuning reduces model and optimizer memory, but dense attention still makes long training sequences expensive. We combine Hierarchical Global Attention (HGA) with segment-wise backpropagation and tiered KV storage. Only the active segment remains differentiable in VRAM; older KV is detached into RAM or NVMe, and HGA loads a bounded set of exact historical tokens for each query block. On Qwen3-8B with 4-bit QLoRA and PG19, dense training on a 16 GB Quadro RTX 5000 fits 2,048 tokens but fails at 4,096, whereas HGA reaches 16,384 tokens with 15.28 GB peak VRAM. Under evaluation the same adapter runs through 131,072 tokens on this card; VRAM is not constant but grows gently with the resident chunk summaries, so RAM and NVMe capacity set the practical limit beyond these lengths. At the shared 2K training length, HGA-trained and dense-trained adapters obtain 2.7405 and 2.7383 nat under the same dense-attention readout, while the stock model obtains 2.9541. At this boundary HGA training is already marginally faster (217.75 vs. 207.02 tokens/s), and the HGA-to-dense throughput ratio improves from 1K to 2K; because HGA keeps the attended historical set per token approximately constant while dense work per token grows, we expect this lead to widen as context grows. Dense attention is used for the main quality and retrieval comparisons so that they measure the learned weights and remain compatible with standard generation frameworks. HGA can also be used for retrieval and generation; an optimized production-grade serving implementation is under development.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：参数高效微调（如 QLoRA）虽然大幅降低模型和优化器的显存占用，但密集注意力（dense attention）仍需要对全部历史 token 进行交互，导致长上下文训练在有限显存消费级 GPU 上不可行（如 16GB 显存，Qwen3-8B 密集训练仅能容纳 2K 序列，4K 即 OOM）。
- **动机**：希望在不显著牺牲质量的前提下，使训练上下文长度大幅扩展，同时保持训练效率，从而让低 VRAM 设备也能处理长序列微调任务。
- **整体含义**：提出一种结合层级全局注意力（HGA）、分段反向传播（TBPTT）和外部 KV 存储的框架，将 GPU 工作集与完整训练历史分离，使训练上下文从 2K 扩展到 16K（同一 GPU），且评估时可达到 131K tokens。

## 2. 方法论

### 核心思想
- 用 **Hierarchical Global Attention (HGA)** 替换标准密集注意力，不改变预训练的 Q/K/V/O 投影层。HGA 通过两级路由，仅为每个查询块选择一小部分精确历史 token 进行注意力计算，避免全量计算。
- 训练时采用 **分段反向传播（TBPTT）**：将长序列切割为固定长度 segment（本文用 2048 tokens），每个 segment 独立前向和反向，segment 结束后历史 KV 被分离并卸载到 RAM 或 NVMe，梯度不跨 segment。
- 构建 **三级存储层级**：VRAM 存放量化模型+适配器、当前可微 segment、选中的精确 token K/V、所有 chunk 摘要和部分 group 摘要缓存；RAM 保存分离的历史 KV 和非缓存的 group 摘要；NVMe 作为可选溢出层（冷数据）。

### 关键技术细节
1. **两级路由**：
   - 上下文分为 **64-token chunk** 和 **8-token group**（可配置）。
   - 第一级：每个 chunk 生成一个紧凑 key 摘要（常驻 VRAM），查询块对所有 chunk 摘要评分，选择 top-k 相关 chunk（加上固定的 sink 和 local window）。
   - 第二级：被选中的 chunk 展开为 group 摘要（部分缓存），再次选择最佳 group。
   - 最终仅加载所选 group 的 **精确 token K/V** 进行注意力计算。路由本身不参与梯度计算，但 K/V 收集和注意力可微。

2. **内存公式**：  
   \( M_{GPU} \approx M_{model} + M_{adapters/optimizer} + O(S) + O(B_{route}) \)  
   其中 \( S \) 为 segment 长度，\( B_{route} \) 为选中的历史 token 数。外部历史记录随总长度 \( L \) 线性增长，但显存不增长（仅摘要常驻）。

3. **训练一致性**：HGA 训练和密集训练使用完全相同的数据顺序、种子、超参，仅注意力模块不同。评估时均采用密集注意力读出来衡量学习到的权重，排除路由选择的影响。

## 6. 主要结论与发现

1. **显存大幅降低**：在 16GB GPU 上，HGA 可训练 16K tokens（峰值 15.28 GB），而密集注意力仅能训练 2K（4K OOM）。
2. **质量几乎无损**：在 2K 共同长度下，HGA-trained 和 Dense-trained 适配器在密集注意力评估下的 loss 差仅 0.0022 nat（PPL 差 0.02）；相对原始模型提升约 19.4% PPL。
3. **训练速度略优**：在 2K 下 HGA 吞吐 217.75 tok/s，Dense 207.02 tok/s（快约 5%）；且 HGA 速度优势随上下文增长更明显（因路由预算恒定）。
4. **检索能力保持**：RULER 检索任务中两种适配器几乎持平（各 1/96 失误），无退化。
5. **路由稀疏性**：默认 8 chunks 已足够，选择密度从 2K 的 59.4% 降至 16K 的 8.7%，但质量稳定。
6. **评估扩展性**：同一适配器在 HGA 评估下可达 131K tokens，突破 GPU 显存限制。
7. **训练稳定性**：在约 100M token 内无问题，超过后可能存在因果泄漏（chunk 内路由共享导致早期 token 获取后期信息）。

## 7. 优点

- **方法创新**：HGA 与 TBPTT、外部 KV 存储有机结合，首次在消费级 GPU 上实现 16K 长上下文微调，且明显优于 LongLoRA 等稀疏方法（因为基于内容而非固定模式）。
- **即插即用**：不改变原始模型结构和预训练权重，仅替换注意力模块；适配器可与标准密集推理框架兼容。
- **质量-效率双赢**：在 2K 下质量和速度均不低于密集训练，且随上下文增长优势扩大。
- **完整开源**：提供了 GitHub 仓库和实验配置，便于复现。
- **多层次存储设计**：从 VRAM 到 RAM 到 NVMe 的层级，理论上支持任意长度。

## 8. 不足与局限

- **长期训练因果泄漏**：当前路由实现在 >100M tokens 的训练后因 chunk 内路由共享导致因果不完整性，不适合预训练。作者提到已有严格因果变体但未在本文评估。
- **实验规模有限**：仅使用一个模型（Qwen3-8B）和一块 GPU，未在更大模型（如 70B）或多卡场景下验证；训练步数仅 100 步，不足以代表完整微调场景。
- **缺乏更长上下文的密集基线**：8K/16K 下无法与密集训练直接比较质量（因 GPU OOM），只能横向对比 HGA 自身不同长度。作者也承认这不是替代基线。
- **TBPTT 梯度截断**：虽然后序 segment 可访问历史精确 KV，但梯度不跨 segment，可能影响长距离依赖学习。
- **NVMe 实现未基准测试**：尽管代码支持文件系统存储，但实验中只用 RAM 作为历史存储，NVMe 性能未汇报。
- **HGA 推理路径未生产化**：当前 HGA 推理仍为研究原型，无法直接用于 vLLM 等生产框架，推理效率尚未优化。
- **选择偏差风险**：路由选择基于摘要，可能遗漏关键 token 导致质量下降（但实验中通过足够大的 top-k 缓解）。
