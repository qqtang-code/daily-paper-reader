## 摘要
高效的长上下文推理不仅关乎降低内存成本，还关乎在生成过程中保持有用的上下文证据可访问。然而，现有的面向压缩的方法，如键值（KV）缓存压缩和上下文压缩，通常要么提前决定保留哪些上下文信息，要么依赖外部压缩器。这种设计使得压缩后的上下文难以适应后续推理步骤所需的证据。本文介绍了PReM（保留与刷新记忆），一种上下文压缩框架，它将长上下文维护为模型内部逐层的KV记忆，并学习保留什么以及何时刷新它。具体地，PReM使用一个专门的记忆层来做出记忆选择决策，并使用一个特殊的记忆令牌<m>在生成过程中触发刷新。为了训练这种行为，PReM引入了相分离刷新训练，将记忆选择与记忆条件下的生成对齐，同时保持跨刷新的连续性。在32K令牌上下文上的实验表明，PReM在16倍和32倍压缩下均优于强基线，同时在答案质量和推理效率之间保持了有利的平衡。

## Abstract
Efficient long-context inference is not only about reducing memory cost, but also about keeping useful contextual evidence accessible as generation proceeds. However, existing compression-oriented approaches, such as key-value (KV) cache compression and context compression, often either make an early decision about which contextual information to keep or rely on an external compressor. Such designs make it difficult to adapt the compressed context to the evidence needed by later reasoning steps. This paper introduces PReM (Preserve and Refresh Memory), a context-compression framework that maintains the long context as the model's internal layer-wise KV memory and learns what to preserve and when to refresh it. Specifically, PReM uses a dedicated memory layer to make memory-selection decisions, and a special memory token <m> to trigger refreshes during generation. To train this behavior, PReM introduces Phase-Separated Refresh Training, aligning memory selection with memory-conditioned generation while preserving continuity across refreshes. Experiments with 32K-token contexts show that PReM outperforms strong baselines under both 16x and 32x compression, while maintaining a favorable balance between answer quality and inference efficiency.

## 一、论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：长上下文推理中，LLM 需要处理大量上下文证据并生成特定输出，但推理成本随上下文长度快速增长（KV 缓存膨胀、解码延迟增加）。现有压缩方法（KV 缓存压缩、上下文压缩）主要存在两个局限：
  1. **静态证据保留**：早期决定保留哪些上下文后，在整个生成过程中复用，但不同推理步骤（尤其是多跳任务）可能需要不同证据，导致保留无关信息、丢弃后续所需证据。
  2. **解耦的证据选择**：依赖外部压缩器，压缩与生成分离，增加额外计算，且证据选择不能直接适配每个推理步骤的需求。
- **核心问题**：如何实现**动态**的上下文压缩，使模型能够根据当前生成状态和步骤特定的证据需求，自主决定**保留什么**以及**何时刷新**记忆。
- **整体含义**：PReM 将上下文压缩视为一个**学习的记忆保留与刷新问题**，把长上下文维护为模型内部的逐层 KV 记忆，无需外部压缩器，通过专用记忆层和特殊 token 实现动态、可学习的记忆更新，兼顾效率与准确性。

## 二、论文提出的方法论

### 核心思想
- 将长上下文编码为逐层的 KV 记忆（layer-wise KV memory），生成过程中由模型内部机制控制**保留哪些 chunk 的完整 KV**、**压缩哪些 chunk 为平均代表**，并在必要时通过特殊 token `<m>` 触发**刷新**（重新选择 chunk）。

### 关键技术细节
1. **上下文记忆表示**：
   - 对 L 层 LLM，每层 ℓ 的记忆为 Mℓ = (K_Mℓ, V_Mℓ)，形状为 H×N×d（H 注意力头数，N 上下文 token 数，d 头维度）。
   - 将 N 个 token 划分为 C 个连续 chunk，每个 chunk 大小 c。

2. **记忆选择与刷新**：
   - 在专用的**记忆层 ℓ_m**（选在中后层）添加一个选择查询投影矩阵 W_cℓ，用于将隐藏状态映射为选择查询。
   - 对于每个步骤 s，利用当前查询位置（问题 token 或 `<m>` token）的查询向量，与每个 chunk 的平均键计算注意力分数，取 Top-k 作为保留 chunk（`Ss`）。
   - **保留-池化规则**：对选中 chunk 保留所有 token 的 KV；对未选中 chunk，用平均池化后的单一代表 pair 替换。
   - 记忆层产生的选择 `Ss` 被**复用到所有其他层**：低于 ℓ_m 的层先临时压缩，收到 `Ss` 后刷新；高于 ℓ_m 的层直接使用 `Ss` 压缩自身 KV 记忆。
   - 刷新由生成的特殊 token `<m>` 触发：模型输出 `<m>` 时，记忆层用其隐藏状态重新选择 chunk，更新全局压缩记忆。

3. **相位分离刷新训练（Phase-Separated Refresh Training）**：
   - 训练数据构造为：`[x, y0, <m>, y1, <m>, ..., y_{T-1}]`，其中 `ys` 是第 s 步的答案段，`<m>` 是刷新标记。
   - 每个步骤分为两个前向阶段：
     - **记忆选择阶段**：仅处理 `zs`（步骤 s 的查询位置：s=0 为问题，s>0 为 `<m>`），计算 `Ss`，优化记忆选择损失。
     - **记忆条件生成阶段**：在刷新后的记忆下，处理强制教师答案段 `ys`，优化语言建模损失。
   - **记忆选择损失**（公式 2）：对每个步骤 s，令 Gs 为支持答案的证据 chunk 索引，使用对比损失训练记忆层将证据 chunk 排在非证据前。
   - **语言建模损失**（公式 3）：每个答案段内的交叉熵。
   - **边界交叉熵**（公式 4）：补充两个缺失的自回归链接（从 zs 到第一个答案 token，从答案段末尾到下一个 `<m>` token）。
   - 总损失（公式 5）：`L = 1/N_LM * (∑ L_ans^s + L_bd^s) + λ_ms * 1/T * ∑ L_ms^s`。

4. **推理过程**：
   - 初始压缩记忆由问题 token 的选择查询构建。
   - 正常解码使用当前压缩记忆；当模型生成 `<m>` 时，触发刷新，记忆层重新选择 Top-k chunk，更新全局压缩记忆。

- 公式 (1)：计算每个 chunk 的注意力分数：`a_{ℓ,s,j} = (1/H|Ps|) * Σ_h Σ_p (q_{ℓ,h,p}^T * k̅_{Mℓ,h,j}) / √d`。

## 六、论文的主要结论与发现

1. **PReM 在 16x 和 32x 压缩下均全面优于强基线**：在 Qwen2.5-3B 和 7B 上，平均 EM/F1 超过所有 Qwen 匹配的基线（KV 缓存压缩和文本压缩方法）。
2. **PReM 特别擅长多跳问答**：在 2WikiMQA、MuSiQue 上优势显著，归功于步骤特定的记忆刷新。
3. **学习到的压缩可超越全上下文提示**：即使 32x 压缩，PReM 仍优于直接输入 32K 上下文的 Prompting，表明选择紧凑、刷新的记忆比暴露全上下文更有效。
4. **PReM-3B 甚至超越发布版 7B 软记忆压缩基线**：说明小 backbone 通过动态记忆保留可以弥补参数规模劣势。
5. **证据选择能力**：在 HotpotQA 两步骤证据检索中，PReM 的 Recall@10 接近甚至超过最强嵌入检索模型 Qwen3-Embedding-8B，远优于原生注意力 top-k 和 SnapKV。
6. **记忆层选择在中后层最佳**：早期层学习记忆选择慢，中后层更适合作为记忆层。
7. **中等 chunk 粒度（50-100 token）最佳**，太粗（200 token）导致性能下降。
8. **效率方面**：PReM 端到端延迟低于全 KV 推理，峰值内存显著降低（19.57GB vs 30.0GB），刷新频率增加会略增延迟但仍在可控范围。
