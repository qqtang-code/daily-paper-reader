## 摘要
视觉语言模型（VLMs）在多模态推理任务中展现了卓越的能力，但仍然存在反复出现的失败，例如跳过关键视觉检查、误用领域规则以及幻觉不存在的概念。现有的大多数解决方案依赖于监督微调（SFT）和强化学习（RL），这些方法迭代成本高昂，且容易在分布偏移下变得脆弱。为此，我们提出多模态错误笔记本学习（Multimodal Mistake Notebook Learning, M2Note），一种无需训练的持续进化框架，将学习过程外部化为可编辑的记忆。M2Note将失败的轨迹转化为紧凑的主题-指导笔记：主题总结潜在的领域和概念，而指导提供可在未来推理中复用的可操作验证步骤。在测试时，M2Note通过多模态检索增强生成（RAG）检索相关笔记，并将其附加到模型上下文中，引导推理远离先前观察到的陷阱。为了稳定持续进化，我们采用带有回滚的批量级后验证，仅当同一批次的性能提升时才提交笔记本编辑，从而减少噪声更新并防止性能退化。M2Note既支持自我进化（同一VLM同时作为求解器和监督者），也支持跨模型进化（更强的监督者指导较弱的求解器），实现能力迁移而无需更新权重。在六个多模态推理基准上的实验表明，该方法在不同领域和骨干网络上均有一致改进，同时实现了强大的成本和样本效率，并能与思维链（CoT）提示互补。

## Abstract
Vision Language Models (VLMs) have demonstrated remarkable capabilities in multimodal reasoning tasks, yet they still suffer from recurring failures, such as skipping key visual checks, misapplying domain rules, and hallucinating unsupported concepts. Most existing solutions rely on supervised fine-tuning (SFT) and reinforcement learning (RL), which are expensive to iterate and can be brittle under distribution shift. To this end, we propose Multimodal Mistake Notebook Learning (M2Note), a training-free continual evolution framework that externalizes learning into an editable memory. M2Note transforms failed trajectories into compact subject-guidance notes: the subject summarizes the underlying domain and concept, while the guidance provides actionable verification steps that can be reused in future inference. At test time, M2Note retrieves relevant notes via multimodal retrieval-augmented generation (RAG) and appends them to the model context, steering reasoning away from previously observed pitfalls. To stabilize continual evolution, we adopt batch-level post-verification with rollback, which commits notebook edits only if they improve performance on the same batch, reducing noisy updates and preventing regressions. M2Note supports both self-evolving, where the same VLM acts as solver and supervisor, and cross-model evolving, where a stronger supervisor guides a weaker solver, enabling capability transfer without weight updates. Experiments on six multimodal reasoning benchmarks show consistent improvements across domains and backbones, while achieving strong cost and sample efficiency and remaining complementary to Chain-of-Thought (CoT) prompting.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- **M2Note（Multimodal Mistake Notebook Learning）**：一种**无需训练**的持续进化框架，通过维护一个外部错误笔记本（mistake notebook）来存储和检索结构化经验，将模型适应从参数更新转换为语义记忆更新。
- **关键角色**：
  - **Tuning Model（πθ）**：求解多模态问题，可借助笔记本指导。
  - **Tuner Model（πtuner）**：分析Tuning Model的错误，总结并写入新笔记。
- **封闭循环协议**：生成 → 反思 → 更新 → 验证。

### 2.2 关键技术细节

- **结构化笔记**：每一条笔记由三元组 `(subject, guidance, embedding)` 构成：
  - **subject**：抽象的任务领域和概念（如“化学计量学中的限制性试剂”）。
  - **guidance**：可操作的验证步骤（如“逐元素计算可用数/所需数，取最小值”）。
  - **embedding**：由多模态嵌入模型（Qwen3-VL-Embedding）生成的向量，用于检索。

- **多模态RAG检索**：
  - 对查询 `(q, i)` 使用嵌入模型获取特征向量 `e`，计算与所有笔记嵌入的余弦相似度。
  - 返回相似度高于阈值 `α` 且排名 top-K 的笔记，作为上下文附加到模型输入 `z = (q, i) ⊕ (subject, guidance)`。
  - 默认 K=1，α=0.4。

- **批次级内存精炼（Batch-Level Memory Refinement）**：
  1. **主题提取**：对每个失败样本，Tuner 使用 prompt 推断 subject。
  2. **指导合成**：将失败响应与反馈转化为验证导向的指导。
  3. **主题合并**：Tuner 决定添加（add）还是合并（merge）新笔记。
  4. **批次级后验证**：用更新后的笔记本重新运行同一批次，仅当准确率不降低时才接受更新，否则回滚（rollback）。该机制保证了单调非退化，抑制噪声更新。

- **奖励函数 R(·)**：
  - **监督设置**：与真实标签对比，匹配得 1，否则 0。
  - **无标签设置**：使用 LLM-as-a-Judge 评估正确性。

- **支持两种进化模式**：
  - **自我进化（Self-Evolving）**：同一 VLM 同时充当求解器和监督者。
  - **跨模型进化（Cross-Model Evolving）**：更强模型作为 Tuner 指导弱模型，实现能力迁移（无需权重更新）。

### 2.3 算法流程（文字说明）

1. 初始化空笔记本 M0。
2. 从数据分布中采样批次，对每个查询进行多模态RAG检索，Tuning Model生成带指导的初始响应，计算批次准确率作为基线。
3. 对失败样本，Tuner Model提取 subject 并合成 guidance，决定添加或合并。
4. 生成更新后的笔记本 M'i。
5. 使用 M'i 重新运行相同批次，计算新准确率。
6. 若新准确率 ≥ 基线，则接受更新（Mi+1 = M'i），否则回滚（Mi+1 = Mi）。

## 7. 优点

- **训练自由**：无需模型权重更新，仅维护外部内存，极大降低计算和迭代成本。
- **可编辑性与可解释性**：笔记以自然语言形式存储，易于审查、修改和迁移。
- **持续进化**：通过封闭循环实现动态改进，适应分布变化。
- **稳定性机制**：批次级后验证与回滚确保了进化过程不退化。
- **跨模型迁移能力**：笔记本可在不同 VLM 间复用，具有一定泛化性。
- **多模态友好**：利用多模态嵌入进行检索，充分捕捉视觉和文本线索。

## 8. 不足与局限

- **领域局限性**：M2Note 在**具有重复错误结构**的较窄领域（如数学推理）效果最佳；在视觉多样性极高的开放领域，笔记检索可能不准确，甚至引入误导性上下文，演化不稳定。
- **依赖高质量反射**：Tuner Model 的总结质量直接影响笔记有效性，若 Tuner 弱可能导致噪声笔记。
- **实验覆盖**：未在更大规模模型（如70B/100B+）或更多样化的 VLM 架构（如 LLaVA、InternVL）上进行实验，通用性有待验证。
- **横向对比有限**：与参数化方法的比较主要基于表格数据，缺乏在完全相同设置下（如相同骨干+相同数据量）的公平复现。
- **未讨论内存增长风险**：虽然合并机制控制笔记数量，但对极端多样场景是否需要更大内存未作分析。
- **应用限制**：仅适用于推理错误，无法处理感知层退化或分布外视觉输入；依赖多模态嵌入模型，其性能会影响检索质量。
