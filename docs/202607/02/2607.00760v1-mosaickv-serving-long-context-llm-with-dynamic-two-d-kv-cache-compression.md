## 摘要
长上下文LLM服务现在要处理数十万到数百万token的提示，使得键值（KV）缓存成为首要的服务开销。由于缓存随上下文长度线性增长，它可能会耗尽GPU内存，迫使批处理规模减小，并降低服务吞吐量。以往的KV缓存压缩技术通常仅针对序列维度或仅针对通道维度，这导致随着上下文窗口的扩展，压缩空间有限。同时压缩两个维度有望带来更高的内存减少，但直接应用这两种压缩形式会导致显著的精度损失。本文介绍了MosaicKV，一种面向极长上下文服务的动态二维KV缓存压缩系统。MosaicKV利用KV缓存中元素重要性分布的非均匀性，通过动态二维压缩来解决精度挑战。MosaicKV不是全局应用一种压缩模式，而是为每个KV向量识别重要元素，并以KV缓存段为粒度选择压缩策略。为了解决性能挑战（细粒度稀疏性和压缩管理开销可能抵消压缩带来的收益），MosaicKV引入了压缩KV缓存管理机制。该机制利用未充分利用的GPU和CPU资源来维护压缩的KV缓存并加速注意力计算。在H800 GPU上使用多个LLM进行的评估表明，与未压缩的基线相比，MosaicKV实现了高达16倍的注意力加速、4.8倍更低的解码延迟和7.3倍更高的吞吐量。同时，它将内存使用减少了3倍，在LongBench和RULER上带来的平均精度损失仅为1.76%。

## Abstract
Long-context LLM services now sustain prompts with hundreds of thousands to millions of tokens, making the key-value (KV) cache a first-order serving cost. Because the cache grows linearly with context length, it can exhaust GPU memory, force smaller batches, and reduce serving throughput. Prior KV cache compression techniques typically target only the sequence dimension or only the channel dimension, which leaves limited headroom as context windows scale. Compressing both dimensions promises higher memory reduction, but applying the two forms of compression directly leads to significant accuracy loss.   This paper introduces MosaicKV, a dynamic two-D (dimensional) KV cache compression system for extremely long-context serving. MosaicKV uses dynamic two-D compression to address the accuracy challenge, exploiting the non-uniform importance distribution of elements within the KV cache. Instead of applying one compression pattern globally, MosaicKV identifies important elements for each KV vector and selects compression strategies at the granularity of KV cache segments. To address the performance challenge, where fine-grained sparsity and compression management overhead can offset the gains from compression, MosaicKV introduces compressed KV cache management. This mechanism uses underutilized GPU and CPU resources to maintain compressed KV caches and accelerate attention computation.   Evaluation on an H800 GPU with multiple LLMs shows that MosaicKV delivers up to 16x attention speedup, 4.8x lower decode latency, and 7.3x higher throughput than the uncompressed baseline. At the same time, it reduces memory usage by 3x and incurs only 1.76% average accuracy loss on LongBench and RULER.

# MosaicKV: 面向长上下文LLM的动态二维KV缓存压缩

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大型语言模型（LLM）越来越多地部署在需要极长上下文的场景中（如代码库级代码生成、多文档问答、多轮代理工作流），前沿模型已支持高达1M token的上下文窗口。
- **核心问题**：长上下文导致KV缓存大小随序列长度线性增长，成为首要的内存瓶颈。例如，LLaMA-3.1-8B模型处理8个并行请求、128K token时需要128GB KV缓存（模型权重的8倍）；1M token时飙升至1TB（64倍）。这不仅耗尽GPU内存，还迫使批处理减小，导致解码延迟增加和吞吐量下降。
- **现有方法不足**：现有KV缓存压缩技术通常仅针对序列维度（保留重要token）或通道维度（保留重要通道），单一维度压缩率有限，难以跟上上下文窗口的指数级扩张。直接联合两个维度进行压缩（称为“二维压缩”）会导致严重的精度损失（如加70%通道压缩后精度损失达82.8%）。
- **论文目标**：提出MosaicKV，通过动态二维压缩在实现高压缩率的同时保持模型精度，并利用高效管理机制将压缩收益转化为实际性能提升。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 利用KV缓存中元素重要性分布的非均匀性：不仅不同token的KV向量重要性不同，同一向量内不同通道的重要性也不同，且这种分布模式在不同缓存区域（segment）也有差异。
- 避免“一刀切”的压缩策略，而是为每个KV向量单独识别重要元素，并为每个缓存段定制压缩策略（动态压缩策略）。

### 关键技术细节

#### （1）动态二维压缩（Dynamic Two-D Compression）
- **两阶段压缩**：
  - **阶段1：每向量通道压缩（Per-Vector Channel Compression）**：对每个KV向量，先通过SVD旋转将重要信息集中到少数通道，然后为每个向量独立选择top-r个重要元素（绝对值最大）。不同于现有方法使用全局统一的通道掩码。
  - **阶段2：动态序列压缩（Dynamic Sequence Compression）**：在解码阶段，将KV缓存划分为token块（block），对每个块计算块键（block key），然后基于当前查询与块键的点积得分选择top-k个块进行注意力计算。
- **动态压缩策略**：将KV缓存划分为多个段（segment），每个段单独分配压缩策略参数（通道压缩率、元素分组大小、序列压缩率、token块大小等）。输入token的段在预填充阶段固定大小划分（如64K token）；输出token的段在解码阶段动态划分，通过监控“压缩损失”来触发新段生成，并限制最大段长。压缩损失超过阈值时，在CPU端异步生成新策略并压缩。

#### （2）压缩KV缓存管理（Compressed KV Cache Management）
- **打包稀疏注意力（Packed Sparse Attention）**：将稀疏的KV缓存打包为稠密格式（packed KV format），使用位置位图（position bitmap）标识保留的元素位置，并支持可变粒度位图（可对相邻元素分组）。在注意力计算时，直接对打包向量进行点积，利用CUDA核心（而非张量核心）逐位重建元素位置，避免解压缩。同时将SVD反旋转合并到单个查询向量中，避免对大矩阵旋转。
- **异构双缓冲压缩（Heterogeneous Double Compression Buffering）**：利用GPU和CPU的空闲资源异步管理压缩缓存。GPU侧缓冲区临时存放新生成token的压缩版本（使用上一段策略），CPU侧缓冲区存放未经压缩的原始KV向量，CPU异步执行段划分、SVD计算、策略生成和压缩。当段完成时，切换替换GPU侧的临时压缩段。
- **增量策略生成（Incremental Strategy Generation）**：对每个新生成的token，增量更新协方差矩阵和压缩损失，避免全量重算，加速SVD矩阵和策略参数的生成。
