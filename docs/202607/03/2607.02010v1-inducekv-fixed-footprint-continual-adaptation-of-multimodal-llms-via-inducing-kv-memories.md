## 摘要
多模态大语言模型必须适应不断变化的任务和领域，但在有限的部署足迹下持续改进仍然困难，因为重复的参数更新或不断增长的回忆存储会随时间累积适应状态。我们研究了固定足迹持续适应：部署的适应状态保持在固定内存预算下，同时骨干模型保持不变，任务特定的更新被外部化。我们提出InduceKV，一种基于检索的方法，将每个选定的训练前缀存储为一个注意力就绪的记忆条目，包括冻结的检索键和紧凑的逐层键值（KV）有效负载，这些可以附加到模型的自注意力缓存中。在严格的内存预算下，InduceKV通过双层选择构建紧凑的诱导集：轻量级校准适用于检索，而选定的记忆在当前任务似然、基于锚点的保留和冻结检索空间中的覆盖之间取得平衡。在任务增量指令调整、连续VQA、领域增量适应和终身多模态指令调整中，InduceKV在匹配的内存预算下持续优于PEFT、MoE、回忆和提示检索基线。我们进一步报告了骨干匹配、阶段1 CoIN、计算匹配和可扩展性诊断，表明增益并非来自更强的骨干、仅回忆或无限制的候选池。

## Abstract
Multimodal large language models must adapt to evolving tasks and domains, yet continual improvement under bounded deployment footprint remains difficult because repeated parameter updates or growing replay stores can accumulate adaptation state over time. We study fixed-footprint continual adaptation: the deployed adaptation state is kept under a fixed memory budget, while the backbone model is left unchanged and task-specific updates are externalized. We propose InduceKV, a retrieval-based method that stores each selected training prefix as an attention-ready memory entry, consisting of a frozen retrieval key and compact layerwise key--value (KV) payloads that can be appended to the model's self-attention cache. Under a strict memory budget, InduceKV constructs a compact inducing set through bilevel selection: a lightweight calibration is fit for retrieval, while the selected memory balances current-task likelihood, anchor-based retention, and coverage in the frozen retrieval space. Across task-incremental instruction tuning, continual VQA, domain-incremental adaptation, and lifelong multimodal instruction tuning, InduceKV consistently improves over PEFT, MoE, replay, and prompt-retrieval baselines under matched memory budgets. We further report backbone-matched, stage-1 CoIN, compute-matched, and scalability diagnostics, showing that the gains are not due to a stronger backbone, replay alone, or an unbounded candidate pool.

## 1. 论文的核心问题与整体含义（研究动机和背景）

多模态大语言模型（MLLMs）在视觉理解和指令跟随等应用中日益普及，但新任务、新领域持续到来，而反复重训或联合调优所有历史数据因计算、隐私和产品迭代约束而不可行。关键挑战是：在**固定的部署适应足迹**下持续适应MLLM，同时保留先前能力。现有方法（如PEFT、MoE、回放）大多内部累积参数或数据，导致足迹增长或稳定性-可塑性难以权衡。本文提出**InduceKV**，通过将任务增量外部化为注意力兼容的KV记忆，在固定内存预算下实现持续适应，无需更新骨干参数。

## 2. 论文提出的方法论

### 2.1 核心思想
将持续适应转化为**预算约束的在线诱导集选择**问题：冻结MLLM所有参数，将每个选定训练前缀提取为注意力就绪的存储条目（包含冻结检索键和紧凑的逐层KV载荷），通过双层优化构建紧凑的诱导集，实现新任务适应和旧知识保留的权衡。

### 2.2 关键技术细节
- **冻结检索键与KV载荷提取**：对每个前缀输入，用冻结模型计算归一化的池化表示作为检索键，对各层自注意力中的键/值进行分段平均池化（固定长度m）得到压缩KV载荷。
- **检索加权KV注入**：学习一个极小的校准参数ϕ（检索温度τ和逐层值门控λ），在推理时根据查询键与存储键的余弦相似度加权组合记忆条目的KV载荷，并作为额外缓存令牌注入每一层的掩码自注意力中（值侧由λ缩放）。
- **双层诱导集选择**：给定候选池（当前任务+先前记忆）和固定预算B，外层优化连续选择权重w，同时优化当前任务似然、历史锚点保留损失和检索空间中的对数行列式覆盖正则项；内层优化校准参数ϕ以最小化当前任务似然（带L2正则）。采用交替优化，最终取Top-B权重对应的条目作为新记忆。
- **理论分析**：在松弛选择器的在线凸优化框架下，证明了固定预算下的次线性静态遗憾和动态遗憾，以及锚点控制历史风险的界。

## 8. 不足与局限

- **推理开销**：相比无记忆方法，InduceKV需额外一次前缀前向计算检索键，且自注意力矩阵增大，预填延迟高于无记忆的PEFT，但低于提示检索方法。
- **依赖模型内部接口**：需访问冻结模型的隐藏状态和KV缓存，对于非标准注意力实现或受限推理API可能需要工程适配。
- **锚点假设**：理论依赖锚点能均匀近似历史风险，虽然实验验证了高相关性（Pearson r>0.92），但非百分百保证。
- **超参数选择**：需在首任务上进行β、γ等超参数的网格搜索，虽不敏感但增加了少许工作量。
- **隐私与安全**：外部KV记忆来源于训练前缀，部署时需要与骨干模型同等谨慎对待。
- **实验范围**：主要基于公开研究基准和检查点，未覆盖工业级用户数据或实时部署环境。
- **无无监督或在线设置**：方法目前假设任务边界已知且批量更新，未探索完全在线或流式场景。
