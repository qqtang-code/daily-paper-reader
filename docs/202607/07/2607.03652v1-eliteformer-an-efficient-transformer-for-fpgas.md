## 摘要
Transformer模块在大型语言模型（LLM）中普遍存在，但由于其苛刻的计算和内存需求，部署面临挑战。虽然先前的工作通常分别优化注意力机制或前馈网络（FFN），但很少有硬件架构通过协同设计的硬件加速来同时处理这两个组件。我们提出了ELiTeFormer（高效线性三元Transformer），这是第一个将混合线性注意力与超低精度（三元）线性投影相结合的Transformer模型架构，专门为现场可编程门阵列（FPGA）部署而协同设计。与LLaMA 3相比，ELiTeFormer实现了10倍的模型权重压缩和12.8倍的键值（KV）缓存压缩，同时保持了有竞争力的精度（在MMLU基准测试中为31.9%，与BitNet b1.58相差3.0%以内）。我们的关键架构贡献是一种新颖的处理单元（PE）微架构，通过位屏蔽操作消除了三元线性投影中的所有乘法运算，通过完全避免专用数字信号处理（DSP）块，显著降低了资源利用率。我们使用高级综合（HLS）流程，针对Xilinx VCK5000 Versal板对ELiTeFormer进行仿真、综合和部署。与标准实现相比，模块级仿真显示FFN操作加速9.6倍，注意力加速4.4倍。在长上下文长度下，端到端部署相比NVIDIA A100图形处理器（GPU）上的LLaMA 3，延迟降低高达3.9倍，能效提升高达3.2倍。这是首个将线性注意力与三元量化相结合的FPGA实现，证明了算法-架构协同设计用于下一代LLM加速的可行性。

## Abstract
Transformer blocks are prevalent in large language model (LLM) but present deployment challenges due to their challenging computational and memory demands. While prior work has typically optimized attention mechanisms or feed-forward networks (FFNs) separately, few hardware (HW) architecture have jointly addressed both components with co-designed hardware acceleration. We present ELiTeFormer (Efficient Linear Ternary Transformer), the first Transformer model architecture that unifies hybrid linear attention with ultra-low-precision (ternary) linear projections, specifically co-designed for field-programmable gate array (FPGA) deployment. ELiTeFormer achieves 10x model weight compression and 12.8x key-value (KV) cache compression compared to LLaMA 3, while maintaining competitive accuracy (31.9% on the MMLU benchmark, within 3.0% of BitNet b1.58). Our key architectural contribution is a novel processing element (PE) micro-architecture that eliminates all multiplications in ternary linear projections through bitmasking operations, significantly reducing resource utilization by completely avoiding dedicated digital signal processing (DSP) blocks. We simulate, synthesize, and deploy ELiTeFormer targeting a Xilinx VCK5000 Versal board using high-level synthesis (HLS) flows. Block-level simulations show 9.6x speedup for FFN operations and 4.4x speedup for attention compared to standard implementations. End-to-end deployment achieves up to 3.9x lower latency and 3.2x better energy efficiency than LLaMA 3 on an NVIDIA A100 graphics processing unit (GPU) at long context lengths. This represents the first FPGA realization combining linear attention with ternary quantization, demonstrating the viability of algorithm-architecture co-design for next-generation LLM acceleration.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：Transformer 大语言模型（LLM）在推理时面临极高的计算和内存开销，尤其在 FPGA 等资源受限平台上部署困难。现有工作要么单独优化注意力机制（如稀疏注意力、线性注意力），要么单独优化前馈网络（FFN，如低精度量化），但缺乏将两者联合优化并协同设计硬件加速器的工作。
- **整体含义**：ELiTeFormer 首次在 FPGA 上实现了混合线性注意力与三元（ternary）精度线性投影的统一设计，通过算法-硬件协同优化（HW/SW co-design），在保持可接受模型质量的同时大幅压缩模型权重和 KV 缓存，显著提升推理速度和能效，体现了 FPGA 在 LLM 加速中的潜力。

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：结合两种高效技术——① **混合线性注意力**（Hedgehog linear attention + 滑动窗口稀疏注意力）使注意力计算复杂度从二次降至线性且静态内存占用；② **BitNet b1.58 风格的三元量化**（权重取值{-1,0,1}）将线性投影计算简化为位掩码操作。两者协同消除乘法器，适配 FPGA 特性。
- **关键技术细节**：
  - **混合注意力块**：每个注意力头并行执行 Hedgehog 线性注意力（基于可分离核的重组公式）和滑动窗口稀疏注意力（窗口大小 256），具体公式见原文 Eq.3。
  - **三元量化线性投影**：所有权重矩阵（Q、K、V 投影及 FFN 投影）采用三元值，仅需 2 bits 存储；激活值保持半精度（float16）。
  - **ELTF PE 微架构**：新型处理单元通过位掩码实现三元乘法（-1/0/1），完全避免 DSP 块使用。8-bit weight dataframe 通过移位解码出四个 2-bit 索引，每个索引选择对应掩码（nullify、sign flip、pass through），然后经归约树累加。
  - **推理流水线**：每层划分为 QKV 投影、注意力计算（attn）、FFN 前两半（FFN1,2）、FFN 后半（FFN3）四个阶段，各阶段间同步（因 RMSNorm 需要全 token 统计量）。流水线支持多 batch 并行。
  - **内存接口**：使用 AXI 内存映射接口，权重以 8-bit 压缩（每 8-bit 存 4 个三元权重），批量读取 256 个权重。注意力缓存仅需线性大小（依赖窗口 w 和头维度 d），压缩比达 12.8×。

## 7. 优点

- **算法-硬件协同设计**：一次性优化注意力与 FFN 两大计算瓶颈，利用 FPGA 可编程性定制位掩码 PE，实现乘法器消除。
- **极高的模型与缓存压缩**：10× 权重 + 12.8× KV 缓存，使大 batch、长上下文推理成为可能，显著降低内存瓶颈。
- **静态推理剖面**：线性注意力在生成阶段内存占用完全由超参数决定，适合 FPGA 静态编译，消除动态优化开销。
- **能源效率突出**：FPGA 上 0.30 tokens/J，远优于 GPU 方案（0.03–0.07 tokens/J），适合边缘与能效敏感场景。
- **实验设计全面**：涵盖算法精度、硬件仿真、实际部署，并与多种基线（vLLM、bitnet.cpp）对比，具有说服力。

## 8. 不足与局限

- **模型质量有限**：在 HellaSwag、ARC-c 等复杂推理任务上准确率下降显著（如 HellaSwag 从 79.1% 降至 58.0%），仅适合对精度要求不高的应用。
- **训练数据差异**：ELiTeFormer 的预训练数据（FineWeb-edu 100B tokens）远少于基线（LLaMA 3 使用专有数据），可能导致“数据质量差距”，影响公平对比。
- **实验覆盖不足**：
  - 未在更大参数规模（如 8B）上进行实际 FPGA 部署（仅仿真），实际延迟/吞吐量可能受内存带宽限制。
  - 未与最新的高效注意力变体（如 Mamba、基于状态空间的模型）对比。
  - 未评估长上下文（>4k tokens）下的实际端到端性能（实验中仅测到 214 tokens 或 4096 维度的模拟）。
- **部署限制**：FPGA 板卡仅 DDR 内存，未使用 HBM（如 U280），若能使用可进一步提升吞吐量。
- **技术局限性**：低秩适配器（LoRA）在 ELiTeFormer 上无效（训练后质量下降），说明三元量化模型对微小参数调整敏感；滑动窗口大小需 256 才能获得合理质量，限制了缓存压缩的上限。
- **未讨论其他优化**：如稀疏注意力中的动态稀疏模式、混合精度训练等未涉及。
