## 摘要
大型语言模型使用的KV缓存具有线性增长的时间复杂度，因此在处理长上下文时，LLMs面临内存膨胀和解码效率降低的问题。当前的KV缓存驱逐已成为重要的研究方向；然而，基于固定软标记（如Judge Q）的现有方法依赖于静态参数集作为查询来评估KV对的重要性，因此它们无法动态适应不同的输入提示，也无法精确捕获复杂且变化的任务相关性。此外，被驱逐的KV对被永久丢弃，导致不可逆的信息丢失和上下文断裂。为解决这个问题，我们提出Meta-Soft，一种基于探针驱动上下文集成的动态压缩框架。具体来说，我们构建了一个具有可学习正交基矩阵L的元库，并使用带有Gumbel-Softmax的选择器网络生成可微分的稀疏组合权重，从而从输入提示特征中动态合成最具有针对性的k个软标记。我们将这些软标记附加到输入序列的末尾以探测关键信息。我们还引入了一种基于注意力流的集成机制，该机制将移除标记的语义信息重新分配到保留标记中，从而有效保持被丢弃的上下文信息。在多个数据集上的实验表明，我们的方法优于现有的最先进驱逐方法，并为KV缓存压缩提供了新的解决方案。

## Abstract
The KV cache used in large language models has linearly growing time complexity, so LLMs face memory blow-up and reduced decoding efficiency when they process long contexts.Current KV Cache eviction has become an important research direction; however, existing methods based on fixed Soft Tokens (e.g., Judge Q) rely on a static parameter set as the query to evaluate the importance of KV pairs, so they cannot adapt dynamically to different input prompts, and they cannot precisely capture complex and changing task relevance.Also, evicted KV pairs are discarded permanently, so this causes irreversible information loss and context breaks. To address this problem, we propose Meta-Soft, a dynamic compression framework based on probe-driven context integration. Specifically, we build a meta-library with a learnable orthogonal basis matrix $\mathcal{L}$, and we use a selector network with Gumbel-Softmax to produce differentiable sparse combination weights, so we dynamically synthesize the most targeted $k$ Soft Tokens from the input prompt features.We append these Soft Tokens to the end of the input sequence to probe key information. We also introduce an attention-flow based integration mechanism, which redistributes the semantic information of removed tokens into retained tokens, and this keeps the dropped context information effectively.Experiments on multiple datasets show that our method outperforms existing state-of-the-art eviction methods and provides a new solution for KV Cache compression.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题所在**：大型语言模型（LLM）在长上下文推理中，KV缓存（Key-Value Cache）的GPU内存消耗随序列长度线性增长，导致内存溢出和解码吞吐量下降。现有KV缓存压缩方法主要分为两类：基于**硬驱逐（Hard Eviction）** 的方法（如H2O、SnapKV、Judge Q）和基于**合并（Merging）** 的方法（如CaM、ZeroMerge）。
- **现有方法的局限**：
  - **静态查询不适应输入**：Judge Q等方法使用固定、可学习的“评判词元”（Judge Tokens）作为查询来评估KV对的重要性，但无法动态适应不同输入提示的语义模式，导致跨任务不匹配。
  - **信息不可逆损失**：硬驱逐直接丢弃被视为不重要的KV对，造成上下文断裂和信息丢失；简单合并则可能导致“语义混叠”（semantic aliasing），影响精度。
- **整体目标**：提出一种**动态压缩框架**，既能根据输入提示自适应地合成查询（软词元），又能将丢弃词元的信息重新整合到保留词元中，实现高保真压缩。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **动态软词元生成**：通过一个可学习的正交基矩阵元库（Meta-Library）和一个轻量选择器网络（Selector Network），根据输入提示的全局特征动态合成任务相关的软词元（Soft Tokens）。这些软词元附加到输入序列末尾，通过一次前向注意力计算作为“探针”来全局定位重要KV对。
- **上下文整合（Contextual Consolidation）**：利用注意力流（Attention Flow）计算被丢弃词元与保留词元之间的语义相似度，将丢弃词元的信息加权融合到最相似的保留词元中，而非直接丢弃或盲目平均。

### 2.2 关键技术细节
- **元库构建**：元库由可学习正交基矩阵 $\mathcal{L} \in \mathbb{R}^{M \times d}$ 构成（$M=512$，$d$为隐藏维度），并施加正交正则化约束 $\lambda_{\text{div}} \|\mathcal{L}\mathcal{L}^\top - I\|_F^2$ 以确保基向量多样性。
- **软词元合成**：选择器网络 $f_\theta$ 提取输入提示全局特征，通过Gumbel-Softmax输出稀疏组合权重，从元库中线性组合出 $k$ 个软词元（默认$k=32$）。过程可微分，支持端到端训练。
- **探针与重要性评分**：软词元作为查询，对全量KV缓存进行注意力计算，得到探测得分 $S \in \mathbb{R}^{k \times L}$，再聚合为每个词元的全局重要性分布 $A_{\text{soft}} \in \mathbb{R}^L$。
- **缓存分区**：根据 $A_{\text{soft}}$ 保留Top-B个词元（$I_{\text{keep}}$），其余为 $I_{\text{drop}}$。
- **注意力流上下文整合**：
  - 计算丢弃词元与保留词元在Key空间上的相似度 $S_{\text{sim}} = K_{\text{drop}} K_{\text{keep}}^\top / \sqrt{d_k}$。
  - 对每个丢弃词元，仅选取Top-m个最相似保留词元进行稀疏软分配（Taj Mahal式稀疏化）。
  - 引入负载均衡：对过度承载的保留词元进行列重加权（$b_j = 1/(\ell_j + \epsilon)$），避免信息坍缩。
  - 最终将丢弃词元的Value值加权合并到保留词元的Value中（$\Delta V = W_{\text{flow}}^\top V_{\text{drop}}$），并用量化门控 $g_j = \text{clip}(\alpha \ell_j + \epsilon, 0, 1)$ 控制注入程度，Key保持不变。
- **训练流程**：两阶段训练。
  - **阶段一：联合优化**。使用SlimPajama 4万样本，联合训练元库 $\mathcal{L}$ 和选择器 $f_\theta$，监督信号为从冻结LLM中提取的“地面真值注意力分布” $A_{\text{gold}}$（公式1）。损失函数为 $J = L_{\text{MSE}}(A_{\text{soft}}, A_{\text{gold}}) + \lambda_{\text{div}} \|\mathcal{L}\mathcal{L}^\top - I\|_F^2$。
  - **阶段二：选择器微调**。冻结元库和LLM，微调选择器 $f_\theta$，使用ShareGPT 1万样本，进一步优化任务特定的组合策略。
