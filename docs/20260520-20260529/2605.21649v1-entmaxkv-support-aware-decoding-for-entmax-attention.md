## 摘要
长上下文解码越来越受到KV缓存内存流量的限制，因为每个生成的令牌都会关注一个缓存，其大小随上下文长度线性增长。现有的稀疏解码方法通过选择令牌或页面的子集来降低这一成本，但它们是针对softmax注意力设计的，而softmax注意力的密集尾部使得任何截断都会丢弃非零概率质量。相比之下，α-entmax生成精确的零，将稀疏解码从密集尾部近似转变为支持恢复：如果所选候选包含entmax支持，则稀疏解码保持精确。虽然最近的entmax内核实现了高效训练，但它们并未解决自回归解码瓶颈，即在知道稀疏性之前，密集推理仍然流式传输整个KV缓存。在这项工作中，我们引入了EntmaxKV，这是一个entmax原生的稀疏解码框架，它在加载KV页面之前利用稀疏性。EntmaxKV结合了查询感知的页面评分、支持感知的候选选择和稀疏entmax注意力。我们通过丢弃的概率质量δ分析截断误差，表明输出误差由δ控制，并在恢复entmax支持时消失。我们进一步引入了一个高斯感知的entmax选择器，它从轻量级页面统计中估计entmax阈值，并根据分数分布调整所选预算。实验上，在匹配的KV预算下，EntmaxKV比基于softmax的稀疏解码丢弃更少的概率质量，保留更多的支持令牌，并实现更低的输出误差。在长上下文和语言建模基准测试中，它使用一小部分KV缓存紧密匹配全缓存entmax，在1M上下文长度下相对于全注意力基线实现了高达3.36倍（softmax）和5.43倍（entmax）的加速。代码地址：https://github.com/deep-spin/entmaxkv。

## Abstract
Long-context decoding is increasingly limited by KV-cache memory traffic since each generated token attends over a cache whose size grows linearly with context length. Existing sparse decoding methods reduce this cost by selecting subsets of tokens or pages, but are designed for softmax attention, whose dense tails make any truncation discard nonzero probability mass. In contrast, $α$-entmax produces exact zeros, turning sparse decoding from dense-tail approximation into support recovery: if the selected candidates contain the entmax support, sparse decoding remains exact. While recent entmax kernels enable efficient training, they do not address the autoregressive decoding bottleneck, where dense inference still streams the full KV cache before sparsity is known. In this work, we introduce EntmaxKV, an entmax-native sparse decoding framework that exploits sparsity before KV pages are loaded. EntmaxKV combines query-aware page scoring, support-aware candidate selection, and sparse entmax attention. We analyze truncation error through the dropped probability mass $δ$, showing that output error is controlled by $δ$ and vanishes when the entmax support is recovered. We further introduce a Gaussian-aware entmax selector that estimates the entmax threshold from lightweight page statistics, adapting the selected budget to the score distribution. Empirically, EntmaxKV drops less probability mass, retains more support tokens, and achieves lower output error than softmax-based sparse decoding at matched KV budgets. On long-context and language modeling benchmarks, it closely matches full-cache entmax while using a small fraction of the KV cache, achieving up to $3.36\times$ (softmax) and $5.43\times$ (entmax) speedup over full attention baselines at 1M context length. Code available at: https://github.com/deep-spin/entmaxkv.

## 1. 核心问题与整体含义（研究动机和背景）

- **瓶颈**：大语言模型（LLM）长上下文自回归解码时，KV缓存大小随序列长度线性增长，导致GPU内存带宽成为主要瓶颈，每次生成新token都需要读取整个KV缓存。
- **现有局限**：现有稀疏解码方法（如Quest、H2O）针对softmax注意力设计，但softmax分配非零概率给所有token，任何截断都会丢弃尾部概率质量（δ>0），属于“密集尾部近似”问题，无法做到精确。
- **α-entmax的优势**：α-entmax（包括α=2的sparsemax）能够产生精确零概率，即只有部分token（支持集S）有非零权重。这改变了稀疏解码的目标：从近似尾部概率转变为“支持恢复”——只要选中的缓存页面包含entmax支持集，稀疏解码结果与全缓存entmax完全相同。
- **问题**：虽然已有高效的entmax训练内核，但解码时仍需先加载整个KV缓存才能知道哪些token有零权重，未能利用稀疏性减少内存读取。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将稀疏解码重新定义为支持恢复问题，在加载KV页面之前利用轻量级页面元数据预测entmax支持集，只读取可能包含支持token的页面，实现精确或低误差的稀疏注意力。

### 关键技术细节

- **分页KV缓存**：将KV缓存划分为大小P的页面，每个页面存储元数据：坐标最小/最大key（用于确定性上限分）、坐标均值和二阶矩（用于高斯近似）。
- **查询感知页面评分**：
  - **确定性上界**：基于每个页面的key最小/最大向量，计算查询-页面的最大可能得分上界\(\bar{s}^{(p)}_{box}(q)\)（公式14），用于Top-k选择。
  - **高斯近似评分**：假设页面内token得分服从高斯分布\(S^{(p)} \sim N(\mu_p(q), \sigma_p^2(q))\)，其中均值和方差由页面统计量和查询计算得到（公式17）。用于估计页面内最大得分以及entmax阈值。
- **候选选择策略**：
  - **Top-k页面选择**：选择页面得分上界最大的k个页面。
  - **高斯感知entmax选择**：利用高斯近似求解entmax阈值\(\hat{\tau}\)（通过归一化方程（19）迭代求解），然后估计每个页面至少有一个token得分超过阈值的置信度，选择估计最大得分大于阈值的页面（公式22），可选安全余量\(\Delta\)。
- **稀疏entmax注意力计算**：仅在选中的token集合上计算α-entmax，得到注意力输出。若选中集包含完整支持集，则输出精确等于全缓存entmax。
- **理论分析**：

## 6. 主要结论与发现

- **支持恢复优势**：entmax稀疏解码在相同KV预算下丢弃更少概率质量、保留更多支持token、输出误差远低于softmax稀疏解码。
- **精确性**：当候选页面包含完整支持集时，entmax稀疏解码输出精确等于全缓存结果。
- **下游性能**：Passkey检索中，entmax在极低预算（<0.1% KV）下保持完美检索，而softmax快速退化。RULER上entmax稀疏解码接近全缓存性能，优于softmax。
- **效率**：在1M上下文长度下，EntmaxKV Gaussian-aware相比全缓存softmax加速3.36倍，相比全缓存entmax加速5.43倍。速度优势主要来自避免读取大部分KV页面。

## 7. 优点

- **理论创新**：首次将entmax的精确稀疏性引入解码阶段，将问题从“尾部近似”重新定义为“支持恢复”，具有严格误差界。
- **方法高效**：高斯感知选择器通过轻量级页面统计估计entmax阈值，自适应调整预算，避免固定top-k的盲目性。
- **兼容性**：基于分页KV缓存，易于集成到现有推理系统（如vLLM）。
- **实验全面**：从近似质量到下游任务到效率，多角度验证。
- **公平对比**：所有稀疏方法在同一分页元数据和Triton基础设施上实现，消除工程差异。

## 8. 不足与局限

- **模型规模限制**：实验仅使用1B参数模型（NAPE），未测试更大模型（如7B、70B），实际应用时支持集分布可能变化。

- **理论误差界**：命题1的\(2B\delta\)界是宽松的，实际误差可能更小；但论文未探讨更紧的界。
- **安全余量Δ**：论文未系统研究Δ的选择方法，依赖经验设置。
- **仅关注解码**：EntmaxKV不处理预填充（prefill）阶段的加速，这是另一个重要场景。
- **超参数敏感性**：页面大小P、预算k、置信度q_page等需人工设定，缺乏自适应机制。
