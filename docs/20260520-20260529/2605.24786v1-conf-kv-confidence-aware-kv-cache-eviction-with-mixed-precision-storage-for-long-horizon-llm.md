## 摘要
长程大语言模型推理使键值（KV）缓存成为GPU内存的主要消耗者，并使每个token的注意力计算日益昂贵。许多常见的淘汰策略使用静态的最近窗口或历史注意力，忽略了每个解码步骤中可用的一个信号：模型当前的不确定性。我们提出CONF-KV，一种KV缓存管理器，它将下一个token分布转换为标量置信度分数，并用于选择每步缓存预算，在模型不确定时保留更多上下文，在模型确信时积极剪枝。在每个预算内，token根据累积注意力权重和最近性的复合指标进行排序，同时一个受保护的最近窗口保持局部连贯性。我们将该策略与分块在线softmax注意力、混合FP16/INT8存储以及逐层金字塔预算变体相结合。在四种模型家族和长达4K的生成长度上，CONF-KV的占用空间接近固定512token滑动窗口，同时与全KV相比，困惑度仅相差1.5-2.1个点。在长达32K token的“大海捞针”测试中，CONF-KV达到91.4%的检索准确率，而滑动窗口为53.8%，H2O为80.6%；在75个VisualWebArena任务中，CONF-KV保留了全KV成功的95.3%，同时峰值内存降低了2.8倍。

## Abstract
Long-horizon LLM inference turns the key--value (KV) cache into the dominant GPU memory consumer and makes per-token attention increasingly expensive. Many common eviction policies use static recency windows or historical attention, leaving unused a signal computed on every decoding step: the model's current uncertainty. We introduce CONF-KV, a KV-cache manager that converts the next-token distribution into a scalar confidence score and uses it to choose the per-step cache budget, retaining more context when the model is uncertain and pruning aggressively when it is confident. Within each budget, tokens are ranked by a composite of accumulated attention mass and recency, while a protected recent window preserves local coherence. We combine the policy with blockwise online-softmax attention, mixed FP16/INT8 storage, and a pyramidal per-layer budget variant. Across four model families and generated lengths up to 4K, CONF-KV stays near the footprint of a fixed 512-token sliding window while remaining within 1.5--2.1 perplexity points of full KV. On Needle-in-a-Haystack up to 32K tokens, CONF-KV reaches 91.4% retrieval accuracy versus 53.8% for sliding windows and 80.6% for H2O; on 75 VisualWebArena tasks it retains 95.3% of full-KV success at 2.8 times lower peak memory.

- **核心问题**：能否利用解码时模型自身的置信度信号来指导KV缓存的逐步骤预算分配，从而在保持质量的同时大幅降低内存占用？
- **整体含义**：提出一种无需训练、不改变模型权重、可插入标准生成循环的置信度感知KV缓存管理器CONF-KV，通过动态预算显著改善了内存-质量帕累托前沿。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 在每步解码时，将下一个token分布（logits softmax后的概率）转换为一个标量置信度分数c。
- 根据c与阈值τ的比较，选择宽松或紧凑的缓存预算：当模型不确定（c < τ）时保留更多上下文；当模型自信（c ≥ τ）时积极淘汰。
- 在选定预算内，根据累积注意力质量（EMA）和最近性的复合评分对token进行排序，同时保留一个受保护的近期窗口以确保局部连贯性。

### 2.2 关键技术细节
- **置信度估计器**：结合三个分量——归一化熵的补数（1 - Ĥ）、log概率边际的sigmoid σ(m)、top-1 token概率p₁，加权求和：  
  c = w_H(1-Ĥ) + w_m σ(m) + w_p p₁，权重(w_H, w_m, w_p) = (0.4, 0.3, 0.3)（通过小范围网格搜索确定，对多种模型稳定）。
- **预算与排序器**：  
  - 若c ≥ τ，使用紧凑预算N_high；否则使用宽松预算N_low（N_high ≤ N_low）。  
  - 每个候选token的排序分数 s_i = α·â_i + (1-α)·r̂_i，其中â_i是归一化的EMA注意力质量，r̂_i是归一化的最近性（原始生成步骤）。注意力质量按层和头平均后以指数移动平均更新（λ=0.9）。两种特征在非受保护候选内进行min-max归一化。  
  - 始终保留最近的P个token（保护窗口）。
- **混合精度存储**：最近W个保留token保持FP16，更旧的token按每头每通道对称量化为INT8（缩放因子s = max(|x|)/127）。反量化融合到分块注意力读取中。
- **分块在线softmax注意力**：读取压缩后的缓存分块，维护running max和normalizer以实现精确在线softmax。
- **金字塔逐层预算（CONF-KV-L变体）**：将选定预算按层非均匀分配：N(ℓ)_high = max(N_min, N(0)_high · β^(ℓ/L))，β=0.5，N_min=96，类似应用于N_low。这利用了深层往往将有用信息集中在更少token上的观察。

### 2.3 算法流程（文字说明）
1. 输入当前token x_t，从缓存C前向传播得到logits和注意力A。
2. 计算softmax概率p，进而计算置信度c。
3. 根据c选择预算N。
4. 对每层：更新EMA注意力元数据；若缓存长度超过N，则对非保护token按复合分数排名，淘汰最低得分者直到长度≤N；对保留但非最近W个token进行INT8量化。
5. 将新的K/V以FP16追加到缓存；采样下一个token。

## 8. 不足与局限

- **对短上下文无效**：当上下文长度从未超过淘汰阈值时，CONF-KV无操作，此时方法没有带来额外收益。
- **速度优势受计算瓶颈影响**：当MLP计算而非注意力或KV带宽主导延迟时，速度提升较小。
- **高温采样下信息来源受限**：熵饱和时置信度信号变得不可靠，可能需要重新调整阈值τ。
- **连续压缩策略向分页布局迁移的挑战**：基于token的淘汰会导致部分已死块，除非采用粗化淘汰或稀疏掩码。
- **剩余失败模式**：NIAH/VWA中的失败主要在模型对稀有实体查找前过度自信时发生（选择紧凑预算而非保留上下文），可通过调高τ缓解，但伴随内存成本。
- **实验性基础设施限制**：吞吐量扫描中全KV在批>8时OOM，因此速度提升主张基于批≤8的比较；原型分配器可能不完全反映生产环境。
- **量化精度选择**：仅测试INT8、NF4、INT4，未探索更先进的位宽选择方法。
- **置信度权重稳定性**：虽然声称跨模型稳定，但(grid search)主要在GPT-2上进行，更大模型是否需要重新调整未充分讨论（仅给出相关性）。
- **无模型权重发布**：论文不涉及新模型发布，但公平性讨论指出更低KV缓存内存可能降低有害自动化成本。
- **实验覆盖**：未在更长上下文（>32K）或更多任务（如代码生成、对话）上评估；只报告了单块H100上的结果，未在多卡分布式推理场景中测试。
