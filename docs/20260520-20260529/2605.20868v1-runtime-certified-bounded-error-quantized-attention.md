## 摘要
KV缓存量化减少了长上下文大语言模型推理的内存成本，但引入了近似误差，通常仅凭经验验证。现有系统依赖于平均情况下的鲁棒性，缺乏在运行时检测或恢复失败的机制。我们提出了一种分层KV缓存架构，实现了运行时认证的注意力：INT8键和INT4值存储在GPU内存中，而FP16原始值保留在系统RAM中用于确定性回退。一个两项误差分解给出了每头、每步的界限，包括(i)键量化导致的注意力分布失真和(ii)值重建误差。这些界限在线计算，并用于驱动自适应精度选择和多阶段回退阶梯，从而保证在需要时恢复精确的密集注意力输出。在LLaMA 3.1-8B上，针对PG-19、NIAH和RULER基准测试，上下文长度达128K，该系统在语言建模和检索任务中与密集FP16 KV质量在噪声范围内匹配，同时恢复了朴素INT8/INT4基线中观察到的灾难性失败。短上下文下的值敏感任务展示了压缩与保真度之间的可控权衡，可以通过更紧的值容限或FP16值回退来消除。该认证是局部的（每头、每步），并不能保证端到端的模型正确性，但确保每个注意力计算要么相对于FP16参考有界，要么通过回退精确恢复。这将KV缓存量化为一个运行时验证的计算，而非固定的近似。目标不是原始加速，而是在严格质量约束下实现激进KV压缩的安全部署。

## Abstract
KV cache quantization reduces the memory cost of long-context LLM inference, but introduces approximation error that is typically validated only empirically. Existing systems rely on average-case robustness, with no mechanism to detect or recover from failures at runtime. We present a tiered KV cache architecture that enables runtime-certified attention: INT8 keys and INT4 values are stored in GPU memory, while FP16 originals are retained in system RAM for deterministic fallback. A two-term error decomposition yields per-head, per-step bounds on (i) attention distribution distortion from key quantization and (ii) value reconstruction error. These bounds are computed online and used to drive adaptive precision selection and a multi-stage fallback ladder, which guarantees recovery to the exact dense attention output when required. Across PG-19, NIAH, and RULER benchmarks on LLaMA~3.1-8B with contexts up to 128K, the system matches dense FP16 KV quality within noise for language modelling and retrieval tasks, while recovering catastrophic failures observed in naive INT8/INT4 baselines. Value-sensitive tasks at short context expose a controlled trade-off between compression and fidelity, which can be eliminated via tighter value tolerances or FP16-value fallback. The certification is local (per-head, per-step) and does not guarantee end-to-end model correctness, but ensures that each attention computation is either bounded relative to an FP16 reference or exactly recovered via fallback. This reframes KV cache quantization as a runtime-verified computation rather than a fixed approximation. The goal is not raw speedups, but enabling safe deployment of aggressive KV compression under strict quality constraints.

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
采用**分层KV缓存架构**：
- **Tier-1（GPU VRAM）**：存储INT8键（每通道量化）和INT4值（每组16个元素量化），附带元数据。存储成本约为FP16的56%。
- **Tier-2（CPU系统RAM）**：保留FP16全精度原始键值，作为确定性回退的“安全网”。
- **运行时监控**：在线计算每头每步的两项误差界（键压缩误差 + 值重建误差），驱动自适应精度选择和四级回退阶梯，保证必要时返回精确密集输出。

#### 关键技术细节

1. **量化方案**：
   - **键**：每通道INT8量化（每块B=16个令牌），每个通道独立标度和偏移，避免低幅通道被压缩。量化后重建误差≤σ_c/2。
   - **值**：每组g=16个元素INT4量化，两个INT4打包成一个字节，附带FP16标度和偏移。存储时记录每块最大每令牌重建误差η_b和最大范数ν_b。

2. **自适应精度选择（Adaptive Top-K）**：
   - 第一阶段：使用INT8键计算每块的对数质量（log-mass）。
   - 选择覆盖阈值τ_cov=0.995，按块质量降序选择前K*个块（K*∈[K_min,K_max]），使其估计累计质量占比≥τ_cov。
   - 被选中的块使用FP16键（从Tier-2调入）；剩余块继续用INT8键。保证尾块真实质量≤e^{3Δ}(1-τ_cov)。

3. **两项误差分解**：
   - **键压缩误差**：影响注意力分布扭曲。通过总变差（TV）界：E_key ≤ 2V_max · e^{3Δ}(1-τ_cov)(e^{2Δ}-1)。
   - **值重建误差**：E_val = ∑_b ρ_b η_b，其中ρ_b为块注意力质量，η_b为最大重建误差。该界可在运行时精确计算。

4. **分级回退阶梯（Fallback Ladder）**：
   - **Rung 1**：扩大FP16键覆盖（增加K*）。
   - **Rung 2**：若块贡献估计ρ_b·η_b超过阈值v_tol，调入FP16值。
   - **Rung 3**：排名一致性检查——比较INT8和FP16下前r个块排名，若不一致或尾块上界可能进入前r，则对该头使用全FP16重计算（调用torch.scaled_dot_product_attention）。
   - **Rung 4**：全层全头FP16重计算（作为无条件终端）。

5. **预条件与监控**：
   - P1：量化元数据最新（块填充时计算，不可变）。
   - P2：量化误差有界（每通道round-to-nearest，无饱和）。
   - P3：无算术溢出（FP32累加器）。
   - P4：Tier-2原始数据可用。
   任一条件不满足则触发回退，最终输出密集FP16结果。

### 7. 优点

1. **首次提供运行时可计算的、每头每步的量化误差界**，且界是紧的、可独立监控。
2. **分层架构设计合理**：GPU存低精度，CPU RAM留全精度原始值，回退路径直接调用标准密集注意力，正确性无条件保证。
3. **自适应精度选择**基于注意力质量分布动态决定FP16键/INT8键，质量集中在少数块时节省大幅。
4. **排名一致性检查**检测到INT8评分噪声导致的注意力分布扭曲并逐头回退，解决了非均匀精度导致的注意稀释问题（关键创新）。
5. **实验对统计显著性进行了严格检验**（McNemar检验、Bootstrap CI），消融实验系统分解误差源，结论可信。
6. **可复现性**：提供完整源代码、配置和种子，MIT协议开源。

### 8. 不足与局限

1. **认证局部性**：仅保证每步每头误差有界，不提供端到端模型正确性保证。值敏感任务（如8K RULER）中每步小误差多层累积仍可导致正确率下降（虽然可通过调参数缓解）。
2. **延迟开销大**：2.7×~4.8×于密集，主要瓶颈为排名一致性检查（28%）和H2D页入（11%），适合质量优先场景，不适合延迟敏感生产环境。
3. **VRAM节省仅在超长上下文（≥64K）时显著**：短上下文时固定大小的FP16暂存缓存使总VRAM可能超过密集。
4. **Tier-2内存要求高**：必须保留完整FP16 KV在系统RAM（约16 GB @128K），对于系统RAM有限的环境无法部署。
5. **数值路径差异**：认证内核与Flash Attention的算术路径不同，产生~4pp的可复现差异（通过排名检查部分恢复，但未完全消除）。
6. **值量化组大小无中间选项**：g=16表现好但值敏感任务仍有问题，g=32质量急剧下降（Δppl=+26.4）。
7. **实验仅在单模型（LLaMA 3.1-8B）单GPU上完成**，未验证跨模型架构（如Mamba、非GQA）、跨硬件（H100、AMD）的泛化性。
