## 摘要
长上下文语言模型受限于键值（KV）缓存的内存占用。现有的免训练KV压缩方法通常根据单一重要性信号（注意力、新近性、逐层分配或键独特性）对token进行排序，当有用上下文具有全局独特性、局部片段性或即时相关性时，这种排序会变得脆弱。我们提出NestedKV，一种受嵌套学习中的连续内存系统启发的仅键KV缓存压缩方法。NestedKV维护全局、块级和滑动窗口键锚点，通过多时间尺度余弦异常对token评分，并利用无训练外部学习器结合头自适应混合和惊喜门控token路由合并生成的排名。该评分与自适应每头预算配对，无需训练或修改LLM。在Qwen3和Llama-3.2模型上的RULER（4k–32k）、LooGLE、LongBench、LongBench-E、InfiniteBench和MMLU-Pro基准测试中，NestedKV在保留缓存较小时表现最佳。在Qwen3-4B上，在r=0.75时，NestedKV在RULER上比KeyDiff提升高达19.10分，在LongBench上提升19.29分；在r=0.95时，NestedKV在LongBench上保留37.32分，而KeyDiff为17.55分。

## Abstract
Long-context language models are limited by the memory footprint of the key-value (KV) cache. Existing training-free KV compression methods usually rank tokens by one importance signal -- attention, recency, layer-wise allocation, or key distinctiveness -- which becomes brittle when useful context is globally distinctive, locally episodic, or immediately relevant. We introduce NestedKV, a key-only KV cache compression method inspired by the Continuum Memory System in Nested Learning. NestedKV maintains global, block-level, and sliding-window key anchors, scores tokens by multi-time-scale cosine anomaly, and combines the resulting rankings with a training-free outer learner using head-adaptive mixing and surprise-gated token routing. The score is paired with adaptive per-head budgets and requires no training or LLM modification. Across RULER (4k--32k), LooGLE, LongBench, LongBench-E, InfiniteBench, and MMLU-Pro on Qwen3 and Llama-3.2 models, NestedKV is strongest when the retained cache is small. On Qwen3-4B, it improves over KeyDiff by up to 19.10 points on RULER and 19.29 on LongBench at $r=0.75$; at $r=0.95$, it retains 37.32 on LongBench versus 17.55 for KeyDiff.

- **问题**：长上下文语言模型（LLM）的推理成本受限于KV缓存，其大小随上下文长度和批次线性增长。现有训练免费的KV压缩方法（如StreamingLLM、SnapKV、KeyDiff等）通常依赖单一重要性信号（注意力、新近性、逐层分配或键的几何独特性）对token排序。
- **限制**：单一信号在以下场景中脆弱：
  - 全局独特的上下文（文档级异常但局部常见）
  - 局部片段（局部主题转移但全局重复）
  - 即时相关（近期流但早期证据缺失）
- **图1示例**：注意力信号集中在prompt尾部和注意力“下沉”前缀，而答案载体token位于低注意力区域，导致基于注意力的压缩器在激进压缩时逐出错误token。
- **整体含义**：提出将KV缓存压缩重新定义为**连续内存系统中的异常检测**，通过多时间尺度视角判断token是否应保留，从而在低保留预算下表现更优雅地退化。

## 2. 论文提出的方法论

### 核心思想
- 受**Nested Learning**中的**Continuum Memory System**启发，为每个层和注意力头构建三个时间尺度的“内存摘要”作为内部学习者，每个token获得三个余弦异常评分（而非一个）。再通过**外部学习者**（免训练）将三个评分组合，自适应于头（head）级别和token级别，最终决定是否保留。

### 关键技术细节

1. **连续内存状态**（针对每个token i）
   - **稳定内存** μ_s：整个预填充上下文的平均键向量（全局）。
   - **片段内存** μ_e(i)：token i所在局部块的平均键向量。块大小 b = clip(⌊N/32⌋, 128, 256) 。
   - **当前内存** μ_c(i)：以i结尾的滑动窗口（窗口大小W=64）的平均键向量。

2. **每尺度异常评分**（余弦相似度的负值）
   - a_s(i) = -cos(ˆk_i, μ_s)； a_e(i) = -cos(ˆk_i, μ_e(i))； a_c(i) = -cos(ˆk_i, μ_c(i))
   - 高表示异常（应保留），低表示可被该内存尺度预测（可逐出）。评分做min-max归一化到每头尺度。

3. **外部学习者**——头自适应混合与惊喜门控路由
   - **头自适应混合**：对每个头，计算每个尺度对前10%与后10% token的差异Δ_k，通过softmax（含固定对数先验w0_s=0.4, w0_e=0.4, w0_c=0.2和温度β=3.0）得到混合权重w_k，然后计算混合得分 a_blend(i) = w_s·ã_s + w_e·ã_e + w_c·ã_c。
   - **压缩诱导的“惊喜”信号**：三个评分排名的标准差 s(i) = std(ã_s, ã_e, ã_c)，当三个尺度之间高度不一致时，认为该token的评分不可靠。
   - **路由**：若惊喜高，则采用最强单个尺度得分 a_win(i) = max(ã_s, ã_e, ã_c)；若惊喜低，采用混合得分。最终得分 a*(i) = (1-α(i))·a_blend(i) + α(i)·a_win(i)，其中α(i) = σ(κ·(s(i)-τ))，门控阈值τ=0.6，锐度κ=10.0。
   - **全局锚点**：前4个token固定保留（sink pinning）。

4. **头内预算分配**
   - 不采用均匀分配，而是让所有头-位置对按归一化残差（即最终得分）竞争全局预算Bℓ，同时每头至少保留其最高得分token的20%作为安全网。这实现了自适应头预算。

5. **公式流程**：
   - 输入：层ℓ、头h的键矩阵K、值V，预算B。
   - 计算三个尺度均值 → 三个余弦异常 → 归一化 → 头自适应混合 → 惊喜路由 → 得到最终得分a*。
   - 执行TopB选择（保留最高分位置），得到压缩缓存。

- **特点**：无需训练、不修改LLM参数、仅依赖键（key），与标准注意力内核兼容。
