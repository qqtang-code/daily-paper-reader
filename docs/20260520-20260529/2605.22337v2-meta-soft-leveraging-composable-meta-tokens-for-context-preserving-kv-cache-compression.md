## 摘要
大型语言模型中使用的KV缓存时间复杂度线性增长，因此在处理长上下文时，LLMs面临内存爆炸和解码效率降低的问题。当前的KV缓存淘汰已成为重要的研究方向；然而，现有基于固定软令牌（如Judge Q）的方法依赖于静态参数集作为查询来评估KV对的重要性，因此无法动态适应不同的输入提示，也无法精确捕获复杂且变化的任务相关性。此外，被淘汰的KV对被永久丢弃，导致不可逆的信息丢失和上下文断裂。为解决这一问题，我们提出Meta-Soft，一种基于探测驱动上下文集成的动态压缩框架。具体来说，我们构建了一个包含可学习正交基矩阵$\mathcal{L}$的元库，并使用带有Gumbel-Softmax的选择器网络生成可微的稀疏组合权重，从而从输入提示特征中动态合成最具针对性的$k$个软令牌。我们将这些软令牌附加到输入序列末尾以探测关键信息。我们还引入了一种基于注意力流的集成机制，将移除令牌的语义信息重新分配到保留令牌中，从而有效保留被丢弃的上下文信息。在多个数据集上的实验表明，我们的方法优于现有最先进的淘汰方法，并为KV缓存压缩提供了新的解决方案。

## Abstract
The KV cache used in large language models has linearly growing time complexity, so LLMs face memory blow-up and reduced decoding efficiency when they process long contexts. Current KV Cache eviction has become an important research direction; however, existing methods based on fixed Soft Tokens (e.g., Judge Q) rely on a static parameter set as the query to evaluate the importance of KV pairs, so they cannot adapt dynamically to different input prompts, and they cannot precisely capture complex and changing task relevance. Also, evicted KV pairs are discarded permanently, so this causes irreversible information loss and context breaks. To address this problem, we propose Meta-Soft, a dynamic compression framework based on probe-driven context integration. Specifically, we build a meta-library with a learnable orthogonal basis matrix $\mathcal{L}$, and we use a selector network with Gumbel-Softmax to produce differentiable sparse combination weights, so we dynamically synthesize the most targeted $k$ Soft Tokens from the input prompt features. We append these Soft Tokens to the end of the input sequence to probe key information. We also introduce an attention-flow based integration mechanism, which redistributes the semantic information of removed tokens into retained tokens, and this keeps the dropped context information effectively. Experiments on multiple datasets show that our method outperforms existing state-of-the-art eviction methods and provides a new solution for KV Cache compression.

  - **丢弃信息不可逆**：主流方法采用硬淘汰（hard eviction），将判定为不重要的KV对永久丢弃，导致信息丢失和上下文断裂，可能引发幻觉。部分合并方法（如CaM、ZipCache）试图合并，但存在语义混叠（semantic aliasing）问题。

## 2. 方法论

### 2.1 核心思想
构建一个可学习的**元库**（Meta-Library，正交基矩阵 \(\mathcal{L} \in \mathbb{R}^{M \times d}\)），通过一个轻量级选择器网络（含Gumbel-Softmax）根据输入提示的全局语义特征，动态合成 \(k\) 个**软令牌**（Soft Tokens）。这些软令牌被附加到输入序列末尾，作为查询（probe）进行一次前向注意力计算，以评估所有历史KV对的重要性，并据此划分保留集与丢弃集。最后，利用**注意力流聚合**（Attention Flow Aggregation）将丢弃令牌的语义信息重新分配到最相似的保留令牌中，而非直接丢弃。

### 2.2 关键技术细节
- **元库构建与两阶段训练**：
  - 从SlimPajama数据集采集40k样本，提取真实注意力分布 \(A_{gold}\) 作为监督信号。
  - 损失函数：\(\mathcal{J} = \mathcal{L}_{MSE}(A_{soft}, A_{gold}) + \lambda_{div} \|\mathcal{L}\mathcal{L}^T - I\|_F^2\)，包含均方误差项和正交正则项。
  - 第一阶段：联合优化元库 \(\mathcal{L}\) 和选择器 \(f_\theta\)，梯度通过Gumbel-Softmax传播。
  - 第二阶段：冻结元库和LLM骨干，仅微调选择器，防止表示漂移。
- **软令牌探测与缓存分区**：
  - 预填充阶段：软令牌 \(E_{soft}\) 拼接到提示嵌入后，得到 \(X_{input} = [X_{emb}; E_{soft}] \in \mathbb{R}^{(L+k)\times d}\)。
  - 计算探测得分 \(S = Q_{probe} (K_{prompt})^T / \sqrt{d_k}\)，得到预测分布 \(A_{soft}\)。
  - 按 \(A_{soft}\) 进行Top-K保留：\(I_{keep} = \text{TopK}(A_{soft}, B)\)，\(I_{drop}\) 为其余。
- **上下文合并**：
  - 利用注意力流（attention flow）计算丢弃令牌与保留令牌之间的键空间相似度，采用**负载均衡稀疏路由**策略：
    - 计算 \(S_{sim} = K_{drop} K_{keep}^T / \sqrt{d_k}\)，对每个丢弃token只考虑top-m个最相似的保留token。
    - 计算分配权重 \(A_{ij}\)，并对列进行重加权以避免个别保留token吸收过多信息，最后行归一化得到 \(W_{flow}\)。
    - 聚合：\(\Delta V = W_{flow}^T V_{drop}\)，并用负载自适应门控更新 \(\hat{V}_{keep}\)（保留 \(K_{keep}\) 不变）。
- **推理流水线**：软令牌生成、探测和上下文合并全部在预填充阶段完成，解码阶段无额外开销。
