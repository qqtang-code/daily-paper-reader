## 摘要
键值（KV）缓存仍然是部署大型语言模型（LLM）在长生成任务中的主要瓶颈。先前的工作通常对预填充和解码缓存应用统一压缩，但压缩预填充缓存会破坏关键上下文，从而降低性能。虽然保留预填充缓存至关重要，但解码阶段的压缩仍未被充分探索，现有方法依赖于严格的最近窗口或瞬时注意力。我们对注意力动态的分析揭示了强烈的时间模式：关键标记在长时间跨度内获得持续关注，而局部推理则涉及短时的爆发。静态启发式方法无法捕捉这种行为，导致重要标记被过早驱逐或陈旧标记被保留。我们提出Moment-KV，一种基于动量驱动的时间注意力聚合的解码时KV缓存压缩方法。我们的方法将标记重要性建模为一个持续演化的状态，其中注意力通过衰减进行聚合，同时捕捉长期影响和近期相关性。实验表明，Moment-KV在长生成任务中显著提高了生成保真度（2.3-3.2%），同时保持了解码延迟。

## Abstract
Key-Value (KV) cache remains a major bottleneck for deploying Large Language Models (LLMs) in long-generation tasks. Prior work often applies uniform compression across both prefill and decoding caches, but compressing the prefill cache degrades performance by corrupting critical context. While preserving the prefill cache is essential, decoding-phase compression remains underexplored, with existing methods relying on rigid recency windows or instantaneous attention. Our analysis of attention dynamics reveals strong temporal patterns: critical tokens receive sustained attention over long horizons, while local reasoning involves short-lived bursts. Static heuristics fail to capture this behavior, leading to premature eviction of important tokens or retention of stale ones. We propose Moment-KV, a decoding-time KV cache compression method based on momentum-driven temporal attention aggregation. Our method models token importance as a continuously evolving state, where attention is aggregated with decay, capturing both long-term influence and recent relevance. Experiments show that Moment-KV significantly improves generation fidelity in long-generation tasks (2.3-3.2 %) while maintaining decoding latency.

- **问题**：
  - 现有方法常对 **预填充（prefill）** 和 **解码（decoding）** 两阶段缓存采用统一压缩策略，但压缩预填充缓存会破坏关键指令与上下文，严重损害生成质量。
  - 解码阶段压缩虽重要，但方法较少，现有方法要么依赖固定的**最近窗口**，要么依赖**瞬时注意力**，无法捕捉注意力的**时序动态**：
    - 重要 token（heavy hitters）在长时段内获得持续关注，但可能存在短暂注意力下降，静态策略会误将其驱逐。
    - 固定最近窗口保留了大量低效用 token，浪费缓存空间。
- **研究动机**：针对上述局限，提出**解码时**的 KV 缓存压缩方法，保留完整预填充缓存，仅压缩解码缓存，并利用**动量驱动**的时序注意力聚合来动态评估 token 重要性。

## 2. 方法论

### 核心思想
- 将 token 重要性建模为**连续演化的状态**，通过**带衰减的注意力聚合**（动量）同时捕捉长期影响和近期相关性。

### 关键技术细节
- **缓存架构**：
  - **预填充缓存 Φp**：完整保留，永不驱逐。
  - **解码缓存 Φd**：固定容量 `Bd`（如 512 或 1024 个 token），超限时驱逐重要性最低的 token。
- **重要性更新**（公式 5）：
  - 每步 t，计算当前 token 在所有注意力头上的平均注意力权重 `āi(t)`。
  - 采用动量更新：`Ii(t) = α * Ii(t-1) + āi(t)`，其中 α ∈ (0,1) 控制历史贡献的保留程度。
- **新 token 处理**：
  - 新生成 token 的重要性初始化为 0，但在本步更新后立即获得正值（因为 self-attention 至少 >0），防止刚生成就被驱逐。
- **驱逐策略**：
  - 当 `|Φd| > Bd` 时，移除重要性得分最低的 overflow 个 token（`arg min overflow`）。

### 算法流程（文字描述）
1. 计算当前 token 的 Q、K、V。
2. 将 `(K, V)` 追加到解码缓存 Φd，初始化其重要性为 0。
3. 在完整缓存上计算注意力，取所有头的平均，得到解码缓存部分的注意力向量。
4. 按动量公式更新每个解码缓存 token 的重要性。
5. 若解码缓存超出预算 Bd，则驱逐重要性最低的 overflow 个 token。
6. 返回更新后的缓存。
